# AddNodeApp — Node.js Script Resources

Use AddNodeApp(name, appDirectory, scriptPath) to run Node.js scripts as Aspire resources.

## The API

```csharp
public static IResourceBuilder<NodeAppResource> AddNodeApp(
    this IDistributedApplicationBuilder builder,
    string name,
    string appDirectory,
    string scriptPath)
```

- **`name`** — resource name for the Dashboard and service discovery
- **`appDirectory`** — path to the Node.js project directory (resolved relative to AppHost)
- **`scriptPath`** — the JavaScript file to run (e.g., `server.js`)

## Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var api = builder.AddNodeApp("express-api", "../node-api", "server.js")
                 .WithHttpEndpoint(env: "PORT");

builder.AddProject<Projects.Web>("web")
       .WithReference(api);

builder.Build().Run();
```

## How It Works

`AddNodeApp` creates an `ExecutableResource` that runs `node server.js` in the specified directory. The orchestrator:

1. Finds the Node.js executable on the system PATH
2. Sets the working directory to `appDirectory`
3. Runs `node scriptPath` with any injected environment variables

## Injecting the Port

Node.js apps typically read their port from an environment variable. Use the `env` parameter on `WithHttpEndpoint` to inject the allocated port:

```csharp
builder.AddNodeApp("api", "../node-api", "server.js")
       .WithHttpEndpoint(targetPort: 3000, env: "PORT");
```

Your Node.js code reads it:

```javascript
const port = process.env.PORT || 3000;
app.listen(port);
```

## When to Use AddNodeApp vs AddJavaScriptApp

| Method | Runs | Best For |
|--------|------|----------|
| `AddNodeApp` | `node scriptPath` directly | Express, Fastify, custom scripts |
| `AddJavaScriptApp` | `npm run <script>` | Projects with `package.json` scripts |
| `AddViteApp` | `npm run dev` with Vite config | Vite-based frontends |

Use `AddNodeApp` when you want to run a specific `.js` file directly with Node.js.
