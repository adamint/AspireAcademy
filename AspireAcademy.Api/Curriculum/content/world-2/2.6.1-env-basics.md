# WithEnvironment Basics — Strings and Endpoints

`WithEnvironment()` is the primary way to inject configuration into resources. It sets environment variables that are available at process startup.

## String Values

The simplest form sets a static string:

```csharp
builder.AddProject<Projects.ApiService>("api")
    .WithEnvironment("LOG_LEVEL", "Information")
    .WithEnvironment("FEATURE_FLAG", "enabled");
```

## Endpoint References

Pass an `EndpointReference` to inject the resolved URL at runtime:

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithHttpEndpoint(name: "http");

builder.AddProject<Projects.Worker>("worker")
    .WithEnvironment("API_URL", api.GetEndpoint("http"));
```

At runtime, `API_URL` resolves to something like `http://localhost:5123`. The actual port is allocated by DCP and resolved when the AppHost starts.

## Interpolated String Expressions

You can use C# interpolated string handlers with endpoint properties:

```csharp
var endpoint = api.GetEndpoint("http");

builder.AddProject<Projects.Worker>("worker")
    .WithEnvironment("API_HOST", $"{endpoint.Property(EndpointProperty.Host)}");
```

## How Environment Variables Flow

1. The AppHost defines environment variables via `WithEnvironment()`
2. DCP injects them into the resource's process at startup
3. The process reads them via `Environment.GetEnvironmentVariable()` or `IConfiguration`
4. The dashboard displays all environment variables for each resource

## Null Values

Passing `null` as the value means the environment variable is set but with an empty string:

```csharp
.WithEnvironment("OPTIONAL_KEY", (string?)null)
```

## Key Points

- String overload: `WithEnvironment("KEY", "value")` for static values
- Endpoint overload: `WithEnvironment("KEY", endpointRef)` for dynamic URLs
- Interpolated string handler: compose expressions inline
- Environment variables are visible in the dashboard for debugging
