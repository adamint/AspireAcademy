# Arguments, Replicas, and External Endpoints

Three methods you'll use frequently with project resources: `WithArgs()` for CLI arguments, `WithReplicas()` for horizontal scaling, and `WithExternalHttpEndpoints()` for external access.

## Passing CLI Arguments

`WithArgs()` passes command-line arguments to the project process:

```csharp
var worker = builder.AddProject<Projects.Worker>("worker")
    .WithArgs("--mode", "background", "--verbose");
```

Arguments can also include dynamic values — endpoint references and parameter resources resolve at runtime:

```csharp
var worker = builder.AddProject<Projects.Worker>("worker")
    .WithArgs("--api-url", api.GetEndpoint("http"));
```

There's also a callback overload for conditional logic:

```csharp
.WithArgs(context =>
{
    context.Args.Add("--workers");
    context.Args.Add("4");
});
```

## Scaling with Replicas

`WithReplicas()` launches multiple instances of a project, load-balanced behind shared endpoints:

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithReplicas(3);
```

This creates three processes. Aspire's reverse proxy distributes incoming requests across all replicas. Each replica gets a unique port, but consumers see a single logical endpoint. The dashboard shows each replica as a child of the resource.

## External Endpoints

By default, project endpoints are **internal** — reachable only within the Aspire network. `WithExternalHttpEndpoints()` marks all HTTP/HTTPS endpoints as externally accessible:

```csharp
var frontend = builder.AddProject<Projects.WebApp>("frontend")
    .WithExternalHttpEndpoints();
```

External endpoints appear as clickable links in the dashboard and are accessible from your host machine's browser.

## Combining All Three

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithArgs("--enable-swagger")
    .WithReplicas(3)
    .WithExternalHttpEndpoints();
```

This creates a 3-replica API with swagger enabled and externally accessible endpoints — a production-like topology for local development.
