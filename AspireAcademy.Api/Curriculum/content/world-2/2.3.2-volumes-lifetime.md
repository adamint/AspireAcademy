# Volumes, Bind Mounts, and Lifetime

Attach named volumes, mount host directories, and control container lifetime (Session vs Persistent).

## Named Volumes

Volumes persist data between container restarts. Aspire auto-generates a volume name:

```csharp
builder.AddPostgres("pg")
       .WithDataVolume();  // Auto-named persistent volume
```

For custom volumes:

```csharp
builder.AddContainer("myapp", "myimage")
       .WithVolume("my-data", "/app/data");
```

- First parameter: volume name (Docker manages the storage)
- Second parameter: mount path inside the container

## Bind Mounts

Bind mounts map a **host directory** into the container. Useful for config files:

```csharp
builder.AddContainer("nginx", "nginx")
       .WithBindMount("./nginx.conf", "/etc/nginx/nginx.conf", isReadOnly: true);
```

The source path is resolved relative to the AppHost directory. Set `isReadOnly: true` to prevent the container from modifying the file.

## Container Lifetime

By default, containers are **session-scoped** — created when the AppHost starts, removed when it stops. For development databases, this means losing data on every restart.

Use `WithLifetime` to make containers **persistent**:

```csharp
builder.AddPostgres("pg")
       .WithDataVolume()
       .WithLifetime(ContainerLifetime.Persistent);
```

The `ContainerLifetime` enum has two values:

| Value | Behavior |
|-------|----------|
| `Session` | Create on start, remove on shutdown (default) |
| `Persistent` | Reuse existing container by name; don't remove on shutdown |

## Combining Them

A typical development database setup:

```csharp
var db = builder.AddPostgres("pg")
                .WithDataVolume()                          // Persist data in a volume
                .WithLifetime(ContainerLifetime.Persistent) // Don't destroy on restart
                .AddDatabase("mydb");
```

This gives you a PostgreSQL container that survives AppHost restarts with all your test data intact.
