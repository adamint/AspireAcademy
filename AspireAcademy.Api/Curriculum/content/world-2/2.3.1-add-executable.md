# Adding Executables with AddExecutable

`AddExecutable()` adds a non-.NET process to the Aspire app model. This enables polyglot orchestration — Node.js, Python, Go, or any CLI tool can participate alongside .NET projects.

## Basic Usage

```csharp
var api = builder.AddExecutable(
    "pyapi",           // resource name
    "python",          // command/executable
    "./python-api",    // working directory
    "app.py"           // arguments
);
```

The signature is: `AddExecutable(name, command, workingDirectory, params args)`.

## How Executables Participate

Executable resources are first-class citizens in the app model:

- **Dashboard**: They appear with status, logs, and endpoints
- **Endpoints**: Add HTTP endpoints for service discovery
- **Environment variables**: Inject config via `WithEnvironment()`
- **Dependencies**: Use `WithReference()` and `WaitFor()`
- **Arguments**: Pass additional args via `WithArgs()`

## Adding Endpoints

Since executables don't have `launchSettings.json`, you must define endpoints explicitly:

```csharp
var api = builder.AddExecutable("pyapi", "python", "./python-api", "app.py")
    .WithHttpEndpoint(port: 5001, name: "http")
    .WithEnvironment("PORT", "5001");
```

## Polyglot Examples

```csharp
// Node.js frontend
var frontend = builder.AddExecutable("frontend", "node", "./web-app", "server.js")
    .WithHttpEndpoint(targetPort: 3000)
    .WithReference(api);

// Go service
var goSvc = builder.AddExecutable("gosvc", "go", "./go-service", "run", ".")
    .WithHttpEndpoint(targetPort: 8080);

// Python worker with arguments
var worker = builder.AddExecutable("worker", "python", "./worker", "main.py")
    .WithArgs("--concurrency", "4");
```

## Key Points

- The `command` is any executable on the system PATH (or an absolute path)
- The working directory is relative to the AppHost project
- Executable resources support all environment, endpoint, and reference extensions
- The process lifecycle is managed by Aspire — it starts and stops with the AppHost
