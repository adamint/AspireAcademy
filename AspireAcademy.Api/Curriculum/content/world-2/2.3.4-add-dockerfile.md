# Building Images with AddDockerfile

Build custom images with AddDockerfile(name, contextPath) and WithDockerfile(contextPath) for existing containers.

## AddDockerfile — New Container from Dockerfile

```csharp
var app = builder.AddDockerfile("myapp", "../src/MyApp")
                 .WithHttpEndpoint(targetPort: 8080);
```

This builds a Docker image from the `Dockerfile` in `../src/MyApp`, then runs it as a container. The context path is resolved relative to the AppHost directory.

## Custom Dockerfile Path

If your Dockerfile isn't named `Dockerfile` or is in a different location:

```csharp
builder.AddDockerfile("myapp", "../src/MyApp",
    dockerfilePath: "docker/Dockerfile.production");
```

## Multi-Stage Builds

Target a specific stage:

```csharp
builder.AddDockerfile("myapp", "../src",
    dockerfilePath: "Dockerfile",
    stage: "runtime");
```

## WithDockerfile — Convert an Existing Container

If you already have a `ContainerResource` and want to build from a Dockerfile instead of pulling an image:

```csharp
builder.AddContainer("myapp", "placeholder")
       .WithDockerfile("../src/MyApp")
       .WithHttpEndpoint(targetPort: 8080);
```

## Build Pipeline

When you use `AddDockerfile` or `WithDockerfile`, Aspire:

1. Generates a unique image name and tag
2. Adds build and push steps to the pipeline
3. Builds the image during the `Building` resource state
4. Starts the container from the built image

You'll see the resource state transition: `NotStarted → Building → Starting → Running`.

## Example: Full Stack with Custom Image

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var db = builder.AddPostgres("pg").AddDatabase("mydb");

// Build API from Dockerfile
var api = builder.AddDockerfile("api", "../src/Api")
                 .WithHttpEndpoint(targetPort: 5000)
                 .WithReference(db);

builder.AddProject<Projects.Web>("web")
       .WithReference(api);

builder.Build().Run();
```
