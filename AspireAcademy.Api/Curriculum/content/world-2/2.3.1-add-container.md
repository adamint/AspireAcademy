# Adding Containers with AddContainer

Add container resources with AddContainer(name, image) or AddContainer(name, image, tag).

## The API

```csharp
// Basic — uses 'latest' tag
builder.AddContainer("nginx", "nginx");

// With explicit tag
builder.AddContainer("nginx", "nginx", "1.27");

// Same as above using chaining
builder.AddContainer("nginx", "nginx").WithImageTag("1.27");
```

The first parameter is the resource name (used in the Dashboard and service discovery). The second is the Docker image name. The optional third is the image tag.

## Adding Endpoints

Containers don't expose endpoints by default. Add them explicitly:

```csharp
var nginx = builder.AddContainer("nginx", "nginx")
                   .WithHttpEndpoint(port: 8080, targetPort: 80);
```

- **`port`** — the port exposed to the host (what you connect to)
- **`targetPort`** — the port inside the container (what the process listens on)

## Environment Variables

```csharp
builder.AddContainer("nginx", "nginx")
       .WithEnvironment("NGINX_HOST", "example.com")
       .WithEnvironment("NGINX_PORT", "80");
```

## Full Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var db = builder.AddPostgres("pg").AddDatabase("mydb");

var api = builder.AddProject<Projects.Api>("api")
                 .WithReference(db);

// Custom Nginx reverse proxy container
builder.AddContainer("proxy", "nginx", "1.27")
       .WithBindMount("./nginx.conf", "/etc/nginx/nginx.conf")
       .WithHttpEndpoint(port: 8080, targetPort: 80)
       .WithReference(api);

builder.Build().Run();
```

## When to Use AddContainer

Use `AddContainer` when you need a container that isn't covered by a hosting integration package. For common infrastructure (Redis, PostgreSQL, RabbitMQ), prefer the dedicated `AddRedis`, `AddPostgres`, etc. methods — they configure health checks, connection strings, and ports automatically.
