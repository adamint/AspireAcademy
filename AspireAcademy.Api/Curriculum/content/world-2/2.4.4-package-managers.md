# Package Managers: WithNpm, WithYarn, WithPnpm, WithBun

Configure package manager behavior with install flags and custom install arguments.

## Default: npm

By default, `AddJavaScriptApp` and `AddViteApp` use **npm** to install dependencies and run scripts. The orchestrator runs `npm install` before `npm run <script>`.

## Switching Package Managers

Chain the appropriate extension method:

```csharp
// Use yarn
builder.AddViteApp("frontend", "../app")
       .WithYarn();

// Use pnpm
builder.AddViteApp("frontend", "../app")
       .WithPnpm();

// Use bun
builder.AddViteApp("frontend", "../app")
       .WithBun();
```

## How It Works

Each method changes two things:
1. The **install command** — `yarn install`, `pnpm install`, or `bun install`
2. The **run command** — `yarn dev`, `pnpm dev`, or `bun run dev`

## Example: Pnpm + Vite

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var api = builder.AddProject<Projects.Api>("api");

builder.AddViteApp("frontend", "../react-app")
       .WithPnpm()
       .WithReference(api);

builder.Build().Run();
```

The orchestrator detects the package manager and runs:
1. `pnpm install` — installs dependencies
2. `pnpm run dev` — starts the Vite dev server

## When to Switch

Use whichever package manager your project already uses. The choice is determined by your lockfile:

| Lockfile | Package Manager |
|----------|----------------|
| `package-lock.json` | npm |
| `yarn.lock` | yarn |
| `pnpm-lock.yaml` | pnpm |
| `bun.lockb` | bun |

Aspire doesn't auto-detect the lockfile — you must explicitly call the matching `With*` method.
