# Understanding EndpointAnnotation

Every endpoint in Aspire is backed by an `EndpointAnnotation` — a resource annotation that describes a network endpoint with its protocol, ports, scheme, and visibility.

## What EndpointAnnotation Contains

When you call `WithHttpEndpoint()`, `WithHttpsEndpoint()`, or `WithEndpoint()`, Aspire creates an `EndpointAnnotation` with these key properties:

| Property | Description |
|----------|-------------|
| `Name` | Identifier (e.g., `"http"`, `"https"`, `"grpc"`) |
| `Protocol` | TCP or UDP (`ProtocolType.Tcp`) |
| `UriScheme` | URL scheme (`"http"`, `"https"`, `"tcp"`) |
| `Port` | External-facing proxy port (auto-allocated if null) |
| `TargetPort` | Port the app listens on inside the resource |
| `IsExternal` | Whether the endpoint is accessible from outside Aspire |
| `IsProxied` | Whether traffic goes through the Aspire proxy (default: true) |

## How Endpoints Surface

Endpoints appear in the Aspire dashboard with clickable URLs for external endpoints. They're also used by:

- **Service discovery**: `WithReference()` reads endpoint annotations to inject connection info
- **Health checks**: Aspire monitors endpoint availability
- **Replicas**: The proxy load-balances across replica endpoints

## Viewing Endpoints in Code

You can inspect annotations on any resource:

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithHttpEndpoint(port: 5000, name: "public");

// EndpointAnnotation is attached to the resource's Annotations collection
```

## Key Points

- Every `WithHttpEndpoint()` / `WithEndpoint()` call creates an `EndpointAnnotation`
- `Name` identifies the endpoint — resources can have multiple named endpoints
- `Port` is what consumers connect to; `TargetPort` is where the app binds
- `IsProxied = true` (default) routes traffic through Aspire's reverse proxy
- The dashboard renders endpoints from these annotations
