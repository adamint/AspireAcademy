# IResourceBuilder<T> — The Fluent Builder

When you call `builder.AddRedis("cache")`, you do not get back a `RedisResource`. You get an `IResourceBuilder<RedisResource>` — a fluent wrapper that lets you chain configuration calls.

---

## The Pattern

```csharp
IResourceBuilder<RedisResource> cache = builder.AddRedis("cache");

cache.WithLifetime(ContainerLifetime.Persistent)    // configure
     .WithEndpoint("redis", e => e.Port = 6380);    // chain another
```

Every `With*` method returns the same `IResourceBuilder<T>`, enabling fluent chaining.

## Cross-Language Builders

The builder pattern works in every AppHost language:

**C#**
```csharp
builder.AddRedis("cache")
    .WithLifetime(ContainerLifetime.Persistent);
```

**TypeScript**
```typescript
const cache = await builder.addRedis("cache");
await cache.withLifetime(ContainerLifetime.Persistent);
```

**Python**
```python
cache = builder.add_redis("cache")
cache.with_lifetime(ContainerLifetime.PERSISTENT)
```

The API is the same — only the casing convention changes (camelCase, snake_case).

## How Extension Methods Work

The `IResourceBuilder<T>` interface is extended by hosting integration packages. When you install `Aspire.Hosting.Redis`, it adds extension methods like `AddRedis()` to the builder and `WithLifetime()` to `IResourceBuilder<ContainerResource>`.

This is why you need to install the right NuGet package — `AddRedis()` does not exist until `Aspire.Hosting.Redis` is referenced.

## Accessing the Underlying Resource

If you need the raw resource object:

```csharp
IResourceBuilder<RedisResource> cacheBuilder = builder.AddRedis("cache");
RedisResource cacheResource = cacheBuilder.Resource;
string name = cacheResource.Name; // "cache"
```
