# Custom Endpoints with WithEndpoint

Define custom protocol endpoints with full control via callback delegates and named endpoints.

## The Full WithEndpoint API

```csharp
public static IResourceBuilder<T> WithEndpoint<T>(
    this IResourceBuilder<T> builder,
    int? port = null,
    int? targetPort = null,
    string? scheme = null,
    string? name = null,
    string? env = null,
    bool isProxied = true,
    bool? isExternal = null,
    ProtocolType? protocol = null)
```

This is the most flexible endpoint method. `WithHttpEndpoint` and `WithHttpsEndpoint` are convenience wrappers.

## TCP Endpoints

For non-HTTP protocols (databases, message brokers, custom TCP):

```csharp
builder.AddContainer("mydb", "custom-db")
       .WithEndpoint(targetPort: 9042, scheme: "tcp", name: "cql");
```

## Callback Overload

For advanced configuration, use the callback overload to modify an `EndpointAnnotation`:

```csharp
builder.AddContainer("myapp", "myimage")
       .WithEndpoint("api", endpoint =>
       {
           endpoint.Port = 8080;
           endpoint.TargetPort = 80;
           endpoint.UriScheme = "http";
           endpoint.IsExternal = true;
       });
```

## isProxied Parameter

When `isProxied: true` (default), Aspire routes traffic through a reverse proxy. The `port` is the proxy port, and `targetPort` is the actual process port.

When `isProxied: false`, the process must listen on the allocated `port` directly:

```csharp
builder.AddContainer("myapp", "myimage")
       .WithEndpoint(targetPort: 5000, scheme: "http",
                     name: "direct", isProxied: false);
```

## isExternal Parameter

Mark an endpoint as externally accessible (visible outside the Aspire network):

```csharp
builder.AddProject<Projects.Web>("web")
       .WithHttpEndpoint(name: "public")
       .WithEndpoint("public", e => e.IsExternal = true);
```

`WithExternalHttpEndpoints()` is a shortcut that marks all HTTP/HTTPS endpoints as external.

## Multiple Endpoints

Resources can have multiple endpoints:

```csharp
builder.AddContainer("kafka", "confluentinc/cp-kafka")
       .WithEndpoint(targetPort: 9092, scheme: "tcp", name: "broker")
       .WithEndpoint(targetPort: 9093, scheme: "tcp", name: "controller");
```
