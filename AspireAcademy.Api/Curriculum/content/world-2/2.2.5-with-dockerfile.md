# Building Images with WithDockerfile

Instead of pulling pre-built images, `WithDockerfile()` builds a container image from a Dockerfile at startup. This is essential for custom services, sidecar containers, and polyglot apps.

## Basic Usage

```csharp
var sidecar = builder.AddContainer("sidecar", "sidecar-image")
    .WithDockerfile("./src/Sidecar");
```

The argument is the **build context directory** — the directory sent to the Docker daemon. Aspire looks for a `Dockerfile` in that directory by default.

## Specifying a Dockerfile Path

If your Dockerfile isn't named `Dockerfile` or lives in a different location:

```csharp
var sidecar = builder.AddContainer("sidecar", "sidecar-image")
    .WithDockerfile("./src", dockerfilePath: "docker/Sidecar.Dockerfile");
```

## Build Arguments

Pass build-time arguments with `WithBuildArg()`:

```csharp
var sidecar = builder.AddContainer("sidecar", "sidecar-image")
    .WithDockerfile("./src/Sidecar")
    .WithBuildArg("APP_VERSION", "1.0.0")
    .WithBuildArg("BUILD_CONFIG", "Release");
```

Build arguments can also reference Aspire parameters:

```csharp
var version = builder.AddParameter("app-version");
var sidecar = builder.AddContainer("sidecar", "sidecar-image")
    .WithDockerfile("./src/Sidecar")
    .WithBuildArg("APP_VERSION", version);
```

## Multi-Stage Builds

Use the `stage` parameter to target a specific build stage:

```csharp
var sidecar = builder.AddContainer("sidecar", "sidecar-image")
    .WithDockerfile("./src/Sidecar", stage: "runtime");
```

## When to Use WithDockerfile

- **Custom sidecar containers** (log forwarders, proxies)
- **Polyglot services** (Node.js, Python, Go apps with Dockerfiles)
- **Modified base images** (adding certificates, tools)

For standard .NET projects, prefer `AddProject<T>()` — it's faster and doesn't require Docker builds.

## Key Points

- Build context path is relative to the AppHost project
- `WithBuildArg()` passes arguments available in the Dockerfile
- Images are built on first startup and cached by Docker
- Combine with `WithVolume()`, `WithEndpoint()`, and other container extensions
