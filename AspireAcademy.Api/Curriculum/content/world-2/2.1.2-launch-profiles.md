# Launch Profiles and Configuration

Aspire reads `launchSettings.json` from referenced projects to configure environment variables and endpoint URLs. Understanding how launch profiles integrate with Aspire gives you control over how projects start.

## Default Behavior

By default, `AddProject<T>()` uses the **first** launch profile defined in `launchSettings.json`. This determines:

- The `applicationUrl` endpoints (e.g., `http://localhost:5000;https://localhost:5001`)
- Environment variables like `ASPNETCORE_ENVIRONMENT`
- Launch browser settings

## Selecting a Launch Profile

Pass a specific profile name to `AddProject<T>()`:

```csharp
var api = builder.AddProject<Projects.ApiService>("api", launchProfileName: "Development");
```

Or use the `Action<ProjectResourceOptions>` overload for more control:

```csharp
var api = builder.AddProject<Projects.ApiService>("api", options =>
{
    options.LaunchProfileName = "Development";
});
```

## Disabling Launch Profiles

Sometimes you want Aspire to fully control configuration — ignoring `launchSettings.json` entirely. Aspire uses an `ExcludeLaunchProfileAnnotation` for this. You can add it via the options:

```csharp
var api = builder.AddProject<Projects.ApiService>("api", options =>
{
    options.ExcludeLaunchProfile = true;
});
```

When excluded, Aspire won't apply any environment variables or URLs from launch profiles. This is useful when your AppHost should be the single source of truth for endpoint and environment configuration.

## Key Points

- Launch profiles provide default URLs and environment variables
- The profile is selected at `AddProject` time, not later in the chain
- Excluding launch profiles gives Aspire full control over the resource's configuration
- Environment variables from the selected profile are applied at process startup
