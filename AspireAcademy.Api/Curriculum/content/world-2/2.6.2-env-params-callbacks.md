# WithEnvironment — Parameters and Callbacks

Beyond strings and endpoints, `WithEnvironment()` supports parameter resources and callback delegates for dynamic configuration.

## Parameter Resources

Pass an `IResourceBuilder<ParameterResource>` to inject a parameter's resolved value:

```csharp
var apiKey = builder.AddParameter("api-key", secret: true);

builder.AddProject<Projects.ApiService>("api")
    .WithEnvironment("API_KEY", apiKey);
```

The environment variable resolves to the parameter's configured value. If the parameter is secret, the value is masked in the dashboard.

## Callback Overload

The callback overload provides an `EnvironmentCallbackContext` for dynamic, conditional, or bulk assignment:

```csharp
builder.AddProject<Projects.ApiService>("api")
    .WithEnvironment(context =>
    {
        context.EnvironmentVariables["LOG_LEVEL"] = "Debug";
        context.EnvironmentVariables["REGION"] = "us-east-1";

        if (builder.ExecutionContext.IsRunMode)
        {
            context.EnvironmentVariables["ENV"] = "Development";
        }
    });
```

## Async Callback

For values that require async resolution:

```csharp
builder.AddProject<Projects.ApiService>("api")
    .WithEnvironment(async context =>
    {
        context.EnvironmentVariables["COMPUTED"] = "value";
        await Task.CompletedTask;
    });
```

## Execution Context

The callback pattern is especially powerful with `ExecutionContext`:

```csharp
.WithEnvironment(context =>
{
    if (builder.ExecutionContext.IsPublishMode)
    {
        context.EnvironmentVariables["ASPNETCORE_ENVIRONMENT"] = "Production";
    }
    else
    {
        context.EnvironmentVariables["ASPNETCORE_ENVIRONMENT"] = "Development";
    }
});
```

`IsRunMode` is true during `dotnet run`; `IsPublishMode` is true during manifest generation for deployment.

## Key Points

- Parameter overload: `WithEnvironment("KEY", paramBuilder)` for externalized config
- Callback overload: `WithEnvironment(ctx => { ... })` for dynamic/conditional logic
- Async callback available for I/O-bound resolution
- Use `ExecutionContext` to differentiate run mode vs publish mode
