# EndpointReference Properties and Expressions

`EndpointReference` is your handle to a specific endpoint on a resource. It provides properties and expression builders that resolve at runtime when actual ports and hosts are known.

## Getting an EndpointReference

Use `GetEndpoint()` on a resource builder:

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithHttpEndpoint(name: "http");

EndpointReference httpEndpoint = api.GetEndpoint("http");
```

## Runtime Properties

`EndpointReference` exposes properties that resolve when DCP allocates endpoints:

```csharp
httpEndpoint.Url      // "http://localhost:5123"
httpEndpoint.Host     // "localhost"
httpEndpoint.Port     // 5123
httpEndpoint.Scheme   // "http"
```

These properties are only available after allocation. Use them in `WithEnvironment()`:

```csharp
builder.AddProject<Projects.Worker>("worker")
    .WithEnvironment("API_URL", httpEndpoint);
// Injects the full URL at runtime
```

## EndpointProperty Expressions

For composed values, use `Property()` with the `EndpointProperty` enum:

```csharp
var hostExpr = httpEndpoint.Property(EndpointProperty.Host);
var portExpr = httpEndpoint.Property(EndpointProperty.Port);
var urlExpr  = httpEndpoint.Property(EndpointProperty.Url);
```

Available properties: `Url`, `Host`, `IPV4Host`, `Port`, `Scheme`, `TargetPort`, `HostAndPort`, `TlsEnabled`.

## Composing Custom Connection Strings

Combine endpoint properties into `ReferenceExpression` for custom formats:

```csharp
var connStr = ReferenceExpression.Create(
    $"Server={httpEndpoint.Property(EndpointProperty.Host)};Port={httpEndpoint.Property(EndpointProperty.Port)}"
);

builder.AddProject<Projects.Worker>("worker")
    .WithEnvironment("CUSTOM_CONN", connStr);
```

The expression resolves at runtime: `Server=localhost;Port=5123`.

## Multiple Endpoints

Resources can have multiple named endpoints. Always specify the name:

```csharp
var publicEndpoint = api.GetEndpoint("public");
var adminEndpoint = api.GetEndpoint("admin");
```

## Key Points

- `GetEndpoint("name")` returns an `EndpointReference` for a specific named endpoint
- Properties like `Url`, `Host`, `Port` resolve at runtime after port allocation
- `Property(EndpointProperty.X)` returns expression fragments for composition
- Use with `WithEnvironment()` and `ReferenceExpression.Create()` for custom wiring
