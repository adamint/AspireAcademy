# AddJavaScriptApp — Generic JavaScript Apps

Use AddJavaScriptApp(name, appDirectory, runScriptName) for any JS framework with a run script.

## The API

```csharp
public static IResourceBuilder<JavaScriptAppResource> AddJavaScriptApp(
    this IDistributedApplicationBuilder builder,
    string name,
    string appDirectory,
    string runScriptName = "dev")
```

- **`name`** — resource name
- **`appDirectory`** — path to the project directory (must contain `package.json`)
- **`runScriptName`** — the npm script to run (defaults to `"dev"`)

## How It Works

`AddJavaScriptApp` runs the specified npm script from `package.json`. For a project with:

```json
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start"
  }
}
```

Calling `AddJavaScriptApp("frontend", "../next-app")` runs `npm run dev`.

## Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var api = builder.AddProject<Projects.Api>("api");

// Next.js frontend
var frontend = builder.AddJavaScriptApp("frontend", "../next-app")
                      .WithHttpEndpoint(targetPort: 3000, env: "PORT")
                      .WithReference(api);

builder.Build().Run();
```

## Custom Script Name

Override the default `"dev"` script:

```csharp
// Run "npm run serve" instead of "npm run dev"
builder.AddJavaScriptApp("frontend", "../vue-app", runScriptName: "serve");
```

## Passing the API URL

When the frontend needs to call your API, inject the URL as an environment variable:

```csharp
var api = builder.AddProject<Projects.Api>("api");

builder.AddJavaScriptApp("frontend", "../react-app")
       .WithHttpEndpoint(targetPort: 3000, env: "PORT")
       .WithEnvironment("REACT_APP_API_URL", api.GetEndpoint("https"));
```

## Package Manager

By default, `AddJavaScriptApp` uses npm. You can switch to yarn, pnpm, or bun with dedicated extension methods (covered in the next lesson).
