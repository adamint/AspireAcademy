# HTTP and HTTPS Endpoints

`WithHttpEndpoint()` and `WithHttpsEndpoint()` are convenience methods for adding web endpoints to any resource that implements `IResourceWithEndpoints`.

## Adding HTTP Endpoints

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithHttpEndpoint(port: 8080, targetPort: 5000, name: "api");
```

Parameters:
- **`port`**: The external-facing port (what consumers connect to). Auto-allocated if omitted.
- **`targetPort`**: The port your app listens on. Auto-allocated if omitted.
- **`name`**: Endpoint identifier. Defaults to `"http"` for HTTP, `"https"` for HTTPS.
- **`env`**: Optional environment variable name to inject the allocated port.
- **`isProxied`**: Whether to route through Aspire's proxy (default: `true`).

## Adding HTTPS Endpoints

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithHttpsEndpoint(port: 8443, targetPort: 5001, name: "secure");
```

The only difference from `WithHttpEndpoint()` is the URI scheme — `https` instead of `http`.

## Multiple Named Endpoints

A single resource can expose multiple endpoints:

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithHttpEndpoint(targetPort: 5000, name: "public")
    .WithHttpEndpoint(targetPort: 5001, name: "admin")
    .WithHttpsEndpoint(targetPort: 5002, name: "secure");
```

Each endpoint has a distinct name, allowing consumers to reference specific endpoints:

```csharp
var publicUrl = api.GetEndpoint("public");
```

## Auto-Allocation

When you omit both `port` and `targetPort`, Aspire allocates free ports automatically:

```csharp
// Aspire picks available ports for both proxy and target
var api = builder.AddProject<Projects.ApiService>("api")
    .WithHttpEndpoint(name: "api");
```

## Key Points

- `WithHttpEndpoint()` sets scheme to `http`; `WithHttpsEndpoint()` sets scheme to `https`
- Both are convenience wrappers around `WithEndpoint()`
- Named endpoints allow multiple distinct endpoints per resource
- Omitting ports enables auto-allocation, preventing port conflicts
