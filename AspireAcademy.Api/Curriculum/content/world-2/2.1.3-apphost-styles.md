# AppHost Styles Across Languages

There is no single way to write an AppHost. Aspire supports C# (two styles) and TypeScript. This lesson shows every option.

---

## C# Single-File AppHost

The simplest C# style. A single `apphost.cs` file with special directives:

```csharp
#:sdk Aspire.AppHost
#:package Aspire.Hosting.Redis
#:package Aspire.Hosting.PostgreSQL

var builder = DistributedApplication.CreateBuilder(args);
var cache = builder.AddRedis("cache");
var db = builder.AddPostgres("pg").AddDatabase("mydb");
builder.Build().Run();
```

Directives:
- `#:sdk Aspire.AppHost` — uses the Aspire AppHost SDK
- `#:package` — adds a NuGet package
- `#:project ../path/to/project.csproj` — adds a project reference

## C# .csproj AppHost

The traditional style with a `.csproj` file and `Program.cs`:

```xml
<Project Sdk="Aspire.AppHost.Sdk/...">
  <ItemGroup>
    <PackageReference Include="Aspire.Hosting.Redis" />
    <ProjectReference Include="../MyApi/MyApi.csproj" />
  </ItemGroup>
</Project>
```

```csharp
// Program.cs
var builder = DistributedApplication.CreateBuilder(args);
builder.AddProject<Projects.MyApi>("api");
builder.Build().Run();
```

Project references generate `Projects.MyApi` types for type-safe `AddProject<T>()`.

## TypeScript AppHost

```typescript
// apphost.ts
import { createBuilder } from './.modules/aspire.js';

const builder = await createBuilder();
const cache = await builder.addRedis("cache");
const api = await builder.addProject("api", "../MyApi");
await api.withReference(cache);
await builder.build().run();
```

The `.modules/aspire.js` module is **generated** by the Aspire CLI. It provides type-safe bindings to the .NET hosting SDK via the RemoteHost bridge.

> 💡 Aspire can **orchestrate** services written in Python, Go, Java, Rust, and any other language — but the **AppHost itself** must be written in C# or TypeScript.

> 🐛 **Common Gotcha:** Don't confuse the AppHost language with the languages of your services. Your AppHost can be C# while orchestrating a Go API, a Python worker, and a Node.js frontend. The AppHost only *describes* the app — it doesn't constrain what languages the services use.

## Choosing a Style

| Style | Best For |
|---|---|
| C# single-file | Quick prototypes, simple apps |
| C# .csproj | Type-safe project references, CI/CD integration, large .NET solutions |
| TypeScript | Teams who prefer TypeScript, JavaScript-heavy projects |

Every style produces the same app model and uses the same DCP orchestrator. The choice is about developer preference, not capability.
