# WithEnvironment — Static and Dynamic Values

Set environment variables with static strings, ReferenceExpression interpolation, and callback delegates.

## Static Values

The simplest form — a key-value pair:

```csharp
builder.AddProject<Projects.Api>("api")
       .WithEnvironment("ASPNETCORE_ENVIRONMENT", "Development")
       .WithEnvironment("LOG_LEVEL", "Debug");
```

## From Parameters

Inject a parameter resource:

```csharp
var apiKey = builder.AddParameter("api-key", secret: true);

builder.AddProject<Projects.Api>("api")
       .WithEnvironment("API_KEY", apiKey);
```

## From Endpoints

Inject an endpoint reference:

```csharp
var api = builder.AddProject<Projects.Api>("api");

builder.AddViteApp("frontend", "../app")
       .WithEnvironment("API_URL", api.GetEndpoint("https"));
```

## From Connection Strings

Inject a connection string resource:

```csharp
var redis = builder.AddRedis("cache");

builder.AddContainer("worker", "myworker")
       .WithEnvironment("REDIS_URL", redis.Resource);
```

## Callback Delegates

For complex logic, use the callback overload:

```csharp
builder.AddProject<Projects.Api>("api")
       .WithEnvironment(context =>
       {
           context.EnvironmentVariables["COMPUTED_VAR"] = "computed-value";
           context.EnvironmentVariables["TIMESTAMP"] = DateTimeOffset.UtcNow.ToString("o");
       });
```

## Async Callbacks

For values that require async resolution:

```csharp
builder.AddProject<Projects.Api>("api")
       .WithEnvironment(async context =>
       {
           var secret = await GetSecretFromVault();
           context.EnvironmentVariables["VAULT_SECRET"] = secret;
       });
```

## WithEnvironment Overload Summary

| Overload | Use Case |
|----------|----------|
| `(string name, string value)` | Static values |
| `(string name, IResourceBuilder<ParameterResource>)` | Parameters |
| `(string name, EndpointReference)` | Endpoint URLs |
| `(Action<EnvironmentCallbackContext>)` | Computed values |
| `(Func<EnvironmentCallbackContext, Task>)` | Async values |

All environment variables are visible in the Dashboard's resource detail view.
