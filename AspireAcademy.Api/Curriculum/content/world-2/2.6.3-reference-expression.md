# ReferenceExpression and Composite Values

`ReferenceExpression.Create()` builds composite values from multiple dynamic references that resolve at runtime. This is how you construct connection strings and custom URLs from endpoint properties and parameters.

## Why ReferenceExpression?

Individual values like a host, port, or password aren't known until DCP allocates endpoints. `ReferenceExpression` lets you compose them into a single string that resolves when all parts are available.

## Creating Expressions

Use the interpolated string handler overload:

```csharp
var dbEndpoint = postgres.GetEndpoint("tcp");
var dbPassword = builder.AddParameter("db-pass", secret: true);

var connStr = ReferenceExpression.Create(
    $"Server={dbEndpoint.Property(EndpointProperty.Host)};Port={dbEndpoint.Property(EndpointProperty.Port)};Password={dbPassword}"
);
```

At runtime, this resolves to: `Server=localhost;Port=5432;Password=SuperSecret123`

## Using Expressions in WithEnvironment

```csharp
builder.AddProject<Projects.ApiService>("api")
    .WithEnvironment("DATABASE_URL", connStr);
```

## Composing Multiple References

Expressions can combine any number of `IValueProvider` references:

```csharp
var host = builder.AddParameter("db-host");
var port = builder.AddParameter("db-port");
var user = builder.AddParameter("db-user");
var pass = builder.AddParameter("db-pass", secret: true);

var connStr = ReferenceExpression.Create(
    $"Host={host};Port={port};Username={user};Password={pass};Database=myapp"
);
```

Static text (like `"Database=myapp"`) is included literally; parameter and endpoint references resolve dynamically.

## Inline Expressions

You can also use the interpolated string handler directly in `WithEnvironment()`:

```csharp
var endpoint = api.GetEndpoint("http");

builder.AddProject<Projects.Worker>("worker")
    .WithEnvironment("CALLBACK_URL",
        $"{endpoint.Property(EndpointProperty.Scheme)}://{endpoint.Property(EndpointProperty.Host)}:{endpoint.Property(EndpointProperty.Port)}/webhook");
```

## Key Points

- `ReferenceExpression.Create()` composes multiple dynamic values into a single string
- Uses C# interpolated string handlers — looks like normal string interpolation
- All references resolve at runtime when actual values are allocated
- Essential for building connection strings from dynamic hosts, ports, and credentials
- Can mix static text with dynamic references
