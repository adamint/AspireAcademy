# HTTP and HTTPS Endpoints

Add HTTP and HTTPS endpoints with WithHttpEndpoint(port, targetPort, name) and WithEndpoint.

## WithHttpEndpoint

```csharp
public static IResourceBuilder<T> WithHttpEndpoint<T>(
    this IResourceBuilder<T> builder,
    int? port = null,
    int? targetPort = null,
    string? name = null,
    string? env = null,
    bool isProxied = true) where T : IResourceWithEndpoints
```

Example:

```csharp
builder.AddContainer("nginx", "nginx")
       .WithHttpEndpoint(port: 8080, targetPort: 80, name: "web");
```

## WithHttpsEndpoint

Same signature, but forces `scheme: "https"`:

```csharp
builder.AddProject<Projects.Api>("api")
       .WithHttpsEndpoint(port: 5001, targetPort: 443, name: "secure");
```

## Port vs TargetPort

| Parameter | Meaning |
|-----------|---------|
| `port` | The port on the **host** (what external clients connect to) |
| `targetPort` | The port **inside the container/process** (what the app listens on) |

If you omit `port`, Aspire allocates one automatically. If you omit `targetPort`, it defaults to the same as `port`.

## The env Parameter

Inject the allocated port into an environment variable — essential for polyglot apps:

```csharp
builder.AddViteApp("frontend", "../app")
       .WithHttpEndpoint(targetPort: 3000, env: "PORT");
```

The app reads `PORT` to know which port to listen on.

## Endpoint Names

Names default to the scheme (`"http"` or `"https"`). Explicit names are needed when a resource has multiple endpoints of the same scheme:

```csharp
builder.AddContainer("myapp", "myimage")
       .WithHttpEndpoint(targetPort: 8080, name: "api")
       .WithHttpEndpoint(targetPort: 8081, name: "admin");
```

## Getting an Endpoint Reference

Retrieve an endpoint reference for use in other resources:

```csharp
var apiEndpoint = api.GetEndpoint("https");
builder.AddViteApp("frontend", "../app")
       .WithEnvironment("API_URL", apiEndpoint);
```
