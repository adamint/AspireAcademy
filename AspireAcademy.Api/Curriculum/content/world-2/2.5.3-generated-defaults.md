# Generated Defaults and Connection Strings

Aspire supports auto-generating parameter values and registering external connection strings — two patterns for different configuration scenarios.

## Generated Defaults with GenerateParameterDefault

Instead of requiring a configured value, you can auto-generate one:

```csharp
var password = builder.AddParameter("db-password", new GenerateParameterDefault
{
    MinLength = 22,
    Lower = true,
    Upper = true,
    Numeric = true,
    Special = false,
    MinLower = 1,
    MinUpper = 1,
    MinNumeric = 1
}, secret: true);
```

`GenerateParameterDefault` produces a random string matching the specified constraints. Properties include:

- `MinLength`: Minimum character count
- `Lower`, `Upper`, `Numeric`, `Special`: Character class toggles
- `MinLower`, `MinUpper`, `MinNumeric`, `MinSpecial`: Minimum per class

This is commonly used for database passwords that don't need to be predetermined.

## External Connection Strings with AddConnectionString

When infrastructure already exists (managed databases, cloud services), register its connection string directly:

```csharp
var legacyDb = builder.AddConnectionString("legacy-db");
```

This reads from `ConnectionStrings:legacy-db` in configuration:

```json
{
  "ConnectionStrings": {
    "legacy-db": "Server=prod-db.example.com;Database=legacy;User=app;Password=..."
  }
}
```

The returned resource implements `IResourceWithConnectionString`, so other resources can reference it with `WithReference()`:

```csharp
builder.AddProject<Projects.ApiService>("api")
    .WithReference(legacyDb);
// Injects ConnectionStrings__legacy-db
```

## When to Use Each

| Scenario | Approach |
|----------|----------|
| New infrastructure managed by Aspire | `AddParameter()` + `GenerateParameterDefault` |
| Existing external infrastructure | `AddConnectionString()` |
| Values varying per environment | `AddParameter()` |

## Key Points

- `GenerateParameterDefault` auto-generates passwords/tokens with configurable constraints
- `AddConnectionString()` registers externally-managed connection strings from configuration
- Both patterns integrate with `WithReference()` and `WithEnvironment()`
- Generated defaults are useful for local development; external connection strings for production
