# EndpointReference Properties and Expressions

Access endpoint details via EndpointReference.Property(EndpointProperty) to compose URLs.

## Getting an EndpointReference

```csharp
var api = builder.AddProject<Projects.Api>("api");
EndpointReference endpoint = api.GetEndpoint("https");
```

## Using EndpointReference

The most common use is injecting an endpoint URL into another resource:

```csharp
builder.AddViteApp("frontend", "../app")
       .WithEnvironment("API_URL", api.GetEndpoint("https"));
```

This injects the full URL (e.g., `https://localhost:5001`) into the `API_URL` environment variable.

## EndpointReference Properties

You can access individual parts of the endpoint:

```csharp
var ep = api.GetEndpoint("https");

// In WithEnvironment, these resolve at runtime:
builder.AddContainer("proxy", "nginx")
       .WithEnvironment("API_HOST", ep.Property(EndpointProperty.Host))
       .WithEnvironment("API_PORT", ep.Property(EndpointProperty.Port))
       .WithEnvironment("API_SCHEME", ep.Property(EndpointProperty.Scheme))
       .WithEnvironment("API_URL", ep.Property(EndpointProperty.Url));
```

Available properties:

| Property | Example Value |
|----------|--------------|
| `Host` | `localhost` |
| `Port` | `5001` |
| `Scheme` | `https` |
| `Url` | `https://localhost:5001` |
| `TargetPort` | `5001` |
| `IPV4Host` | `127.0.0.1` |

## String Interpolation with ReferenceExpression

You can compose complex URLs using interpolated string handlers:

```csharp
var pg = builder.AddPostgres("pg");
var pgEndpoint = pg.GetEndpoint("tcp");

builder.AddContainer("admin", "adminer")
       .WithEnvironment("DB_URL",
           $"postgresql://{pgEndpoint.Property(EndpointProperty.Host)}:{pgEndpoint.Property(EndpointProperty.Port)}/mydb");
```

## EndpointReference in WithReference

When you call `WithReference(api)`, Aspire injects **all** of the API's endpoints as service discovery environment variables. When you call `WithReference(api.GetEndpoint("https"))`, only that specific endpoint is injected.
