# External Endpoints and Visibility

By default, Aspire endpoints are **internal** — accessible only within the Aspire network. Making endpoints external allows access from your browser or external tools.

## Internal vs External

- **Internal**: Reachable by other resources in the app model, but not from the host machine
- **External**: Reachable from the host machine (your browser, curl, Postman)

## WithExternalHttpEndpoints

The simplest way to make all HTTP/HTTPS endpoints external:

```csharp
var frontend = builder.AddProject<Projects.WebApp>("frontend")
    .WithExternalHttpEndpoints();
```

This marks every HTTP and HTTPS endpoint on the resource as `IsExternal = true`. External endpoints appear as clickable links in the dashboard.

## Fine-Grained Control

For per-endpoint control, use `WithEndpoint()`:

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithHttpEndpoint(name: "public")
    .WithEndpoint("public", e => e.IsExternal = true)
    .WithHttpEndpoint(name: "internal-admin");
// "public" is external, "internal-admin" stays internal
```

Or set `isExternal` directly in the `WithEndpoint()` parameter overload:

```csharp
.WithEndpoint(port: 8080, scheme: "http", name: "public", isExternal: true)
```

## When to Use External Endpoints

- **Web frontends**: Users access these from a browser
- **API services with Swagger**: Developers browse the Swagger UI
- **Debugging**: Direct access for testing with curl or Postman
- **Webhooks**: External services need to reach your endpoint

## When to Keep Endpoints Internal

- **Backend services**: Only called by other services in the model
- **Databases and caches**: Should never be directly exposed
- **Internal APIs**: Admin or health endpoints that should be secured

## Key Points

- Endpoints default to internal — explicit action is needed for external access
- `WithExternalHttpEndpoints()` is the quick way to expose all HTTP endpoints
- Use `WithEndpoint()` callback for per-endpoint visibility control
- External endpoints appear as clickable links in the dashboard
