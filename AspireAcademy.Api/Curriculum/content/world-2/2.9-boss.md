# Boss: Polyglot App

Build a polyglot app with .NET API + Vite frontend + Python worker, all wired with WaitFor and endpoints.

## The Challenge

Build an AppHost that orchestrates four different technologies:

1. **PostgreSQL** — database with persistent volume
2. **.NET API** — references the database
3. **Vite React frontend** — references the API
4. **Python worker** — processes background jobs from the database

## The Solution

```csharp
var builder = DistributedApplication.CreateBuilder(args);

// Infrastructure
var db = builder.AddPostgres("pg")
                .WithDataVolume()
                .WithLifetime(ContainerLifetime.Persistent)
                .AddDatabase("mydb");

// .NET API
var api = builder.AddProject<Projects.Api>("api")
                 .WithReference(db)
                 .WaitFor(db);

// Vite React frontend
builder.AddViteApp("frontend", "../frontend")
       .WithReference(api)
       .WithEnvironment("VITE_API_URL", api.GetEndpoint("https"))
       .WaitFor(api);

// Python background worker
builder.AddPythonApp("worker", "../python-worker", "worker.py")
       .WithPip()
       .WithReference(db)
       .WithEnvironment("DB_CONNECTION", db.Resource.ConnectionStringExpression)
       .WaitFor(db);

builder.Build().Run();
```

## What Each Resource Demonstrates

| Resource | Concepts |
|----------|----------|
| PostgreSQL | `WithDataVolume`, `WithLifetime`, `AddDatabase` |
| .NET API | `AddProject<T>`, `WithReference`, `WaitFor` |
| Vite frontend | `AddViteApp`, `WithEnvironment`, endpoint injection |
| Python worker | `AddPythonApp`, `WithPip`, connection string injection |

## Startup Order

```
PostgreSQL starts
  ├── API waits for DB, then starts
  │     └── Frontend waits for API, then starts
  └── Worker waits for DB, then starts
```

The API and worker start in parallel once the database is healthy. The frontend waits for the API.

## Key Takeaways

- **All four technologies** use the same `WithReference` and `WaitFor` patterns
- **Environment variable injection** works identically across .NET, JavaScript, and Python
- **The Dashboard** shows all resources regardless of language — traces span across them
- **One AppHost** orchestrates everything
