# Custom Endpoints with WithEndpoint

`WithEndpoint()` provides full control over endpoint configuration via a callback delegate. Use it for non-HTTP protocols or when you need fine-grained control.

## Callback Overload

```csharp
var grpc = builder.AddProject<Projects.GrpcService>("grpc")
    .WithEndpoint("grpc", endpoint =>
    {
        endpoint.Port = 5050;
        endpoint.UriScheme = "http";
        endpoint.Transport = "http2";
        endpoint.Protocol = ProtocolType.Tcp;
    });
```

The callback receives an `EndpointAnnotation` instance. If the named endpoint already exists (e.g., from launch profiles), you modify it. If not, `createIfNotExists: true` (default) creates a new one.

## Parameter Overload

For simpler cases, use the parameter-based overload:

```csharp
var tcp = builder.AddContainer("cache", "redis:7")
    .WithEndpoint(
        port: 6379,
        targetPort: 6379,
        scheme: "tcp",
        name: "redis",
        protocol: ProtocolType.Tcp
    );
```

## When to Use WithEndpoint vs WithHttpEndpoint

| Scenario | Method |
|----------|--------|
| Standard HTTP endpoint | `WithHttpEndpoint()` |
| Standard HTTPS endpoint | `WithHttpsEndpoint()` |
| gRPC (HTTP/2) | `WithEndpoint()` with `transport: "http2"` |
| TCP protocol (Redis, databases) | `WithEndpoint()` with `scheme: "tcp"` |
| Modify existing endpoint | `WithEndpoint(name, callback)` |

## Modifying Existing Endpoints

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithEndpoint("https", endpoint =>
    {
        endpoint.IsExternal = true;
        endpoint.Port = 443;
    });
```

This modifies the existing `"https"` endpoint (from launch profiles) instead of creating a new one.

## Key Points

- `WithEndpoint()` is the base method — `WithHttpEndpoint()` and `WithHttpsEndpoint()` wrap it
- The callback gives access to all `EndpointAnnotation` properties
- Use for non-HTTP protocols: gRPC, TCP, UDP
- Set `createIfNotExists: false` to fail if the endpoint doesn't already exist
