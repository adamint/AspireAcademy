# WaitBehavior: WaitOnResourceUnavailable vs StopOnResourceUnavailable

Choose between waiting for recovery or stopping when a dependency becomes unavailable.

## The WaitBehavior Enum

```csharp
builder.AddProject<Projects.Api>("api")
       .WaitFor(db, WaitBehavior.WaitOnResourceUnavailable);
```

## WaitOnResourceUnavailable (Default)

When a dependency goes unhealthy or stops, the dependent resource **waits** for it to recover:

```
DB Running (Healthy) → API Running
DB Crashes           → API continues but DB is Unhealthy
DB Restarts          → DB becomes Healthy again
```

The API keeps running through the dependency outage. This is resilient but means the API may experience errors while the database is down.

## StopOnResourceUnavailable

When a dependency goes unhealthy or stops, the dependent resource **also stops**:

```
DB Running (Healthy) → API Running
DB Crashes           → API Stops
DB Restarts          → API Restarts
```

Use this when the service can't function at all without the dependency:

```csharp
builder.AddProject<Projects.Api>("api")
       .WaitFor(db, WaitBehavior.StopOnResourceUnavailable);
```

## Choosing a Behavior

| Scenario | Recommended Behavior |
|----------|---------------------|
| API with database (can queue/retry) | `WaitOnResourceUnavailable` |
| Worker that processes DB records | `StopOnResourceUnavailable` |
| Frontend with API backend | `WaitOnResourceUnavailable` |
| Tightly coupled service pair | `StopOnResourceUnavailable` |

## Default Behavior

If you don't specify a `WaitBehavior`, Aspire uses `WaitOnResourceUnavailable`. This is the safest default — resources start when dependencies are healthy and continue running even if dependencies become temporarily unavailable.

## Example

```csharp
var db = builder.AddPostgres("pg").AddDatabase("mydb");

// Resilient — keeps running if DB goes down temporarily
var api = builder.AddProject<Projects.Api>("api")
                 .WaitFor(db);  // Default: WaitOnResourceUnavailable

// Strict — stops if DB goes down
var worker = builder.AddProject<Projects.Worker>("worker")
                    .WaitFor(db, WaitBehavior.StopOnResourceUnavailable);
```
