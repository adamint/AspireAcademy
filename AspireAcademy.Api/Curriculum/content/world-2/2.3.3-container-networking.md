# Container Networking and DNS

How Aspire assigns network aliases and DNS resolution for containers.

## The Aspire Container Network

Aspire creates a shared Docker network called `aspire-container-network`. All containers are attached to this network, enabling container-to-container communication by name.

## Network Identifiers

Aspire defines three network contexts:

```csharp
public static class KnownNetworkIdentifiers
{
    public static readonly NetworkIdentifier LocalhostNetwork;         // host machine
    public static readonly NetworkIdentifier PublicInternet;            // external access
    public static readonly NetworkIdentifier DefaultAspireContainerNetwork; // container-to-container
}
```

- **Containers** default to `DefaultAspireContainerNetwork`
- **Projects** (non-containerized) default to `LocalhostNetwork`

## How DNS Resolution Works

When Container A needs to reach Container B, Aspire uses Docker's built-in DNS:

1. Container B is added to the `aspire-container-network`
2. Aspire assigns a DNS alias matching the resource name
3. Container A resolves `container-b-name` to the container's IP

## Custom Aliases

Add DNS aliases with `WithContainerNetworkAlias`:

```csharp
builder.AddContainer("pg", "postgres")
       .WithContainerNetworkAlias("database");
```

Now other containers can reach PostgreSQL using either `pg` or `database` as the hostname.

## Container-to-Project Communication

When a container needs to reach a .NET project (which runs on the host, not in Docker), Aspire uses the `host.docker.internal` hostname or an equivalent mechanism. The endpoint URLs injected by `WithReference` automatically use the correct address for the network context.

## Port vs TargetPort in Context

```csharp
builder.AddContainer("redis", "redis")
       .WithEndpoint(port: 6380, targetPort: 6379, scheme: "tcp", name: "redis");
```

- **`targetPort: 6379`** — the port Redis listens on inside the container
- **`port: 6380`** — the port mapped to the host

Containers on the same network use the `targetPort`. Host processes use the `port`.
