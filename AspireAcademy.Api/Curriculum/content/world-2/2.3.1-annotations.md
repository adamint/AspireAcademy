# Annotations: Behavior Without Inheritance

Instead of creating subclasses for every combination of features, Aspire uses **annotations** — small metadata objects attached to a resource's `Annotations` collection.

---

## The Interface

```csharp
public interface IResourceAnnotation { }
```

A marker interface. Any class implementing it can be attached to any resource. This is the **composition over inheritance** principle.

## How Annotations Are Added

When you call an extension method like `WithHttpEndpoint`, it creates an annotation:

```csharp
// What you write:
builder.AddContainer("nginx", "nginx")
       .WithHttpEndpoint(port: 80);

// What happens internally:
resource.Annotations.Add(new EndpointAnnotation
{
    UriScheme = "http",
    Port = 80,
    Name = "http"
});
```

Every `With*` method follows this pattern — create an annotation, add it to the collection.

## In TypeScript and Python

The same pattern works in polyglot AppHosts. When TypeScript calls `cache.withLifetime(ContainerLifetime.Persistent)`, the RemoteHost bridge adds a `ContainerLifetimeAnnotation` on the .NET side.

## Why Annotations Matter

Annotations make Aspire **extensible without touching base classes**:

1. **DCP reads annotations** to know how to start resources (image, ports, volumes, env vars)
2. **Publishers read annotations** to generate deployment artifacts
3. **Health checks are annotations** — `HealthCheckAnnotation` tells WaitFor what to check
4. **The Dashboard reads annotations** to display resource details

To add a new capability:
1. Define an annotation class implementing `IResourceAnnotation`
2. Add an extension method that attaches it
3. Read it wherever needed (pipeline step, event handler, publisher)

No changes to `Resource`, `ContainerResource`, or any base class.

## Custom Annotations

```csharp
// Define
public class FeatureFlagAnnotation(string configPath) : IResourceAnnotation
{
    public string ConfigPath { get; } = configPath;
}

// Attach
public static IResourceBuilder<T> WithFeatureFlags<T>(
    this IResourceBuilder<T> builder, string configPath)
    where T : IResource
{
    return builder.WithAnnotation(new FeatureFlagAnnotation(configPath));
}

// Use
builder.AddProject<Projects.Api>("api")
    .WithFeatureFlags("./flags.json");

// Read
builder.Eventing.Subscribe<BeforeStartEvent>((evt, ct) =>
{
    foreach (var resource in evt.Model.Resources)
    {
        if (resource.TryGetAnnotationsOfType<FeatureFlagAnnotation>(out var flags))
        {
            Console.WriteLine($"{resource.Name}: flags at {flags.First().ConfigPath}");
        }
    }
    return Task.CompletedTask;
});
```
