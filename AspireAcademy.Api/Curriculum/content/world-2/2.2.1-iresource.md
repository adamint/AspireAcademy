# IResource — Everything Is a Resource

In Aspire, every component of your distributed application is a **resource**. A PostgreSQL database is a resource. A Redis cache is a resource. Your C# API project is a resource. A Python script is a resource. Even a configuration parameter is a resource.

---

## The Interface

```csharp
public interface IResource
{
    string Name { get; }
    ResourceAnnotationCollection Annotations { get; }
}
```

That is it. Two members:
- **Name** — a unique identifier used in the Dashboard, service discovery, and deployment artifacts
- **Annotations** — a collection of metadata objects that describe the resource's behavior

Every resource in Aspire implements this interface. The simplicity is intentional — behavior is attached through **annotations** (composition), not through deep class hierarchies (inheritance).

## Why Everything Is a Resource

By treating everything as a resource, Aspire can:

1. **Display it in the Dashboard** — every resource gets a row in the Resources table
2. **Track its state** — Starting, Running, Exited, FailedToStart
3. **Wire it to other resources** — `WithReference()` works between any two resources
4. **Publish it** — the publishing pipeline knows how to generate artifacts for each resource type
5. **Health-check it** — any resource can have health check annotations

When you call `builder.AddPostgres("pg")`, you are adding a resource named `"pg"` to the app model. When you call `builder.AddViteApp("frontend", "./web")`, you are adding another resource. They are different types, but they share the same `IResource` foundation.

## The Resource Name Contract

Resource names must be unique within an AppHost. The name is used as:
- The key in the Dashboard's resource table
- The hostname for service discovery (`http://api` resolves to the resource named `"api"`)
- The key in connection string environment variables (`ConnectionStrings__cache`)
- The identifier in deployment manifests
