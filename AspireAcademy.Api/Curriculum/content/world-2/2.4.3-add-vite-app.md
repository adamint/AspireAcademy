# AddViteApp — Vite Frontend Resources

Use AddViteApp(name, appDirectory) for Vite-based frontends with dev server integration.

## The API

```csharp
public static IResourceBuilder<ViteAppResource> AddViteApp(
    this IDistributedApplicationBuilder builder,
    string name,
    string appDirectory,
    string runScriptName = "dev")
```

Same parameters as `AddJavaScriptApp`, but `AddViteApp` adds Vite-specific behavior:

- Wires the Vite dev server port automatically
- Configures the HTTP endpoint for the dev server
- Supports HTTPS configuration

## Basic Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var api = builder.AddProject<Projects.Api>("api");

var frontend = builder.AddViteApp("frontend", "../vite-app")
                      .WithReference(api);

builder.Build().Run();
```

## How Vite Port Injection Works

Vite reads its port from the `PORT` environment variable (or Vite config). `AddViteApp` automatically:

1. Allocates a port for the Vite dev server
2. Creates an HTTP endpoint
3. Injects the port into the process environment

Your `vite.config.ts`:

```typescript
export default defineConfig({
  server: {
    port: parseInt(process.env.PORT || '5173'),
  },
});
```

## Passing Backend URLs to Vite

Vite apps often need environment variables prefixed with `VITE_`:

```csharp
var api = builder.AddProject<Projects.Api>("api");

builder.AddViteApp("frontend", "../vite-react-app")
       .WithEnvironment("VITE_API_URL", api.GetEndpoint("https"))
       .WithReference(api);
```

In your React code:

```typescript
const apiUrl = import.meta.env.VITE_API_URL;
```

## AddViteApp vs AddJavaScriptApp

| Feature | AddViteApp | AddJavaScriptApp |
|---------|-----------|-----------------|
| Auto HTTP endpoint | ✅ | ❌ Manual setup |
| Vite-specific config | ✅ | ❌ |
| Any JS framework | ❌ Vite only | ✅ Any |
| Default script | `"dev"` | `"dev"` |

Use `AddViteApp` for Vite-based projects (React + Vite, Vue + Vite, SvelteKit). Use `AddJavaScriptApp` for Next.js, Angular, or other frameworks.
