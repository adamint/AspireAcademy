# Port Allocation and Proxying

Understand automatic port allocation, Port vs TargetPort, and the isProxied parameter.

## Automatic Port Allocation

When you omit the `port` parameter, Aspire picks an available port automatically:

```csharp
// Aspire chooses a random available port for the host
builder.AddContainer("redis", "redis")
       .WithEndpoint(targetPort: 6379, scheme: "tcp", name: "redis");
```

This prevents port conflicts when running multiple services. The allocated port appears in the Dashboard's endpoint URLs.

## Port vs TargetPort Explained

```
Host Machine                 Container/Process
┌──────────────────┐        ┌──────────────────┐
│   port: 15432   ─┼────────┼→  targetPort: 5432│
│  (you connect    │        │  (app listens     │
│   to this)       │        │   on this)        │
└──────────────────┘        └──────────────────┘
```

- **`port`** — the externally visible port on the host
- **`targetPort`** — the port the application process actually listens on

## The Proxy Layer

When `isProxied: true` (default), Aspire runs a reverse proxy:

```
Client → port (proxy) → targetPort (app)
```

Benefits of proxying:
- **Port isolation** — the app doesn't need to know its external port
- **Health checking** — the proxy can check if the app is healthy
- **Automatic port allocation** — no conflicts

## isProxied: false

For cases where the app must bind directly:

```csharp
builder.AddContainer("myapp", "myimage")
       .WithHttpEndpoint(port: 5000, targetPort: 5000, isProxied: false);
```

The app must listen on the exact allocated port. Use this when:
- The protocol doesn't support proxying
- You need direct connections (e.g., WebSocket-heavy apps)
- Performance is critical

## Project Endpoints

.NET projects get their endpoints from `launchSettings.json` automatically. You usually don't need `WithEndpoint` for projects unless adding extra endpoints:

```csharp
// Project already has HTTP/HTTPS from launchSettings
var api = builder.AddProject<Projects.Api>("api");

// Add an additional gRPC endpoint
api.WithEndpoint(targetPort: 5001, scheme: "https", name: "grpc");
```
