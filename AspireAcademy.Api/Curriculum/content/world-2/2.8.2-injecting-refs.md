# Injecting Endpoints and Parameters

Pass EndpointReference and ParameterResource values as environment variables.

## Endpoint Injection

When you call `WithReference(api)` on a resource, Aspire injects **all** of the API's endpoints as service discovery environment variables:

```
services__api__https__0 = https://localhost:5001
services__api__http__0 = http://localhost:5000
```

For more control, inject a specific endpoint:

```csharp
var api = builder.AddProject<Projects.Api>("api");
var httpsEndpoint = api.GetEndpoint("https");

builder.AddContainer("proxy", "nginx")
       .WithEnvironment("BACKEND_URL", httpsEndpoint);
```

Result: `BACKEND_URL=https://localhost:5001`

## Endpoint Property Extraction

Extract individual parts of an endpoint:

```csharp
var ep = api.GetEndpoint("https");

builder.AddContainer("proxy", "nginx")
       .WithEnvironment("BACKEND_HOST", ep.Property(EndpointProperty.Host))
       .WithEnvironment("BACKEND_PORT", ep.Property(EndpointProperty.Port));
```

## Parameter Injection

Parameters are injected the same way:

```csharp
var apiKey = builder.AddParameter("api-key", secret: true);

builder.AddProject<Projects.Api>("api")
       .WithEnvironment("API_KEY", apiKey);
```

The parameter's resolved value becomes the environment variable value.

## Combining References

A single resource can receive endpoints, parameters, and connection strings:

```csharp
var db = builder.AddPostgres("pg").AddDatabase("mydb");
var cache = builder.AddRedis("cache");
var apiKey = builder.AddParameter("api-key", secret: true);
var api = builder.AddProject<Projects.Api>("api");

builder.AddProject<Projects.Worker>("worker")
       .WithReference(db)                               // ConnectionStrings__mydb
       .WithReference(cache)                             // ConnectionStrings__cache
       .WithReference(api)                               // services__api__...
       .WithEnvironment("API_KEY", apiKey)               // API_KEY
       .WithEnvironment("CUSTOM_API_URL",
           api.GetEndpoint("https"));                    // CUSTOM_API_URL
```

## Viewing in the Dashboard

All injected environment variables appear in the Dashboard's resource detail view. Secret values are masked. You can copy variable names and values for debugging.
