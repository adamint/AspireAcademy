# Key Built-In Annotations

Aspire ships with dozens of annotations. Here are the ones you will encounter most often.

---

| Annotation | Added By | Purpose |
|---|---|---|
| `EndpointAnnotation` | `WithEndpoint`, `WithHttpEndpoint` | Declares a network endpoint (port, scheme, name) |
| `ContainerMountAnnotation` | `WithVolume`, `WithBindMount` | Mounts storage into a container |
| `ContainerLifetimeAnnotation` | `WithLifetime` | Controls Session vs Persistent lifetime |
| `HealthCheckAnnotation` | Hosting integrations | Registers health checks (e.g., PostgreSQL ping) |
| `WaitAnnotation` | `WaitFor`, `WaitForCompletion` | Declares startup dependencies |
| `EnvironmentCallbackAnnotation` | `WithEnvironment` | Injects environment variables via callbacks |
| `ConnectionStringRedirectAnnotation` | `WithReference` (databases) | Specifies which resource provides the connection string |
| `ResourceSnapshotAnnotation` | Orchestrator | Tracks current resource state |
| `PipelineStepAnnotation` | Publishing pipeline | Registers pipeline steps for a resource |

## Reading Annotations

You can query annotations by type:

```csharp
var endpoints = resource.Annotations.OfType<EndpointAnnotation>();
var hasHealthCheck = resource.Annotations.OfType<HealthCheckAnnotation>().Any();
```

## Custom Annotations

Creating your own annotation is straightforward:

```csharp
public class MetricsPortAnnotation : IResourceAnnotation
{
    public int Port { get; set; }
}

// Usage
builder.AddProject<Projects.Api>("api")
    .WithAnnotation(new MetricsPortAnnotation { Port = 9090 });
```

Then read it wherever you need — in a lifecycle hook, event handler, or custom pipeline step.
