# Monitoring Commands

These commands give you visibility into running Aspire applications — viewing logs, listing instances, checking environment health, and exporting telemetry.

## aspire logs

Streams or snapshots console logs from a resource.

```bash
aspire logs api                    # snapshot of api logs
aspire logs api --follow           # live tail
aspire logs --tail 50              # last 50 lines
aspire logs api --timestamps       # include timestamps
aspire logs --format json          # JSON output
```

| Option | Description |
|--------|-------------|
| `resource` | Resource name (optional — omit for all resources) |
| `--follow/-f` | Continuously stream new log entries |
| `--tail/-n` | Number of recent lines to show |
| `--timestamps/-t` | Include timestamps in output |
| `--format` | `table` (human-readable) or `json` (NDJSON stream) |
| `--apphost/--project` | AppHost project path |

In follow mode, `LogsCommand` maintains a live connection to the AppHost backchannel and streams console log entries as they arrive. JSON mode outputs newline-delimited JSON for piping to tools like `jq`.

## aspire ps

Lists all running Aspire AppHost instances on the machine.

```bash
aspire ps                      # list running AppHosts
aspire ps --format json        # JSON output with details
aspire ps --resources          # include resource info
```

| Option | Description |
|--------|-------------|
| `--format` | Output format (`table`/`json`) |
| `--resources` | Include resource payloads in JSON output |

`PsCommand` discovers all running AppHost processes by scanning for active backchannel connections. The table view shows AppHost name, PID, project path, and status. JSON mode can include full resource state when `--resources` is specified.

## aspire doctor

Validates your development environment for Aspire readiness.

```bash
aspire doctor                  # human-readable report
aspire doctor --format json    # machine-readable
```

| Option | Description |
|--------|-------------|
| `--format` | Output format (`table`/`json`) |

`DoctorCommand` runs environment checks through `IEnvironmentChecker`, validating:
- .NET SDK version and installation
- Container runtime availability (Docker, Podman)
- Required workloads and tools
- Network connectivity

Results are grouped by category with pass/warning/fail indicators. JSON output provides structured results for CI/CD pipelines.

## aspire config & aspire secret

`config` manages CLI-level configuration:

```bash
aspire config list                     # show all settings
aspire config get telemetry.enabled    # get a value
aspire config set telemetry.enabled false
aspire config delete some.key
aspire config info                     # show config file paths
```

`secret` manages AppHost user secrets:

```bash
aspire secret list                     # list all secrets
aspire secret get ConnectionStrings:db # get a secret value
aspire secret set ApiKey my-key        # set a secret
aspire secret delete ApiKey            # remove a secret
aspire secret path                     # show secrets file path
```

Both use `--apphost/--project` to target specific AppHost projects. Secrets are stored using .NET's User Secrets infrastructure, scoped to the AppHost project's `UserSecretsId`.

## aspire certificates

Manages development HTTPS certificates:

```bash
aspire certificates trust    # trust the dev certificate
aspire certificates clean    # remove all dev certificates
```
