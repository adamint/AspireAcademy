# Configuration Overlays and Test Fixtures

Share test infrastructure across test classes and override configuration for test scenarios.

---

## Shared Fixtures

xUnit's `ICollectionFixture` lets multiple test classes share one AppHost:

```csharp
[CollectionDefinition("Atlas")]
public class AtlasCollection : ICollectionFixture<AtlasFixture> { }

public class AtlasFixture : IAsyncLifetime
{
    public DistributedApplication App { get; private set; } = null!;

    public async Task InitializeAsync()
    {
        var builder = await DistributedApplicationTestingBuilder
            .CreateAsync<Projects.AtlasAppHost>();
        App = await builder.BuildAsync();
        await App.StartAsync();
    }

    public async Task DisposeAsync() => await App.DisposeAsync();
}

[Collection("Atlas")]
public class SearchTests(AtlasFixture fixture)
{
    [Fact]
    public async Task SearchWorks()
    {
        var client = fixture.App.CreateHttpClient("api");
        // ...
    }
}
```

All test classes in the `"Atlas"` collection share one running AppHost.

## Configuration Overlays

Override `appsettings.json` values for tests:

```csharp
var builder = await DistributedApplicationTestingBuilder
    .CreateAsync<Projects.AtlasAppHost>();

builder.Configuration["Parameters:api-key"] = "test-api-key-12345";
```

## Parallel Test Execution

By default, xUnit runs test classes in parallel. Since each shared fixture gets its own AppHost:
- Tests within a collection run sequentially (shared state)
- Tests across collections run in parallel (separate AppHosts)

Configure with `xunit.runner.json`:
```json
{
    "parallelizeTestCollections": true,
    "maxParallelThreads": 4
}
```
