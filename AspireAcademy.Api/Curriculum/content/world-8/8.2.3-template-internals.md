# Template Internals

Understanding how templates scaffold projects helps you troubleshoot creation issues and extend the template system. The two runtimes — DotNet and Cli — use fundamentally different approaches.

## DotNet Template Path

When you run `aspire new aspire-starter`:

1. **Version resolution** — `ResolveCliTemplateVersionAsync` checks:
   - Explicit `--channel` or `--version` flags
   - Global config channel setting
   - Implicit channel detection (checking for PR "hive" packages)

2. **Template installation** — Installs the appropriate Aspire project templates for the selected channel

3. **Project generation** — Scaffolds the project using the selected template with the given name and options

4. **Post-generation** — Trusts dev HTTPS certificates, optionally creates/updates `NuGet.config` for custom feeds

5. **Agent chain** — Offers `AgentInitCommand.PromptAndChainAsync()` to set up AI agent integration

## CLI Template Path

CLI-native templates use embedded resource trees. When you run `aspire new aspire-ts-starter`:

1. **Embedded files** — The template payload lives under `src/Aspire.Cli/Templating/Templates/`:
   - `ts-starter/` — TypeScript starter files
   - `java-starter/` — Java starter files
   - `empty-apphost/` — C# empty AppHost files

2. **Token replacement** — Files are processed with placeholder substitution:
   | Token | Replacement |
   |-------|-------------|
   | `{{projectName}}` | The `--name` value |
   | `{{projectNameLower}}` | Lowercased project name |
   | `{{aspireVersion}}` | Resolved Aspire package version |
   | `{{hostName}}` | Generated hostname |
   | Generated port tokens | Random available ports |

3. **Channel configuration** — Writes `aspire.config.json` with the resolved channel and version

4. **Localhost TLD** — When `--localhost-tld` is set, rewrites `localhost` references to `*.dev.localhost`

5. **Package restore** — For TypeScript, runs the language-specific package restore after scaffolding

## TemplateNuGetConfigService

Both paths may need NuGet feed configuration. `TemplateNuGetConfigService.PromptToCreateOrUpdateNuGetConfigAsync()` checks if the project needs a `NuGet.config` for accessing preview or internal package feeds. It prompts the user and creates or updates the config file with the appropriate feed URL for the selected channel.

## How Templates Register as Commands

`NewCommand` calls `ITemplateProvider.GetTemplates()` during initialization and creates a subcommand for each template. Each template's `ApplyOptions()` method adds template-specific options to its subcommand. This is why `aspire new --help` shows all available templates as subcommands.

## Error Handling

Template failures surface as non-zero `TemplateResult.ExitCode` values. Common issues:
- SDK not found (DotNet templates)
- Runtime not available (Node.js/Java for CLI templates)
- Output directory already exists
- Network issues during package resolution

The CLI provides helpful error messages and suggests fixes (like installing the required SDK or runtime).
