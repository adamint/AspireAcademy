# Testing Pipeline Steps and Custom Resources

How to test your own pipeline steps, custom resources, and hosting integration packages.

---

## Testing Custom Resources

```csharp
[Fact]
public async Task CustomResourceRegistersCorrectly()
{
    var builder = await DistributedApplicationTestingBuilder.CreateAsync<TestAppHost>();
    var app = await builder.BuildAsync();

    var model = app.Services.GetRequiredService<DistributedApplicationModel>();
    var resource = model.Resources.OfType<MeilisearchResource>().Single();

    resource.Name.Should().Be("search");
    resource.Annotations.OfType<EndpointAnnotation>().Should().HaveCount(1);
    resource.Annotations.OfType<HealthCheckAnnotation>().Should().HaveCount(1);
}
```

## Testing Pipeline Steps

```csharp
[Fact]
public async Task CustomPipelineStepGeneratesArtifacts()
{
    var builder = await DistributedApplicationTestingBuilder.CreateAsync<TestAppHost>(args =>
    {
        args.Args = ["--operation", "publish", "--output-path", outputDir];
    });

    var app = await builder.BuildAsync();
    await app.StartAsync();

    // Verify artifacts were generated
    File.Exists(Path.Combine(outputDir, "custom-output.json")).Should().BeTrue();
}
```

## Testing Health Checks

```csharp
[Fact]
public async Task MeilisearchHealthCheckPasses()
{
    var builder = await DistributedApplicationTestingBuilder.CreateAsync<TestAppHost>();
    var app = await builder.BuildAsync();
    await app.StartAsync();

    var rns = app.Services.GetRequiredService<ResourceNotificationService>();
    var state = await rns.WaitForResourceAsync("search", KnownResourceStates.Running);

    state.Should().Be(KnownResourceStates.Running);
}
```

## The Aspire Integration Test Pattern

Every integration in the Aspire repo follows this pattern:

1. Create a test AppHost that uses the integration
2. Build and start it with `DistributedApplicationTestingBuilder`
3. Verify the resource model (annotations, connection strings)
4. Verify runtime behavior (health checks, HTTP responses)
5. Clean up in `DisposeAsync`

This is the same pattern you follow when testing your own integrations.
