# Your First Integration Test

Walk through writing a complete integration test for an Atlas API endpoint.

---

## The Test

```csharp
public class SearchEndpointTests : IAsyncLifetime
{
    private DistributedApplication _app = null!;

    public async Task InitializeAsync()
    {
        var builder = await DistributedApplicationTestingBuilder
            .CreateAsync<Projects.AtlasAppHost>();

        _app = await builder.BuildAsync();

        // Wait for resources to be ready
        var rns = _app.Services.GetRequiredService<ResourceNotificationService>();
        await rns.WaitForResourceAsync("api", KnownResourceStates.Running);
        await _app.StartAsync();
    }

    public async Task DisposeAsync() => await _app.DisposeAsync();

    [Fact]
    public async Task SearchReturnsResults()
    {
        // Arrange
        var client = _app.CreateHttpClient("api");

        // Act
        var response = await client.GetAsync("/api/search?q=aspire");

        // Assert
        response.StatusCode.Should().Be(HttpStatusCode.OK);
        var results = await response.Content.ReadFromJsonAsync<SearchResult[]>();
        results.Should().NotBeEmpty();
    }

    [Fact]
    public async Task SearchWithEmptyQueryReturns400()
    {
        var client = _app.CreateHttpClient("api");
        var response = await client.GetAsync("/api/search?q=");
        response.StatusCode.Should().Be(HttpStatusCode.BadRequest);
    }
}
```

## Key Patterns

1. **`IAsyncLifetime`** — xUnit interface for async setup/teardown
2. **`WaitForResourceAsync`** — ensures resources are ready before tests run
3. **`CreateHttpClient("name")`** — gets a client for the named resource
4. **Real responses** — you are testing against real services with real databases

## Running Tests

```bash
dotnet test
```

Tests start DCP, pull container images (first run only), start services, run assertions, and tear everything down.
