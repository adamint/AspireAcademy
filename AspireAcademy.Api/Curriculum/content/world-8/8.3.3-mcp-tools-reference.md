# MCP Tools Reference

The Aspire MCP server exposes 14 built-in tools that AI agents can call. These tools map directly to functionality in the Aspire CLI and Dashboard.

## Resource Management Tools

### list_resources
Lists all application resources with their type, state, endpoints, health status, and relationships.
- **Parameters:** none
- **Returns:** Resource array with name, type, state, endpoints, health

### execute_resource_command
Executes a command on a specific resource (start, stop, restart, or custom).
- **Parameters:** `resourceName` (required), `commandName` (required)
- **Returns:** Command execution result

### list_console_logs
Retrieves console output (stdout/stderr) for a resource. Includes logs from resource commands like start/stop/restart.
- **Parameters:** `resourceName` (required)
- **Returns:** Log entries array

## Observability Tools

### list_structured_logs
Lists structured log entries for one or all resources. These are application-level logs (not console output).
- **Parameters:** `resourceName` (optional — omit for all resources)
- **Returns:** Structured log entries with severity, message, attributes

### list_traces
Lists distributed traces across resources. Each trace tracks an operation spanning multiple services.
- **Parameters:** `resourceName` (optional — omit for all resources)
- **Returns:** Trace summaries with ID, involved resources, duration, error status

### list_trace_structured_logs
Gets structured logs for a specific distributed trace, organized by span ID.
- **Parameters:** `traceId` (required)
- **Returns:** Logs belonging to the trace, grouped by span

## AppHost Management Tools

### list_apphosts
Lists all AppHost connections detected by the MCP server, showing which are in scope and which are outside the working directory.
- **Parameters:** none
- **Returns:** AppHost connection info array

### select_apphost
Selects which AppHost to use when multiple are running. Accepts fully qualified or workspace-relative paths.
- **Parameters:** `appHostPath` (required)
- **Returns:** Confirmation of selected AppHost

## Integration & Environment Tools

### list_integrations
Lists all available Aspire hosting integrations (NuGet packages). Does not require a running AppHost.
- **Parameters:** none
- **Returns:** Integration list with names and package IDs

### doctor
Runs comprehensive environment diagnostic checks. Returns status (pass/warning/fail), messages, and actionable fix suggestions.
- **Parameters:** none
- **Returns:** Diagnostic check results

### refresh_tools
Triggers the MCP server to re-scan for available tools (including dynamic resource-exposed tools) and emits a `tool_list_changed` notification.
- **Parameters:** none
- **Returns:** Confirmation

## Documentation Tools

### list_docs
Lists all available Aspire documentation pages from aspire.dev with titles, slugs, and summaries.
- **Parameters:** none
- **Returns:** Documentation page catalog

### search_docs
Searches aspire.dev documentation using keyword-based lexical search across titles, headings, code identifiers, and body text.
- **Parameters:** `query` (required), `topK` (optional, default 5, max 10)
- **Returns:** Ranked search results with relevance scores

### get_doc
Retrieves the full content of a documentation page by slug, optionally filtered to a specific section.
- **Parameters:** `slug` (required), `section` (optional)
- **Returns:** Full page content or specific section
