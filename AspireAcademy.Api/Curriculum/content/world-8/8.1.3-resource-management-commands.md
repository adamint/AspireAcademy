# Resource Management Commands

These commands let you interact with resources in a running Aspire application — adding integrations, executing commands, inspecting state, and waiting for readiness.

## aspire add

Adds an Aspire integration package to your AppHost project.

```bash
aspire add redis
aspire add postgresql --version 9.3.0
```

| Option | Description |
|--------|-------------|
| `integration` | Integration name or package ID (argument) |
| `--apphost/--project` | AppHost project path |
| `--version` | Specific package version |
| `--source/-s` | NuGet package source |

`AddCommand` fuzzy-matches your input against all available integration packages by friendly name and package ID. It prompts for version selection, stops any running AppHost (since package changes require rebuild), then runs `dotnet add package`. Community Toolkit packages are surfaced alongside first-party integrations.

## aspire exec

Executes a command inside a running resource's context.

```bash
aspire exec --resource cache -- redis-cli PING
aspire exec -r pg -- psql -U postgres
```

| Option | Description |
|--------|-------------|
| `--resource/-r` | Target resource name |
| `--start-resource/-s` | Start the resource if not running |
| `--workdir/-w` | Working directory |
| `--` | Delimiter before the command |

`ExecCommand` streams the command through the AppHost backchannel. Everything after `--` is passed as the command and its arguments.

## aspire describe (alias: resources)

Shows detailed information about resources in a running AppHost.

```bash
aspire describe              # all resources
aspire describe api          # specific resource
aspire describe --follow     # watch mode
aspire describe --format json
```

| Option | Description |
|--------|-------------|
| `resource` | Optional resource name (argument) |
| `--follow/-f` | Continuously watch for state changes |
| `--format` | Output format (`table`/`json`) |
| `--apphost/--project` | AppHost project path |

In table mode, it displays resource name, type, state, endpoints, and health status. In JSON mode, it outputs the full resource model. Watch mode (`--follow`) streams real-time updates as resources start, stop, or change state.

## aspire resource

Executes lifecycle commands on a specific resource.

```bash
aspire resource cache start
aspire resource api restart
aspire resource worker stop
```

| Argument | Description |
|----------|-------------|
| `resource` | Resource name |
| `command` | Command to execute (`start`/`stop`/`restart` or custom) |

`ResourceCommand` maps friendly labels to resource commands and executes them through the backchannel. Custom commands beyond `start`/`stop`/`restart` are passed through generically.

## aspire wait

Blocks until a resource reaches a specified state.

```bash
aspire wait api                           # waits for "healthy" (default)
aspire wait pg --status running           # waits for "running"
aspire wait cache --timeout 60            # 60-second timeout
```

| Option | Description |
|--------|-------------|
| `resource` | Resource name (argument) |
| `--status` | Target state (default: `healthy`) |
| `--timeout` | Timeout in seconds (default: `120`) |
| `--apphost/--project` | AppHost project path |
