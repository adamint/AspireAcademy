# CLI Architecture

The Aspire CLI (`aspire`) is a native AOT-compiled tool that manages the full lifecycle of Aspire applications — from scaffolding to deployment. Every command inherits from `BaseCommand` and is organized into help groups for discoverability.

## BaseCommand

All CLI commands extend `BaseCommand`, which provides:

- **Global option handling** — `--debug`, `--log-level`, `--non-interactive`, `--nologo`, `--banner`
- **JSON output routing** — when `--format json` is used, output switches to structured JSON
- **Update notifications** — checks for newer CLI versions on supported commands
- **Telemetry** — reports command usage via `AspireCliTelemetry`

```csharp
public abstract class BaseCommand : Command
{
    protected abstract Task<int> ExecuteAsync(
        ParseResult parseResult,
        CancellationToken cancellationToken);
}
```

## Help Groups

Commands are organized into six `HelpGroup` categories displayed in `aspire --help`:

| Group | Commands |
|-------|----------|
| **AppCommands** | `new`, `init`, `run`, `start`, `stop`, `add`, `update`, `restore`, `ps` |
| **ResourceManagement** | `resource`, `wait` |
| **Monitoring** | `describe`, `logs`, `otel`, `export` |
| **Deployment** | `publish`, `deploy`, `do` |
| **ToolsAndConfiguration** | `config`, `secret`, `certs`, `cache`, `docs`, `doctor`, `agent`, `mcp` |

## RootCommand

`RootCommand` is the top-level entry point that registers all subcommands and defines global options:

- `--debug` — enables debug-level logging
- `--log-level/-l` — sets log verbosity (`Trace`, `Debug`, `Information`, `Warning`, `Error`)
- `--non-interactive` — disables interactive prompts
- `--nologo` / `--banner` — controls startup banner display
- `--wait-for-debugger` — pauses until a debugger attaches

`GetChildProcessArgs(ParseResult)` forwards selected globals to detached child processes, ensuring consistency when `aspire run --detach` spawns a background instance.

## AppHostLauncher

Many commands need to find, build, and run an AppHost project. `AppHostLauncher` provides shared infrastructure:

- **`--apphost/--project`** — specify the AppHost project path
- **`--format`** — `table` or `json` output
- **`--isolated`** — run without connecting to a backchannel
- **`LaunchDetachedAsync()`** — spawns the AppHost as a background process

Both `RunCommand` and `StartCommand` use `AppHostLauncher` to locate and launch AppHost projects.

## OutputFormat

Commands that display tabular data support two output formats via the `OutputFormat` enum:
- `Table` — human-readable formatted tables (default)
- `Json` — machine-readable JSON output for scripting and automation
