# Dev Tunnels

Dev Tunnels let you expose your local Aspire endpoints to the public internet — essential for testing webhooks, sharing work-in-progress, or connecting to services that require a public URL. The `Aspire.Hosting.DevTunnels` package wraps the `devtunnel` CLI into a first-class Aspire resource.

## Adding a Dev Tunnel

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var tunnel = builder.AddDevTunnel("tunnel");

var api = builder.AddProject<Projects.Api>("api")
    .WithHttpsEndpoint(port: 5001, name: "https");

// Expose the API's HTTPS endpoint through the tunnel
tunnel.WithReference(api);

// Use the tunnel URL in other services
builder.AddProject<Projects.Frontend>("frontend")
    .WithReference(api, tunnel);
```

## DevTunnelResource

`AddDevTunnel(name, tunnelId?, options?)` creates a `DevTunnelResource` — an executable resource that runs the `devtunnel host` CLI command. If no `tunnelId` is provided, one is derived from the resource name and AppHost identity hash, ensuring stable tunnel IDs across restarts.

## DevTunnelOptions

Configure the tunnel with `DevTunnelOptions`:

| Property | Default | Purpose |
|----------|---------|---------|
| `Description` | `null` | Human-readable tunnel description |
| `AllowAnonymous` | `false` | Allow unauthenticated access |
| `Labels` | `null` | Metadata labels for the tunnel |

Per-port options (`DevTunnelPortOptions`) allow fine-grained control:

| Property | Purpose |
|----------|---------|
| `Description` | Port-specific description |
| `AllowAnonymous` | Override tunnel-level anonymous access |
| `Protocol` | Protocol hint (`http`, `https`) |
| `Labels` | Port-specific labels |

## Exposing Endpoints

Several `WithReference` overloads control what gets exposed:

```csharp
// Expose all endpoints on a resource
tunnel.WithReference(api);

// Expose a specific endpoint
tunnel.WithReference(api.GetEndpoint("https"));

// Expose with anonymous access
tunnel.WithReference(api.GetEndpoint("https"), allowAnonymous: true);

// Use tunnel URL in another resource's service discovery
builder.AddProject<Projects.Frontend>("frontend")
    .WithReference(api, tunnel);
```

## How It Works

When the AppHost starts:

1. **Login** — `DevTunnelLoginManager` ensures you're authenticated with the `devtunnel` CLI
2. **Tunnel creation** — Creates (or reuses) a tunnel with the derived/specified ID
3. **Port forwarding** — Creates tunnel ports for each exposed endpoint
4. **URL rewriting** — Replaces localhost URLs with public tunnel URLs in service discovery
5. **Health checks** — `DevTunnelHealthCheck` and `DevTunnelPortHealthCheck` monitor tunnel availability

The tunnel endpoint is always HTTPS on port 443, regardless of the local endpoint's port. Service discovery environment variables are updated with the tunnel URL in the standard Aspire format: `services__{ResourceName}__{Scheme}__{Index}`.

## Anonymous Access

By default, tunnels require Microsoft authentication. Enable anonymous access for public-facing scenarios:

```csharp
tunnel.WithAnonymousAccess(); // tunnel-wide
// or per-port:
tunnel.WithReference(api.GetEndpoint("https"), allowAnonymous: true);
```
