# Devcontainers and Codespaces

Aspire automatically detects when your AppHost is running inside a **devcontainer**, **GitHub Codespace**, or **SSH remote** session, and adjusts endpoint URLs and port forwarding to work seamlessly in these environments. This support is built into `DistributedApplicationBuilder` — no configuration required.

## Automatic Detection

During startup, Aspire registers several internal services that detect the environment:

| Service | Environment | Detection |
|---------|-------------|-----------|
| `ConfigureCodespacesOptions` | GitHub Codespaces | Checks `CODESPACES` environment variable |
| `ConfigureDevcontainersOptions` | Local devcontainers | Checks `REMOTE_CONTAINERS` environment variable |
| `ConfigureSshRemoteOptions` | SSH Remote (VS Code) | Checks `SSH_CONNECTION` environment variable |

## Codespaces URL Rewriting

In GitHub Codespaces, `localhost` URLs don't work because the environment runs in a cloud VM. `CodespacesResourceUrlRewriterService` rewrites all non-internal HTTP/HTTPS endpoint URLs to the Codespaces port-forwarding format:

```
https://{codespaceName}-{port}.{portForwardingDomain}{path}{query}
```

For example, if your API runs on `http://localhost:5000`, the Codespace URL becomes:
```
https://my-codespace-5000.app.github.dev
```

This rewriting applies to all resource endpoints exposed through the Dashboard and service discovery.

## Port Forwarding

`DevcontainerPortForwardingLifecycleHook` listens for `ResourceEndpointsAllocatedEvent` and automatically configures VS Code port forwarding settings when running in any remote environment.

It writes to VS Code's machine settings file:

| Environment | Settings Path |
|-------------|--------------|
| Codespaces | `~/.vscode-remote/data/Machine/settings.json` |
| Devcontainer | `~/.vscode-server/data/Machine/settings.json` |
| SSH Remote | `~/.vscode-server/data/Machine/settings.json` |

### Settings Written

For each endpoint, `DevcontainerSettingsWriter` adds a `remote.portsAttributes` entry:

```json
{
  "remote.portsAttributes": {
    "5000": {
      "label": "api (http)",
      "protocol": "https",
      "onAutoForward": "notify"
    },
    "5432": {
      "label": "pg (tcp)",
      "protocol": "https",
      "onAutoForward": "silent"
    }
  }
}
```

Port labels are derived from the resource name and endpoint scheme. Entries are deduplicated by label to avoid conflicts when multiple resources share ports across restarts.

## Limitations

- Only HTTP and HTTPS endpoints are forwarded in Codespaces — TCP-only endpoints (like raw database connections) require alternative access methods
- The port forwarding settings are written once when endpoints are allocated and may need a VS Code reload to take effect
- URL rewriting only affects non-internal endpoints — internal service-to-service communication within the Codespace uses standard localhost

## Why It Matters

Devcontainer and Codespace support means you can run your full Aspire application in a cloud development environment with zero configuration. The Dashboard, service endpoints, and all resource URLs just work — whether you're on your laptop, in a Codespace, or connected via SSH. This makes Aspire ideal for team onboarding and standardized development environments.
