# Resource Substitution

Replace production resources with test doubles to speed up tests or isolate behavior.

---

## Why Substitute?

- **Speed** — in-memory databases start instantly; PostgreSQL containers take seconds
- **Isolation** — test a service without its real dependencies
- **Determinism** — mock services return predictable responses

## Substituting via DI

The testing builder exposes the same `Services` collection as your AppHost. Register test doubles before calling `BuildAsync()`:

```csharp
var builder = await DistributedApplicationTestingBuilder
    .CreateAsync<Projects.BazaarAppHost>();

// Replace an external dependency with a test double
builder.Services.AddSingleton<IPaymentGateway, FakePaymentGateway>();

// Add resilience handlers to HTTP clients used in tests
builder.Services.ConfigureHttpClientDefaults(http =>
{
    http.AddStandardResilienceHandler();
});
```

## Manifest-Only Testing

Test your app model configuration without starting any containers:

```csharp
var builder = await DistributedApplicationTestingBuilder
    .CreateAsync<Projects.BazaarAppHost>(args =>
    {
        args.Args = ["--publisher", "manifest"];
    });

var app = await builder.BuildAsync();
var model = app.Services.GetRequiredService<DistributedApplicationModel>();

// Assert resource configuration without container overhead
model.Resources.OfType<PostgresServerResource>().Should().HaveCount(1);
```

This runs the AppHost in manifest publishing mode — it builds the full resource model without starting DCP or any containers. Useful for fast assertions on resource configuration, annotations, and connection string shapes.

## Fake Services

For the Python ML service in Bazaar tests, you might not want to start the real ML model:

```csharp
// In your test AppHost, replace the Python service with a mock HTTP server
builder.Services.AddHostedService<FakeMlService>();
```

## Best Practices

1. Test with real dependencies when possible (higher confidence)
2. Substitute only when real dependencies are too slow or non-deterministic
3. Use fake services for external APIs you do not control
4. Use persistent containers with `ContainerLifetime.Persistent` for faster test iterations

> 🌍 **Real World:** Major Aspire users typically run real containers for all first-party infrastructure (databases, caches, queues) and fake only external SaaS APIs. The `ContainerLifetime.Persistent` option keeps containers alive across test runs, cutting iteration time from minutes to seconds.
