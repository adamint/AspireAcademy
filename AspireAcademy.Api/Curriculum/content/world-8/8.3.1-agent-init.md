# Agent Initialization

The `aspire agent init` command sets up AI agent integration for your Aspire workspace — configuring skills, MCP servers, and agent-specific settings so tools like GitHub Copilot and other AI coding agents can interact with your Aspire application.

## Running Agent Init

```bash
aspire agent init
```

The command is also offered automatically after `aspire new` and `aspire init` complete successfully, via `AgentInitCommand.PromptAndChainAsync()`.

## Initialization Flow

1. **Workspace Discovery** — Finds the workspace root by looking for a git repository root, falling back to the current directory

2. **Agent Environment Detection** — Scans for supported AI agent environments using `IAgentEnvironmentDetector`. This detects:
   - GitHub Copilot (VS Code)
   - GitHub Copilot CLI
   - Other MCP-compatible agent environments

3. **Config Migrations** — Silently applies any configuration schema migrations to existing agent configs (e.g., updating MCP server definitions from old formats)

4. **Skill Configuration** — Prompts for:
   - Skill file locations (where to write agent instruction files)
   - Optional skills to install (project-specific skills, MCP setup)
   - Whether to install Playwright CLI (for E2E testing skills)

5. **File Generation** — Writes:
   - Agent instruction files (AGENTS.md, skill files)
   - MCP configuration in the appropriate format for each detected environment
   - `.github/skills/` directory with specialized skill definitions

## What Gets Configured

Agent init creates several types of configuration:

### MCP Server Configuration
Registers the Aspire MCP server so AI agents can:
- List and inspect running resources
- Execute resource commands
- View console and structured logs
- Browse distributed traces
- Search Aspire documentation

### Skill Files
Project-specific instruction files that teach the AI agent about:
- How to build and test the project
- Aspire-specific conventions and patterns
- Available CLI commands and workflows

## Non-Interactive Mode

When `--non-interactive` is set (or when called from CI), agent init uses defaults without prompting. This is useful for automated project setup in CI/CD pipelines.

## Integration with New/Init

After `aspire new` creates a project or `aspire init` adds Aspire to an existing project, both commands call `PromptAndChainAsync()`:

```
✓ Project created successfully!
? Would you like to set up AI agent integration? [Y/n]
```

This seamless chaining ensures new projects are agent-ready from the start.
