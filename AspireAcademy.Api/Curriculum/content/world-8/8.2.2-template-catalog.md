# Template Catalog

The Aspire CLI ships with templates for multiple languages and project types. Each template has a unique ID defined in `KnownTemplateId` and is available through `aspire new <template-id>`.

## Known Template IDs

| Template ID | Language | Description | Runtime |
|-------------|----------|-------------|---------|
| `aspire-starter` | C# | Full starter app with AppHost, ServiceDefaults, API, and frontend | DotNet |
| `aspire` | C# | Standard AppHost + ServiceDefaults | DotNet |
| `aspire-empty` | C# | Empty AppHost (minimal) | Cli |
| `aspire-apphost` | C# | AppHost project only | DotNet |
| `aspire-servicedefaults` | C# | ServiceDefaults project only | DotNet |
| `aspire-ts-cs-starter` | C#/TS | Hybrid starter with TypeScript frontend | DotNet |
| `aspire-ts-starter` | TypeScript | Full TypeScript starter app | Cli |
| `aspire-ts-empty` | TypeScript | Empty TypeScript AppHost | Cli |
| `aspire-py-starter` | Python | Python starter app | DotNet |
| `aspire-java-empty` | Java | Empty Java AppHost | Cli |
| `aspire-test` | C# | Test project (generic) | DotNet |
| `aspire-mstest` | C# | MSTest integration test project | DotNet |
| `aspire-nunit` | C# | NUnit integration test project | DotNet |
| `aspire-xunit` | C# | xUnit integration test project | DotNet |

## .NET SDK Templates (DotNet Runtime)

These use `dotnet new` under the hood. The CLI:
1. Checks SDK availability
2. Resolves the template package version from `--version` or `--channel`
3. Installs `Aspire.ProjectTemplates` via `dotnet new install`
4. Runs `dotnet new <template-id>` with options
5. Trusts dev certificates
6. Optionally creates/updates `NuGet.config`

Template-specific options include:
- `--use-redis-cache` ŌĆö adds Redis caching to the starter
- `--test-framework` ŌĆö selects test framework (xunit, mstest, nunit)
- `--xunit-version` ŌĆö sets xUnit version (v2 or v3)
- `--localhost-tld` ŌĆö rewrites localhost to `*.dev.localhost`

## CLI-Native Templates (Cli Runtime)

These use embedded file trees with token replacement:

### TypeScript Starter (`aspire-ts-starter`)
Scaffolds a full app with:
- Root: `apphost.ts`, `eslint.config.mjs`, `tsconfig.json`, `aspire.config.json`, `package.json`
- `api/`: Express API with `src/index.ts`, `src/instrumentation.ts` (OpenTelemetry)
- `frontend/`: React + Vite app with `App.tsx`, `vite.config.ts`, Aspire branding

### Java Empty (`aspire-java-empty`)
Scaffolds a Java AppHost with `AppHost.java` and `aspire.config.json`.

### Empty AppHost (`aspire-empty`)
Minimal C# AppHost with `apphost.cs` and `aspire.config.json`.

## Template Availability

Templates are filtered by runtime availability. TypeScript templates only appear when Node.js is detected via `ILanguageDiscovery`. Java templates require a Java runtime. C# .NET templates require the .NET SDK. This prevents users from selecting templates they can't actually use.

## Init Templates

`aspire init` uses a subset of templates for adding Aspire to existing projects. `GetInitTemplatesAsync()` returns only templates suitable for initialization (typically the empty/minimal variants for each language).
