# Template System Architecture

The Aspire CLI's template system is how `aspire new` and `aspire init` scaffold new projects. It's built on two layers: **template factories** that know how to create templates, and the **template provider** that aggregates and exposes them.

## Core Interfaces

### ITemplate

Every template implements `ITemplate`:

```csharp
public interface ITemplate
{
    string Name { get; }
    string Description { get; }
    bool IsEmpty { get; }
    TemplateRuntime Runtime { get; }
    string? LanguageId { get; }
    Func<string, string> PathDeriver { get; }
    void ApplyOptions(Command command);
    Task<TemplateResult> ApplyTemplateAsync(
        TemplateInputs inputs, ParseResult parseResult, CancellationToken ct);
}
```

`ApplyOptions` adds template-specific CLI options (like `--use-redis-cache`). `ApplyTemplateAsync` executes the scaffolding and returns a `TemplateResult` with the exit code and output path.

### ITemplateFactory and ITemplateProvider

`ITemplateFactory` produces templates from a single source (e.g., .NET SDK templates or embedded CLI templates). `ITemplateProvider` aggregates multiple factories into a unified list.

## TemplateInputs

Every template receives a `TemplateInputs` record with the shared options:

| Property | Source |
|----------|--------|
| `Name` | `--name/-n` |
| `Output` | `--output/-o` |
| `Source` | `--source/-s` |
| `Version` | `--version` |
| `Channel` | `--channel` |
| `Language` | `--language` |
| `UseWorkingDirectory` | Whether to scaffold in the current directory |

## DotNetTemplateFactory

This factory wraps .NET SDK templates. It checks SDK availability before exposing templates, installs `Aspire.ProjectTemplates` via `dotnet new install`, then runs `dotnet new <template-id>`. It supports options like `--localhost-tld`, `--use-redis-cache`, `--test-framework`, and `--xunit-version`.

## CliTemplateFactory

This factory provides CLI-native templates using embedded file trees and token replacement. It uses `ILanguageDiscovery` to check runtime availability (Node.js for TypeScript, Java for Java) and only exposes templates for installed runtimes.

Token replacement handles placeholders like `{{projectName}}`, `{{projectNameLower}}`, `{{aspireVersion}}`, and `{{hostName}}`. The factory also writes `aspire.config.json` for channel configuration and can rewrite `localhost` to `*.dev.localhost` when `--localhost-tld` is specified.
