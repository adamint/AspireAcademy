# MCP Server

The `aspire agent mcp` command starts a **Model Context Protocol (MCP) server** that exposes your running Aspire application to AI agents. MCP is a standardized protocol for AI tools to interact with external systems — and Aspire's implementation gives agents full visibility into your distributed application.

## Starting the MCP Server

```bash
aspire agent mcp
```

The server communicates over **stdio** (standard input/output) using the MCP protocol. AI agents like GitHub Copilot connect to this server and can call tools to inspect resources, read logs, and execute commands.

## Architecture

`AgentMcpCommand` sets up:

1. **Stdio Transport** — `StdioMcpTransportFactory` creates the MCP transport layer using stdin/stdout, allowing any MCP-compatible client to connect

2. **Built-in Tools** — A fixed set of `CliMcpTool` instances registered as `KnownTools`

3. **Dynamic Resource Tools** — `McpResourceToolRefreshService` discovers tools exposed by running resources (via their own MCP endpoints) and adds them to the tool list

4. **Tool Refresh** — When resources start/stop or expose new tools, the `refresh_tools` tool emits a `tool_list_changed` notification to connected clients

## Tool Registration

Each tool is a `CliMcpTool` with:
- **Name** — unique identifier (e.g., `list_resources`)
- **Description** — human-readable explanation for the AI agent
- **Input Schema** — JSON Schema defining parameters
- **Handler** — async function that executes the tool and returns results

Tools are registered in `KnownMcpTools` and instantiated in `AgentMcpCommand`.

## Legacy MCP Commands

The CLI also has legacy MCP commands (from before the `agent` subcommand was introduced):

| Legacy Command | Replacement |
|---------------|-------------|
| `aspire mcp start` | `aspire agent mcp` |
| `aspire mcp init` | `aspire agent init` |
| `aspire mcp tools` | Lists tools from running resources |
| `aspire mcp call <tool> <resource>` | Calls a specific tool on a resource |

`McpStartCommand` and `McpInitCommand` display deprecation warnings and delegate to their `agent` counterparts.

## Documentation Subsystem

The MCP server includes a documentation subsystem (`src/Aspire.Cli/Mcp/Docs/`) that provides AI agents with searchable Aspire documentation:

- **`DocsFetcher`** — Downloads `https://aspire.dev/llms-small.txt` with ETag caching
- **`DocsCache`** — Stores content, ETag, and parsed index in memory + disk
- **`DocsIndexService`** — Parses and indexes docs for lexical search
- **`DocsSearchService`** — Provides ranked search results for `search_docs` tool

This enables AI agents to look up Aspire APIs, configuration options, and integration documentation without leaving the coding context.
