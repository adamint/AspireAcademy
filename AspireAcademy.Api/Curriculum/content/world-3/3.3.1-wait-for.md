# WaitFor — Health-Based Dependency Waiting

`WaitFor()` delays a resource's startup until its dependency is healthy. This ensures databases, caches, and other infrastructure are ready before services that depend on them start.

## Basic Usage

```csharp
var postgres = builder.AddPostgres("pg").AddDatabase("mydb");
var api = builder.AddProject<Projects.ApiService>("api")
    .WithReference(postgres)
    .WaitFor(postgres);
```

**Important**: `WithReference()` and `WaitFor()` serve different purposes:
- `WithReference()` injects connection information as environment variables
- `WaitFor()` delays startup until the dependency is healthy

You typically need both.

## What "Healthy" Means

`WaitFor()` waits for the dependency to reach a healthy state as determined by health checks:

1. DCP starts the dependency resource
2. The resource transitions to `Running` state
3. If the resource has registered health checks (via `HealthCheckAnnotation`), Aspire polls them
4. Once health checks pass, the resource is marked **ready**
5. Only then does the waiting resource start

If no health checks are registered, the resource is considered ready as soon as it enters `Running` state.

## The Signature

```csharp
public static IResourceBuilder<T> WaitFor<T>(
    this IResourceBuilder<T> builder,
    IResourceBuilder<IResource> dependency)
    where T : IResourceWithWaitSupport
```

`IResourceWithWaitSupport` is implemented by `ProjectResource`, `ContainerResource`, and `ExecutableResource`.

## Multiple Dependencies

Chain multiple `WaitFor()` calls:

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithReference(postgres)
    .WithReference(redis)
    .WaitFor(postgres)
    .WaitFor(redis);
```

The API service starts only after **both** PostgreSQL and Redis are healthy.

## WaitFor vs WaitForStart

`WaitForStart()` is a lighter variant that only waits for the resource to enter `Running` state — it doesn't check health:

```csharp
.WaitForStart(postgres)  // Starts as soon as postgres process is running
.WaitFor(postgres)       // Waits until postgres health checks pass
```

Use `WaitForStart()` when you need ordering but not health verification.

## Key Points

- `WaitFor()` delays startup until the dependency's health checks pass
- `WithReference()` injects config; `WaitFor()` controls startup ordering — use both
- Resources without health checks are considered ready when they enter `Running` state
- Chain multiple `WaitFor()` calls for multiple dependencies
