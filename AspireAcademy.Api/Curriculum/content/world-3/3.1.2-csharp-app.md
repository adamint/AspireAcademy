# AddCSharpApp — Path-Based Projects

`AddCSharpApp` is an **experimental** path-based alternative to `AddProject<T>()` for .NET projects that aren't part of your solution — when you have a path to a `.csproj` but no project reference. The normal way to add .NET projects is `AddProject<T>()` with a project reference; use `AddCSharpApp` for cross-repository scenarios where that isn't practical.

> ⚠️ **Experimental:** `AddCSharpApp` requires suppressing the `ASPIRECSHARPAPPS001` diagnostic. Add `#pragma warning disable ASPIRECSHARPAPPS001` before the call, or suppress it in your project file.

---

## When to Use AddCSharpApp vs AddProject

| Method | Reference Type | Best For |
|---|---|---|
| `AddProject<T>()` | Project reference in `.csproj` | Projects in the same solution (preferred) |
| `AddCSharpApp()` | File system path | External projects, separate repos, downloaded samples |

## Usage

```csharp
#pragma warning disable ASPIRECSHARPAPPS001
builder.AddCSharpApp("external-api", "../other-repo/Api")
    .WithHttpEndpoint(port: 5100);
#pragma warning restore ASPIRECSHARPAPPS001
```

The first parameter is the resource name. The second is the path to the directory containing the `.csproj` file.

## How It Works

Under the hood, `AddCSharpApp` returns a `CSharpAppResource` and applies `WithProjectDefaults`, so it still processes launch profiles and detects endpoints from `launchSettings.json` — just like `AddProject<T>()`. It:

1. Resolves the `.csproj` path
2. Registers the resource with the app model
3. Applies project defaults (launch profiles, endpoint detection, environment variables)
4. Configures DCP to build and run the project

## Limitations

Because there is no project reference, you lose:
- The generated `Projects.MyApi` type (no generic `AddProject<T>` syntax)
- Compile-time validation that the project exists
- Automatic rebuilds when the referenced project changes

Use `AddProject<T>()` when possible. Reserve `AddCSharpApp` for cross-repo scenarios where a project reference isn't feasible.

> 🐛 **Common Gotcha:** The path in `AddCSharpApp` is relative to the AppHost project directory, not the solution root. If your AppHost is at `src/AppHost/` and the external project is at `../other-repo/Api/`, the path should be `../../other-repo/Api` — count the directory levels carefully.
