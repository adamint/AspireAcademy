# How Service Discovery Works Client-Side

Understanding the request flow from `HttpClient` to the actual endpoint helps you debug connectivity issues and understand what happens under the hood.

## The Request Pipeline

```
Your Code: HttpClient.GetAsync("http://apiservice/api/products")
    ↓
ServiceDiscoveryHttpMessageHandler
    ↓ Extracts hostname: "apiservice"
    ↓ Queries IServiceDiscoveryProvider
    ↓
ConfigurationServiceEndpointProvider
    ↓ Reads IConfiguration["services:apiservice:http:0"]
    ↓ Returns: ServiceEndpoint { Host="localhost", Port=5123, Scheme="http" }
    ↓
ServiceDiscoveryHttpMessageHandler
    ↓ Rewrites URL: http://localhost:5123/api/products
    ↓
HttpClientHandler → Actual HTTP request
```

## ConfigurationServiceEndpointProvider

The default provider reads from `IConfiguration` under the `services` section:

- Looks for keys matching `services:{serviceName}:*`
- These keys are populated from `services__{name}__{endpoint}__{index}` environment variables
- Returns a collection of `ServiceEndpoint` objects

## Multiple Endpoints and Load Balancing

If multiple endpoints exist (e.g., from `WithReplicas(3)`):

```
services__apiservice__http__0=http://localhost:5001
services__apiservice__http__1=http://localhost:5002
services__apiservice__http__2=http://localhost:5003
```

The service discovery provider returns all three endpoints. A load balancer (round-robin by default) distributes requests across them.

## HTTPS Resolution

When you use `https://apiservice`, service discovery looks for HTTPS endpoints:

```csharp
builder.Services.AddHttpClient<ApiClient>(client =>
    client.BaseAddress = new Uri("https://apiservice"));
// Resolves using services__apiservice__https__0
```

## Caching and Refresh

Resolved endpoints are cached to avoid repeated configuration lookups. The cache refreshes when configuration changes, enabling dynamic endpoint updates.

## Debugging Tips

1. **Check environment variables**: Verify `services__` vars are injected (visible in dashboard)
2. **Check configuration**: `builder.Configuration["services:apiservice:http:0"]` should return a URL
3. **Verify AddServiceDefaults()**: Service discovery only works if the handler is registered
4. **Check the scheme**: `http://` vs `https://` must match the endpoint's scheme

## Key Points

- `ServiceDiscoveryHttpMessageHandler` intercepts requests and rewrites URLs
- `ConfigurationServiceEndpointProvider` reads endpoints from `IConfiguration`
- Multiple endpoints support load balancing
- Endpoints are cached and refreshed on configuration changes
- All resolution is transparent to application code
