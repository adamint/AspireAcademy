# ConnectionStringReference

`ConnectionStringReference` wraps an `IResourceWithConnectionString` as both an expression provider and a value provider. It's the intermediary that `WithReference()` uses to inject connection strings.

## The Class

```csharp
public class ConnectionStringReference(IResourceWithConnectionString resource, bool optional)
    : IManifestExpressionProvider, IValueProvider, IValueWithReferences
{
    public IResourceWithConnectionString Resource { get; }
    public bool Optional { get; }
}
```

## What It Does

When `WithReference(postgres)` is called, Aspire creates a `ConnectionStringReference`:

1. **As an `IValueProvider`**: Its `GetValueAsync()` calls through to the wrapped resource's `GetConnectionStringAsync()`, returning the resolved connection string
2. **As an `IManifestExpressionProvider`**: Its `ValueExpression` returns `{resourceName.connectionString}` for deployment manifests
3. **As an `IValueWithReferences`**: Exposes the underlying references (endpoints, parameters) for dependency tracking

## The Optional Flag

```csharp
builder.AddProject<Projects.Api>("api")
    .WithReference(postgres, optional: true);
```

When `optional: true`, if the connection string can't be resolved (resource unavailable), the environment variable is simply not set — rather than throwing an error. This is useful for resources that may not always be present.

When `optional: false` (the default), a missing connection string throws `InvalidOperationException` at startup.

## How It Flows

```
WithReference(postgres)
  → Creates ConnectionStringReference(postgres, optional: false)
    → Registers environment callback
      → At startup: calls GetValueAsync()
        → Calls postgres.GetConnectionStringAsync()
          → Resolves ConnectionStringExpression
            → Returns "Host=localhost;Port=5432;..."
              → Sets ConnectionStrings__mydb = "Host=localhost;Port=5432;..."
```

## Key Points

- `ConnectionStringReference` wraps a resource for both runtime resolution and manifest generation
- The `optional` flag controls whether missing connection strings cause errors
- It implements `IValueProvider` (runtime) and `IManifestExpressionProvider` (publish)
- You rarely create this directly — `WithReference()` creates it automatically
