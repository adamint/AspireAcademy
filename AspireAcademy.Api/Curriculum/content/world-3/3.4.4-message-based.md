# Message-Based Async Communication

Not all service communication should be synchronous. Message brokers like RabbitMQ enable asynchronous, decoupled communication — services publish and consume messages without knowing about each other directly.

## AppHost Setup

```csharp
var messaging = builder.AddRabbitMQ("messaging");

var orderApi = builder.AddProject<Projects.OrderApi>("orderapi")
    .WithReference(messaging)
    .WaitFor(messaging);

var processor = builder.AddProject<Projects.OrderProcessor>("processor")
    .WithReference(messaging)
    .WaitFor(messaging);
```

Both services receive `ConnectionStrings__messaging` with the RabbitMQ connection string (e.g., `amqp://guest:password@localhost:5672`).

## Producer Service

```csharp
// OrderApi/Program.cs
builder.AddServiceDefaults();
builder.AddRabbitMQClient("messaging"); // Registers IConnection in DI

var app = builder.Build();

app.MapPost("/api/orders", async (IConnection rabbit, Order order) =>
{
    using var channel = await rabbit.CreateChannelAsync();
    await channel.QueueDeclareAsync("orders", durable: true);

    var body = JsonSerializer.SerializeToUtf8Bytes(order);
    await channel.BasicPublishAsync("", "orders", body);

    return Results.Accepted();
});
```

## Consumer Service

```csharp
// OrderProcessor/Program.cs
builder.AddServiceDefaults();
builder.AddRabbitMQClient("messaging");
builder.Services.AddHostedService<OrderProcessingWorker>();
```

The worker processes messages asynchronously:

```csharp
public class OrderProcessingWorker(IConnection rabbit) : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        using var channel = await rabbit.CreateChannelAsync(ct);
        await channel.QueueDeclareAsync("orders", durable: true);

        var consumer = new AsyncEventingBasicConsumer(channel);
        consumer.ReceivedAsync += async (_, ea) =>
        {
            var order = JsonSerializer.Deserialize<Order>(ea.Body.Span);
            // Process order...
            await channel.BasicAckAsync(ea.DeliveryTag, false);
        };

        await channel.BasicConsumeAsync("orders", false, consumer, ct);
    }
}
```

## When to Use Message-Based Communication

| Scenario | Pattern |
|----------|---------|
| Immediate response needed | HTTP / gRPC |
| Fire-and-forget workloads | Message queue |
| Event-driven architecture | Message queue |
| Retry / dead-letter handling | Message queue |
| Multiple consumers for same event | Message queue (pub/sub) |

## Key Points

- Message brokers decouple producers from consumers
- Both services reference the broker via `WithReference()` and receive the connection string
- Aspire client integration packages (`Aspire.RabbitMQ.Client`) register the connection in DI
- The AppHost wires the broker; application code handles publishing and consumption
- Use `WaitFor()` to ensure the broker is healthy before services start
