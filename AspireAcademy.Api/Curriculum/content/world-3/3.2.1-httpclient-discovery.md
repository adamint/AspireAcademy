# HttpClient Service Discovery

Using AddServiceDiscovery and typed HttpClient to resolve service endpoints by name.

## How It Works

`AddServiceDefaults()` registers service discovery:

```csharp
builder.Services.AddServiceDiscovery();
builder.Services.ConfigureHttpClientDefaults(http =>
{
    http.AddServiceDiscovery();
});
```

This lets `HttpClient` resolve service names to URLs automatically.

## Using Service Discovery

In your service's `Program.cs`:

```csharp
builder.AddServiceDefaults();

// Named HttpClient
builder.Services.AddHttpClient("catalog", client =>
{
    client.BaseAddress = new Uri("https://catalog");
});
```

The URL `https://catalog` isn't a real hostname — service discovery resolves it using the `services__catalog__https__0` environment variable injected by the AppHost.

## Typed HttpClient

```csharp
builder.Services.AddHttpClient<CatalogClient>(client =>
{
    client.BaseAddress = new Uri("https://catalog");
});

public class CatalogClient(HttpClient client)
{
    public async Task<Product[]> GetProducts()
        => await client.GetFromJsonAsync<Product[]>("/api/products")
           ?? [];
}
```

## Direct Usage

With `IHttpClientFactory`:

```csharp
app.MapGet("/products", async (IHttpClientFactory factory) =>
{
    var client = factory.CreateClient();
    return await client.GetFromJsonAsync<Product[]>(
        "https://catalog/api/products");
});
```

## The Resolution Flow

```
HttpClient.GetAsync("https://catalog/api/products")
  → Service Discovery reads: services__catalog__https__0 = https://localhost:5010
  → Resolves to: https://localhost:5010/api/products
  → HTTP request sent to localhost:5010
```

## Scheme Matching

Service discovery matches the **scheme** in the URL to the endpoint scheme:
- `https://catalog` → looks for `services__catalog__https__*`
- `http://catalog` → looks for `services__catalog__http__*`
