# IResourceWithConnectionString and ConnectionStringReference

How resources expose connection strings and how ConnectionStringReference resolves them.

## The Interface

```csharp
public interface IResourceWithConnectionString : IResource
{
    ReferenceExpression ConnectionStringExpression { get; }

    ValueTask<string?> GetConnectionStringAsync(
        CancellationToken cancellationToken = default);

    string? ConnectionStringEnvironmentVariable => null;
}
```

Resources implementing this interface can produce a connection string that `WithReference` injects automatically.

## How Connection Strings Are Built

Each hosting integration defines its `ConnectionStringExpression` differently:

- **Redis**: `{redis.bindings.tcp.host}:{redis.bindings.tcp.port}`
- **PostgreSQL**: `Host={pg.bindings.tcp.host};Port={pg.bindings.tcp.port};Username={pg-username.value};Password={pg-password.value}`
- **SQL Server**: `Server={sql.bindings.tcp.host},{sql.bindings.tcp.port};...`

These expressions use **manifest expression syntax** — resolved at runtime for local dev and at deploy time for production.

## Connection Properties

Resources can also expose structured **connection properties** beyond the connection string:

```csharp
IEnumerable<KeyValuePair<string, ReferenceExpression>>
    GetConnectionProperties() => [];
```

When `WithReference` is called, both the connection string and any connection properties are injected. Properties follow the pattern `ConnectionStrings__{name}__{propertyName}`.

## Runtime Resolution

At runtime, `GetConnectionStringAsync()` resolves the expression to an actual value. For a local Redis:

```
ConnectionStrings__cache = localhost:52341
```

For a cloud PostgreSQL:
```
ConnectionStrings__mydb = Host=myserver.postgres.database.azure.com;Port=5432;...
```

## AddConnectionString for External Resources

For pre-existing resources not managed by Aspire:

```csharp
var externalDb = builder.AddConnectionString("orders-db");
```

This reads from `ConnectionStrings:orders-db` in configuration and injects it the same way.
