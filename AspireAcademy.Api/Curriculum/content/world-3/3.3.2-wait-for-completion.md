# WaitForCompletion — Init Container Pattern

`WaitForCompletion()` waits for a resource to **exit successfully** before starting the dependent resource. This implements the "init container" pattern — a resource that runs setup tasks and then terminates.

## Basic Usage

```csharp
var migration = builder.AddProject<Projects.DbMigrator>("migrator")
    .WithReference(postgres);

var api = builder.AddProject<Projects.ApiService>("api")
    .WithReference(postgres)
    .WaitForCompletion(migration);
```

The API service doesn't start until the `migrator` project has run to completion and exited with code 0.

## The Signature

```csharp
public static IResourceBuilder<T> WaitForCompletion<T>(
    this IResourceBuilder<T> builder,
    IResourceBuilder<IResource> dependency,
    int exitCode = 0)
    where T : IResourceWithWaitSupport
```

The `exitCode` parameter (default: 0) specifies which exit code indicates success.

## Common Use Cases

**Database migrations:**
```csharp
var migrator = builder.AddProject<Projects.DbMigrator>("migrator")
    .WithReference(postgres)
    .WaitFor(postgres);  // Migrator waits for DB to be healthy

var api = builder.AddProject<Projects.Api>("api")
    .WithReference(postgres)
    .WaitForCompletion(migrator);  // API waits for migrations to finish
```

**Seed data loading:**
```csharp
var seeder = builder.AddExecutable("seeder", "python", "./scripts", "seed.py")
    .WithReference(postgres);

var api = builder.AddProject<Projects.Api>("api")
    .WaitForCompletion(seeder);
```

**Certificate generation:**
```csharp
var certGen = builder.AddExecutable("certgen", "openssl", "./certs", "req", "-x509", ...);

var api = builder.AddProject<Projects.Api>("api")
    .WaitForCompletion(certGen);
```

## WaitFor + WaitForCompletion Together

A common pattern chains both:

```csharp
var api = builder.AddProject<Projects.Api>("api")
    .WaitFor(postgres)           // Wait for DB to be healthy
    .WaitForCompletion(migrator) // Wait for migrations to complete
    .WaitFor(redis);             // Wait for cache to be healthy
```

## Key Points

- `WaitForCompletion()` waits for the dependency to exit with the specified exit code
- Default exit code is 0 (success)
- Ideal for migration scripts, seed data, and initialization tasks
- The dependency runs to completion and then the waiting resource starts
- Combine with `WaitFor()` for complex dependency chains
