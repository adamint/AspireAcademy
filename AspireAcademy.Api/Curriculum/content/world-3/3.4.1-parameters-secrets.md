# AddParameter and Secrets

Parameters are configuration values declared in your AppHost. They can be plain values or secrets — and they behave differently at run time vs publish time.

---

## Declaring Parameters

```csharp
var apiKey = builder.AddParameter("api-key");
```

At run time, Aspire resolves the value from multiple sources (checked in order): the `Parameters:{name}` configuration key (in `appsettings.json`, environment variables, or .NET User Secrets), and finally an interactive prompt if no value is found. At publish time, the parameter appears in the deployment template for the operator to fill in.

## Secret Parameters

```csharp
var dbPassword = builder.AddParameter("db-password", secret: true);
```

Secret parameters are:
- Masked in the Dashboard by default; not logged in plaintext
- Marked as secret in deployment artifacts
- Resolved from the same sources as plain parameters (`Parameters:{name}` config key, environment variables, user secrets) — only prompted if no value is found

## Default Values and Generated Passwords

You can provide a default value for a parameter using `ParameterDefault`. This is especially handy for passwords — `GenerateParameterDefault` creates a random value automatically:

```csharp
var dbPassword = builder.AddParameter("db-password", secret: true,
    defaultValue: GenerateParameterDefault.Create(minLower: 5, minUpper: 5, minNumeric: 3));
```

If you want Aspire to save a generated value so it stays the same across runs, use `persist: true`:

```csharp
var dbPassword = builder.AddParameter("db-password", secret: true,
    defaultValue: GenerateParameterDefault.Create(persist: true));
```

With `persist: true`, Aspire stores the generated value in .NET User Secrets so you get the same password every time you run locally.

## Using Parameters

Inject parameters into resources:

```csharp
builder.AddPostgres("pg", password: dbPassword);
```

Or as environment variables:

```csharp
builder.AddProject<Projects.Api>("api")
    .WithEnvironment("API_KEY", apiKey);
```

## In TypeScript and Python

```typescript
const apiKey = await builder.addParameter("api-key");
const dbPass = await builder.addParameter("db-password", { secret: true });
```

```python
api_key = builder.add_parameter("api-key")
db_pass = builder.add_parameter("db-password", secret=True)
```

## Run Time vs Publish Time

| Phase | Plain Parameter | Secret Parameter |
|---|---|---|
| Run | Resolved from config (Parameters:{name}), env vars, user secrets, or prompted | Same resolution chain — masked in Dashboard and logs |
| Publish | Appears in manifest | Marked as secret in deployment template |

This separation lets you declare configuration shape in the AppHost without hardcoding values.

> 🐛 **Common Gotcha:** Secret parameters are **not** always prompted — they follow the same resolution chain as plain parameters. If you set `Parameters:db-password` in User Secrets or an environment variable (`Parameters__db-password=...`), Aspire uses that value silently. Prompting only happens when no value is found in any config source. In CI/CD, always provide values via environment variables.
