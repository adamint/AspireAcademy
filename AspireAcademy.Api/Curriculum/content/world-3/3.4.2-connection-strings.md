# AddConnectionString — External Infrastructure

`AddConnectionString` references infrastructure that Aspire does not manage — an existing database, a managed cloud service, or a legacy system.

---

## Usage

```csharp
var existingDb = builder.AddConnectionString("legacy-db");
```

This creates a resource that:
- Reads its value from `ConnectionStrings:legacy-db` in `appsettings.json` or environment variables
- Can be passed to services via `WithReference(existingDb)`
- Appears in the Dashboard as a resource (with no lifecycle management)

## When to Use It

Use `AddConnectionString` when:
- The database or service already exists and is not managed by Aspire
- You are connecting to a cloud-managed service (Azure SQL, Amazon RDS)
- You are migrating an existing app to Aspire and want to keep existing connection strings

## Example: Hybrid App

```csharp
// Aspire manages Redis
var redis = builder.AddRedis("cache");

// Production PostgreSQL is managed externally
var db = builder.AddConnectionString("production-db");

builder.AddProject<Projects.Api>("api")
    .WithReference(redis)       // Aspire-managed
    .WithReference(db);         // External
```

The API receives both connection strings as environment variables. It does not know or care which one Aspire manages.
