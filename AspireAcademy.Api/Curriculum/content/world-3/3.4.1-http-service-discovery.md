# HTTP Service-to-Service with Service Discovery

Aspire's service discovery enables services to call each other using friendly resource names instead of hardcoded URLs. No DNS servers or service registries required — it's all configuration-driven.

## The Pattern

In the AppHost:

```csharp
var api = builder.AddProject<Projects.ApiService>("apiservice");
var web = builder.AddProject<Projects.WebFrontend>("webfrontend")
    .WithReference(api);
```

In the web frontend's `Program.cs`:

```csharp
builder.AddServiceDefaults();

builder.Services.AddHttpClient<CatalogClient>(client =>
    client.BaseAddress = new Uri("http://apiservice"));
```

The `HttpClient` uses `http://apiservice` as the base address — the resource name, not an actual hostname. Aspire's service discovery resolves this to the real endpoint.

## How It Works

1. `AddServiceDefaults()` registers `ServiceDiscoveryHttpMessageHandler` in the HTTP pipeline
2. When `HttpClient` makes a request to `http://apiservice/api/products`, the handler intercepts it
3. The handler reads `services:apiservice:http:0` from `IConfiguration` (populated from the `services__apiservice__http__0` environment variable)
4. The request URL is rewritten from `http://apiservice/api/products` to `http://localhost:5123/api/products`
5. The actual HTTP request goes to the resolved endpoint

## Typed HttpClient Pattern

```csharp
// Register
builder.Services.AddHttpClient<CatalogClient>(client =>
    client.BaseAddress = new Uri("http://apiservice"));

// Use via DI
public class CatalogClient(HttpClient http)
{
    public async Task<Product[]> GetProductsAsync()
        => await http.GetFromJsonAsync<Product[]>("/api/products") ?? [];
}
```

## Named HttpClient Pattern

```csharp
builder.Services.AddHttpClient("apiservice", client =>
    client.BaseAddress = new Uri("http://apiservice"));

// Inject IHttpClientFactory and create by name
var client = httpClientFactory.CreateClient("apiservice");
```

## Key Points

- Use `http://resourcename` as the base address — service discovery resolves it
- `AddServiceDefaults()` registers the service discovery handler
- Resolution uses the `services__` environment variables injected by `WithReference()`
- Works transparently with typed and named `HttpClient` patterns
- No external DNS or service registry needed
