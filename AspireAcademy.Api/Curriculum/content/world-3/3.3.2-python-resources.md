# Python Resources

Aspire orchestrates Python applications with `AddPythonApp()` and `AddUvicornApp()`.

---

## AddPythonApp

```csharp
builder.AddPythonApp("ml", "../bazaar-recs", "app:app")
    .WithHttpEndpoint(port: 8000, env: "PORT")
    .WithReference(rabbitmq);
```

Parameters:
- `"ml"` — resource name
- `"../bazaar-recs"` — path to the Python project directory
- `"app:app"` — the ASGI/WSGI entry point (module:application)

Aspire handles:
- Virtual environment creation (if `.venv` does not exist)
- Dependency installation via `pip install -r requirements.txt` or `uv`
- Starting the application with uvicorn

## AddUvicornApp

A convenience wrapper when you are using uvicorn explicitly:

```csharp
builder.AddUvicornApp("ml", "../bazaar-recs", "app:app")
    .WithHttpEndpoint(port: 8000);
```

## Environment Variables

Python services receive connection strings and service URLs as environment variables, just like any other language:

```python
# In your Python service
import os

api_url = os.environ["services__api__http__0"]
rabbitmq_url = os.environ["ConnectionStrings__messaging"]
```

## Virtual Environment Support

Aspire supports both `pip` and `uv` for dependency management:
- If `uv.lock` exists, Aspire uses `uv`
- Otherwise, it uses `pip` with `requirements.txt`

> 🐛 **Common Gotcha:** The entry point format `\"app:app\"` means module `app`, variable `app`. If your FastAPI app is in `main.py` and the variable is called `application`, the entry point should be `\"main:application\"`. Getting this wrong produces a cryptic uvicorn error like "Error loading ASGI app."

> 🎲 **Fun Fact:** Using `uv` instead of `pip` can speed up dependency installation by 10-100x. If your Python project has a `uv.lock` file, Aspire automatically uses `uv` — no configuration needed.
