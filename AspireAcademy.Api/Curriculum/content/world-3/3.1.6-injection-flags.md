# WithReferenceEnvironment and Injection Flags

`ReferenceEnvironmentInjectionFlags` provides fine-grained control over what environment variables `WithReference` injects. This is useful when you need individual connection properties instead of a combined string.

## The Flags

The `ReferenceEnvironmentInjectionFlags` enum defines:

```csharp
public enum ReferenceEnvironmentInjectionFlags
{
    None = 0,
    ConnectionString = 1 << 0,    // ConnectionStrings__{name}
    ConnectionProperties = 1 << 1, // Individual host, port, username, etc.
    ServiceDiscovery = 1 << 2,     // Service discovery config entries
    Endpoints = 1 << 3,            // services__{name}__{endpoint}__{index}
    All = ConnectionString | ConnectionProperties | ServiceDiscovery | Endpoints
}
```

## Default Behavior

By default, `WithReference` uses sensible defaults:
- **Connection string resources**: Injects `ConnectionStrings__{name}`
- **Endpoint resources**: Injects `services__{name}__{endpoint}__{index}` and service discovery entries

## Overriding with WithReferenceEnvironment

Use `WithReferenceEnvironment()` to customize what gets injected:

```csharp
var api = builder.AddProject<Projects.ApiService>("api")
    .WithReference(postgres)
    .WithReferenceEnvironment(ReferenceEnvironmentInjectionFlags.ConnectionProperties);
```

With `ConnectionProperties`, instead of a single combined connection string, individual properties are injected:

```
ConnectionStrings__postgres__Host=localhost
ConnectionStrings__postgres__Port=5432
ConnectionStrings__postgres__Username=postgres
ConnectionStrings__postgres__Password=xyz
```

## Use Cases

**Sidecar proxy needing only host and port:**
```csharp
builder.AddContainer("proxy", "envoyproxy/envoy:v1.30")
    .WithReference(backend)
    .WithReferenceEnvironment(ReferenceEnvironmentInjectionFlags.ConnectionProperties);
```

**Service needing both connection string and individual properties:**
```csharp
.WithReferenceEnvironment(
    ReferenceEnvironmentInjectionFlags.ConnectionString |
    ReferenceEnvironmentInjectionFlags.ConnectionProperties)
```

## Combining Flags

Flags are combinable with bitwise OR:

```csharp
var flags = ReferenceEnvironmentInjectionFlags.ConnectionString
          | ReferenceEnvironmentInjectionFlags.Endpoints;
```

## Key Points

- Default injection behavior works for most scenarios
- `ConnectionProperties` injects individual host, port, username, password as separate variables
- Flags are combinable with `|` for multiple injection patterns
- Useful for sidecar containers and non-.NET consumers that need granular connection details
