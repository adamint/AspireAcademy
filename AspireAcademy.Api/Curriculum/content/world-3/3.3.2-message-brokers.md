# Message Broker Communication

Decoupled communication with RabbitMQ, Kafka, and NATS via WithReference.

## The Pattern

Message brokers decouple producers from consumers. In Aspire, the AppHost wires the connection:

```csharp
var rabbit = builder.AddRabbitMQ("messaging");

builder.AddProject<Projects.Api>("api")
       .WithReference(rabbit);       // Producer

builder.AddProject<Projects.Worker>("worker")
       .WithReference(rabbit);       // Consumer
```

Both services receive `ConnectionStrings__messaging` with the RabbitMQ connection string.

## RabbitMQ

```csharp
// AppHost
var rabbit = builder.AddRabbitMQ("messaging")
                    .WithManagementPlugin();  // Adds web UI on port 15672

// Service — reads ConnectionStrings__messaging
builder.AddRabbitMQClient("messaging");
```

Connection string format: `amqp://guest:guest@localhost:5672`

## Kafka

```csharp
// AppHost
var kafka = builder.AddKafka("kafka");

// Service — reads ConnectionStrings__kafka
builder.AddKafkaProducer<string, string>("kafka");
builder.AddKafkaConsumer<string, string>("kafka");
```

Connection string format: `localhost:9092` (bootstrap servers)

## NATS

```csharp
// AppHost
var nats = builder.AddNats("nats");

// Service — reads ConnectionStrings__nats
builder.AddNatsClient("nats");
```

## Key Difference from HTTP

HTTP communication uses **service discovery** (`services__` variables). Message brokers use **connection strings** (`ConnectionStrings__` variables).

| Communication | Env Var Pattern | Discovery |
|--------------|-----------------|-----------|
| HTTP (projects) | `services__name__scheme__0` | Service discovery |
| Databases | `ConnectionStrings__name` | Connection string |
| Message brokers | `ConnectionStrings__name` | Connection string |

## Async Communication Benefits

Message brokers enable:
- **Decoupling** — producer doesn't wait for consumer
- **Reliability** — messages are queued if consumer is down
- **Scaling** — multiple consumers process messages in parallel
- **Event-driven architecture** — services react to events, not direct calls
