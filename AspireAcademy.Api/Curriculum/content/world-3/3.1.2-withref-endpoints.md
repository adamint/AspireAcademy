# WithReference for Endpoint Resources

When the referenced resource implements `IResourceWithServiceDiscovery` (which `ProjectResource` does) but not `IResourceWithConnectionString`, `WithReference` injects endpoint URLs as environment variables using a different pattern.

## How It Works

```csharp
var api = builder.AddProject<Projects.ApiService>("apiservice");
var web = builder.AddProject<Projects.WebFrontend>("webfrontend")
    .WithReference(api);
```

This injects environment variables into `webfrontend`:

```
services__apiservice__http__0=http://localhost:5123
services__apiservice__https__0=https://localhost:7123
```

## The Naming Pattern

Endpoint injection follows: `services__{resourceName}__{endpointName}__{index}`

- **`services`**: The root section
- **`resourceName`**: The name passed to `AddProject()` — `"apiservice"`, not the variable name `api`
- **`endpointName`**: The endpoint name (e.g., `http`, `https`, `grpc`)
- **`index`**: Zero-based index supporting multiple bindings per endpoint name

## How This Maps to Configuration

In .NET configuration, `__` maps to `:`:

```
services__apiservice__http__0 → services:apiservice:http:0
```

Service code reads: `builder.Configuration["services:apiservice:http:0"]`

## When Endpoint Injection Is Used

Endpoint injection applies when:

- The resource is a `ProjectResource` (implements `IResourceWithServiceDiscovery`)
- The resource is an `ExecutableResource`
- The resource has endpoints but no connection string

This is the standard pattern for project-to-project references — there's no connection string concept, just HTTP/HTTPS endpoints.

## Service Discovery Integration

In practice, you rarely read these environment variables directly. Aspire's service discovery resolves `http://apiservice` automatically:

```csharp
// In the consuming service
builder.Services.AddHttpClient<ApiClient>(c =>
    c.BaseAddress = new Uri("http://apiservice"));
// ServiceDiscoveryHttpMessageHandler resolves "apiservice" using the injected env vars
```

## Key Points

- Endpoint injection uses `services__{name}__{endpoint}__{index}` pattern
- The resource name in the variable comes from the `AddProject()` call, not the C# variable
- Index is zero-based, supporting multiple bindings
- Service discovery uses these variables transparently — you write `http://apiservice`
