# AddProject<T> — .NET Services

`AddProject<T>()` registers a .NET project as an Aspire resource. The generic type parameter `T` comes from auto-generated project references.

---

## Basic Usage

```csharp
var api = builder.AddProject<Projects.AtlasApi>("api");
```

This tells Aspire: "there is a .NET project called AtlasApi — start it, give it a name, and make it discoverable."

## How Project References Work

In a .csproj AppHost, you add a `<ProjectReference>`:

```xml
<ProjectReference Include="../Atlas.Api/Atlas.Api.csproj" />
```

The Aspire SDK generates a `Projects.AtlasApi` type that you use as the generic parameter. In a single-file AppHost, you use a directive:

```csharp
#:project ../Atlas.Api/Atlas.Api.csproj
```

In TypeScript and Python AppHosts, you pass the path directly:

```typescript
const api = await builder.addProject("api", "../Atlas.Api");
```

```python
api = builder.add_project("api", "../Atlas.Api")
```

## Replicas

Run multiple instances of the same service for load testing:

```csharp
builder.AddProject<Projects.AtlasApi>("api")
    .WithReplicas(3);
```

DCP creates three instances, each with its own port. Service discovery load-balances across them.

## External Endpoints

Expose a service publicly (outside the Aspire network):

```csharp
builder.AddProject<Projects.AtlasApi>("api")
    .WithExternalHttpEndpoints();
```

## Launch Profiles

By default, Aspire uses the `http` launch profile from `launchSettings.json`. Override with:

```csharp
builder.AddProject<Projects.AtlasApi>("api")
    .WithLaunchProfile("https");
```
