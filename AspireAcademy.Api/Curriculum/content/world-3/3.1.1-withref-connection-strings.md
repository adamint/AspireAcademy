# WithReference for Connection String Resources

`WithReference()` is the glue that wires Aspire services together. When the referenced resource implements `IResourceWithConnectionString`, Aspire injects a `ConnectionStrings__` environment variable into the consuming service.

## How It Works

```csharp
var postgres = builder.AddPostgres("pgserver").AddDatabase("inventorydb");
var api = builder.AddProject<Projects.ApiService>("api")
    .WithReference(postgres);
```

This injects an environment variable into the `api` service:

```
ConnectionStrings__inventorydb=Host=localhost;Port=5432;Username=postgres;Password=xyz;Database=inventorydb
```

## The Naming Convention

The environment variable name follows the pattern `ConnectionStrings__{resourceName}`. The double underscore (`__`) is .NET's standard environment variable delimiter, mapping to `ConnectionStrings:resourceName` in the `IConfiguration` hierarchy.

For the example above, the key is `inventorydb` — the database resource name, not the server name `pgserver`.

## Which Resources Use This Pattern?

Any resource implementing `IResourceWithConnectionString`:

- **Databases**: PostgreSQL, SQL Server, MongoDB, MySQL
- **Caches**: Redis
- **Message brokers**: RabbitMQ, Kafka
- **Resources created with `AddConnectionString()`**

## The Connection String Value

The value is the fully resolved connection string, constructed by the resource's `ConnectionStringExpression`. After DCP allocates ports and endpoints, the expression resolves to a complete connection string with host, port, credentials, and database name.

## Consuming in Service Code

```csharp
// In the service's Program.cs
var connectionString = builder.Configuration.GetConnectionString("inventorydb");
// Returns: "Host=localhost;Port=5432;Username=postgres;Password=xyz;Database=inventorydb"
```

Or use Aspire client integration packages that abstract this:

```csharp
builder.AddNpgsqlDataSource("inventorydb"); // Reads GetConnectionString("inventorydb") internally
```

## Key Points

- `WithReference()` for `IResourceWithConnectionString` injects `ConnectionStrings__{name}`
- The name comes from the resource name passed to `AddDatabase()`, `AddRedis()`, etc.
- `__` maps to `:` in .NET configuration hierarchy
- Service code reads via `GetConnectionString("name")` or client integration packages
