# HTTP and gRPC Communication

Using HttpClient and gRPC channels with Aspire service discovery.

## HTTP Communication (Default)

Most Aspire services communicate via HTTP using `HttpClient` with service discovery:

```csharp
// AppHost
var api = builder.AddProject<Projects.Api>("api");
builder.AddProject<Projects.Web>("web").WithReference(api);

// Web's Program.cs
builder.Services.AddHttpClient<ApiClient>(client =>
{
    client.BaseAddress = new Uri("https://api");
});
```

Service discovery resolves `https://api` to the actual URL.

## gRPC Communication

For high-performance inter-service communication, use gRPC:

```csharp
// AppHost — same setup
var grpcService = builder.AddProject<Projects.GrpcService>("grpc");
builder.AddProject<Projects.Api>("api").WithReference(grpcService);

// Api's Program.cs
builder.Services.AddGrpcClient<Greeter.GreeterClient>(options =>
{
    options.Address = new Uri("https://grpc");
});
```

gRPC uses HTTP/2 under the hood. Service discovery works the same way — resolve the service name to the actual endpoint.

## When to Use Which

| Factor | HTTP/REST | gRPC |
|--------|-----------|------|
| Simplicity | ✅ Simple, well-known | More setup (proto files) |
| Performance | Good | ✅ Better (binary, HTTP/2) |
| Streaming | Limited | ✅ Bidirectional streaming |
| Browser support | ✅ Universal | Limited (gRPC-Web) |
| Tooling | ✅ curl, Postman | Specialized tools |

## Both Work with Aspire

The key point: **Aspire's service discovery works with any HTTP-based protocol**. Whether your services use REST, gRPC, or GraphQL, the same `WithReference` + service discovery pattern applies.

```csharp
// All three use the same service discovery infrastructure
builder.Services.AddHttpClient("rest-api", c => c.BaseAddress = new Uri("https://api"));
builder.Services.AddGrpcClient<MyService.MyServiceClient>(o => o.Address = new Uri("https://grpc"));
builder.Services.AddGraphQLClient("https://graphql");
```

## Resilience

`AddStandardResilienceHandler()` applies to all `HttpClient` instances, including those used by gRPC clients. Retry, circuit breaker, and timeout are automatic.
