# gRPC Service-to-Service Communication

gRPC services in Aspire follow the same `WithReference` pattern as HTTP services, with additional considerations for HTTP/2 transport.

## AppHost Setup

```csharp
var grpcService = builder.AddProject<Projects.GrpcService>("grpcservice");
var api = builder.AddProject<Projects.ApiService>("api")
    .WithReference(grpcService);
```

The `api` service receives `services__grpcservice__http__0` and/or `services__grpcservice__https__0`.

## Consuming gRPC Services

In the consuming service, create a gRPC channel using the resource name:

```csharp
builder.AddServiceDefaults();

builder.Services.AddGrpcClient<Greeter.GreeterClient>(options =>
    options.Address = new Uri("http://grpcservice"));
```

The service discovery handler resolves `http://grpcservice` to the actual endpoint, just like HTTP.

## HTTP/2 Considerations

gRPC requires HTTP/2. There are two approaches:

**1. HTTPS with ALPN negotiation (recommended):**
```csharp
builder.Services.AddGrpcClient<Greeter.GreeterClient>(options =>
    options.Address = new Uri("https://grpcservice"));
```

HTTPS handles HTTP/2 negotiation automatically via ALPN.

**2. Plain HTTP/2 (for local development):**

When using `http://` without TLS, Kestrel must be configured to accept HTTP/2 directly:

```csharp
// In the gRPC service's Program.cs
builder.WebHost.ConfigureKestrel(options =>
{
    options.ConfigureEndpointDefaults(listenOptions =>
    {
        listenOptions.Protocols = HttpProtocols.Http2;
    });
});
```

Or use `AppContext.SetSwitch("System.Net.Http.SocketsHttpHandler.Http2UnencryptedSupport", true)` on the client side.

## Endpoint Configuration

If the gRPC service uses a dedicated port:

```csharp
var grpcService = builder.AddProject<Projects.GrpcService>("grpcservice")
    .WithEndpoint("grpc", endpoint =>
    {
        endpoint.UriScheme = "http";
        endpoint.Transport = "http2";
    });
```

## Key Points

- gRPC services use the same `WithReference()` and service discovery pattern as HTTP
- `AddGrpcClient<T>()` with `http://resourcename` is the cleanest approach
- HTTPS handles HTTP/2 negotiation automatically
- Plain HTTP requires explicit HTTP/2 configuration on Kestrel
- Service discovery resolves gRPC endpoints the same way as HTTP endpoints
