# How Hosting Integrations Build Connection Strings

Each Aspire hosting integration constructs its connection string from endpoints, parameters, and resource properties. Understanding these patterns helps you predict the injected values and debug connection issues.

## Pattern: Endpoint + Credentials

Most database integrations compose a connection string from an allocated endpoint and credential parameters:

```csharp
// PostgreSQL pattern (simplified)
// Host={endpoint.Host};Port={endpoint.Port};Username=postgres;Password={password};Database={dbName}
var postgres = builder.AddPostgres("pg").AddDatabase("mydb");
// Resolves to: Host=localhost;Port=5432;Username=postgres;Password=abc123;Database=mydb
```

## Pattern: Simple Endpoint

Some resources use just the endpoint:

```csharp
// Redis pattern (simplified)
// {endpoint.Host}:{endpoint.Port}
var redis = builder.AddRedis("cache");
// Resolves to: localhost:6379
```

## Pattern: URI Format

Others use URI-style connection strings:

```csharp
// RabbitMQ pattern (simplified)
// amqp://guest:{password}@{endpoint.Host}:{endpoint.Port}
var rabbit = builder.AddRabbitMQ("messaging");
// Resolves to: amqp://guest:abc123@localhost:5672
```

## Pattern: MongoDB

```csharp
// MongoDB pattern (simplified)
// mongodb://admin:{password}@{endpoint.Host}:{endpoint.Port}?authSource=admin&authMechanism=SCRAM-SHA-256
var mongo = builder.AddMongoDB("mongo");
// Resolves to: mongodb://admin:abc123@localhost:27017?authSource=admin&authMechanism=SCRAM-SHA-256
```

MongoDB defaults to authentication enabled with username `"admin"` and an auto-generated password. The `authSource=admin` and `authMechanism=SCRAM-SHA-256` query parameters are included automatically.

## Tracing a Connection String

To understand what a specific integration produces, check:

1. The resource class's `ConnectionStringExpression` property
2. The hosting extension method (e.g., `AddPostgres`) where the expression is built
3. The endpoint and parameter configurations

## The Parent-Child Pattern

Server resources often have child database resources:

```csharp
var pgServer = builder.AddPostgres("pg");        // Server with password, endpoint
var myDb = pgServer.AddDatabase("mydb");           // Child adds ;Database=mydb

builder.AddProject<Projects.Api>("api")
    .WithReference(myDb);  // ConnectionStrings__mydb includes Database=mydb
```

The child resource appends its specific properties to the parent's connection string.

## Key Points

- Each integration builds connection strings from endpoints + credentials + resource-specific properties
- Database child resources append their name to the parent's connection string
- Actual values resolve at runtime after port allocation
- Understanding the pattern helps debug "connection refused" and "auth failed" errors
