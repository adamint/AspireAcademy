# Custom Connection String Formats

When built-in hosting integrations don't cover your needs, you can create custom connection strings using `AddConnectionString()` with a `ReferenceExpression` or callback.

## AddConnectionString with ReferenceExpression

Build a connection string from dynamic components:

```csharp
var customDb = builder.AddContainer("mydb", "custom-db:latest")
    .WithEndpoint(targetPort: 9042, scheme: "tcp", name: "cql");

var dbEndpoint = customDb.GetEndpoint("cql");
var password = builder.AddParameter("db-pass", secret: true);

var connStr = builder.AddConnectionString("mydb",
    ReferenceExpression.Create(
        $"Contact Points={dbEndpoint.Property(EndpointProperty.Host)};Port={dbEndpoint.Property(EndpointProperty.Port)};Password={password}"
    ));
```

This creates a resource implementing `IResourceWithConnectionString` that other resources can reference:

```csharp
builder.AddProject<Projects.Api>("api")
    .WithReference(connStr);
// Injects: ConnectionStrings__mydb=Contact Points=localhost;Port=9042;Password=secret
```

## AddConnectionString with Callback

Use the callback overload for builder-pattern composition:

```csharp
var connStr = builder.AddConnectionString("mydb", connStrBuilder =>
{
    connStrBuilder.AppendLiteral("Contact Points=");
    connStrBuilder.AppendFormatted(dbEndpoint.Property(EndpointProperty.Host));
    connStrBuilder.AppendLiteral(";Port=");
    connStrBuilder.AppendFormatted(dbEndpoint.Property(EndpointProperty.Port));
});
```

## Simple External Connection String

For externally-managed infrastructure where the connection string is already known:

```csharp
var legacyDb = builder.AddConnectionString("legacy");
// Reads from ConnectionStrings:legacy in configuration
```

## When to Build Custom Connection Strings

- **Niche databases** without Aspire hosting packages (Cassandra, CockroachDB)
- **Custom protocols** with non-standard connection string formats
- **Composite connections** combining multiple resources
- **Proxy connections** that need custom host/port routing

## Key Points

- `AddConnectionString(name, expression)` creates a custom connection string resource
- The expression resolves at runtime with actual endpoint values
- The resulting resource implements `IResourceWithConnectionString` for `WithReference()`
- Use for databases and services without dedicated Aspire hosting packages
