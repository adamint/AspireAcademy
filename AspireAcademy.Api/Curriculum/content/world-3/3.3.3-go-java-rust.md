# Go, Java, and Rust Resources

Aspire supports Go, Java, and Rust services through the Community Toolkit and the underlying `AddExecutable` primitive.

---

## AddExecutable — The Universal Escape Hatch

Every non-.NET resource in Aspire ultimately becomes an `ExecutableResource`. `AddExecutable` is the raw primitive:

```csharp
builder.AddExecutable("worker", "go", "../bazaar-worker", "run", ".")
    .WithReference(rabbitmq)
    .WaitFor(rabbitmq);
```

Parameters: resource name, executable name, working directory, then any arguments. DCP spawns the process directly — no containerization, no build system awareness. It's `Process.Start()` with Aspire's environment variable injection layered on top.

Detects Go modules via go.mod. Sets GOPATH and injects environment variables.\n- **AddJavaApp**: Detects Maven (pom.xml) or Gradle (build.gradle) and runs the appropriate build command. Handles JAVA_HOME detection.\n- **AddRustApp**: Runs 'cargo build' then starts the compiled binary from target/debug or target/release.\n\nCritically, these methods don't add health checks — non-.NET services don't have Aspire's health check middleware. If you need DCP to verify your Go service is healthy before starting dependents, you must add a custom health check:\n\nbuilder.AddGoApp(\"worker\", \"../worker\")\n    .WithHttpHealthCheck(\"/healthz\");\n\nWithout this, .WaitFor() only waits for the process to start — not for it to be ready to accept requests."}

## Go

Using the Community Toolkit:

```csharp
builder.AddGoApp("worker", "../bazaar-worker")
    .WithReference(rabbitmq);
```

Go reads connection info from environment variables:

```go
apiURL := os.Getenv("services__api__http__0")
rabbitURL := os.Getenv("ConnectionStrings__messaging")
```

## Java / Spring

```csharp
builder.AddJavaApp("catalog", "../catalog-service")
    .WithReference(postgres)
    .WaitFor(postgres);
```

Spring Boot services can read Aspire's environment variables directly — Spring's `@Value("${ConnectionStrings__postgres}")` or `Environment.getProperty()` work without any Aspire-specific library.

## Rust

```csharp
builder.AddRustApp("indexer", "../indexer-service")
    .WithReference(redis);
```

Rust services use `cargo run` and read environment variables with `std::env::var()`.

## The Common Pattern

Regardless of language, every service:
1. Receives connection strings as `ConnectionStrings__*` environment variables
2. Receives service URLs as `services__*` environment variables
3. Appears in the Dashboard with state tracking
4. Is included in the publishing pipeline for deployment

> 🌍 **Real World:** Many teams use Aspire to orchestrate polyglot microservice architectures — a .NET API gateway, a Python ML service, a Go worker, and a React frontend — all managed by a single AppHost. The common environment variable pattern makes this seamless.
