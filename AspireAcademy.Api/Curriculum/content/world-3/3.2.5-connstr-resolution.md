# Connection String Resolution Order

Aspire uses a layered resolution strategy for connection strings. Understanding this order helps you override values per environment and debug why a service connects to the wrong endpoint.

## Resolution Priority

When a service calls `GetConnectionString("mydb")`, .NET configuration resolves it in this order (highest priority first):

1. **Environment variable**: `ConnectionStrings__mydb` (injected by the AppHost)
2. **User secrets**: `ConnectionStrings:mydb` in user secrets
3. **appsettings.{Environment}.json**: Environment-specific overrides
4. **appsettings.json**: Base configuration file

In a standard Aspire app, the AppHost injects the environment variable (#1), which takes priority over everything else.

## Overriding in the AppHost

The AppHost can also override connection strings from its own configuration. If `ConnectionStrings:mydb` exists in the AppHost's configuration, it may be used instead of the dynamically-composed value — depending on the resource type.

For `AddConnectionString("mydb")`, the value always comes from configuration:

```json
{
  "ConnectionStrings": {
    "mydb": "Host=prod-server;Database=mydb;..."
  }
}
```

## Dynamic vs Static Resolution

| Resource Type | Resolution |
|--------------|------------|
| `AddPostgres().AddDatabase()` | Dynamic — composed from endpoints and parameters |
| `AddRedis()` | Dynamic — composed from endpoint |
| `AddConnectionString()` | Static — read from configuration |

Dynamic resources compose their connection string at runtime. Static resources (`AddConnectionString()`) read a pre-configured value.

## Debugging Connection Issues

If a service connects to the wrong host/port:

1. Check the dashboard — environment variables are visible per resource
2. Verify the resource name matches what the service expects
3. Check for configuration overrides in `appsettings.json` or user secrets
4. For dynamic resources, verify the endpoint allocation in the dashboard

## Key Points

- Environment variables (from the AppHost) have the highest priority in standard .NET configuration
- Dynamic resources compose connection strings from runtime values
- `AddConnectionString()` reads from `ConnectionStrings:name` in the AppHost's configuration
- The dashboard shows all injected environment variables for debugging
