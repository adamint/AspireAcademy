# Environment Variable Naming Conventions

The ConnectionStrings__ and services__ naming patterns and how consuming services read them.

## Connection String Pattern

For resources with `IResourceWithConnectionString`:

```
ConnectionStrings__{resourceName}={connectionString}
```

Examples:
```
ConnectionStrings__cache=localhost:6379
ConnectionStrings__mydb=Host=localhost;Port=5432;Username=postgres;Password=...
ConnectionStrings__rabbitmq=amqp://guest:guest@localhost:5672
```

Services read these with `IConfiguration`:
```csharp
var connStr = builder.Configuration.GetConnectionString("cache");
```

Or through Aspire client integrations:
```csharp
builder.AddRedisClient("cache");        // reads ConnectionStrings__cache
builder.AddNpgsqlDataSource("mydb");    // reads ConnectionStrings__mydb
```

## Service Discovery Pattern

For resources with `IResourceWithServiceDiscovery`:

```
services__{resourceName}__{scheme}__{index}={url}
```

Examples:
```
services__api__https__0=https://localhost:5001
services__api__http__0=http://localhost:5000
services__catalog__https__0=https://localhost:5010
```

Services resolve these through service discovery:
```csharp
var client = httpClientFactory.CreateClient();
var response = await client.GetAsync("https://api/weatherforecast");
// "api" is resolved to https://localhost:5001 via service discovery
```

## Custom Connection Name

Override the default name in `WithReference`:
```csharp
builder.AddProject<Projects.Api>("api")
       .WithReference(redis, connectionName: "my-redis");
// Injects: ConnectionStrings__my-redis (not ConnectionStrings__cache)
```

## Double Underscore

The `__` separator is .NET's convention for nested configuration keys. `ConnectionStrings__cache` maps to:
```json
{
  "ConnectionStrings": {
    "cache": "localhost:6379"
  }
}
```

This works because .NET configuration flattens environment variables using `__` as the hierarchy separator.
