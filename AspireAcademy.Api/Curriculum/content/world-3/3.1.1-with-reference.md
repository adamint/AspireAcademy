# WithReference — Connection String vs Endpoint Injection

How WithReference decides between injecting ConnectionStrings__ and services__ environment variables.

## The Two Modes

`WithReference` behaves differently based on what the source resource implements:

### Connection String Resources

If the source implements `IResourceWithConnectionString` (databases, caches, message brokers):

```csharp
var redis = builder.AddRedis("cache");
builder.AddProject<Projects.Api>("api").WithReference(redis);
```

Aspire injects: `ConnectionStrings__cache=localhost:6379`

### Service Discovery Resources

If the source implements `IResourceWithServiceDiscovery` (projects, executables):

```csharp
var api = builder.AddProject<Projects.Api>("api");
builder.AddProject<Projects.Web>("web").WithReference(api);
```

Aspire injects service discovery variables:
```
services__api__https__0=https://localhost:5001
services__api__http__0=http://localhost:5000
```

## The Decision Flow

```
WithReference(source)
  ├── source has IResourceWithConnectionString?
  │     └── Yes → inject ConnectionStrings__{name}
  └── source has IResourceWithServiceDiscovery?
        └── Yes → inject services__{name}__{scheme}__{index}
```

## Optional Connection Name

Override the default connection name:

```csharp
builder.AddProject<Projects.Api>("api")
       .WithReference(redis, connectionName: "my-cache");
// Injects: ConnectionStrings__my-cache
```

## Endpoint-Specific Reference

Inject a single endpoint instead of all endpoints:

```csharp
builder.AddProject<Projects.Web>("web")
       .WithReference(api.GetEndpoint("https"));
// Injects only the HTTPS endpoint
```

## Resource Relationships

`WithReference` also creates a **relationship** in the Dashboard between the source and destination resources, visible in the resource graph view.
