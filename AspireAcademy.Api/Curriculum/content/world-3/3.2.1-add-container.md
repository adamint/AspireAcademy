# AddContainer — Running Any Image

`AddContainer` is the raw way to add a Docker or Podman container to your app model. It works with any container image — no hosting integration required.

---

## Basic Usage

```csharp
// Uses 'latest' tag
builder.AddContainer("nginx", "nginx");

// With explicit tag
builder.AddContainer("nginx", "nginx", "1.27");

// Same using chaining
builder.AddContainer("nginx", "nginx").WithImageTag("1.27");
```

The first parameter is the resource name. The second is the Docker image name. The optional third is the image tag.

In TypeScript and Python:

```typescript
const nginx = await builder.addContainer("nginx", "nginx", "1.27");
```

```python
nginx = builder.add_container("nginx", "nginx", "1.27")
```

## Adding Endpoints

Containers do not expose endpoints by default. Add them explicitly:

```csharp
var nginx = builder.AddContainer("nginx", "nginx")
    .WithHttpEndpoint(port: 8080, targetPort: 80);
```

> 📝 **`port`** vs **`targetPort`**: `port` is what you connect to from the host (your browser). `targetPort` is what the process inside the container listens on. These are often different — nginx listens on 80 inside the container, but you might map it to 8080 on your machine.

## Environment Variables

```csharp
builder.AddContainer("nginx", "nginx")
    .WithEnvironment("NGINX_HOST", "example.com")
    .WithEnvironment("NGINX_PORT", "80");
```

## When to Use AddContainer vs Hosting Integrations

| Use `AddContainer` when… | Use hosting integrations when… |
|---|---|
| No integration package exists | One exists (`AddRedis`, `AddPostgres`, etc.) |
| You need a custom or niche image | You want automatic health checks & connection strings |
| You want full control over config | You want consistency with the Aspire ecosystem |

> 📋 **Key Takeaways**
> - `AddContainer` works with any Docker/Podman image — it's the escape hatch for anything without a hosting integration
> - Endpoints must be added explicitly; containers are isolated by default
> - Hosting integrations (`AddRedis`, etc.) are preferred when available — they handle health checks, connection strings, and port config automatically
