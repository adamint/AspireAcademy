# The IResourceWithConnectionString Interface

`IResourceWithConnectionString` is the interface that marks a resource as capable of providing a connection string. Understanding this interface explains how databases, caches, and message brokers expose their connection information.

## The Interface

```csharp
public interface IResourceWithConnectionString : IResource, IManifestExpressionProvider, IValueProvider, IValueWithReferences
{
    ReferenceExpression ConnectionStringExpression { get; }

    ValueTask<string?> GetConnectionStringAsync(CancellationToken cancellationToken = default);

    string? ConnectionStringEnvironmentVariable => null;

    IEnumerable<KeyValuePair<string, ReferenceExpression>> GetConnectionProperties() => [];
}
```

Key members:

- **`ConnectionStringExpression`**: A `ReferenceExpression` that defines how the connection string is composed from dynamic parts (host, port, credentials)
- **`GetConnectionStringAsync()`**: Resolves the expression to the final string value
- **`ConnectionStringEnvironmentVariable`**: Optional override for the environment variable name (defaults to `ConnectionStrings__{name}`)
- **`GetConnectionProperties()`**: Returns individual connection properties (host, port, username, etc.) as key-value pairs, used when `ConnectionProperties` injection is enabled

## How Resources Implement It

Database resources compose their connection string from allocated endpoints and parameters:

```csharp
// Simplified illustration of how PostgresDatabaseResource builds its connection string
ConnectionStringExpression = ReferenceExpression.Create(
    $"Host={primaryEndpoint.Property(EndpointProperty.Host)};Port={primaryEndpoint.Property(EndpointProperty.Port)};Username=postgres;Password={passwordParameter};Database={databaseName}"
);
```

The expression uses `EndpointReference` properties and `ParameterResource` values — all of which resolve at runtime after DCP allocates ports.

## Why It's an Expression

The connection string can't be a static string because:

1. **Ports are allocated dynamically** by DCP
2. **Passwords may be generated** by `GenerateParameterDefault`
3. **Hosts change** between local development and deployed environments

`ReferenceExpression` defers resolution until all dependencies are ready.

## How WithReference Uses It

When `WithReference()` detects `IResourceWithConnectionString`, it:

1. Creates a `ConnectionStringReference` wrapping the resource
2. Registers an environment callback that calls `GetConnectionStringAsync()`
3. Injects the result as `ConnectionStrings__{name}`

## Key Points

- `IResourceWithConnectionString` provides `ConnectionStringExpression` and `GetConnectionStringAsync()`
- Connection strings are composed from dynamic references (endpoints, parameters)
- Resolution happens at runtime when actual ports and values are known
- `WithReference()` detects this interface and uses connection string injection
