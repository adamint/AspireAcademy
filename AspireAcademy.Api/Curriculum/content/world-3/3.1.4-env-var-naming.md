# Environment Variable Naming Conventions

Aspire follows .NET's standard environment variable configuration provider conventions. Knowing the exact naming rules helps you debug injection issues and understand how configuration flows.

## Connection String Variables

Pattern: `ConnectionStrings__{resourceName}`

```
ConnectionStrings__cache → ConnectionStrings:cache
ConnectionStrings__mydb  → ConnectionStrings:mydb
```

Access in code: `builder.Configuration.GetConnectionString("cache")`

The resource name comes from the `Add*()` call — `AddRedis("cache")`, `AddDatabase("mydb")`, etc.

## Endpoint Variables

Pattern: `services__{resourceName}__{endpointName}__{index}`

```
services__apiservice__http__0  → services:apiservice:http:0
services__apiservice__https__0 → services:apiservice:https:0
```

Access in code: `builder.Configuration["services:apiservice:http:0"]`

## The Double Underscore Rule

`__` (double underscore) is the standard .NET delimiter for hierarchical configuration in environment variables. Colons (`:`) are not valid in environment variable names on Linux, so `__` is the cross-platform substitute.

```
Environment variable: ConnectionStrings__mydb
Configuration path:   ConnectionStrings:mydb
```

## Resource Name Preservation

Resource names are used as-is in environment variables — including hyphens:

```csharp
builder.AddRedis("my-cache");
// → ConnectionStrings__my-cache
```

Access: `builder.Configuration.GetConnectionString("my-cache")`

## Database vs Server Names

For parent-child resources, the **child** name is the key:

```csharp
var pg = builder.AddPostgres("pgserver")  // server name
    .AddDatabase("orders");                // database name

builder.AddProject<Projects.Api>("api").WithReference(pg);
// Injects: ConnectionStrings__orders (not ConnectionStrings__pgserver)
```

## Index for Multiple Endpoints

The zero-based index supports multiple bindings:

```
services__api__http__0=http://localhost:5000
services__api__http__1=http://localhost:5001  // if multiple bindings exist
```

## Key Points

- `__` maps to `:` in .NET configuration
- Connection strings: `ConnectionStrings__{name}` → `GetConnectionString("name")`
- Endpoints: `services__{name}__{endpoint}__{index}` → `["services:name:endpoint:index"]`
- Resource names (including hyphens) are preserved exactly
- Child resource names (databases) are used as keys, not parent server names
