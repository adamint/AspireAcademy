# Endpoints: WithEndpoint and WithHttpEndpoint

Endpoints define how resources are reachable over the network. They control ports, URI schemes, and how other resources discover them.

---

## WithHttpEndpoint

The most common method — adds an HTTP or HTTPS endpoint:

```csharp
builder.AddContainer("nginx", "nginx")
    .WithHttpEndpoint(port: 8080, targetPort: 80);
```

| Parameter | Meaning |
|---|---|
| `port` | Host port — what you connect to from outside |
| `targetPort` | Container/process port — what the service listens on |
| `name` | Endpoint name (default: "http") — used in service discovery |
| `env` | Environment variable name to inject the allocated port |
| `isProxied` | Whether DCP proxies the endpoint (default: true) |

## WithEndpoint (Generic)

For non-HTTP protocols:

```csharp
builder.AddContainer("mqtt-broker", "eclipse-mosquitto")
    .WithEndpoint("mqtt", e =>
    {
        e.Port = 1883;
        e.UriScheme = "mqtt";
    });
```

## Port Allocation

If you omit `port`, DCP allocates a random available port. This prevents conflicts when running multiple Aspire apps:

```csharp
builder.AddProject<Projects.Api>("api")
    .WithHttpEndpoint(targetPort: 5000); // DCP picks the host port
```

## Endpoint Environment Variables

Use the `env` parameter to inject the port into the service:

```csharp
builder.AddViteApp("frontend", "../web")
    .WithHttpEndpoint(port: 5173, env: "PORT");
```

The TypeScript process receives `PORT=5173` as an environment variable.

## How Other Resources Discover Endpoints

When you call `.WithReference(api)` on another resource, Aspire injects the API's endpoint URL:

```
services__api__http__0=http://localhost:XXXXX
```

The exact port is resolved by DCP at runtime.
