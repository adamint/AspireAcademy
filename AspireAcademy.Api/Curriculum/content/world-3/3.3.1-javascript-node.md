# JavaScript & Node Resources

Aspire has first-class support for JavaScript and Node.js applications through `AddNodeApp()` and `AddViteApp()`.

---

## AddNodeApp

```csharp
builder.AddNodeApp("api", "../express-api")
    .WithHttpEndpoint(env: "PORT");
```

Aspire detects the package manager (npm, yarn, pnpm) from the lock file, runs `install`, then starts the app with `npm start` (or the equivalent).

## AddViteApp

For frontend apps using Vite (React, Vue, Svelte, etc.):

```csharp
builder.AddViteApp("frontend", "../bazaar-web")
    .WithHttpEndpoint(port: 5173, env: "PORT")
    .WithReference(api);
```

`AddViteApp` starts the Vite development server. It:
- Detects the package manager from the lock file
- Runs dependency installation
- Starts `vite dev` with the correct port

When the API resource is passed via `WithReference`, the API's URL is injected as an environment variable that the frontend can read.

## TypeScript AppHost Equivalent

In a TypeScript AppHost, the same calls look like:

```typescript
const api = await builder.addNodeApp("api", "../express-api");
await api.withHttpEndpoint({ env: "PORT" });

const frontend = await builder.addViteApp("frontend", "../bazaar-web");
await frontend.withReference(api);
```

## Package Manager Detection

Aspire detects the package manager from lock files:

| File | Package Manager |
|---|---|
| `package-lock.json` | npm |
| `yarn.lock` | yarn |
| `pnpm-lock.yaml` | pnpm |
| `bun.lockb` | bun (via Community Toolkit) |
