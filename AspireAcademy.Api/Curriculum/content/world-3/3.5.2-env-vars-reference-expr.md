# Environment Variables and ReferenceExpression

`WithEnvironment` injects environment variables into resources. `ReferenceExpression` resolves dynamic values — like endpoints and connection strings — at runtime.

---

## Static Environment Variables

```csharp
builder.AddProject<Projects.Api>("api")
    .WithEnvironment("LOG_LEVEL", "Debug")
    .WithEnvironment("FEATURE_FLAG_X", "true");
```

## Callback-Based Environment Variables

For dynamic values, use a callback:

```csharp
builder.AddProject<Projects.Api>("api")
    .WithEnvironment(context =>
    {
        context.EnvironmentVariables["STARTUP_TIME"] = DateTime.UtcNow.ToString("O");
    });
```

The callback runs when DCP starts the resource, so the value is resolved at startup.

## ReferenceExpression

When you use `WithReference()`, Aspire uses `ReferenceExpression` internally to build the environment variable value:

```csharp
// This:
api.WithReference(postgres);

// Internally creates a ReferenceExpression that resolves to:
// ConnectionStrings__bazaar-db=Host={pg.bindings.tcp.host};Port={pg.bindings.tcp.port};...
```

The expression contains **placeholders** that DCP resolves to actual values when the resource starts.

## Custom ReferenceExpressions

You can build your own:

```csharp
var api = builder.AddProject<Projects.Api>("api")
    .WithHttpEndpoint(name: "http");

builder.AddContainer("monitor", "monitor-image")
    .WithEnvironment("TARGET_URL", api.GetEndpoint("http"));
```

`GetEndpoint("http")` returns a `ReferenceExpression` that resolves to the API's HTTP endpoint URL at runtime.

## Reading Environment Variables

Every language reads these the same way — `Environment.GetEnvironmentVariable()` in C#, `process.env` in Node, `os.environ` in Python, `os.Getenv()` in Go. This is covered in detail in the [polyglot resources lessons](3.3.1).

> 🎲 **Fun Fact:** ReferenceExpression uses the same placeholder syntax as Aspire's deployment manifests. When you publish, the same `{api.bindings.http.url}` placeholders appear in the generated Bicep/Terraform templates — they're resolved by the cloud provider instead of DCP.
