# Connection String vs Endpoint Injection

When you call `WithReference()`, Aspire chooses between two injection patterns based on the resource's interfaces. Understanding this decision logic is critical for predicting how your services receive configuration.

## The Decision Logic

`WithReference()` has multiple overloads. The compiler selects the overload based on the resource type:

1. **`IResourceWithConnectionString`** → Connection string injection (`ConnectionStrings__{name}`)
2. **`IResourceWithServiceDiscovery`** → Endpoint injection (`services__{name}__{endpoint}__{index}`)

If a resource implements both interfaces, the connection string overload takes priority because it's more specific.

## Built-In Resource Types

| Resource Type | Interface | Injection Pattern |
|--------------|-----------|-------------------|
| `PostgresServerResource` / `PostgresDatabaseResource` | `IResourceWithConnectionString` | `ConnectionStrings__` |
| `RedisResource` | `IResourceWithConnectionString` | `ConnectionStrings__` |
| `SqlServerServerResource` | `IResourceWithConnectionString` | `ConnectionStrings__` |
| `MongoDBServerResource` | `IResourceWithConnectionString` | `ConnectionStrings__` |
| `RabbitMQServerResource` | `IResourceWithConnectionString` | `ConnectionStrings__` |
| `ProjectResource` | `IResourceWithServiceDiscovery` | `services__` |
| `ExecutableResource` | `IResourceWithServiceDiscovery` | `services__` |
| `ContainerResource` (generic) | Depends on configuration | Varies |

## Example: Mixed References

```csharp
var redis = builder.AddRedis("cache");
var api = builder.AddProject<Projects.ApiService>("api");

builder.AddProject<Projects.Worker>("worker")
    .WithReference(redis)   // → ConnectionStrings__cache
    .WithReference(api);    // → services__api__http__0
```

## Database Child Resources

When you call `AddDatabase()` on a server, the child resource name is used:

```csharp
var pg = builder.AddPostgres("pgserver").AddDatabase("mydb");
// WithReference(pg) → ConnectionStrings__mydb (not pgserver)
```

## Key Points

- `IResourceWithConnectionString` → `ConnectionStrings__` (databases, caches, brokers)
- `IResourceWithServiceDiscovery` → `services__` (projects, executables)
- The compiler selects the overload — you don't choose manually
- For database child resources, the connection string key uses the database name
