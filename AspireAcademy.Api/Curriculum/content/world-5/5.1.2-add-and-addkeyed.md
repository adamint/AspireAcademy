# Add and AddKeyed Registration

Client integrations offer two registration styles: `Add{Component}` for a single default client and `AddKeyed{Component}` for multiple named instances using .NET 8's keyed dependency injection.

## Single Client Registration

The standard pattern registers one client per technology. The connection name maps to `ConnectionStrings:{name}` in configuration:

```csharp
// Registers a single NpgsqlDataSource as a singleton
builder.AddNpgsqlDataSource("catalogdb");
```

You resolve this client normally through constructor injection:

```csharp
public class CatalogService(NpgsqlDataSource dataSource)
{
    // dataSource is the single registered instance
}
```

## Keyed Registration for Multiple Instances

When you need multiple clients of the same type — for example, two Redis caches with different purposes — use `AddKeyed{Component}`:

```csharp
builder.AddKeyedRedisClient("session-cache");
builder.AddKeyedRedisClient("output-cache");
```

Each keyed registration reads from its own `ConnectionStrings:{name}` entry. Resolve keyed services using the `[FromKeyedServices]` attribute:

```csharp
public class MyService(
    [FromKeyedServices("session-cache")] IConnectionMultiplexer sessionCache,
    [FromKeyedServices("output-cache")] IConnectionMultiplexer outputCache)
{
    // Two separate Redis connections, each with its own config
}
```

## Connection Name to Configuration Mapping

The connection name you provide drives configuration lookup:

```json
{
  "ConnectionStrings": {
    "session-cache": "localhost:6379,defaultDatabase=0",
    "output-cache": "localhost:6379,defaultDatabase=1"
  }
}
```

## When to Use Each

| Pattern | Use When |
|---|---|
| `Add{Component}` | One instance of that resource type is sufficient |
| `AddKeyed{Component}` | You need multiple distinct connections of the same type |

Keyed DI is a first-class .NET 8 feature, and Aspire integrations leverage it consistently. Both `Add` and `AddKeyed` variants apply the same health checks, tracing, and metrics — the only difference is how you register and resolve the client.
