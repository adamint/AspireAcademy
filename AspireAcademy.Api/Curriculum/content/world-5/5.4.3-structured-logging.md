# Structured Logging

Structured logging captures log data as named properties rather than plain text strings. Aspire's dashboard leverages this structure to enable filtering, searching, and correlation across services.

## Message Templates

Use `ILogger<T>` with message templates containing named placeholders. The placeholder names become searchable properties in the dashboard.

```csharp
public class OrderService(ILogger<OrderService> logger)
{
    public async Task PlaceOrder(Order order)
    {
        logger.LogInformation(
            "Order {OrderId} placed by {CustomerId} for {Total:C}",
            order.Id, order.CustomerId, order.Total);

        // Process order...

        logger.LogInformation("Order {OrderId} fulfilled", order.Id);
    }
}
```

In the dashboard, you can filter logs by `OrderId` or `CustomerId` to see all activity related to a specific order or customer.

## Trace Correlation

When logging occurs inside an `Activity` (span), the `TraceId` and `SpanId` are automatically attached to the log entry. The dashboard renders the TraceId as a clickable link—clicking it navigates directly to the corresponding distributed trace.

This means you can jump from a log entry saying "Payment failed for Order X" straight to the full trace showing exactly which service call failed and why.

## High-Performance Logging

For hot paths, use the `[LoggerMessage]` source generator to avoid boxing and string allocation.

```csharp
public static partial class LogMessages
{
    [LoggerMessage(Level = LogLevel.Information,
        Message = "Order {OrderId} placed by {CustomerId}")]
    public static partial void OrderPlaced(
        ILogger logger, string orderId, string customerId);
}

// Usage
LogMessages.OrderPlaced(logger, order.Id, order.CustomerId);
```

## Log Level Mapping

.NET log levels map to OTLP severity levels: `Trace` → Trace, `Debug` → Debug, `Information` → Info, `Warning` → Warn, `Error` → Error, `Critical` → Fatal. The dashboard color-codes entries by severity for quick visual scanning.
