# Health Checks

Health checks let your application report its readiness and liveness status. Aspire client integrations automatically register health checks for their backing resources, and you can add custom checks for business-specific dependencies.

## Auto-Registered Checks

When you call `AddRedisClient("cache")` or `AddRabbitMQClient("messaging")`, a health check is registered automatically. These checks verify connectivity to the backing resource—Redis responds to `PING`, RabbitMQ confirms the connection is open, Kafka queries broker metadata.

No additional code is needed. The dashboard picks up these checks and displays resource health status.

## Custom Health Checks

For dependencies without built-in checks, implement `IHealthCheck`.

```csharp
public class PaymentProviderHealthCheck(HttpClient client) : IHealthCheck
{
    public async Task<HealthCheckResult> CheckHealthAsync(
        HealthCheckContext context,
        CancellationToken cancellationToken = default)
    {
        try
        {
            var response = await client.GetAsync("/health", cancellationToken);
            return response.IsSuccessStatusCode
                ? HealthCheckResult.Healthy("Payment provider is reachable")
                : HealthCheckResult.Degraded("Payment provider returned non-success status");
        }
        catch (Exception ex)
        {
            return HealthCheckResult.Unhealthy("Payment provider unreachable", ex);
        }
    }
}
```

Register the check in `Program.cs`:

```csharp
builder.Services.AddHealthChecks()
    .AddCheck<PaymentProviderHealthCheck>("payment-provider");
```

## Health Endpoints

`AddServiceDefaults()` maps two health endpoints:

- **`/health`** — Liveness probe. Returns all registered checks. Used by orchestrators to determine if the app should be restarted.
- **`/alive`** — Readiness probe. Returns a simple alive signal. Used to determine if the app is ready to accept traffic.

## Dashboard Display

The dashboard shows health as colored indicators: **green** (Healthy), **yellow** (Degraded), **red** (Unhealthy). Each resource row on the Resources page reflects the aggregate health of all registered checks.
