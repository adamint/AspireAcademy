# Redis Client Integration

Register IConnectionMultiplexer and IDistributedCache with the Redis client integration.

The `Aspire.StackExchange.Redis` package registers `IConnectionMultiplexer` for low-level Redis operations. For simple caching scenarios, a separate package provides `IDistributedCache`.

## Registering the Redis Client

In your service's `Program.cs`:

```csharp
builder.AddRedisClient("cache");
```

This registers `IConnectionMultiplexer` as a singleton with health checks, tracing, and metrics enabled.

## Using IConnectionMultiplexer

Inject it for direct Redis operations:

```csharp
public class SessionStore(IConnectionMultiplexer redis)
{
    private readonly IDatabase _db = redis.GetDatabase();

    public async Task SetAsync(string key, string value)
        => await _db.StringSetAsync(key, value, TimeSpan.FromMinutes(30));

    public async Task<string?> GetAsync(string key)
        => await _db.StringGetAsync(key);
}
```

## Distributed Cache

For `IDistributedCache` (used by ASP.NET Core session, output caching, etc.), use the distributed cache integration:

```csharp
builder.AddRedisDistributedCache("cache");
```

This registers `IDistributedCache` backed by Redis, which integrates with ASP.NET Core's caching middleware automatically.

## Configuration

Settings are under `Aspire:StackExchange:Redis`:

```json
{
  "Aspire": {
    "StackExchange": {
      "Redis": {
        "DisableHealthChecks": false,
        "DisableTracing": false
      }
    }
  }
}
```

Or configure inline:

```csharp
builder.AddRedisClient("cache", settings =>
    settings.DisableHealthChecks = true);
```

## Multiple Redis Instances

Use keyed registration for multiple Redis connections:

```csharp
builder.AddKeyedRedisClient("sessions");
builder.AddKeyedRedisClient("output-cache");
```

Then inject with `[FromKeyedServices("sessions")]` to resolve the correct instance. Each keyed client gets its own health check and telemetry pipeline.
