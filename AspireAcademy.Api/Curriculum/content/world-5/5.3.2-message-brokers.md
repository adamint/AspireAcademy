# RabbitMQ, Kafka, and NATS

Message brokers enable asynchronous, decoupled communication between services. Aspire has hosting integrations for the three most popular options.

---

## RabbitMQ

```csharp
var rabbitmq = builder.AddRabbitMQ("messaging")
    .WithManagementPlugin();  // enables the management UI on port 15672
```

- **Image**: `rabbitmq:4-management`
- **Port**: `5672` (AMQP), `15672` (Management UI)
- **Connection string**: `amqp://guest:{password}@{host}:{port}`

Atlas uses RabbitMQ for:
- API → Worker: background job dispatch
- API → ML: content indexing events

## Kafka

```csharp
var kafka = builder.AddKafka("kafka");
```

- **Image**: `confluentinc/confluent-local:latest`
- **Port**: `9092`
- **Connection string**: `{host}:{port}`

Kafka is ideal for high-throughput event streaming, log aggregation, and event sourcing.

## NATS

```csharp
var nats = builder.AddNats("nats")
    .WithJetStream();  // enables persistence
```

- **Image**: `nats:latest`
- **Port**: `4222`
- **Connection string**: `nats://{host}:{port}`

NATS is lightweight and fast — good for microservice communication and IoT.

## Choosing a Broker

| Feature | RabbitMQ | Kafka | NATS |
|---|---|---|---|
| Pattern | Queues + Pub/Sub | Event streaming | Pub/Sub + Queues |
| Ordering | Per-queue | Per-partition | Per-subject |
| Persistence | Optional | Always | JetStream |
| Performance | High | Very high | Ultra-low latency |
| Use case | Task queues, RPC | Event logs, analytics | Real-time messaging |
