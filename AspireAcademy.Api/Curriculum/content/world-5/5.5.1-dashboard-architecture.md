# Dashboard Architecture

The Aspire dashboard is a Blazor Server application that serves as the central observability hub for your distributed application. It receives, stores, and visualizes all telemetry and resource information in real time.

## Core Components

The dashboard is built on two primary data channels, each served by a separate gRPC endpoint.

### OTLP Collector Endpoint

This endpoint receives telemetry from your application services via the OpenTelemetry Protocol. Three gRPC services handle the three pillars:

- **`OtlpGrpcTraceService`** — Receives distributed trace spans from all services.
- **`OtlpGrpcLogsService`** — Receives structured log entries with severity, properties, and trace correlation.
- **`OtlpGrpcMetricsService`** — Receives metric data points (counters, histograms, gauges).

Each service deserializes the incoming protobuf messages and stores them in the `TelemetryRepository`.

### Resource Service Endpoint

This endpoint receives resource lifecycle information from the AppHost. It provides metadata about every resource in your application model—name, type (Project, Container, Executable), state, endpoints, environment variables, and health status.

The data originates from DCP (Developer Control Plane), which monitors container and process state and reports changes to the AppHost. The AppHost then forwards this information to the dashboard via the Resource Service gRPC channel.

## Storage Model

All telemetry is stored in **circular in-memory buffers** managed by `TelemetryRepository`. Each buffer type (traces, logs, metrics) has a fixed capacity. When the buffer is full, the oldest entries are evicted to make room for new data.

This design means:

- **Fast queries** — No database overhead, everything is in memory.
- **No persistence** — Data is lost when the dashboard restarts.
- **Bounded memory** — The dashboard won't consume unbounded memory regardless of telemetry volume.

The Blazor Server UI subscribes to change notifications from the repository, enabling real-time updates as new telemetry arrives.
