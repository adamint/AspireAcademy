# Resources View

The Resources page is the dashboard landing page and your primary overview of every component in the distributed application. Each resource appears as a row with rich metadata and interactive controls.

## Resource Information

Each row displays:

- **Name** — The resource name as defined in the AppHost (e.g., `"cache"`, `"apiservice"`).
- **Type** — The resource kind: Project, Container, or Executable. An icon distinguishes each type visually.
- **State** — A color-coded badge showing the current lifecycle state: Running (green), Starting (blue), Stopping (yellow), Exited (gray), or Failed (red).
- **Source** — The container image name (e.g., `redis:7.4`) for containers, or the project path for .NET projects.
- **Endpoints** — Clickable HTTP/HTTPS URLs. Click to open the endpoint in a new browser tab directly from the dashboard.

## Expanded Details

Click a resource row to expand it and reveal additional information:

- **Environment Variables** — All environment variables injected into the resource, including connection strings and OTLP configuration. Sensitive values can be revealed on demand.
- **Arguments** — Command-line arguments passed to the resource process.
- **Properties** — Resource-specific metadata such as container IDs, ports, and volumes.

## Actions

Each resource supports lifecycle commands:

- **Stop** — Gracefully stops the resource process or container.
- **Start** — Starts a previously stopped resource.
- **Restart** — Stops and restarts the resource in sequence.

These actions interact with DCP to manage the underlying processes and containers.

## Health Indicators

Resources that register health checks display a health indicator: **green** (Healthy), **yellow** (Degraded), or **red** (Unhealthy). The indicator reflects the aggregate status of all health checks registered by that resource's client integrations and custom checks.
