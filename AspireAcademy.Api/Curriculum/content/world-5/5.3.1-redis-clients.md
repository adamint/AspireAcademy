# Redis Client Integrations

Aspire provides three Redis client integrations, each targeting a different usage pattern. All three share the same `StackExchangeRedisSettings` configuration class and the `Aspire:StackExchange:Redis` configuration key.

## 1. Full Redis Client

`Aspire.StackExchange.Redis` registers an `IConnectionMultiplexer`, giving you access to the complete Redis API—strings, hashes, sets, sorted sets, streams, and pub/sub.

```csharp
builder.AddRedisClient("cache");

app.MapGet("/user/{id}", async (string id, IConnectionMultiplexer redis) =>
{
    var db = redis.GetDatabase();
    var name = await db.StringGetAsync($"user:{id}:name");
    return name.HasValue ? Results.Ok(name.ToString()) : Results.NotFound();
});
```

## 2. Distributed Cache

`Aspire.StackExchange.Redis.DistributedCaching` registers `IDistributedCache`, the standard .NET caching abstraction. Ideal when you want a simple key-value byte store without Redis-specific APIs.

```csharp
builder.AddRedisDistributedCache("cache");

app.MapGet("/catalog/{id}", async (string id, IDistributedCache cache) =>
{
    var cached = await cache.GetStringAsync($"catalog:{id}");
    if (cached is not null) return Results.Ok(cached);

    var data = await FetchFromDatabase(id);
    await cache.SetStringAsync($"catalog:{id}", data,
        new DistributedCacheEntryOptions { AbsoluteExpirationRelativeToNow = TimeSpan.FromMinutes(5) });
    return Results.Ok(data);
});
```

## 3. Output Cache

`Aspire.StackExchange.Redis.OutputCaching` registers a Redis-backed output cache for HTTP response caching. Entire responses are cached and served without re-executing the endpoint.

```csharp
builder.AddRedisOutputCache("cache");

app.MapGet("/products", () => GetProducts())
   .CacheOutput(p => p.Expire(TimeSpan.FromMinutes(10)));
```

## Common Configuration

All three integrations register a health check named `StackExchange.Redis` that verifies connectivity with a `PING` command. Keyed service variants are available for multi-instance scenarios using `AddKeyedRedisClient("name")`.
