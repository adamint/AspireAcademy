# The Enrich Pattern

Sometimes you need custom DI registration — perhaps a specific DbContext factory configuration or a non-standard lifetime. The **Enrich** pattern lets you layer Aspire's observability features onto a manually registered client.

## The Problem

When you use `AddNpgsqlDbContext<T>()`, Aspire controls the entire registration. But what if you need:

- Custom `DbContextFactory` registration
- A specific service lifetime
- Additional `DbContextOptionsBuilder` configuration not exposed by Aspire

If you call `AddDbContext<T>()` manually, you lose health checks, tracing, and metrics.

## The Solution: Enrich

Register your DbContext however you need, then call `Enrich` to add Aspire observability:

```csharp
// Step 1: Manual registration with custom configuration
builder.Services.AddDbContext<CatalogContext>(options =>
{
    options.UseNpgsql(connectionString);
    options.EnableDetailedErrors();
    options.EnableSensitiveDataLogging();
});

// Step 2: Layer on Aspire health checks, tracing, and metrics
builder.EnrichNpgsqlDbContext<CatalogContext>();
```

## What Enrich Does (and Doesn't Do)

| Feature | `Add{Component}` | `Enrich{Component}` |
|---|---|---|
| Registers client in DI | ✅ Yes | ❌ No — you do this |
| Reads `ConnectionStrings:` | ✅ Yes | ❌ No |
| Adds health checks | ✅ Yes | ✅ Yes |
| Enables tracing | ✅ Yes | ✅ Yes |
| Enables metrics | ✅ Yes | ✅ Yes |

**Important**: Enrich does **not** read from the `ConnectionStrings` configuration section. You must provide the connection string yourself during manual registration.

## Available Enrich Methods

Enrich is available for EF Core integrations:

- `EnrichNpgsqlDbContext<TContext>()` — PostgreSQL
- `EnrichSqlServerDbContext<TContext>()` — SQL Server
- `EnrichMySqlDbContext<TContext>()` — MySQL
- `EnrichOracleDatabaseDbContext<TContext>()` — Oracle

## When to Use Enrich

Use `Add{Component}` as your default choice — it handles everything. Reach for `Enrich` only when you have specific DI registration requirements that `Add{Component}` doesn't support. The Enrich pattern ensures you never have to choose between custom configuration and production-grade observability.
