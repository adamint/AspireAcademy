# Npgsql DataSource Integration

The `Aspire.Npgsql` package provides direct PostgreSQL access by registering an `NpgsqlDataSource` into the DI container. This is the foundational PostgreSQL integration — use it when you want raw SQL control without an ORM layer.

## Registration

```csharp
builder.AddNpgsqlDataSource("postgresdb");
```

This registers `NpgsqlDataSource` as a **singleton**. The data source is thread-safe and manages its own connection pool internally, so a single instance efficiently serves all concurrent requests.

## Usage

Inject `NpgsqlDataSource` and open connections as needed:

```csharp
public class ProductRepository(NpgsqlDataSource dataSource)
{
    public async Task<List<Product>> GetAllAsync()
    {
        await using var conn = await dataSource.OpenConnectionAsync();
        await using var cmd = conn.CreateCommand();
        cmd.CommandText = "SELECT id, name, price FROM products";
        // Read results...
    }
}
```

## Settings and Configuration

The settings class is `NpgsqlSettings`, configured under the key `Aspire:Npgsql`:

```json
{
  "Aspire": {
    "Npgsql": {
      "DisableHealthChecks": false,
      "DisableTracing": false,
      "DisableMetrics": false
    }
  }
}
```

Or via inline delegate:

```csharp
builder.AddNpgsqlDataSource("postgresdb", settings =>
{
    settings.DisableHealthChecks = true;
    settings.CommandTimeout = 60;
});
```

## Automatic Observability

The integration configures three observability pillars automatically:

- **Health check**: Registered as `"PostgreSql"` — verifies the database is reachable
- **Tracing**: OpenTelemetry traces via Npgsql's built-in `ActivitySource`, capturing every SQL command
- **Metrics**: Connection pool statistics (active connections, idle connections, wait time) via Npgsql's built-in meters

All three are enabled by default and can be individually toggled through `NpgsqlSettings`. This gives you production-grade PostgreSQL observability with zero additional setup.
