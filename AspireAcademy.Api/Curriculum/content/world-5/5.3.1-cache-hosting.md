# Redis, Valkey, and Garnet

Aspire supports three cache servers: Redis (the original), Valkey (the open-source fork), and Garnet (Microsoft's high-performance cache).

---

## Redis

```csharp
var redis = builder.AddRedis("cache")
    .WithLifetime(ContainerLifetime.Persistent)
    .WithDataVolume("redis-data")
    .WithRedisCommander();  // adds Redis Commander UI
```

- **Image**: `docker.io/library/redis:7.4`
- **Port**: `6379`
- **Connection string**: `{host}:{port}`
- **Health check**: PING → PONG

## Valkey

```csharp
var valkey = builder.AddValkey("cache");
```

Valkey is the Linux Foundation fork of Redis. API-compatible. Use when you want open-source licensing.

## Garnet

```csharp
var garnet = builder.AddGarnet("cache");
```

Garnet is Microsoft's high-performance cache server. RESP-compatible (works with Redis clients). Use for performance-sensitive workloads.

## Client Integration (Service Side)

In your .NET service:

```csharp
builder.AddRedisClient("cache");
// or
builder.AddRedisDistributedCache("cache");
// or
builder.AddRedisOutputCache("cache");
```

All three provide different abstraction levels:
- **RedisClient** — raw `IConnectionMultiplexer` access
- **DistributedCache** — `IDistributedCache` for key-value storage
- **OutputCache** — ASP.NET output caching middleware

In Python:
```python
import os, redis
cache_url = os.environ["ConnectionStrings__cache"]
r = redis.from_url(f"redis://{cache_url}")
```

In TypeScript:
```typescript
import Redis from 'ioredis';
const redis = new Redis(process.env.ConnectionStrings__cache);
```
