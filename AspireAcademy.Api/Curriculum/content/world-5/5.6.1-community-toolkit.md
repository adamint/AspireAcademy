# The Community Toolkit

Beyond the official Aspire integrations, the **Aspire Community Toolkit** provides additional hosting integrations maintained by the community.

---

## Popular Integrations

| Integration | Usage | Description |
|---|---|---|
| **Dapr** | `AddDapr()` | Distributed Application Runtime |
| **Ollama** | `AddOllama()` | Local LLM inference |
| **Go apps** | `AddGoApp()` | Go application hosting |
| **Java/Spring** | `AddJavaApp()` | Java application hosting |
| **Rust apps** | `AddRustApp()` | Rust application hosting |
| **LavinMQ** | `AddLavinMQ()` | Lightweight AMQP broker |
| **Meilisearch** | `AddMeilisearch()` | Full-text search engine |
| **SQLite** | `AddSqlite()` | Embedded SQL database |
| **EventStore** | `AddEventStore()` | Event sourcing database |
| **flagd** | `AddFlagd()` | Feature flag service |
| **MailPit** | `AddMailPit()` | Development email server |

> 🌍 **Real World:** The Community Toolkit fills the gaps that the official Aspire team can't cover. For example, `AddGoApp()` and `AddRustApp()` were community-contributed because Go and Rust developers needed first-class hosting support. If you're using an infrastructure component that doesn't have an integration yet, chances are someone in the community is already working on one.

## Installing Community Integrations

```bash
dotnet add package CommunityToolkit.Aspire.Hosting.Ollama
```

```csharp
var ollama = builder.AddOllama("llm")
    .AddModel("phi3");

builder.AddPythonApp("ml", "../ml-service", "app:app")
    .WithReference(ollama);
```

## Contributing to the Toolkit

The Community Toolkit lives at `github.com/CommunityToolkit/Aspire`. If you need an integration that does not exist, you can:

1. Open an issue proposing the integration
2. Submit a PR following the contribution guidelines
3. Follow the same hosting integration pattern as official packages

The Community Toolkit uses the same project structure, testing patterns, and API conventions as the official Aspire integrations.
