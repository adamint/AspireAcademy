# RabbitMQ Client Integration

The `Aspire.RabbitMQ.Client` package registers an `IConnection` in the DI container, providing the foundation for publishing and consuming messages through RabbitMQ exchanges and queues.

## Registration

```csharp
builder.AddRabbitMQClient("messaging");
```

This registers a singleton `IConnection` configured from the `ConnectionStrings:messaging` entry. The settings class `RabbitMQClientSettings` is loaded from the `Aspire:RabbitMQ:Client` configuration key.

## Publishing Messages

RabbitMQ uses a channel-based model. Create a channel from the connection, then publish messages to exchanges. Aspire uses RabbitMQ.Client 7.x, which provides an async-first API.

```csharp
app.MapPost("/orders", async (Order order, IConnection connection) =>
{
    await using var channel = await connection.CreateChannelAsync();

    await channel.ExchangeDeclareAsync("orders", ExchangeType.Topic, durable: true);

    var body = JsonSerializer.SerializeToUtf8Bytes(order);
    var props = new BasicProperties
    {
        ContentType = "application/json",
        DeliveryMode = DeliveryModes.Persistent
    };

    await channel.BasicPublishAsync(
        exchange: "orders",
        routingKey: "order.created",
        basicProperties: props,
        body: body);

    return Results.Accepted();
});
```

## Consuming Messages

For background consumption, create a hosted service that subscribes to a queue.

```csharp
public class OrderConsumer(IConnection connection) : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        var channel = await connection.CreateChannelAsync(cancellationToken: ct);
        await channel.QueueDeclareAsync("order-processing", durable: true, cancellationToken: ct);
        await channel.QueueBindAsync("order-processing", "orders", "order.#", cancellationToken: ct);

        var consumer = new AsyncEventingBasicConsumer(channel);
        consumer.ReceivedAsync += async (_, ea) =>
        {
            var order = JsonSerializer.Deserialize<Order>(ea.Body.Span);
            // Process the order
            await channel.BasicAckAsync(ea.DeliveryTag, multiple: false);
        };

        await channel.BasicConsumeAsync("order-processing", autoAck: false, consumer: consumer, cancellationToken: ct);

        // Keep the service running
        await Task.Delay(Timeout.Infinite, ct);
    }
}
```

## Retry Configuration

The `RabbitMQClientSettings` class includes `MaxConnectRetryCount` (default: 5), which controls how many times the integration retries connecting to the broker on startup. Set it to 0 to disable retry. Retries use Polly with exponential backoff.

## Health Check

The integration registers a health check that verifies the connection is in the `Open` state, ensuring your application can detect broker connectivity issues through the dashboard.
