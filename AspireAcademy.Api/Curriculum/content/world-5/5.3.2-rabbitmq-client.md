# RabbitMQ Client Integration

The `Aspire.RabbitMQ.Client` package registers an `IConnection` in the DI container, providing the foundation for publishing and consuming messages through RabbitMQ exchanges and queues.

## Registration

```csharp
builder.AddRabbitMQClient("messaging");
```

This registers a singleton `IConnection` configured from the `ConnectionStrings:messaging` entry. The settings class `RabbitMQClientSettings` is loaded from the `Aspire:RabbitMQ:Client` configuration key.

## Publishing Messages

RabbitMQ uses a channel-based model. Create a channel from the connection, then publish messages to exchanges.

```csharp
app.MapPost("/orders", async (Order order, IConnection connection) =>
{
    using var channel = connection.CreateModel();

    channel.ExchangeDeclare("orders", ExchangeType.Topic, durable: true);

    var body = JsonSerializer.SerializeToUtf8Bytes(order);
    var props = channel.CreateBasicProperties();
    props.ContentType = "application/json";
    props.DeliveryMode = 2; // persistent

    channel.BasicPublish(
        exchange: "orders",
        routingKey: "order.created",
        basicProperties: props,
        body: body);

    return Results.Accepted();
});
```

## Consuming Messages

For background consumption, create a hosted service that subscribes to a queue.

```csharp
public class OrderConsumer(IConnection connection) : BackgroundService
{
    protected override Task ExecuteAsync(CancellationToken ct)
    {
        var channel = connection.CreateModel();
        channel.QueueDeclare("order-processing", durable: true);
        channel.QueueBind("order-processing", "orders", "order.#");

        var consumer = new EventingBasicConsumer(channel);
        consumer.Received += (_, ea) =>
        {
            var order = JsonSerializer.Deserialize<Order>(ea.Body.Span);
            // Process the order
            channel.BasicAck(ea.DeliveryTag, multiple: false);
        };

        channel.BasicConsume("order-processing", autoAck: false, consumer);
        return Task.CompletedTask;
    }
}
```

## Health Check

The integration registers a health check that verifies the connection is in the `Open` state, ensuring your application can detect broker connectivity issues through the dashboard.
