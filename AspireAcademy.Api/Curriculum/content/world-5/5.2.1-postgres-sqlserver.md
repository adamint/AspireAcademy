# PostgreSQL and SQL Server

You're standing up a new service and need a relational database. If your team is in the Microsoft ecosystem, SQL Server is the path of least resistance — but if you're building a polyglot system or need first-class JSONB support, PostgreSQL is likely the better fit. Both are first-class citizens in Aspire, and the choice comes down to your data model, licensing, and team expertise.

---

## PostgreSQL

```csharp
var postgres = builder.AddPostgres("pg")
    .WithLifetime(ContainerLifetime.Persistent)
    .WithDataVolume("pg-data")
    .WithPgAdmin();  // adds pgAdmin container

var db = postgres.AddDatabase("bazaar-db");
```

Key details:
- **Image**: `docker.io/library/postgres:17`
- **Port**: `5432`
- **Health check**: Attempts a SQL connection
- **Connection string**: `Host={host};Port={port};Username=postgres;Password={generated};Database=bazaar-db`
- **`.WithPgAdmin()`**: Adds a pgAdmin container for visual database management

> 💥 **Common gotcha:** Notice that the password is `{generated}`. Aspire auto-generates a random password every time you start a new session. If you're using persistent containers (`WithLifetime(ContainerLifetime.Persistent)`), the password is generated only on first run and then reused — but if you delete the container and recreate it, the password changes and any data volumes using the old credentials won't be accessible. If this happens, either delete the data volume too, or pin the password with `.WithParameter("password", "my-dev-password")`.

> 🤓 **Did you know?** `.WithPgAdmin()` doesn't just start a pgAdmin container — it automatically configures pgAdmin with a server entry pointing to your PostgreSQL instance. No manual "add server" step. Open pgAdmin in the Dashboard endpoints, and your database is already there, connected, ready to query.

### Accessing from Python

```python
import os, psycopg2

conn_str = os.environ["ConnectionStrings__bazaar-db"]
conn = psycopg2.connect(conn_str)
```

### Accessing from Go

```go
connStr := os.Getenv("ConnectionStrings__bazaar-db")
db, err := sql.Open("postgres", connStr)
```

## SQL Server

```csharp
var sql = builder.AddSqlServer("sql")
    .WithLifetime(ContainerLifetime.Persistent)
    .WithDataVolume("sql-data");

var db = sql.AddDatabase("catalog");
```

Key details:
- **Image**: `mcr.microsoft.com/mssql/server:2022-latest`
- **Port**: `1433`
- **SA password**: Auto-generated
- **Connection string**: `Server={host},{port};Database=catalog;User Id=sa;Password={generated};TrustServerCertificate=true`

## Choosing Between Them

| Feature | PostgreSQL | SQL Server |
|---|---|---|
| License | Open source | Microsoft license |
| JSON support | Excellent (JSONB) | Good (JSON functions) |
| Full-text search | Built-in | Built-in |
| Cross-platform | Yes | Yes (Linux containers) |
| Azure managed | Azure Database for PostgreSQL | Azure SQL Database |
