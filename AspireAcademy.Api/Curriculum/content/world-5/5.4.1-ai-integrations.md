# AI Integrations: OpenAI, GitHub Models, Foundry

Aspire has hosting integrations for AI model services — both cloud APIs and self-hosted models. The key design challenge: AI services don't follow the same connection string pattern as databases. They need API keys, model deployment names, and endpoint URLs — and the shape differs between providers.

---

## OpenAI

```csharp
var openai = builder.AddOpenAI("openai")
    .AddDeployment("gpt-4o");
```

The client integration:
```csharp
builder.AddOpenAIClient("openai");
```

Under the hood, the OpenAI integration injects the API key and endpoint as environment variables. Unlike database integrations that use `ConnectionStrings__`, AI integrations may use provider-specific configuration keys. The `AddOpenAIClient` on the service side reads these and configures the official OpenAI SDK client.

## GitHub Models

```csharp
var github = builder.AddGitHubModels("github-models")
    .AddModel("deepseek-r1");
```

GitHub Models provides access to open models (DeepSeek, Phi, GPT) via GitHub's API.

## Azure AI Foundry

```csharp
var foundry = builder.AddAzureAIFoundry("foundry")
    .AddProject("my-project")
    .AddDeployment("gpt-4o");
```

Foundry is Azure's enterprise AI platform. Aspire can:
- Provision Foundry projects
- Deploy models
- Create hosted agents

## Ollama (Community Toolkit)

For local model inference without any cloud dependency:

```csharp
var ollama = builder.AddOllama("ollama")
    .AddModel("llama3");
```

Ollama runs models locally — no API key, no cloud costs. The integration handles model pulling automatically on first start: if the model weights aren't cached locally, Ollama downloads them (which can be several GB). Subsequent starts are instant because the model is cached.

The connection string for Ollama is an HTTP endpoint (e.g., `http://localhost:11434`), and the API is OpenAI-compatible. This means you can use the OpenAI SDK to talk to Ollama — you just point it at the Ollama endpoint instead of `api.openai.com`. This is what makes the "Ollama in dev, OpenAI in prod" pattern work without code changes.

## Using AI from Non-.NET Services

All AI integrations inject connection information as environment variables:

```python
# Python service with OpenAI
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["ConnectionStrings__openai"])
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello"}]
)
```

```typescript
// TypeScript with OpenAI
import OpenAI from 'openai';
const client = new OpenAI({ apiKey: process.env.ConnectionStrings__openai });
```
