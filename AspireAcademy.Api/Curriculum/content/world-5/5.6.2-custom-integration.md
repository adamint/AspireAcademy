# Writing Your Own Hosting Integration

When no existing integration covers your infrastructure, you can write your own. This lesson walks through the complete process.

---

## Project Structure

```
Aspire.Hosting.Meilisearch/
├── Aspire.Hosting.Meilisearch.csproj
├── MeilisearchResource.cs
└── MeilisearchBuilderExtensions.cs
```

## Step 1: Define the Resource

```csharp
public class MeilisearchResource(string name)
    : ContainerResource(name), IResourceWithConnectionString
{
    internal const string PrimaryEndpointName = "http";

    private EndpointReference? _primaryEndpoint;

    public EndpointReference PrimaryEndpoint =>
        _primaryEndpoint ??= new(this, PrimaryEndpointName);

    public ReferenceExpression ConnectionStringExpression =>
        ReferenceExpression.Create(
            $"http://{PrimaryEndpoint.Property(EndpointProperty.Host)}:{PrimaryEndpoint.Property(EndpointProperty.Port)}");
}
```

```deepdive
{"title": "Understanding IResourceWithConnectionString", "content": "The IResourceWithConnectionString interface is what makes WithReference() work. When your resource implements it, Aspire knows how to generate a connection string that gets injected as an environment variable.\n\nThe ConnectionStringExpression property uses ReferenceExpression to build the connection string from runtime values. The {PrimaryEndpoint.Property(EndpointProperty.Host)} and {PrimaryEndpoint.Property(EndpointProperty.Port)} placeholders are resolved at runtime to the actual host and port allocated by DCP.\n\nDifferent resources use different formats:\n- Redis: host:port\n- PostgreSQL: Host=...;Port=...;Username=...;Password=...;Database=...\n- MongoDB: mongodb://host:port/database\n\nYour custom integration defines whatever format your service's client library expects."}
```

## Step 2: Write the Extension Method

```csharp
public static class MeilisearchBuilderExtensions
{
    public static IResourceBuilder<MeilisearchResource> AddMeilisearch(
        this IDistributedApplicationBuilder builder,
        string name,
        int? port = null)
    {
        var resource = new MeilisearchResource(name);

        return builder.AddResource(resource)
            .WithImage("getmeili/meilisearch")
            .WithImageTag("v1.7")
            .WithHttpEndpoint(
                port: port,
                targetPort: 7700,
                name: MeilisearchResource.PrimaryEndpointName)
            .WithHealthCheck(MeilisearchResource.PrimaryEndpointName);
    }
}
```

## Step 3: Add Health Check

```csharp
.WithHealthCheck(MeilisearchResource.PrimaryEndpointName)
```

This adds an HTTP health check that hits the endpoint and expects a 200 response.

For custom health checks:

```csharp
builder.Services.AddHealthChecks()
    .AddCheck<MeilisearchHealthCheck>("meilisearch");
```

## Step 4: Use It

```csharp
var search = builder.AddMeilisearch("search");

builder.AddProject<Projects.Api>("api")
    .WithReference(search)
    .WaitFor(search);
```

The API receives `ConnectionStrings__search=http://localhost:XXXXX` and waits for the health check to pass.

## Step 5: Package It

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net9.0</TargetFramework>
    <PackageId>Aspire.Hosting.Meilisearch</PackageId>
    <Description>Aspire hosting integration for Meilisearch</Description>
  </PropertyGroup>
  <ItemGroup>
    <PackageReference Include="Aspire.Hosting" />
  </ItemGroup>
</Project>
```

Publish to NuGet and anyone can use `builder.AddMeilisearch("search")`.

> 🐛 **Common Gotcha:** Don't forget the health check! Without a `HealthCheckAnnotation`, `WaitFor()` won't know when your custom resource is ready. For HTTP services, `.WithHealthCheck(endpointName)` adds a simple HTTP 200 check. For services with custom readiness logic (like waiting for index initialization), write a custom `IHealthCheck` implementation.
