# OTLP Protocol and Configuration

How OTLP endpoints are configured and how telemetry flows from services to the Dashboard.

**OpenTelemetry Protocol (OTLP)** is the standard wire protocol that carries traces, logs, and metrics from your services to the Aspire Dashboard. Understanding this flow is key to debugging telemetry issues.

## The Telemetry Pipeline

When your AppHost starts, Aspire automatically configures each service with OTLP exporter settings via environment variables:

```
OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:18889
OTEL_EXPORTER_OTLP_PROTOCOL=grpc
OTEL_SERVICE_NAME=catalogapi
```

The service's OpenTelemetry SDK reads these variables and exports telemetry to the Dashboard's OTLP endpoint.

## Dashboard OTLP Endpoints

The Dashboard exposes two OTLP ingestion endpoints:

- **gRPC** (default port 18889): High-performance binary protocol
- **HTTP/Protobuf** (default port 18890): HTTP-based alternative

Services send traces, logs, and metrics to these endpoints, where the Dashboard stores them in memory for visualization.

## How Aspire Configures OTLP

The `Aspire.ServiceDefaults` project calls `AddOpenTelemetry()` which wires everything together:

```csharp
// In your ServiceDefaults Extensions.cs
builder.Logging.AddOpenTelemetry(logging =>
    logging.IncludeFormattedMessage = true);

builder.Services.AddOpenTelemetry()
    .WithMetrics(metrics =>
    {
        metrics.AddAspNetCoreInstrumentation();
        metrics.AddHttpClientInstrumentation();
    })
    .WithTracing(tracing =>
    {
        tracing.AddAspNetCoreInstrumentation();
        tracing.AddHttpClientInstrumentation();
    });
```

The OTLP exporter is added automatically when `OTEL_EXPORTER_OTLP_ENDPOINT` is set — no explicit exporter registration needed.

## Troubleshooting

If telemetry isn't appearing in the Dashboard:

1. Check the `OTEL_EXPORTER_OTLP_ENDPOINT` environment variable is set
2. Verify the Dashboard is running and its OTLP port is accessible
3. Ensure `Aspire.ServiceDefaults` is referenced and `AddServiceDefaults()` is called
4. Look at the service's console output for OTLP export errors

You can view the configured OTLP endpoint in the Dashboard's resource details panel under environment variables.
