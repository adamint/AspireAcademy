# MongoDB Client Integration

The `Aspire.MongoDB.Driver` package registers the official MongoDB C# driver into DI, providing access to MongoDB databases and collections with full Aspire observability.

## Registration

```csharp
builder.AddMongoDBClient("mongodb");
```

This registers `IMongoClient` as a **singleton**. The MongoDB driver manages its own connection pool and is fully thread-safe — a single `IMongoClient` instance efficiently serves your entire application.

## Usage

MongoDB uses a navigation pattern: client → database → collection:

```csharp
public class ProductCatalog(IMongoClient client)
{
    private readonly IMongoCollection<Product> _products =
        client.GetDatabase("catalog").GetCollection<Product>("products");

    public async Task<List<Product>> GetAllAsync()
    {
        return await _products.Find(_ => true).ToListAsync();
    }

    public async Task AddAsync(Product product)
    {
        await _products.InsertOneAsync(product);
    }
}
```

## Settings and Configuration

The settings class is `MongoDBSettings`, configured under `Aspire:MongoDB:Driver`:

```json
{
  "Aspire": {
    "MongoDB": {
      "Driver": {
        "DisableHealthChecks": false,
        "DisableTracing": false,
        "HealthCheckTimeout": 5000
      }
    }
  }
}
```

The `HealthCheckTimeout` property (in milliseconds) controls how long the health check waits for a server ping response before reporting unhealthy.

```csharp
builder.AddMongoDBClient("mongodb", settings =>
{
    settings.HealthCheckTimeout = 10000; // 10 seconds
    settings.DisableTracing = false;
});
```

## Observability

- **Health check**: Pings the MongoDB server to verify connectivity
- **Tracing**: Captured via `AddSource("MongoDB.Driver.Core.Extensions.DiagnosticSources")` — traces all database commands including find, insert, update, and delete operations
- **Metrics**: Connection pool and command duration metrics

MongoDB's document model differs from relational databases, but the Aspire integration pattern remains identical. The same `Add`, `AddKeyed`, settings class, and configuration hierarchy patterns apply — making it straightforward to add MongoDB alongside any relational database in your application.
