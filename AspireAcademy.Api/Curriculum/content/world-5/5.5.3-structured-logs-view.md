# Structured Logs View

The Structured Logs page aggregates log entries from all resources into a single, searchable view. Because logs are structured—containing typed properties rather than plain text—the dashboard enables powerful filtering and correlation.

## Log Entry Anatomy

Each log entry displays:

- **Timestamp** — When the log event occurred, shown in local time.
- **Level** — The severity level, color-coded for quick scanning: Debug (gray), Information (blue), Warning (yellow), Error (red), Critical (dark red).
- **Resource** — Which service or container emitted the log.
- **Message** — The rendered log message with template parameters filled in.
- **Category** — The logger category, typically the fully qualified class name (e.g., `OrderService.Controllers.OrderController`).

## Filtering

The Structured Logs view provides several filter mechanisms:

- **Resource dropdown** — Select a specific resource to see only its logs, or view all resources at once.
- **Level selector** — Filter by minimum severity level. Setting this to "Warning" hides Information and Debug entries.
- **Text search** — Free-text search across rendered messages and property values.

Filters combine with AND logic, so selecting a resource and a level shows only that resource's logs at or above the chosen severity.

## Structured Properties

Click any log entry to expand its structured properties panel. Every named placeholder from the message template appears as a key-value pair. For example, a log message `"Order {OrderId} placed by {CustomerId}"` shows `OrderId: "ABC-123"` and `CustomerId: "cust-456"` as discrete, searchable properties.

## Trace Correlation

If a log entry was emitted inside an `Activity` (span), the `TraceId` appears as a clickable link. Clicking it navigates directly to the Traces view, showing the full distributed trace that includes that log entry. This log-to-trace correlation is one of the most powerful debugging features in the dashboard.
