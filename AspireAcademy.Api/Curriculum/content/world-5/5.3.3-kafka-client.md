# Kafka Client Integration

The `Aspire.Confluent.Kafka` package provides typed producer and consumer registrations built on the Confluent .NET client. Unlike RabbitMQ's single connection model, Kafka registers separate producers and consumers with explicit key/value type parameters.

## Registration

```csharp
// Register a producer
builder.AddKafkaProducer<string, string>("kafka");

// Register a consumer
builder.AddKafkaConsumer<string, string>("kafka");
```

Each registration uses separate configuration keys under `Aspire:Confluent:Kafka:Producer` and `Aspire:Confluent:Kafka:Consumer`, allowing independent tuning of producer and consumer behavior.

## Producing Messages

Inject `IProducer<TKey, TValue>` and send messages to topics.

```csharp
app.MapPost("/events", async (AppEvent evt, IProducer<string, string> producer) =>
{
    var message = new Message<string, string>
    {
        Key = evt.Type,
        Value = JsonSerializer.Serialize(evt)
    };

    var result = await producer.ProduceAsync("app-events", message);
    return Results.Ok(new { result.Partition, result.Offset });
});
```

## Consuming Messages

Use `IConsumer<TKey, TValue>` in a background service to poll for messages.

```csharp
public class EventConsumer(IConsumer<string, string> consumer) : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        consumer.Subscribe("app-events");

        while (!ct.IsCancellationRequested)
        {
            var result = consumer.Consume(ct);
            var evt = JsonSerializer.Deserialize<AppEvent>(result.Message.Value);
            // Process event
            consumer.Commit(result);
        }
    }
}
```

## Key Concepts

Kafka topics are divided into **partitions** for parallelism. Messages with the same key go to the same partition, preserving order per key. **Consumer groups** distribute partitions across instances for horizontal scaling.

## Health Check

The health check verifies broker connectivity by querying cluster metadata. This confirms the Kafka bootstrap servers are reachable and the cluster is responding to requests.
