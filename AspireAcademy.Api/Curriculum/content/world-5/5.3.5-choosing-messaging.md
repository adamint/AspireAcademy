# Choosing the Right Messaging System

Aspire integrates with three messaging systems, each designed for different workloads. Choosing the right one depends on your requirements for ordering, persistence, throughput, and routing complexity.

## RabbitMQ: Smart Routing & Work Queues

**Best for:** Task distribution, complex routing, traditional enterprise messaging.

RabbitMQ routes messages through exchanges using bindings and routing keys. It supports direct, topic, fanout, and header-based routing. Messages are acknowledged per-consumer, enabling reliable work queue patterns where each message is processed exactly once.

**Choose when:** You need flexible routing rules, per-message acknowledgment, or classic request/work-queue patterns with moderate throughput.

## Kafka: High-Throughput Event Streaming

**Best for:** Event sourcing, log aggregation, high-volume data pipelines.

Kafka stores messages in ordered, partitioned topic logs. Messages persist for a configurable retention period, allowing consumers to replay history. Consumer groups enable parallel processing with partition-level ordering guarantees.

**Choose when:** You need ordered event streams, message replay, high throughput (millions of messages/second), or multiple independent consumers reading the same data.

## NATS: Lightweight Pub/Sub & Request/Reply

**Best for:** Microservice communication, real-time notifications, low-latency messaging.

Core NATS is a fire-and-forget pub/sub system with subject-based routing and wildcard subscriptions. It has the lowest operational complexity and latency. JetStream adds persistence when needed.

**Choose when:** You want minimal infrastructure overhead, simple pub/sub with wildcards, built-in request/reply, or are starting with lightweight messaging and may scale later.

## Comparison Matrix

| Feature | RabbitMQ | Kafka | NATS |
|---|---|---|---|
| **Ordering** | Per-queue | Per-partition | None (core) |
| **Persistence** | Until consumed | Retention-based | JetStream opt-in |
| **Throughput** | Moderate | Very high | High |
| **Routing** | Exchanges/bindings | Topics/partitions | Subjects/wildcards |
| **Complexity** | Medium | Higher | Low |
