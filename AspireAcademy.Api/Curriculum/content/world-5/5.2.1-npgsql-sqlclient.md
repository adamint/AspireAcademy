# Npgsql and SqlClient

Register NpgsqlDataSource and SqlConnection clients for PostgreSQL and SQL Server.

Aspire provides dedicated client integrations for the two most common relational databases. Each follows the standard `Add` pattern and automatically enables health checks, tracing, and metrics.

## PostgreSQL with Npgsql

The `Aspire.Npgsql` package registers an `NpgsqlDataSource` in DI:

```csharp
// Program.cs
builder.AddNpgsqlDataSource("postgresdb");
```

Then inject it into your services:

```csharp
public class ProductRepository(NpgsqlDataSource dataSource)
{
    public async Task<Product?> GetAsync(int id)
    {
        await using var conn = await dataSource.OpenConnectionAsync();
        await using var cmd = conn.CreateCommand();
        cmd.CommandText = "SELECT * FROM products WHERE id = $1";
        cmd.Parameters.AddWithValue(id);
        await using var reader = await cmd.ExecuteReaderAsync();
        // ...
    }
}
```

Configure via `Aspire:Npgsql`:

```json
{
  "Aspire": {
    "Npgsql": {
      "DisableHealthChecks": false,
      "DisableTracing": false
    }
  }
}
```

## SQL Server with SqlClient

The `Aspire.Microsoft.Data.SqlClient` package registers `SqlConnection`:

```csharp
builder.AddSqlServerClient("sqldb");
```

Inject and use `SqlConnection` directly:

```csharp
public class OrderRepository(SqlConnection connection)
{
    public async Task<Order?> GetAsync(int id)
    {
        await connection.OpenAsync();
        using var cmd = connection.CreateCommand();
        cmd.CommandText = "SELECT * FROM Orders WHERE Id = @Id";
        cmd.Parameters.AddWithValue("@Id", id);
        // ...
    }
}
```

## AppHost Wiring

Both integrate seamlessly with hosting packages:

```csharp
// AppHost Program.cs
var postgres = builder.AddPostgres("pg").AddDatabase("postgresdb");
var sql = builder.AddSqlServer("sql").AddDatabase("sqldb");

builder.AddProject<Projects.Api>()
    .WithReference(postgres)
    .WithReference(sql);
```

The connection names `"postgresdb"` and `"sqldb"` automatically flow through to the service's `ConnectionStrings` configuration.
