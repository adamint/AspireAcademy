# IHealthCheck and Registration

How health checks are registered, executed, and reported to the Aspire runtime.

Health checks let Aspire know whether each service and its dependencies are functioning correctly. Every client integration automatically registers health checks — but understanding how they work helps you extend and troubleshoot them.

## Automatic Health Check Registration

When you call any Aspire client integration method, a health check is registered automatically:

```csharp
builder.AddNpgsqlDataSource("postgresdb");   // Registers a PostgreSQL health check
builder.AddRedisClient("cache");              // Registers a Redis health check
builder.AddRabbitMQClient("messaging");       // Registers a RabbitMQ health check
```

Each health check periodically tests connectivity to its backing resource and reports `Healthy`, `Degraded`, or `Unhealthy`.

## The IHealthCheck Interface

Health checks implement `IHealthCheck`:

```csharp
public class CustomDependencyHealthCheck : IHealthCheck
{
    public async Task<HealthCheckResult> CheckHealthAsync(
        HealthCheckContext context,
        CancellationToken cancellationToken = default)
    {
        try
        {
            // Test connectivity to your dependency
            var isHealthy = await TestConnectionAsync(cancellationToken);
            return isHealthy
                ? HealthCheckResult.Healthy("Dependency is reachable")
                : HealthCheckResult.Unhealthy("Cannot reach dependency");
        }
        catch (Exception ex)
        {
            return HealthCheckResult.Unhealthy("Health check failed", ex);
        }
    }
}
```

Register custom health checks in `Program.cs`:

```csharp
builder.Services.AddHealthChecks()
    .AddCheck<CustomDependencyHealthCheck>("custom-dep");
```

## Health Check Endpoint

`Aspire.ServiceDefaults` maps a `/health` endpoint that ASP.NET Core's health check middleware serves. The Aspire runtime polls this endpoint to determine resource state.

## Dashboard Integration

The Dashboard displays health status for each resource with color-coded indicators. A resource shows:

- 🟢 **Healthy**: All health checks pass
- 🟡 **Degraded**: Some checks report degraded
- 🔴 **Unhealthy**: One or more checks fail

## Disabling Health Checks

Disable per client when running in environments where the backing resource may not be available:

```csharp
builder.AddRedisClient("cache", settings =>
    settings.DisableHealthChecks = true);
```
