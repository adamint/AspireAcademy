# Settings Classes

Every Aspire client integration exposes a settings class that controls connection behavior and observability features. These classes provide a consistent, typed surface for configuring integrations programmatically or through configuration files.

## Naming Convention

Settings classes follow the pattern `{Technology}{Component}Settings`:

- **`NpgsqlSettings`** — for `Aspire.Npgsql`
- **`RabbitMQClientSettings`** — for `Aspire.RabbitMQ.Client`
- **`StackExchangeRedisSettings`** — for `Aspire.StackExchange.Redis`
- **`MongoDBSettings`** — for `Aspire.MongoDB.Driver`

## Common Properties

While each settings class has technology-specific properties, most share these standard toggles:

| Property | Type | Default | Purpose |
|---|---|---|---|
| `ConnectionString` | `string?` | `null` | Override the connection string |
| `DisableHealthChecks` | `bool` | `false` | Prevents `IHealthCheck` registration |
| `DisableTracing` | `bool` | `false` | Disables OpenTelemetry trace instrumentation |
| `DisableMetrics` | `bool` | `false` | Disables OpenTelemetry metrics collection |

## The configureSettings Delegate

Every `Add{Component}` method accepts an optional `Action<TSettings>` delegate for programmatic control:

```csharp
builder.AddNpgsqlDataSource("db", settings =>
{
    settings.DisableHealthChecks = true;
    settings.DisableTracing = false;
    settings.DisableMetrics = false;
});
```

This delegate runs **after** all other configuration sources (JSON, environment variables), so it acts as the final override.

## configureSettings vs. configureOptions

Some integrations also expose a `configureOptions` delegate for the underlying SDK client options:

```csharp
builder.AddNpgsqlDataSource("db",
    configureSettings: settings => settings.DisableHealthChecks = true,
    configureDataSourceBuilder: dsBuilder => dsBuilder.ConnectionStringBuilder.Timeout = 30);
```

The distinction matters: `configureSettings` controls **Aspire behaviors** (health checks, tracing, metrics), while `configureOptions` controls the **underlying SDK client** (timeouts, buffer sizes, protocol settings). Use both together for full control over integration behavior.
