# Structured Logging and Metrics

ILogger structured logging, IMeterFactory custom metrics, and OTLP export configuration.

Aspire automatically configures structured logging and metrics for every service. Logs include structured properties that can be searched and filtered in the Dashboard, while metrics provide real-time counters and histograms.

## Structured Logging with ILogger

Aspire configures `ILogger` with OpenTelemetry export by default. Use structured log templates with named placeholders:

```csharp
public class OrderService(ILogger<OrderService> logger)
{
    public async Task ProcessOrderAsync(Order order)
    {
        logger.LogInformation(
            "Processing order {OrderId} for {CustomerId} with {ItemCount} items",
            order.Id, order.CustomerId, order.Items.Count);

        // ... process order

        logger.LogInformation("Order {OrderId} processed successfully", order.Id);
    }
}
```

The named placeholders (`{OrderId}`, `{CustomerId}`) become searchable properties in the Dashboard's structured log viewer, not just interpolated strings.

## Custom Metrics with IMeterFactory

Create custom metrics using `IMeterFactory`, which integrates with OpenTelemetry automatically:

```csharp
public class OrderMetrics
{
    private readonly Counter<long> _ordersProcessed;
    private readonly Histogram<double> _orderProcessingDuration;

    public OrderMetrics(IMeterFactory meterFactory)
    {
        var meter = meterFactory.Create("OrderApi");
        _ordersProcessed = meter.CreateCounter<long>("orders.processed");
        _orderProcessingDuration = meter.CreateHistogram<double>(
            "orders.processing.duration", unit: "ms");
    }

    public void RecordOrderProcessed(double durationMs)
    {
        _ordersProcessed.Add(1);
        _orderProcessingDuration.Record(durationMs);
    }
}
```

Register `OrderMetrics` in DI and inject it where needed.

## Built-in Metrics

Aspire client integrations expose metrics automatically:

- **HTTP**: Request duration, active requests, error rates
- **Database**: Query duration, connection pool usage
- **Redis**: Command duration, connection count

## Disabling Per Client

Disable metrics or logging export for specific integrations:

```csharp
builder.AddNpgsqlDataSource("db", settings =>
    settings.DisableMetrics = true);
```

All telemetry is exported via OTLP to the Dashboard, where you can view logs and metrics side-by-side with traces.
