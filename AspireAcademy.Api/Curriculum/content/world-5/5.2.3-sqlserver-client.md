# SQL Server Client Integration

The `Aspire.Microsoft.Data.SqlClient` package registers a `SqlConnection` factory for connecting to SQL Server and Azure SQL databases. It follows the same Aspire patterns you've learned but with SQL Server-specific behaviors.

## Registration

```csharp
builder.AddSqlServerClient("sqldb");
```

Unlike `NpgsqlDataSource` (which is a singleton), `SqlConnection` is registered as **Scoped**. Each DI scope (typically one per HTTP request) resolves a fresh `SqlConnection` from the underlying connection pool.

## Usage

```csharp
public class OrderRepository(SqlConnection connection)
{
    public async Task<Order?> GetByIdAsync(int id)
    {
        await connection.OpenAsync();
        using var cmd = connection.CreateCommand();
        cmd.CommandText = "SELECT Id, Total FROM Orders WHERE Id = @id";
        cmd.Parameters.AddWithValue("@id", id);
        // Read results...
    }
}
```

## Settings and Configuration

The settings class is `MicrosoftDataSqlClientSettings`, under the configuration key `Aspire:Microsoft:Data:SqlClient`:

```json
{
  "Aspire": {
    "Microsoft": {
      "Data": {
        "SqlClient": {
          "DisableHealthChecks": false,
          "DisableTracing": false
        }
      }
    }
  }
}
```

```csharp
builder.AddSqlServerClient("sqldb", settings =>
{
    settings.DisableHealthChecks = true;
});
```

## Health Check Behavior

The health check is registered as `"SqlServer"` and executes a lightweight `SELECT 1` query against the database. This validates both network connectivity and authentication without imposing significant load.

## Observability

- **Tracing**: Captured via `SqlClientInstrumentation` — every SQL command appears as a span with command text and duration
- **Health checks**: The `SELECT 1` probe runs on the configured health check interval

Note that unlike some other database integrations (like Npgsql and MySqlConnector), the SQL Server client integration does not expose a separate `DisableMetrics` toggle — metrics are controlled at the SqlClient instrumentation level.

The scoped lifetime ensures each request gets a clean connection state, preventing accidental state leakage between requests while still benefiting from ADO.NET's built-in connection pooling.
