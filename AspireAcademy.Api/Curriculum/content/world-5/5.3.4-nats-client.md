# NATS Client Integration

The `Aspire.NATS.Net` package registers an `INatsConnection` for lightweight, high-performance messaging. NATS excels at simple pub/sub and request/reply patterns with minimal configuration overhead.

## Registration

```csharp
builder.AddNatsClient("nats");
```

Settings are loaded from the `Aspire:NATS:Net` configuration key via `NatsClientSettings`.

## Publish/Subscribe

NATS uses subject-based addressing with dot-separated hierarchies. Publishers send to subjects; subscribers receive from matching subjects.

```csharp
// Publishing
app.MapPost("/orders", async (Order order, INatsConnection nats) =>
{
    await nats.PublishAsync("orders.new", order);
    return Results.Accepted();
});

// Subscribing in a background service
public class OrderSubscriber(INatsConnection nats) : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        await foreach (var msg in nats.SubscribeAsync<Order>("orders.new", cancellationToken: ct))
        {
            // Process the order
            Console.WriteLine($"Received order: {msg.Data?.Id}");
        }
    }
}
```

## Wildcard Subscriptions

NATS supports two wildcard tokens for flexible routing:

- `*` matches a single token: `orders.*` matches `orders.new` and `orders.updated`
- `>` matches one or more tokens: `orders.>` matches `orders.new`, `orders.us.east`, etc.

## Request/Reply

NATS has built-in request/reply for RPC-style communication.

```csharp
var response = await nats.RequestAsync<OrderRequest, OrderResponse>(
    "orders.validate", new OrderRequest(orderId));
```

## JetStream for Persistence

Core NATS is fire-and-forget. For durable messaging with persistence and replay, add JetStream.

```csharp
builder.AddNatsJetStream();
```

JetStream adds streams (durable storage), consumers (pull/push delivery), and exactly-once semantics—bridging the gap between lightweight pub/sub and full-featured event streaming.
