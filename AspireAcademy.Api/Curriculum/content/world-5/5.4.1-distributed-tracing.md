# Distributed Tracing with Activities

How Activity and ActivitySource create spans that flow across services via trace context propagation.

Distributed tracing lets you follow a single request as it crosses service boundaries. Aspire client integrations automatically instrument outgoing calls with trace context, so traces appear end-to-end in the Dashboard.

## How It Works

.NET uses `System.Diagnostics.Activity` to represent a **span** — a unit of work within a trace. Each span has a trace ID, span ID, and parent span ID that link spans across services.

```csharp
// Creating a custom activity source for your service
private static readonly ActivitySource s_activitySource = new("CatalogApi");

public async Task<Product> GetProductAsync(int id)
{
    using var activity = s_activitySource.StartActivity("GetProduct");
    activity?.SetTag("product.id", id);

    var product = await _db.Products.FindAsync(id);

    activity?.SetTag("product.found", product is not null);
    return product;
}
```

## Automatic Instrumentation

Aspire client integrations add tracing automatically. When you call `AddNpgsqlDataSource`, `AddRedisClient`, or any other client integration, the underlying library's `ActivitySource` is registered with the OpenTelemetry SDK. This means:

- HTTP calls via `HttpClient` generate spans automatically
- Database queries create child spans with SQL details
- Redis commands produce spans with command names
- gRPC calls are traced end-to-end

## Trace Context Propagation

When service A calls service B via HTTP, the `W3C TraceContext` header (`traceparent`) is injected automatically:

```
traceparent: 00-4bf92f3577b34da6a3ce929d0e0e4736-00f067aa0ba902b7-01
```

Service B extracts this header and creates child spans under the same trace ID, linking the entire request chain.

## Viewing Traces

The Aspire Dashboard collects all spans via OTLP and visualizes them as a waterfall timeline. You can see exactly how long each database query, cache lookup, or HTTP call took — and where errors occurred in the chain.

To disable tracing for a specific client:

```csharp
builder.AddRedisClient("cache", settings =>
    settings.DisableTracing = true);
```
