# MongoDB Client

Register MongoClient with the MongoDB client integration.

The `Aspire.MongoDB.Driver` package registers a `IMongoClient` in the DI container, with automatic health checks, tracing, and metrics — following the same pattern as all Aspire client integrations.

## Registration

In your service's `Program.cs`, register the MongoDB client:

```csharp
builder.AddMongoDBClient("mongodb");
```

This reads the connection string from `ConnectionStrings:mongodb` and registers `IMongoClient` as a singleton.

## Using the Client

Inject `IMongoClient` and access your databases and collections:

```csharp
public class ProductRepository(IMongoClient client)
{
    private readonly IMongoCollection<Product> _products =
        client.GetDatabase("catalog").GetCollection<Product>("products");

    public async Task<List<Product>> GetAllAsync()
        => await _products.Find(_ => true).ToListAsync();

    public async Task CreateAsync(Product product)
        => await _products.InsertOneAsync(product);
}
```

## AppHost Wiring

Add a MongoDB resource in the AppHost and reference it:

```csharp
var mongo = builder.AddMongoDB("mongo").AddDatabase("mongodb");

builder.AddProject<Projects.CatalogApi>()
    .WithReference(mongo);
```

The connection name `"mongodb"` flows to the service's `ConnectionStrings` automatically.

## Configuration

Configure via `Aspire:MongoDB:Driver`:

```json
{
  "Aspire": {
    "MongoDB": {
      "Driver": {
        "DisableHealthChecks": false,
        "DisableTracing": false,
        "HealthCheckTimeout": 5000
      }
    }
  }
}
```

Or inline:

```csharp
builder.AddMongoDBClient("mongodb", settings =>
    settings.DisableTracing = true);
```

The health check verifies connectivity by issuing a `ping` command to the MongoDB server, ensuring the database is reachable before the service reports itself as healthy.
