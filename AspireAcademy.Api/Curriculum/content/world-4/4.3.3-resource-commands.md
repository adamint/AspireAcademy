# Custom Resource Commands

Resources can expose **commands** — actions that users trigger from the Dashboard or CLI. Commands let you interact with running resources beyond start/stop.

---

## Registering a Command

```csharp
builder.Eventing.Subscribe<AfterResourcesCreatedEvent>((evt, ct) =>
{
    var commandService = evt.Services.GetRequiredService<ResourceCommandService>();

    commandService.RegisterCommand("api", new ResourceCommand("seed-data", "Seed Data",
        async context =>
        {
            // Run database seeding logic
            var httpClient = context.ServiceProvider.GetRequiredService<HttpClient>();
            await httpClient.PostAsync("http://api/admin/seed", null);
            return CommandResult.Success();
        },
        iconName: "Database"));

    return Task.CompletedTask;
});
```

## Using Commands

### From the Dashboard
Commands appear as buttons in the resource's action menu. Click to execute.

### From the CLI
```bash
aspire do api seed-data
```

## Built-In Commands

Some commands are built-in:
- **Start** / **Stop** / **Restart** — lifecycle management
- **View Logs** — open the log viewer
- **Browse** — open the resource's endpoint in a browser

## The Interaction Service

For commands that need user input, use the **Interaction Service**:

```csharp
commandService.RegisterCommand("db", new ResourceCommand("run-query", "Run SQL Query",
    async context =>
    {
        var interaction = context.ServiceProvider.GetRequiredService<IInteractionService>();
        var query = await interaction.PromptAsync("Enter SQL query:");
        // Execute the query...
        return CommandResult.Success();
    }));
```

The Dashboard shows an input dialog. The CLI prompts in the terminal.
