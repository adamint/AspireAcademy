# Garnet and Valkey

Add Garnet with AddGarnet(name) and Valkey with AddValkey(name) as Redis-compatible alternatives.

## Garnet

Garnet is Microsoft's Redis-compatible cache server, built for high performance:

```csharp
var cache = builder.AddGarnet("cache");

builder.AddProject<Projects.Api>("api")
       .WithReference(cache)
       .WaitFor(cache);
```

Injects: `ConnectionStrings__cache = localhost:{port}`

Garnet uses the same wire protocol as Redis, so your Redis client code works unchanged.

## Valkey

Valkey is the community fork of Redis (after Redis changed its license):

```csharp
var cache = builder.AddValkey("cache");

builder.AddProject<Projects.Api>("api")
       .WithReference(cache)
       .WaitFor(cache);
```

Same connection string format and client compatibility as Redis.

## Client Compatibility

All three (Redis, Garnet, Valkey) work with the same .NET client:

```csharp
// In your service — same code regardless of backend
builder.AddRedisClient("cache");
```

The `IConnectionMultiplexer` and `IDatabase` APIs are identical.

## Comparison

| Feature | Redis | Garnet | Valkey |
|---------|-------|--------|--------|
| License | SSPL (source-available) | MIT | BSD |
| Performance | Excellent | High (built on .NET) | Excellent |
| Compatibility | Native | Redis-compatible | Redis-compatible |
| Aspire method | `AddRedis` | `AddGarnet` | `AddValkey` |

## When to Choose

- **Redis** — industry standard, broadest ecosystem
- **Garnet** — MIT licensed, high performance, Microsoft-backed
- **Valkey** — community-driven Redis fork, permissive license
