# Milvus for AI Workloads

Milvus is a large-scale vector database designed for production AI workloads. It supports the server-plus-database pattern with an optional Attu admin UI.

## Adding Milvus

The `AddMilvus` extension registers a `MilvusServerResource` backed by `docker.io/milvusdb/milvus:v2.5.27`:

```csharp
var milvus = builder.AddMilvus("milvus");
```

**Signature:**
```csharp
IResourceBuilder<MilvusServerResource> AddMilvus(
    this IDistributedApplicationBuilder builder,
    string name,
    IResourceBuilder<ParameterResource>? apiKey = null,
    int? grpcPort = null)
```

The default gRPC port is **19530**.

## Adding Databases

Like relational databases, Milvus supports named databases:

```csharp
var vectorDb = milvus.AddDatabase("embeddings");
```

This creates a `MilvusDatabaseResource` as a child of the server.

## Connection String

Milvus uses an endpoint-based connection string:
```
Endpoint=http://{host}:{grpcPort};Key={apiKey}
```

## Attu Admin UI

Attach the Attu web interface (`docker.io/zilliz/attu:v2.5`) for visual collection management:

```csharp
var milvus = builder.AddMilvus("milvus")
    .WithAttu();
```

Attu provides:
- Collection browsing and schema inspection
- Vector search testing
- Index management
- Data import/export

## Configuration Files

Customize Milvus configuration:

```csharp
var milvus = builder.AddMilvus("milvus")
    .WithConfigurationFile("./milvus-config.yaml");
```

## Milvus vs. Qdrant

| Feature        | Qdrant                    | Milvus                        |
|---------------|---------------------------|-------------------------------|
| **Scale**     | Single-node optimized     | Distributed, multi-node       |
| **Ports**     | gRPC 6334, HTTP 6333      | gRPC 19530                    |
| **Databases** | Single-tier               | Server + Database (two-tier)  |
| **Admin UI**  | Built-in web dashboard    | Attu (`WithAttu`)             |
| **Image**     | `qdrant:v1.16.3`          | `milvus:v2.5.27`              |
| **Best for**  | Lightweight, fast startup | Large-scale, production AI    |

Choose Qdrant for simplicity and fast development. Choose Milvus for large-scale vector workloads with database partitioning.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var milvus = builder.AddMilvus("milvus")
    .WithAttu()
    .WithDataVolume();

var vectorDb = milvus.AddDatabase("embeddings");

builder.AddProject<Projects.AiService>("ai-service")
    .WithReference(vectorDb);

builder.Build().Run();
```

## Key Takeaways

- **Resource types**: `MilvusServerResource` → `MilvusDatabaseResource` (two-tier model).
- **Container image**: `docker.io/milvusdb/milvus:v2.5.27`.
- **`WithAttu`**: Adds the `AttuResource` web UI for collection management.
- **gRPC default port**: 19530 — different from Qdrant's 6334.
- **Database partitioning**: Use `AddDatabase` for logical separation of vector collections.
