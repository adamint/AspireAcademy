# SQL Server: AddSqlServer and AddDatabase

Add a SQL Server instance with AddSqlServer(name) and databases with AddDatabase(databaseName).

## Basic Setup

```csharp
var sql = builder.AddSqlServer("sql");
var db = sql.AddDatabase("mydb");

builder.AddProject<Projects.Api>("api")
       .WithReference(db)
       .WaitFor(db);
```

Injects: `ConnectionStrings__mydb = Server=localhost,1433;Database=mydb;User Id=sa;Password=...;TrustServerCertificate=true`

## Custom Password

```csharp
var password = builder.AddParameter("sql-password", secret: true);
var sql = builder.AddSqlServer("sql", password: password);
```

## Persistent Data

```csharp
var db = builder.AddSqlServer("sql")
                .WithDataVolume()
                .WithLifetime(ContainerLifetime.Persistent)
                .AddDatabase("mydb");
```

## Container Image

By default, `AddSqlServer` uses the `mcr.microsoft.com/mssql/server` image. The SA password is auto-generated and meets SQL Server complexity requirements.

## Multiple Databases

```csharp
var sql = builder.AddSqlServer("sql");
var ordersDb = sql.AddDatabase("orders");
var inventoryDb = sql.AddDatabase("inventory");
```

## Health Checks

SQL Server health checks verify the database accepts connections. `WaitFor` uses this to ensure dependent services don't start until SQL Server is ready.
