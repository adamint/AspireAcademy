# RabbitMQ & the Management Plugin

RabbitMQ is Aspire's primary AMQP message broker integration, with a built-in management UI plugin for development.

## Adding RabbitMQ

The `AddRabbitMQ` extension registers a `RabbitMQServerResource` backed by `docker.io/library/rabbitmq:4.2`:

```csharp
var messaging = builder.AddRabbitMQ("messaging");
```

**Signature:**
```csharp
IResourceBuilder<RabbitMQServerResource> AddRabbitMQ(
    this IDistributedApplicationBuilder builder,
    string name,
    IResourceBuilder<ParameterResource>? userName = null,
    IResourceBuilder<ParameterResource>? password = null,
    int? port = null)
```

Defaults: username `"guest"`, auto-generated password, port **5672** (AMQP).

## Connection String Format

RabbitMQ uses the AMQP URI format:
```
amqp://guest:{password}@{host}:{port}
```

This is a standardized URI scheme recognized by all AMQP client libraries.

## The Management Plugin

Enable the web-based management UI with `WithManagementPlugin`:

```csharp
var messaging = builder.AddRabbitMQ("messaging")
    .WithManagementPlugin();
```

This switches the container image to `rabbitmq:4.2-management` and exposes the management console on port **15672**. The UI provides:

- Queue and exchange monitoring
- Message publishing and consumption testing
- Connection and channel inspection
- User and permission management

You can also specify a custom host port:

```csharp
.WithManagementPlugin(port: 15672)
```

## Data Persistence

Use `WithDataVolume` to persist queue state, messages, and configuration:

```csharp
var messaging = builder.AddRabbitMQ("messaging")
    .WithManagementPlugin()
    .WithDataVolume();
```

Without a volume, all queued messages are lost when the container stops.

## Credential Configuration

Supply explicit credentials for connecting to an existing RabbitMQ instance:

```csharp
var user = builder.AddParameter("rmq-user");
var pass = builder.AddParameter("rmq-pass", secret: true);
var messaging = builder.AddRabbitMQ("messaging", userName: user, password: pass);
```

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var messaging = builder.AddRabbitMQ("messaging")
    .WithManagementPlugin()
    .WithDataVolume();

builder.AddProject<Projects.OrderProcessor>("order-processor")
    .WithReference(messaging);

builder.AddProject<Projects.NotificationService>("notifications")
    .WithReference(messaging);

builder.Build().Run();
```

## Key Takeaways

- **Resource type**: `RabbitMQServerResource` (container, no child database resources).
- **Connection string**: `amqp://user:pass@host:port` — the AMQP URI format.
- **`WithManagementPlugin`**: Switches to the `-management` image tag and exposes port 15672.
- **No server+database model**: RabbitMQ is a single-tier resource (no `AddDatabase` equivalent).
- **`WithDataVolume`**: Persists queues, exchanges, and messages across restarts.
