# Broker Hosting Knowledge Check

Review the key concepts from RabbitMQ, Kafka, and NATS hosting before the quiz.

## Quick Reference

| Broker   | Method        | Resource Type            | Image                          | Admin UI              |
|----------|---------------|--------------------------|--------------------------------|-----------------------|
| RabbitMQ | `AddRabbitMQ` | `RabbitMQServerResource` | `rabbitmq:4.2`                 | `WithManagementPlugin`|
| Kafka    | `AddKafka`    | `KafkaServerResource`    | `confluent-local:8.1.1`       | `WithKafkaUI`         |
| NATS     | `AddNats`     | `NatsServerResource`     | `nats:2.12`                    | None                  |

## Key Differences

- RabbitMQ: AMQP protocol, queue-based messaging, `WithManagementPlugin` for web UI
- Kafka: Log-based streaming, `WithKafkaUI` for topic/consumer management
- NATS: Lightweight pub/sub, `WithJetStream` adds durable persistence
- Connection string formats differ: AMQP URI vs. host:port vs. nats:// URI
