# Oracle Database Hosting

Oracle hosting in Aspire provides a containerized Oracle Database Free instance, following the server-plus-database pattern with Oracle-specific configuration options.

## Adding Oracle

The `AddOracle` extension registers an `OracleDatabaseServerResource` backed by `container-registry.oracle.com/database/free:23.26.1.0`:

```csharp
var oracle = builder.AddOracle("oracle");
```

**Signature:**
```csharp
IResourceBuilder<OracleDatabaseServerResource> AddOracle(
    this IDistributedApplicationBuilder builder,
    string name,
    IResourceBuilder<ParameterResource>? password = null,
    int? port = null)
```

The default container port is **1521**. Oracle's password is auto-generated following Oracle's complexity rules.

## Adding Databases

```csharp
var hrDb = oracle.AddDatabase("hr");
```

This creates an `OracleDatabaseResource` child resource. In Oracle's architecture, databases map to pluggable databases (PDBs) within the container database.

## Connection String Format

Oracle uses the `Data Source` format with TNS-style descriptors:
```
user id=system;password={password};data source={host}:{port}/hr
```

This differs significantly from both the key-value format of SQL Server and the URI format of MongoDB.

## Init & Setup Scripts

Oracle supports two script directories:

- **`WithInitBindMount`**: Runs scripts after database initialization
- **`WithDbSetupBindMount`**: Runs setup scripts during database creation

```csharp
var oracle = builder.AddOracle("oracle")
    .WithInitBindMount("./oracle-init")
    .WithDbSetupBindMount("./oracle-setup");
```

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var oracle = builder.AddOracle("oracle")
    .WithDataVolume();

var hrDb = oracle.AddDatabase("hr");

builder.AddProject<Projects.HrApi>("hr-api")
    .WithReference(hrDb);

builder.Build().Run();
```
