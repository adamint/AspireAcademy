# Polyglot Architecture Patterns

Aspire's app model unifies .NET, Python, JavaScript, and container-based services under a single orchestration layer. This lesson synthesizes the polyglot hosting patterns.

## The Unified Resource Model

Every application type uses the same patterns:

| App Type     | Method                | Resource Type           | Runs As    |
|-------------|------------------------|-------------------------|------------|
| .NET        | `AddProject<T>`       | `ProjectResource`       | Process    |
| Python      | `AddPythonApp`        | `PythonAppResource`     | Process    |
| JavaScript  | `AddJavaScriptApp`    | `JavaScriptAppResource` | Process    |
| Node.js     | `AddNodeApp`          | `NodeAppResource`       | Process    |
| Container   | `AddContainer`        | `ContainerResource`     | Container  |

`WithReference` works identically across all types, injecting connection strings as `ConnectionStrings__{name}` environment variables.

## Cross-Runtime Service Discovery

For .NET projects, Aspire injects service discovery URLs that the `Microsoft.Extensions.ServiceDiscovery` client handles automatically. For Python and JavaScript, the service URLs are injected as environment variables:

```csharp
var api = builder.AddProject<Projects.WebApi>("api");
var frontend = builder.AddJavaScriptApp("frontend", "../react-app")
    .WithReference(api);
```

In JavaScript, access the API URL:
```javascript
const apiUrl = process.env["services__api__https__0"] 
            || process.env["services__api__http__0"];
```

## Environment Variable Convention

All non-.NET services read Aspire configuration via environment variables:

| Variable Pattern                    | Source                  |
|------------------------------------|-------------------------|
| `ConnectionStrings__{name}`        | `WithReference(db)`     |
| `services__{name}__http__0`        | `WithReference(svc)`    |
| `services__{name}__https__0`       | `WithReference(svc)`    |
| Custom name                        | `WithEnvironment(k, v)` |

## Observability Across Runtimes

OpenTelemetry works across all runtimes. Aspire auto-configures the OTLP endpoint:

- **.NET**: ServiceDefaults configures OTel automatically
- **Python**: Use `opentelemetry-sdk` and read `OTEL_EXPORTER_OTLP_ENDPOINT`
- **JavaScript**: Use `@opentelemetry/sdk-node` and read the same env var

## Architecture Guidance

### When to Use Each Approach

- **`AddProject<T>`**: .NET services — best integration, auto-health checks
- **`AddPythonApp`**: ML models, data pipelines, FastAPI services
- **`AddJavaScriptApp`**: React/Vue/Angular frontends, Express APIs
- **`AddContainer`**: Pre-built images, third-party services

### Common Polyglot Patterns

1. **.NET API + React Frontend**: API via `AddProject`, frontend via `AddJavaScriptApp`
2. **.NET API + Python ML**: API via `AddProject`, ML service via `AddPythonApp`
3. **Microservices mix**: Each team uses their preferred runtime, unified by Aspire

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var db = builder.AddPostgres("pg").AddDatabase("appdb");
var cache = builder.AddRedis("cache");

// .NET API
var api = builder.AddProject<Projects.WebApi>("api")
    .WithReference(db)
    .WithReference(cache);

// Python ML service
var ml = builder.AddPythonApp("ml", "../python-ml", "app.py")
    .WithPip()
    .WithReference(db);

// React frontend
var frontend = builder.AddJavaScriptApp("frontend", "../react-app", "dev")
    .WithNpm()
    .WithReference(api)
    .WithHttpEndpoint(env: "PORT");

builder.Build().Run();
```

## Key Takeaways

- **Unified model**: `WithReference` works the same for .NET, Python, JavaScript, and containers.
- **Environment variables**: Non-.NET services read configuration from `ConnectionStrings__` and `services__` env vars.
- **Host-based apps**: Python and JavaScript run on the host; .NET projects and containers are managed by Aspire.
- **OpenTelemetry**: Cross-runtime observability via the shared OTLP endpoint.
- **Team autonomy**: Each team picks their runtime; Aspire provides the integration layer.
