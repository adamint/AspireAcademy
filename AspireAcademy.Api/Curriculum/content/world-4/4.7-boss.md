# Boss: Multi-Database + AI-Powered App

Build an app with PostgreSQL, Redis, and a GitHub Model — all orchestrated together.

## The Challenge

Combine hosting integrations into a realistic application:

```csharp
var builder = DistributedApplication.CreateBuilder(args);

// Databases
var db = builder.AddPostgres("pg")
                .WithDataVolume()
                .WithLifetime(ContainerLifetime.Persistent)
                .AddDatabase("mydb");

var cache = builder.AddRedis("cache");

// AI
var model = builder.AddGitHubModel("ai", "gpt-4o");

// Services
var api = builder.AddProject<Projects.Api>("api")
                 .WithReference(db)
                 .WithReference(cache)
                 .WithReference(model)
                 .WaitFor(db)
                 .WaitFor(cache);

var web = builder.AddProject<Projects.Web>("web")
                 .WithReference(api)
                 .WaitFor(api);

builder.Build().Run();
```

## What This Demonstrates

- **Multiple database types** — relational (PostgreSQL) + cache (Redis) + AI (GitHub Model)
- **Startup ordering** — API waits for both databases; web waits for API
- **Persistent storage** — PostgreSQL data survives restarts
- **AI integration** — the API can call an LLM for intelligent features

## Architecture

```
PostgreSQL ──┐
Redis ───────┼──→ API ──→ Web Frontend
GitHub Model ┘
```

Each service receives only the connections it needs. The web frontend doesn't get database access — it only talks to the API.
