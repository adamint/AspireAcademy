# Orleans Distributed Actors

Orleans is Microsoft's virtual actor framework. Aspire's hosting integration configures Orleans clustering, grain storage, and streaming through the resource model.

## Adding Orleans

The `AddOrleans` extension creates an `OrleansService` configuration builder:

```csharp
var orleans = builder.AddOrleans("my-cluster");
```

**Signature:**
```csharp
OrleansService AddOrleans(
    this IDistributedApplicationBuilder builder,
    string name)
```

Note: `OrleansService` is **not** a resource type — it's a configuration builder that attaches Orleans settings to project resources.

## Configuring Clustering

Orleans silos need a clustering provider to discover each other. Use an Aspire resource:

```csharp
var redis = builder.AddRedis("redis");
var orleans = builder.AddOrleans("my-cluster")
    .WithClustering(redis);
```

For local development with a single silo:
```csharp
var orleans = builder.AddOrleans("my-cluster")
    .WithDevelopmentClustering();
```

## Grain Storage

Configure where Orleans persists grain state:

```csharp
var storage = builder.AddRedis("grain-state");
var orleans = builder.AddOrleans("my-cluster")
    .WithClustering(storage)
    .WithGrainStorage("Default", storage);
```

For in-memory storage during development:
```csharp
orleans.WithMemoryGrainStorage("Default");
```

## Streaming

Wire streaming providers to Aspire resources:

```csharp
var kafka = builder.AddKafka("kafka");
orleans.WithStreaming("Default", kafka);
```

Or use in-memory streams:
```csharp
orleans.WithMemoryStreaming("Default");
```

## Wiring to Projects

Apply Orleans configuration to project resources:

```csharp
var silo = builder.AddProject<Projects.SiloHost>("silo")
    .WithReference(orleans);

// For client-only projects
var webApi = builder.AddProject<Projects.WebApi>("web-api")
    .WithReference(orleans.AsClient());
```

`AsClient()` creates an `OrleansServiceClient` that connects to the cluster without hosting grains.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var redis = builder.AddRedis("redis");

var orleans = builder.AddOrleans("shopping")
    .WithClustering(redis)
    .WithGrainStorage("Default", redis)
    .WithMemoryStreaming("Default");

builder.AddProject<Projects.SiloHost>("silo")
    .WithReference(orleans)
    .WithReference(redis);

builder.AddProject<Projects.WebApi>("web-api")
    .WithReference(orleans.AsClient());

builder.Build().Run();
```

## Key Takeaways

- **`OrleansService`**: A configuration builder, not a resource — it configures Orleans on projects.
- **`WithClustering`**: Sets the cluster membership provider (Redis, SQL, etc.).
- **`WithGrainStorage`**: Configures persistent grain state storage.
- **`WithDevelopmentClustering`**: In-memory clustering for single-silo development.
- **`AsClient()`**: Creates a client-only configuration for non-silo projects.
- **Resource-backed**: Clustering and storage use standard Aspire resources (Redis, databases).
