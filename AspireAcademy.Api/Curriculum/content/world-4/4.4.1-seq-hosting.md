# Structured Logging with Seq

Seq is a centralized structured log server that complements the Aspire Dashboard with advanced log querying and analysis.

## Adding Seq

The `AddSeq` extension registers a `SeqResource` backed by `docker.io/datalust/seq:2025.2`:

```csharp
var seq = builder.AddSeq("seq");
```

**Signature:**
```csharp
IResourceBuilder<SeqResource> AddSeq(
    this IDistributedApplicationBuilder builder,
    string name,
    IResourceBuilder<ParameterResource>? adminPassword,
    int? port = null)
```

There's also a simpler overload without authentication:
```csharp
IResourceBuilder<SeqResource> AddSeq(
    this IDistributedApplicationBuilder builder,
    string name,
    int? port = null)
```

When `adminPassword` is null, Seq runs without authentication — suitable for local development.

## Connection String

Seq exposes an HTTP endpoint for log ingestion. The connection string is the Seq server URL:
```
http://{host}:{port}
```

Services reference Seq with `WithReference`, which injects this URL for log forwarding.

## Wiring Services to Seq

```csharp
var seq = builder.AddSeq("seq");

builder.AddProject<Projects.WebApi>("web-api")
    .WithReference(seq);
```

The consuming service receives the Seq endpoint and can configure its logging pipeline to send structured logs there.

## Seq vs. Aspire Dashboard

| Feature           | Aspire Dashboard       | Seq                        |
|-------------------|------------------------|----------------------------|
| **Log viewing**   | Basic structured logs  | Advanced query language    |
| **Filtering**     | Resource-based         | Property-based expressions |
| **Retention**     | In-memory              | Persistent (with volume)   |
| **Alerting**      | None                   | Built-in alerts            |
| **Dashboards**    | Pre-built              | Customizable SQL queries   |

Seq is valuable when you need persistent log storage, complex queries, or alerting beyond what the Aspire Dashboard provides.

## Data Persistence

```csharp
var seq = builder.AddSeq("seq")
    .WithDataVolume();
```

This preserves logs across container restarts. Without a volume, all ingested logs are lost on shutdown.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var seq = builder.AddSeq("seq")
    .WithDataVolume();

builder.AddProject<Projects.WebApi>("web-api")
    .WithReference(seq);

builder.AddProject<Projects.Worker>("worker")
    .WithReference(seq);

builder.Build().Run();
```

## Key Takeaways

- **Resource type**: `SeqResource` (container, implements `IResourceWithConnectionString`).
- **Container image**: `docker.io/datalust/seq:2025.2` — automatically accepts EULA.
- **Connection string**: HTTP URL (`http://host:port`) for log ingestion.
- **Single-tier**: No child resources — one Seq server receives logs from all services.
- **Complements the Dashboard**: Use Seq for persistent storage, advanced queries, and alerting.
