# Multi-Tier Caching Architecture

Configure an AppHost with a two-tier caching strategy using Redis and Garnet.

## Your Mission

Design a caching architecture where Redis serves as the primary application cache with full monitoring, and Garnet acts as a high-performance session store.

## Requirements

1. Add Redis `"primary-cache"` with RedisInsight, persistence, and a data volume
2. Add Garnet `"session-store"` with a data volume
3. Add an `"api"` project referencing both cache resources

## Hints

- Chain methods: `AddRedis("primary-cache").WithRedisInsight().WithPersistence().WithDataVolume()`
- Garnet uses `AddGarnet`, not `AddRedis`
- Both produce the same connection string format, but they are separate resources
