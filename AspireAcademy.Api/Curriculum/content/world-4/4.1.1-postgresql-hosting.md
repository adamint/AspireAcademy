# PostgreSQL & the AddDatabase Pattern

PostgreSQL hosting in Aspire follows a **two-tier resource model**: you create a *server* resource, then add one or more *database* child resources. This pattern is shared across all relational databases in Aspire and is the canonical example to learn first.

## Adding a PostgreSQL Server

The `AddPostgres` extension method registers a `PostgresServerResource` backed by the `docker.io/library/postgres:17.6` container image:

```csharp
var postgres = builder.AddPostgres("pg");
```

**Signature:**
```csharp
IResourceBuilder<PostgresServerResource> AddPostgres(
    this IDistributedApplicationBuilder builder,
    string name,
    IResourceBuilder<ParameterResource>? userName = null,
    IResourceBuilder<ParameterResource>? password = null,
    int? port = null)
```

By default, Aspire auto-generates a strong password and uses `"postgres"` as the username. You can supply explicit credentials via parameter resources.

## Adding a Database

Call `AddDatabase` on the server builder to create a `PostgresDatabaseResource`:

```csharp
var catalogDb = postgres.AddDatabase("catalog");
```

The database name defaults to the resource name (`"catalog"`), but you can override it:

```csharp
var catalogDb = postgres.AddDatabase("catalog", databaseName: "product_catalog");
```

## Wiring to Services

Use `WithReference` to inject the **per-database** connection string into a consuming service:

```csharp
var api = builder.AddProject<Projects.CatalogApi>("catalog-api")
    .WithReference(catalogDb);
```

The injected connection string follows the format:
```
Host={host};Port=5432;Username=postgres;Password={password};Database=catalog
```

## Admin Tools

Aspire provides two admin UI extensions for PostgreSQL:

- **`WithPgAdmin`** — attaches the pgAdmin 4 web UI (`docker.io/dpage/pgadmin4:9.12.0`)
- **`WithPgWeb`** — attaches the lightweight pgweb client (`docker.io/sosedoff/pgweb:0.17.0`)

```csharp
var postgres = builder.AddPostgres("pg")
    .WithPgAdmin();
```

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var postgres = builder.AddPostgres("pg")
    .WithDataVolume()
    .WithPgAdmin();

var catalogDb = postgres.AddDatabase("catalog");
var ordersDb = postgres.AddDatabase("orders");

builder.AddProject<Projects.CatalogApi>("catalog-api")
    .WithReference(catalogDb);

builder.AddProject<Projects.OrdersApi>("orders-api")
    .WithReference(ordersDb);

builder.Build().Run();
```

## Key Takeaways

- **Server + Database**: `AddPostgres` creates the server; `AddDatabase` creates logical databases within it.
- **Resource types**: `PostgresServerResource` (container) → `PostgresDatabaseResource` (child).
- **Connection strings**: Each database resource produces its own connection string targeting that specific database.
- **Credentials**: Auto-generated by default; override with `userName` and `password` parameters.
