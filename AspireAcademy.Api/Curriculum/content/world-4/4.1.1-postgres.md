# PostgreSQL: AddPostgres and AddDatabase

Add a PostgreSQL server with AddPostgres(name) and create databases with AddDatabase(databaseName).

## Adding a PostgreSQL Server

```csharp
var postgres = builder.AddPostgres("pg");
```

This starts a PostgreSQL container with auto-generated credentials. The server resource itself isn't usually referenced directly — you add databases to it.

## Adding Databases

```csharp
var db = builder.AddPostgres("pg").AddDatabase("mydb");

builder.AddProject<Projects.Api>("api")
       .WithReference(db)
       .WaitFor(db);
```

`AddDatabase` creates a logical database on the server. The connection string injected includes the database name:

```
ConnectionStrings__mydb = Host=localhost;Port=5432;Username=postgres;Password=...;Database=mydb
```

## Custom Credentials

```csharp
var username = builder.AddParameter("pg-user");
var password = builder.AddParameter("pg-password", secret: true);

var postgres = builder.AddPostgres("pg", userName: username, password: password);
```

## Persistent Storage

```csharp
var db = builder.AddPostgres("pg")
                .WithDataVolume()
                .WithLifetime(ContainerLifetime.Persistent)
                .AddDatabase("mydb");
```

This preserves your data between AppHost restarts.

## pgAdmin

Add the pgAdmin web UI:

```csharp
builder.AddPostgres("pg")
       .WithPgAdmin();
```

## Multiple Databases

```csharp
var pg = builder.AddPostgres("pg");
var ordersDb = pg.AddDatabase("orders");
var catalogDb = pg.AddDatabase("catalog");

builder.AddProject<Projects.OrderService>("orders")
       .WithReference(ordersDb);

builder.AddProject<Projects.CatalogService>("catalog")
       .WithReference(catalogDb);
```

## Health Checks

`AddPostgres` automatically registers health checks. The database reports healthy when it accepts connections. `WaitFor(db)` uses this to delay dependent services.
