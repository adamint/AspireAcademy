# OpenAI & Model Hosting

Unlike container-based integrations, the OpenAI hosting integration registers a **connection resource** pointing to the OpenAI API. It models API connections and model deployments in the Aspire resource graph.

## Adding OpenAI

The `AddOpenAI` extension registers an `OpenAIResource`:

```csharp
var openai = builder.AddOpenAI("openai");
```

**Signature:**
```csharp
IResourceBuilder<OpenAIResource> AddOpenAI(
    this IDistributedApplicationBuilder builder,
    string name)
```

This is a **connection resource**, not a container resource. No Docker image is pulled — it represents an external API endpoint.

## Adding Models

Register model references with `AddModel`:

```csharp
var chatModel = openai.AddModel("chat", "gpt-4o");
```

**Signature:**
```csharp
IResourceBuilder<OpenAIModelResource> AddModel(
    this IResourceBuilder<OpenAIResource> builder,
    string name,
    string model)
```

The first parameter is the resource name (used for `WithReference`); the second is the actual model identifier (e.g., `"gpt-4o"`, `"gpt-4o-mini"`).

## Configuration

### API Key

```csharp
var apiKey = builder.AddParameter("openai-key", secret: true);
var openai = builder.AddOpenAI("openai")
    .WithApiKey(apiKey);
```

### Custom Endpoint

For OpenAI-compatible services (Ollama, LM Studio, Azure OpenAI):

```csharp
var openai = builder.AddOpenAI("openai")
    .WithEndpoint("http://localhost:11434/v1");
```

## Wiring to Services

Reference the model resource, not the OpenAI resource:

```csharp
builder.AddProject<Projects.ChatApi>("chat-api")
    .WithReference(chatModel);
```

The consuming service receives the connection details needed to call the specified model.

## Health Checks

Enable health checking for model availability:

```csharp
var chatModel = openai.AddModel("chat", "gpt-4o")
    .WithHealthCheck();
```

## Resource Types

| Type               | Base Class | Purpose                          |
|--------------------|------------|----------------------------------|
| `OpenAIResource`   | `Resource` | Represents the OpenAI connection |
| `OpenAIModelResource` | `Resource` | Represents a specific model   |

Both implement `IResourceWithConnectionString`. The model is a child of the OpenAI resource, similar to the database-within-server pattern.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var apiKey = builder.AddParameter("openai-key", secret: true);

var openai = builder.AddOpenAI("openai")
    .WithApiKey(apiKey);

var chatModel = openai.AddModel("chat", "gpt-4o");
var embeddingModel = openai.AddModel("embeddings", "text-embedding-3-small");

builder.AddProject<Projects.ChatApi>("chat-api")
    .WithReference(chatModel);

builder.AddProject<Projects.SearchApi>("search-api")
    .WithReference(embeddingModel);

builder.Build().Run();
```

## Key Takeaways

- **Not a container**: `OpenAIResource` is a connection resource pointing to an external API.
- **Two-tier model**: `OpenAIResource` → `OpenAIModelResource` (parent → child).
- **`AddModel`**: Registers a named model reference with the actual model identifier.
- **`WithApiKey`**: Sets the authentication key via a secret parameter.
- **`WithEndpoint`**: Redirects to OpenAI-compatible services (Ollama, LM Studio).
- **Reference models**: Use `WithReference(modelResource)`, not the parent OpenAI resource.
