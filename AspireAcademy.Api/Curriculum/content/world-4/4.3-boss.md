# Event-Driven Order Pipeline

Build an AppHost for an event-driven order processing system using all three message brokers.

## Your Mission

Design a system where:
- **RabbitMQ** handles command messages (order placement, cancellation)
- **Kafka** streams events (order created, order shipped) for analytics
- **NATS** delivers real-time notifications to connected clients

## Requirements

1. Add RabbitMQ `"commands"` with management plugin and data volume
2. Add Kafka `"events"` with KafkaUI
3. Add NATS `"notifications"` with JetStream
4. Add `"order-api"` referencing RabbitMQ and Kafka
5. Add `"analytics"` referencing Kafka
6. Add `"notifications-svc"` referencing NATS

## Hints

- A project can reference multiple broker resources via chained `WithReference` calls
- Use `WithDataVolume()` on RabbitMQ to persist queue state
- NATS needs `WithJetStream()` for durable messaging
