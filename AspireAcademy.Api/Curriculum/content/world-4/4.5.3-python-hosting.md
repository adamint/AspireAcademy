# Python Apps in the Aspire Model

Aspire can orchestrate Python applications alongside .NET services, enabling polyglot architectures with unified resource management.

## Adding a Python App

The `AddPythonApp` extension registers a `PythonAppResource`:

```csharp
var mlService = builder.AddPythonApp("ml-service", "../python-api", "main.py");
```

**Signature:**
```csharp
IResourceBuilder<PythonAppResource> AddPythonApp(
    this IDistributedApplicationBuilder builder,
    string name,
    string appDirectory,
    string scriptPath)
```

- `name`: Resource name in the Aspire model
- `appDirectory`: Relative path to the Python project directory
- `scriptPath`: The Python script to execute (relative to appDirectory)

## Not a Container

Python apps run **directly on the host** using your local Python interpreter — no Docker container is involved. This means Python must be installed on the development machine.

## Virtual Environments

Configure a virtual environment for dependency isolation:

```csharp
var mlService = builder.AddPythonApp("ml-service", "../python-api", "main.py")
    .WithVirtualEnvironment();
```

By default, Aspire looks for a `.venv` directory in the app directory. Specify a custom path:
```csharp
.WithVirtualEnvironment("./my-venv")
```

## Package Management

Install dependencies automatically with pip or uv:

```csharp
// Using pip
var mlService = builder.AddPythonApp("ml-service", "../python-api", "main.py")
    .WithPip();

// Using uv (faster alternative)
var mlService = builder.AddPythonApp("ml-service", "../python-api", "main.py")
    .WithUv();
```

## Passing References

`WithReference` injects connection strings as environment variables readable by Python:

```csharp
var postgres = builder.AddPostgres("pg").AddDatabase("mldata");

var mlService = builder.AddPythonApp("ml-service", "../python-api", "main.py")
    .WithReference(postgres);
```

In Python, access the connection string:
```python
import os
conn_str = os.environ["ConnectionStrings__mldata"]
```

## Custom Environment Variables

```csharp
var mlService = builder.AddPythonApp("ml-service", "../python-api", "main.py")
    .WithEnvironment("MODEL_PATH", "/models/v2")
    .WithEnvironment("LOG_LEVEL", "DEBUG");
```

## Uvicorn Apps

For ASGI applications (FastAPI, Starlette):

```csharp
var api = builder.AddUvicornApp("fastapi", "../fastapi-app", "app:app");
```

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var db = builder.AddPostgres("pg").AddDatabase("mldata");

var mlService = builder.AddPythonApp("ml-service", "../python-ml", "app.py")
    .WithPip()
    .WithReference(db)
    .WithHttpEndpoint(env: "PORT");

builder.AddProject<Projects.WebApi>("web-api")
    .WithReference(mlService);

builder.Build().Run();
```

## Key Takeaways

- **Resource type**: `PythonAppResource` — runs on host, not in a container.
- **Three parameters**: name, app directory path, script path.
- **`WithVirtualEnvironment`**: Isolates Python dependencies in a venv.
- **`WithPip`/`WithUv`**: Auto-installs dependencies from requirements.txt.
- **Environment variables**: `WithReference` injects `ConnectionStrings__name` for Python to read.
- **Host requirement**: Python must be installed on the development machine.
