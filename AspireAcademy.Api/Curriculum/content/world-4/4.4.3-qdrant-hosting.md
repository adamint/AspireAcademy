# Qdrant for Vector Search

Qdrant is a vector similarity search engine, essential for AI/ML applications like semantic search and Retrieval-Augmented Generation (RAG).

## Adding Qdrant

The `AddQdrant` extension registers a `QdrantServerResource` backed by `docker.io/qdrant/qdrant:v1.16.3`:

```csharp
var vectors = builder.AddQdrant("vectors");
```

**Signature:**
```csharp
IResourceBuilder<QdrantServerResource> AddQdrant(
    this IDistributedApplicationBuilder builder,
    string name,
    IResourceBuilder<ParameterResource>? apiKey = null,
    int? grpcPort = null,
    int? httpPort = null)
```

Qdrant exposes two endpoints:
- **gRPC** on port **6334** — high-performance operations
- **HTTP/REST** on port **6333** — REST API access

An API key is auto-generated by default for authentication.

## Connection String

Qdrant's connection string targets the gRPC endpoint:
```
Endpoint=http://{host}:{grpcPort};Key={apiKey}
```

## API Key Authentication

Qdrant requires an API key. Aspire auto-generates one, or you can supply your own:

```csharp
var apiKey = builder.AddParameter("qdrant-key", secret: true);
var vectors = builder.AddQdrant("vectors", apiKey: apiKey);
```

## Dual Endpoints

The gRPC endpoint is used for high-throughput vector operations (search, upsert), while the HTTP endpoint provides a REST API and web dashboard:

```csharp
// Access the HTTP endpoint for the web UI
var vectors = builder.AddQdrant("vectors");
// The Qdrant dashboard is available at the HTTP port
```

## Data Persistence

```csharp
var vectors = builder.AddQdrant("vectors")
    .WithDataVolume();
```

Persists vector collections and indexes. Without a volume, all vector data is lost on restart.

## Use Cases

Qdrant is commonly used for:
- **Semantic search**: Find documents by meaning, not keywords
- **RAG patterns**: Retrieve relevant context for LLM prompts
- **Recommendation engines**: Find similar items based on embeddings
- **Image search**: Match images by visual similarity

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var vectors = builder.AddQdrant("vectors")
    .WithDataVolume();

builder.AddProject<Projects.SearchApi>("search-api")
    .WithReference(vectors);

builder.Build().Run();
```

## Key Takeaways

- **Resource type**: `QdrantServerResource` (container, implements `IResourceWithConnectionString`).
- **Dual endpoints**: gRPC (6334) for performance, HTTP (6333) for REST API.
- **API key**: Auto-generated; override with a `ParameterResource`.
- **Connection string**: `Endpoint=http://host:port;Key={key}`.
- **AI/ML essential**: Primary use cases are semantic search and RAG patterns.
