# SQL Server & Password Parameters

SQL Server hosting follows the same two-tier pattern as PostgreSQL — a server resource with child database resources — but uses a different container image and connection string format.

## Adding SQL Server

The `AddSqlServer` extension registers a `SqlServerServerResource` backed by `mcr.microsoft.com/mssql/server:2022-latest`:

```csharp
var sql = builder.AddSqlServer("sql");
```

**Signature:**
```csharp
IResourceBuilder<SqlServerServerResource> AddSqlServer(
    this IDistributedApplicationBuilder builder,
    string name,
    IResourceBuilder<ParameterResource>? password = null,
    int? port = null)
```

Aspire auto-generates a strong SA password meeting SQL Server's complexity requirements (uppercase, lowercase, numeric, and symbol characters).

## Adding Databases

```csharp
var ordersDb = sql.AddDatabase("orders");
```

This creates a `SqlServerDatabaseResource` as a child of the server. The database name defaults to the resource name.

## Connection String Format

SQL Server uses a semicolon-delimited key-value format that differs from PostgreSQL:

```
Server={host},{port};User ID=sa;Password={password};Database=orders;TrustServerCertificate=true
```

Notice: SQL Server uses `Server=host,port` (comma-separated) while PostgreSQL uses `Host=host;Port=port` (separate keys).

## Password Parameters

You can supply an explicit password for scenarios like connecting to an existing SQL Server:

```csharp
var saPassword = builder.AddParameter("sa-password", secret: true);
var sql = builder.AddSqlServer("sql", password: saPassword);
```

## Database Creation Scripts

Use `WithCreationScript` to run T-SQL when the database is first created:

```csharp
var ordersDb = sql.AddDatabase("orders")
    .WithCreationScript("""
        CREATE TABLE Orders (
            Id INT PRIMARY KEY IDENTITY,
            CustomerId NVARCHAR(50) NOT NULL
        );
    """);
```

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var sql = builder.AddSqlServer("sql")
    .WithDataVolume();

var ordersDb = sql.AddDatabase("orders");

builder.AddProject<Projects.OrdersApi>("orders-api")
    .WithReference(ordersDb);

builder.Build().Run();
```

## Key Takeaways

- **Resource types**: `SqlServerServerResource` → `SqlServerDatabaseResource`.
- **Container**: `mcr.microsoft.com/mssql/server:2022-latest` with `ACCEPT_EULA=Y` set automatically.
- **Connection string**: `Server={host},{port};Database={db};User ID=sa;Password={pass}` format.
- **Password complexity**: SQL Server requires mixed case, numbers, and symbols — Aspire handles this automatically.
- **`WithCreationScript`**: Runs T-SQL to initialize a database on first creation.
