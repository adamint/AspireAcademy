# Broker Connection Strings Compared

Understanding connection string formats is essential for debugging and configuring message broker integrations. Each broker uses a different convention.

## Connection String Quick Reference

| Broker   | Format                           | Example                              |
|----------|----------------------------------|--------------------------------------|
| RabbitMQ | `amqp://user:pass@host:port`     | `amqp://guest:abc123@localhost:5672` |
| Kafka    | `host:port`                      | `localhost:9092`                     |
| NATS     | `nats://host:port`               | `nats://localhost:4222`              |

## How WithReference Injects Connection Strings

When you call `WithReference(broker)`, Aspire sets an environment variable in the consuming service:

```
ConnectionStrings__{resourceName}={connectionString}
```

For example:
```csharp
var rabbit = builder.AddRabbitMQ("messaging");
builder.AddProject<Projects.Worker>("worker")
    .WithReference(rabbit);
```

This injects `ConnectionStrings__messaging=amqp://guest:{pass}@{host}:5672` into the worker's environment.

## Default Ports

| Broker   | Default Port | Protocol        |
|----------|-------------|-----------------|
| RabbitMQ | 5672        | AMQP            |
| RabbitMQ | 15672       | Management HTTP |
| Kafka    | 9092        | Kafka protocol  |
| NATS     | 4222        | NATS protocol   |

## Common Pitfalls

### RabbitMQ: URI Encoding
Special characters in passwords must be URI-encoded in AMQP connection strings. Aspire handles this automatically.

### Kafka: Advertised Listeners
Kafka clients connect to the advertised listener address, not the bootstrap address. Aspire configures internal/external listeners automatically so container-to-container and host-to-container connectivity both work.

### NATS: No Auth by Default
NATS runs without authentication by default. For production, supply `userName` and `password` parameters to enable credential-based access.

## Using Connection Strings in Code

In your service's `Program.cs`, access the injected connection string:

```csharp
// RabbitMQ client integration
builder.AddRabbitMQClient("messaging");

// Kafka - use the connection string name
var kafkaConn = builder.Configuration.GetConnectionString("events");

// NATS client integration
builder.AddNatsClient("nats");
```

## Key Takeaways

- **Three formats**: AMQP URI (RabbitMQ), plain host:port (Kafka), NATS URI (NATS).
- **`WithReference`** injects `ConnectionStrings__{name}` environment variables automatically.
- **Port awareness**: Know the default ports for debugging connectivity issues.
- **Aspire handles encoding**: Special characters and advertised listeners are managed automatically.
