# MongoDB & MongoExpress

MongoDB hosting uses the same server-plus-database pattern but with a **URI-based** connection string format instead of key-value pairs.

## Adding MongoDB

The `AddMongoDB` extension registers a `MongoDBServerResource` backed by `docker.io/library/mongo:8.2`:

```csharp
var mongo = builder.AddMongoDB("mongo");
```

**Signature:**
```csharp
IResourceBuilder<MongoDBServerResource> AddMongoDB(
    this IDistributedApplicationBuilder builder,
    string name,
    int? port = null,
    IResourceBuilder<ParameterResource>? userName = null,
    IResourceBuilder<ParameterResource>? password = null)
```

The default container port is **27017**. Username defaults to `"admin"`.

## Adding Databases

```csharp
var productsDb = mongo.AddDatabase("products");
```

MongoDB databases are logical partitions — they're created on first use rather than requiring explicit DDL.

## Connection String Format

MongoDB uses the `mongodb://` URI format:
```
mongodb://admin:{password}@{host}:{port}/products?authSource=admin
```

This single-string URI approach differs from the key-value format used by relational databases. The database name appears as a path segment in the URI.

## MongoExpress Admin UI

Attach MongoExpress (`docker.io/library/mongo-express:1.0`) for visual administration:

```csharp
var mongo = builder.AddMongoDB("mongo")
    .WithMongoExpress();
```

MongoExpress provides a web UI for browsing collections, running queries, and managing indexes.

## Init Scripts

MongoDB supports `WithInitBindMount` for running `.js` or `.sh` scripts on first startup:

```csharp
var mongo = builder.AddMongoDB("mongo")
    .WithInitBindMount("./mongo-init");
```

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var mongo = builder.AddMongoDB("mongo")
    .WithDataVolume()
    .WithMongoExpress();

var productsDb = mongo.AddDatabase("products");

builder.AddProject<Projects.ProductsApi>("products-api")
    .WithReference(productsDb);

builder.Build().Run();
```

## Key Takeaways

- **Resource types**: `MongoDBServerResource` → `MongoDBDatabaseResource`.
- **Container image**: `docker.io/library/mongo:8.2` on port 27017.
- **Connection string**: `mongodb://user:pass@host:port/dbname?authSource=admin` — a URI, not key-value pairs.
- **`WithMongoExpress`**: Adds the `MongoExpressContainerResource` web UI.
- **Databases are logical**: No DDL required — MongoDB creates databases on first write.
