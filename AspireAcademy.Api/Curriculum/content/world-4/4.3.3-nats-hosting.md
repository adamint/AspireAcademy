# NATS & JetStream

NATS is a lightweight, high-performance messaging system. Aspire supports both core NATS pub/sub and JetStream for durable messaging.

## Adding NATS

The `AddNats` extension registers a `NatsServerResource` backed by `docker.io/library/nats:2.12`:

```csharp
var nats = builder.AddNats("nats");
```

**Signature:**
```csharp
IResourceBuilder<NatsServerResource> AddNats(
    this IDistributedApplicationBuilder builder,
    string name,
    int? port = null,
    IResourceBuilder<ParameterResource>? userName = null,
    IResourceBuilder<ParameterResource>? password = null)
```

Default port is **4222**. No authentication by default.

## Connection String Format

NATS uses its own URI scheme:
```
nats://{host}:{port}
```

This is the simplest connection string among the three brokers.

## JetStream: Durable Messaging

Core NATS is fire-and-forget pub/sub. `WithJetStream` enables the persistence layer for durable streams, consumers, and exactly-once delivery:

```csharp
var nats = builder.AddNats("nats")
    .WithJetStream();
```

JetStream adds:
- **Streams**: Named, append-only message logs
- **Consumers**: Durable subscriptions with acknowledgment
- **Key-value store**: Distributed key-value via JetStream
- **Object store**: Large object storage via JetStream

## JetStream with Volume Mount

For custom JetStream storage, specify a source mount path:

```csharp
var nats = builder.AddNats("nats")
    .WithJetStream(srcMountPath: "./nats-data");
```

## Data Persistence

Persist JetStream data across restarts:

```csharp
var nats = builder.AddNats("nats")
    .WithJetStream()
    .WithDataVolume();
```

Without a data volume, JetStream streams and their messages are lost on container restart.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var nats = builder.AddNats("nats")
    .WithJetStream()
    .WithDataVolume();

builder.AddProject<Projects.NotificationPublisher>("publisher")
    .WithReference(nats);

builder.AddProject<Projects.NotificationSubscriber>("subscriber")
    .WithReference(nats);

builder.Build().Run();
```

## Comparing the Three Brokers

| Feature       | RabbitMQ               | Kafka                    | NATS                    |
|---------------|------------------------|--------------------------|-------------------------|
| **Protocol**  | AMQP                   | Custom binary            | NATS protocol           |
| **Model**     | Queue-based            | Log-based                | Pub/sub + streams       |
| **Conn. str.**| `amqp://user:pass@h:p` | `host:port`              | `nats://host:port`      |
| **Admin UI**  | Management Plugin      | KafkaUI                  | None built-in           |
| **Persistence**| Default (durable)     | Default (log-based)      | JetStream (opt-in)      |

## Key Takeaways

- **Resource type**: `NatsServerResource` (container, single-tier resource).
- **Connection string**: `nats://{host}:{port}` — URI format with `nats://` scheme.
- **`WithJetStream`**: Enables durable streams, consumers, and KV store.
- **Lightweight**: NATS has the smallest footprint of the three brokers.
- **No admin UI**: Unlike RabbitMQ and Kafka, NATS has no built-in web management extension.
