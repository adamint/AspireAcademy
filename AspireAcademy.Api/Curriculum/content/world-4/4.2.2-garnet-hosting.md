# Garnet: High-Performance Cache

Garnet is Microsoft's high-performance, Redis-compatible cache server. In Aspire, it's a drop-in alternative that uses the same client libraries.

## Adding Garnet

The `AddGarnet` extension registers a `GarnetResource` backed by `ghcr.io/microsoft/garnet:1.0`:

```csharp
var cache = builder.AddGarnet("cache");
```

**Signature:**
```csharp
IResourceBuilder<GarnetResource> AddGarnet(
    this IDistributedApplicationBuilder builder,
    string name,
    int? port = null,
    IResourceBuilder<ParameterResource>? password = null)
```

The default port is **6379** — the same as Redis. This means downstream services use the same Redis client libraries (StackExchange.Redis, etc.) without code changes.

## Wire Compatibility

Garnet implements the Redis RESP protocol, so services consuming a Garnet resource use the exact same `AddRedisClient` or `AddRedisDistributedCache` client integrations. Switching between Redis and Garnet is purely an AppHost decision.

## Persistence

Like Redis, Garnet supports `WithPersistence` for RDB-style snapshotting:

```csharp
var cache = builder.AddGarnet("cache")
    .WithPersistence(interval: TimeSpan.FromMinutes(1))
    .WithDataVolume();
```

> **Note:** An older overload of `WithPersistence` accepted a `keysChangedThreshold` parameter, but this is now deprecated. Use the single-parameter overload with just `interval`.

## When to Choose Garnet

| Aspect         | Redis                          | Garnet                         |
|----------------|--------------------------------|--------------------------------|
| **Origin**     | Redis Ltd.                     | Microsoft Research             |
| **Performance**| Excellent                      | Higher throughput on multi-core|
| **Protocol**   | RESP native                    | RESP compatible                |
| **Admin UIs**  | Commander, Insight             | None built-in                  |
| **Maturity**   | Decades of production use      | Newer, growing ecosystem       |

Choose Garnet when you need maximum throughput on modern hardware. Choose Redis when you need the admin tool ecosystem.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var cache = builder.AddGarnet("cache")
    .WithPersistence()
    .WithDataVolume();

builder.AddProject<Projects.SessionApi>("session-api")
    .WithReference(cache);

builder.Build().Run();
```

## Key Takeaways

- **Resource type**: `GarnetResource` (container, implements `IResourceWithConnectionString`).
- **Container image**: `ghcr.io/microsoft/garnet:1.0`.
- **Wire compatible**: Uses Redis protocol — same client libraries, no code changes.
- **`WithPersistence`** and **`WithDataVolume`**: Work identically to Redis.
- **No admin UIs**: Unlike Redis, Garnet doesn't have Commander or Insight extensions.
