# WithReference — Connection Strings vs Endpoints

`WithReference()` is how you wire resources together. What it injects depends on the type of resource being referenced.

---

## Database References → Connection Strings

When you reference a database resource, Aspire injects its **connection string**:

```csharp
var db = builder.AddPostgres("pg").AddDatabase("atlas-db");
var api = builder.AddProject<Projects.Api>("api")
    .WithReference(db);
```

The API receives: `ConnectionStrings__atlas-db=Host=...;Port=5432;Database=atlas-db;...`

## Service References → Endpoint URLs

When you reference a service resource, Aspire injects its **endpoint URL**:

```csharp
var api = builder.AddProject<Projects.Api>("api");
var frontend = builder.AddViteApp("frontend", "../web")
    .WithReference(api);
```

The frontend receives: `services__api__http__0=http://localhost:XXXX`

## The Same in Every Language

**C#** reads connection strings:
```csharp
builder.Configuration.GetConnectionString("atlas-db");
```

**TypeScript** reads from env:
```typescript
const apiUrl = process.env.services__api__http__0;
```

**Python** reads from env:
```python
api_url = os.environ["services__api__http__0"]
```

**Go** reads from env:
```go
apiURL := os.Getenv("services__api__http__0")
```

## How It Works Under the Hood

`WithReference()` adds annotations to the referencing resource:
- For databases: `ConnectionStringRedirectAnnotation` pointing to the database's connection string
- For services: `EndpointReferenceAnnotation` pointing to the service's endpoints

At runtime, DCP resolves these annotations into actual environment variables with real hostnames and ports.
