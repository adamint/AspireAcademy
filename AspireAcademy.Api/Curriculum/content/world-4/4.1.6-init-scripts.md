# Seeding with Init Bind Mounts

Init bind mounts let you run SQL scripts automatically when a database container starts for the first time. This is essential for seeding schema and test data during development.

## The WithInitBindMount Pattern

`WithInitBindMount` maps a local directory into the container's init script directory:

```csharp
var postgres = builder.AddPostgres("pg")
    .WithInitBindMount("./pg-scripts");
```

**Signature:**
```csharp
IResourceBuilder<PostgresServerResource> WithInitBindMount(
    this IResourceBuilder<PostgresServerResource> builder,
    string source,
    bool isReadOnly = true)
```

The `source` path is relative to the AppHost project directory.

## How It Works

Each database engine maps init scripts to a specific container directory:

| Database   | Container Init Directory               |
|------------|----------------------------------------|
| PostgreSQL | `/docker-entrypoint-initdb.d/`         |
| MySQL      | `/docker-entrypoint-initdb.d/`         |
| MongoDB    | `/docker-entrypoint-initdb.d/`         |
| Oracle     | `/opt/oracle/scripts/startup/`         |

Scripts execute **only on first startup** when the data directory is empty. If you've attached a data volume with existing data, init scripts won't re-run.

## File Naming for Execution Order

Scripts execute in alphabetical order. Use numeric prefixes to control sequencing:

```
./pg-scripts/
  01-create-schema.sql
  02-seed-data.sql
  03-create-indexes.sql
```

## WithInitFiles Alternative

`WithInitFiles` copies files into the container rather than using bind mounts:

```csharp
var postgres = builder.AddPostgres("pg")
    .WithInitFiles("./pg-scripts");
```

This is useful when the script directory needs to be read-only or when working with remote Docker hosts.

## WithCreationScript for Per-Database Init

For database-level scripts, use `WithCreationScript` on the database resource (available for PostgreSQL and SQL Server):

```csharp
var db = postgres.AddDatabase("catalog")
    .WithCreationScript("""
        CREATE TABLE Products (
            Id SERIAL PRIMARY KEY,
            Name TEXT NOT NULL
        );
        INSERT INTO Products (Name) VALUES ('Widget'), ('Gadget');
    """);
```

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var postgres = builder.AddPostgres("pg")
    .WithInitBindMount("./pg-scripts")
    .WithDataVolume();

var catalogDb = postgres.AddDatabase("catalog");

builder.AddProject<Projects.CatalogApi>("catalog-api")
    .WithReference(catalogDb);

builder.Build().Run();
```

## Key Takeaways

- **`WithInitBindMount`**: Maps a local directory to the container's init script path.
- **First-run only**: Scripts execute only when the data directory is empty.
- **Alphabetical order**: Name files with numeric prefixes (`01-`, `02-`) for sequencing.
- **`WithInitFiles`**: Copies files instead of bind-mounting — useful for remote Docker.
- **`WithCreationScript`**: Inline SQL for per-database initialization.
