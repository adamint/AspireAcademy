# YARP: AddYarp

Use AddYarp(name) to add a YARP reverse proxy with WithConfiguration for routes and clusters.

## What is YARP?

YARP (Yet Another Reverse Proxy) is a .NET library for building reverse proxies. Aspire provides hosting integration to configure YARP with routes and clusters.

## Basic Setup

```csharp
var api = builder.AddProject<Projects.Api>("api");
var web = builder.AddProject<Projects.Web>("web");

var proxy = builder.AddYarp("proxy")
                   .AddRoute("api-route", "/api/{**catch-all}", api)
                   .AddRoute("web-route", "/{**catch-all}", web)
                   .WithHttpEndpoint(port: 8080);
```

## How It Works

YARP sits in front of your services and routes requests based on URL patterns:

- `/api/*` → routes to the API service
- `/*` → routes to the web frontend

## JSON Configuration

For more complex setups, use YARP's JSON configuration:

```csharp
builder.AddYarp("proxy")
       .WithConfiguration(configBuilder =>
       {
           configBuilder.AddRoute("api", "/api/{**catch-all}")
                        .WithCluster("api-cluster");
       });
```

## Use Cases

- **Single entry point** — one URL for multiple backend services
- **SSL termination** — HTTPS at the proxy, HTTP internally
- **Load balancing** — distribute requests across multiple instances
- **Path-based routing** — different services for different URL prefixes
