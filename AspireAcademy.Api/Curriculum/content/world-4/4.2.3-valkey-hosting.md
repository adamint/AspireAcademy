# Valkey: Community Redis Fork

Valkey is the Linux Foundation's open-source fork of Redis, created after Redis changed its licensing. In Aspire, it's another Redis-compatible drop-in option.

## Adding Valkey

The `AddValkey` extension registers a `ValkeyResource` backed by `docker.io/valkey/valkey:9.0`:

```csharp
var kv = builder.AddValkey("kv");
```

**Signature:**
```csharp
IResourceBuilder<ValkeyResource> AddValkey(
    this IDistributedApplicationBuilder builder,
    string name,
    int? port = null,
    IResourceBuilder<ParameterResource>? password = null)
```

Default port is **6379**. Like Garnet, Valkey is fully wire-compatible with Redis clients.

## Persistence & Data Volumes

The same persistence methods work identically:

```csharp
var kv = builder.AddValkey("kv")
    .WithPersistence(interval: TimeSpan.FromMinutes(2))
    .WithDataVolume();
```

## Three Redis-Compatible Options Compared

Aspire offers three Redis-compatible hosting integrations:

| Feature         | Redis                  | Garnet                | Valkey                |
|----------------|------------------------|-----------------------|-----------------------|
| **Package**    | `Aspire.Hosting.Redis` | `Aspire.Hosting.Garnet` | `Aspire.Hosting.Valkey` |
| **Image**      | `redis:8.6`            | `garnet:1.0`          | `valkey:9.0`          |
| **Origin**     | Redis Ltd.             | Microsoft             | Linux Foundation      |
| **License**    | SSPL / RSAL            | MIT                   | BSD                   |
| **Admin UIs**  | Commander, Insight     | None                  | None                  |
| **Persistence**| ✅ WithPersistence      | ✅ WithPersistence     | ✅ WithPersistence     |
| **Protocol**   | RESP native            | RESP compatible       | RESP native (fork)    |

All three produce the same connection string format (`host:port`) and work with the same client libraries.

## Choosing Between Them

- **Redis**: Best ecosystem, most admin tools, largest community
- **Garnet**: Best raw performance, MIT licensed, Microsoft-backed
- **Valkey**: BSD licensed, community-maintained, closest to Redis feature parity

The choice is primarily about licensing and organizational preferences — the application code doesn't change.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var kv = builder.AddValkey("kv")
    .WithDataVolume();

builder.AddProject<Projects.CacheApi>("cache-api")
    .WithReference(kv);

builder.Build().Run();
```

## Key Takeaways

- **Resource type**: `ValkeyResource` (container, implements `IResourceWithConnectionString`).
- **Container image**: `docker.io/valkey/valkey:9.0`.
- **Drop-in replacement**: Same protocol, same client libraries, same connection string format.
- **Licensing**: BSD-licensed — important for organizations sensitive to Redis's SSPL/RSAL terms.
