# YARP Load Balancing & Routing

YARP (Yet Another Reverse Proxy) in Aspire provides a containerized reverse proxy with code-first route configuration that integrates with Aspire's service discovery.

## Adding YARP

The `AddYarp` extension registers a `YarpResource`:

```csharp
var gateway = builder.AddYarp("gateway");
```

**Signature:**
```csharp
IResourceBuilder<YarpResource> AddYarp(
    this IDistributedApplicationBuilder builder,
    string name)
```

YARP runs as a container resource using `mcr.microsoft.com/dotnet/nightly/yarp:2.3-preview`.

## Configuring Routes

Use `WithConfiguration` and its fluent API to define routes and clusters:

```csharp
var api = builder.AddProject<Projects.WebApi>("api");
var frontend = builder.AddProject<Projects.Frontend>("frontend");

var gateway = builder.AddYarp("gateway")
    .WithConfiguration(config =>
    {
        config.AddRoute("api-route", route =>
        {
            route.WithMatch(match => match.WithPath("/api/{**catch-all}"));
            route.WithCluster("api-cluster", cluster =>
            {
                cluster.WithDestination(api);
            });
        });

        config.AddRoute("frontend-route", route =>
        {
            route.WithMatch(match => match.WithPath("/{**catch-all}"));
            route.WithCluster("frontend-cluster", cluster =>
            {
                cluster.WithDestination(frontend);
            });
        });
    });
```

## Service Discovery Integration

YARP destinations reference Aspire resources directly via `WithDestination`. Aspire resolves the actual endpoints at runtime — no hardcoded URLs needed.

## Port Configuration

```csharp
var gateway = builder.AddYarp("gateway")
    .WithHostPort(8080)
    .WithHostHttpsPort(8443);
```

## Static File Serving

YARP can serve static files alongside proxied routes:

```csharp
var gateway = builder.AddYarp("gateway")
    .WithStaticFiles("./wwwroot");
```

## Common Patterns

### API Gateway
Route all API traffic through a single entry point:
```csharp
// /api/orders → orders service
// /api/products → products service
// /  → frontend
```

### Backend-for-Frontend (BFF)
YARP acts as the BFF layer, aggregating backend calls and handling authentication.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var api = builder.AddProject<Projects.WebApi>("api");

var gateway = builder.AddYarp("gateway")
    .WithConfiguration(config =>
    {
        config.AddRoute("api-route", route =>
        {
            route.WithMatch(match => match.WithPath("/api/{**catch-all}"));
            route.WithCluster("api-cluster", cluster =>
            {
                cluster.WithDestination(api);
            });
        });
    });

builder.Build().Run();
```

## Key Takeaways

- **Resource type**: `YarpResource` (container, implements `IResourceWithServiceDiscovery`).
- **`WithConfiguration`**: Fluent API for defining routes, clusters, and destinations.
- **Service discovery**: `WithDestination(resource)` resolves endpoints automatically.
- **Container-based**: Runs as a standalone YARP proxy container.
- **API gateway pattern**: Natural fit for microservice entry points and BFF architectures.
