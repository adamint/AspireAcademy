# Node.js & JavaScript Apps

Aspire can orchestrate JavaScript applications using `AddJavaScriptApp` for package-manager-based apps and `AddNodeApp` for direct Node.js script execution.

## Adding a JavaScript App

The `AddJavaScriptApp` extension registers a `JavaScriptAppResource` that runs via a package manager script:

```csharp
var frontend = builder.AddJavaScriptApp("frontend", "../react-app", "dev");
```

**Signature:**
```csharp
IResourceBuilder<JavaScriptAppResource> AddJavaScriptApp(
    this IDistributedApplicationBuilder builder,
    string name,
    string appDirectory,
    string runScriptName = "dev")
```

- `name`: Resource name in the Aspire model
- `appDirectory`: Relative path to the JavaScript project directory
- `runScriptName`: The package.json script to run (defaults to `"dev"`)

## Adding a Node.js App

For direct Node.js script execution:

```csharp
var api = builder.AddNodeApp("node-api", "../node-server", "server.js");
```

**Signature:**
```csharp
IResourceBuilder<NodeAppResource> AddNodeApp(
    this IDistributedApplicationBuilder builder,
    string name,
    string appDirectory,
    string scriptPath)
```

## Vite Apps

For Vite-based frontend applications with optimized configuration:

```csharp
var spa = builder.AddViteApp("spa", "../vue-app");
```

## Package Manager Configuration

Choose your package manager:

```csharp
// npm (default)
var app = builder.AddJavaScriptApp("app", "../my-app")
    .WithNpm();

// yarn
var app = builder.AddJavaScriptApp("app", "../my-app")
    .WithYarn();

// pnpm
var app = builder.AddJavaScriptApp("app", "../my-app")
    .WithPnpm();

// bun
var app = builder.AddJavaScriptApp("app", "../my-app")
    .WithBun();
```

Each method auto-installs dependencies before running the app.

## Passing References

Like Python, `WithReference` injects connection strings as environment variables:

```csharp
var redis = builder.AddRedis("cache");

var frontend = builder.AddJavaScriptApp("frontend", "../react-app")
    .WithReference(redis);
```

In JavaScript:
```javascript
const redisUrl = process.env.ConnectionStrings__cache;
```

## HTTP Endpoints

Specify the port your JavaScript app listens on:

```csharp
var frontend = builder.AddJavaScriptApp("frontend", "../react-app")
    .WithHttpEndpoint(env: "PORT");
```

This sets the `PORT` environment variable and registers the endpoint in Aspire's service discovery.

## Not a Container

JavaScript apps run **directly on the host** — Node.js must be installed. No Docker container is created.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var redis = builder.AddRedis("cache");
var api = builder.AddProject<Projects.WebApi>("api");

var frontend = builder.AddJavaScriptApp("frontend", "../react-app", "dev")
    .WithNpm()
    .WithReference(api)
    .WithReference(redis)
    .WithHttpEndpoint(env: "PORT");

builder.Build().Run();
```

## Key Takeaways

- **`AddJavaScriptApp`**: Runs a package.json script via a configured package manager.
- **`AddNodeApp`**: Runs a Node.js script directly.
- **`AddViteApp`**: Optimized for Vite-based frontend apps.
- **Package managers**: `WithNpm`, `WithYarn`, `WithPnpm`, `WithBun` for dependency installation.
- **Host-based**: Runs on local Node.js, not in a container.
- **Environment variables**: `ConnectionStrings__name` injected via `WithReference`.
