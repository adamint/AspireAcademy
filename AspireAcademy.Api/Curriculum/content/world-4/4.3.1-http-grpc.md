# HTTP and gRPC Communication

Bazaar's services communicate via HTTP and gRPC. Each language uses its native HTTP client — Aspire provides the endpoint URLs via environment variables.

---

## C# — HttpClient with Service Discovery

```csharp
// In Program.cs
builder.Services.AddHttpClient("ml", client =>
{
    client.BaseAddress = new Uri("http://ml");
});

// In a handler
app.MapGet("/recommendations", async (IHttpClientFactory factory) =>
{
    var client = factory.CreateClient("ml");
    var result = await client.GetFromJsonAsync<List<Recommendation>>("/recommend");
    return result;
});
```

The hostname `ml` is resolved by Aspire's service discovery.

## TypeScript — fetch

```typescript
const apiUrl = process.env.services__api__http__0;

const response = await fetch(`${apiUrl}/products`);
const products = await response.json();
```

## Python — requests / httpx

```python
import os
import httpx

api_url = os.environ["services__api__http__0"]

async with httpx.AsyncClient(base_url=api_url) as client:
    response = await client.get("/products")
    products = response.json()
```

## Go — net/http

```go
apiURL := os.Getenv("services__api__http__0")
resp, err := http.Get(apiURL + "/products")
if err != nil {
    log.Fatal(err)
}
defer resp.Body.Close()
```

## gRPC

For gRPC communication between .NET services:

```csharp
// AppHost
var grpcService = builder.AddProject<Projects.GrpcService>("grpc-service");
builder.AddProject<Projects.Api>("api")
    .WithReference(grpcService);

// In API service
builder.Services.AddGrpcClient<Greeter.GreeterClient>(o =>
{
    o.Address = new Uri("http://grpc-service");
});
```

gRPC clients use the same service discovery as HTTP — the resource name resolves to the endpoint URL.

> 🎲 **Fun Fact:** gRPC uses HTTP/2 under the hood, which means it supports multiplexing — multiple requests can be in-flight over a single TCP connection simultaneously. This makes it significantly more efficient than HTTP/1.1 for high-frequency service-to-service calls. In Aspire, you don't need to configure HTTP/2 explicitly — when a .NET gRPC client connects to a service, Kestrel negotiates HTTP/2 automatically.
