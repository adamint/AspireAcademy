# Kafka & KafkaUI

Kafka hosting in Aspire provides a single-broker setup using Confluent's container image, with an optional KafkaUI for visual topic management.

## Adding Kafka

The `AddKafka` extension registers a `KafkaServerResource` backed by `docker.io/confluentinc/confluent-local:8.1.1`:

```csharp
var kafka = builder.AddKafka("kafka");
```

**Signature:**
```csharp
IResourceBuilder<KafkaServerResource> AddKafka(
    this IDistributedApplicationBuilder builder,
    string name,
    int? port = null)
```

The default port is **9092** (PLAINTEXT listener).

## Connection String Format

Kafka uses a simple bootstrap server format:
```
{host}:{port}
```

This is the `bootstrap.servers` value that Kafka producers and consumers need. Unlike RabbitMQ's AMQP URI, Kafka uses a plain `host:port` list.

## KafkaUI

Add a web UI for managing topics, consumer groups, and messages:

```csharp
var kafka = builder.AddKafka("kafka")
    .WithKafkaUI();
```

This adds a `KafkaUIContainerResource` using `docker.io/kafbat/kafka-ui:v1.4.2`. The UI provides:

- Topic creation and configuration
- Message browsing and production
- Consumer group monitoring
- Broker configuration inspection

## Advertised Listeners

Kafka requires correct advertised listener configuration for container networking. Aspire handles this automatically — the container is configured with both internal (container-to-container) and external (host) listeners.

## Data Persistence

```csharp
var kafka = builder.AddKafka("kafka")
    .WithDataVolume();
```

This persists topic data, consumer offsets, and broker state.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var kafka = builder.AddKafka("kafka")
    .WithKafkaUI()
    .WithDataVolume();

builder.AddProject<Projects.EventProducer>("producer")
    .WithReference(kafka);

builder.AddProject<Projects.EventConsumer>("consumer")
    .WithReference(kafka);

builder.Build().Run();
```

## Key Takeaways

- **Resource type**: `KafkaServerResource` (container, single-tier resource).
- **Container image**: `docker.io/confluentinc/confluent-local:8.1.1`.
- **Connection string**: `{host}:{port}` — the bootstrap server address.
- **`WithKafkaUI`**: Adds `KafkaUIContainerResource` for web-based topic management.
- **Advertised listeners**: Handled automatically by Aspire for container networking.
- **No credentials parameter**: Unlike RabbitMQ, the default Kafka setup has no authentication.
