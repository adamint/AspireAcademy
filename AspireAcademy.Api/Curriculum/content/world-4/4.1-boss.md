# Multi-Database E-Commerce Platform

Build an AppHost that configures a multi-database e-commerce backend using PostgreSQL, SQL Server, MongoDB, and Redis.

## Your Mission

You're architecting an e-commerce platform that uses the right database for each job:

- **PostgreSQL** for the product catalog (relational, with full-text search)
- **SQL Server** for order processing (transactional integrity)
- **MongoDB** for user sessions (flexible document storage)

Each database needs data volumes for persistence, and PostgreSQL needs init scripts for schema seeding.

## Requirements

1. Add PostgreSQL `"pg"` with a `"catalog"` database, init scripts from `"./pg-scripts"`, and a data volume
2. Add SQL Server `"sql"` with an `"orders"` database and a data volume
3. Add MongoDB `"mongo"` with a `"sessions"` database
4. Add a `"webapi"` project that references all three databases

## Hints

- Use `AddPostgres("pg").AddDatabase("catalog")` — the database is a child of the server
- `WithInitFiles` takes a relative path to a directory containing `.sql` files
- Each `WithReference(db)` injects a connection string named after the database resource
