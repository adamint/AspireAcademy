# Cache Hosting Mastery

Review the key concepts from Redis, Garnet, and Valkey hosting before testing your knowledge in the quiz.

## Quick Reference

| Integration | Method | Resource Type | Image | Port |
|-------------|--------|---------------|-------|------|
| Redis | `AddRedis` | `RedisResource` | `redis:8.6` | 6379 |
| Garnet | `AddGarnet` | `GarnetResource` | `garnet:1.0` | 6379 |
| Valkey | `AddValkey` | `ValkeyResource` | `valkey:9.0` | 6379 |

All three share the same connection string format (`host:port`) and support `WithPersistence` and `WithDataVolume`.

Only Redis supports admin UI extensions: `WithRedisCommander` (web CLI) and `WithRedisInsight` (graphical management).

## Key Concepts to Remember

- **`WithPersistence`** enables RDB snapshotting but needs `WithDataVolume` to survive restarts
- **Wire compatibility** means all three work with the same Redis client libraries
- **Connection strings** are `host:port` format, optionally with `password={pass}`
- Garnet is from Microsoft; Valkey is the Linux Foundation Redis fork
