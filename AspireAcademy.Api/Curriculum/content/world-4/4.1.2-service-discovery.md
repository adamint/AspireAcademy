# Service Discovery at Runtime

When the C# API needs to call the Python ML service, how does it know the URL? Aspire uses **configuration-based service discovery** — environment variables injected by `WithReference`.

---

## How It Works in .NET

In a .NET service, register `AddServiceDefaults()` in your `Program.cs`:

```csharp
builder.AddServiceDefaults();
```

Then use `HttpClient` with the resource name:

```csharp
app.MapGet("/recommendations", async (HttpClient client) =>
{
    var response = await client.GetAsync("http://ml/recommend");
    return await response.Content.ReadAsStringAsync();
});
```

The hostname `ml` is resolved by Aspire's service discovery middleware. Behind the scenes:

1. The AppHost injected `services__ml__http__0=http://localhost:8000` into the API
2. `AddServiceDefaults()` registered a service discovery provider that reads these variables
3. When `HttpClient` sees `http://ml`, it resolves `ml` to `http://localhost:8000`

## How It Works in TypeScript

TypeScript services read the endpoint URL from environment variables directly:

```typescript
const mlUrl = process.env.services__ml__http__0;
const response = await fetch(`${mlUrl}/recommend`);
```

There is no automatic hostname resolution — you use the full URL from the environment variable.

## How It Works in Python

```python
import os, requests

ml_url = os.environ["services__ml__http__0"]
response = requests.get(f"{ml_url}/recommend")
```

## How It Works in Go

```go
mlURL := os.Getenv("services__ml__http__0")
resp, err := http.Get(mlURL + "/recommend")
```

## Environment Variable Format

Aspire injects service endpoints as:

```
services__{resource-name}__{scheme}__{index}
```

Examples:
- `services__api__http__0` — first HTTP endpoint of the "api" resource
- `services__api__https__0` — first HTTPS endpoint
- `ConnectionStrings__cache` — Redis connection string

## DNS-Based Discovery

In Kubernetes deployment, service discovery uses DNS instead of environment variables. The `services__api__http__0` variable is replaced by the Kubernetes service DNS name. Your code does not change — only the resolution mechanism.
