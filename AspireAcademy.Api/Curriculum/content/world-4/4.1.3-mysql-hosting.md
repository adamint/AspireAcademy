# MySQL & phpMyAdmin

MySQL hosting in Aspire follows the familiar server-plus-database pattern and adds `WithPhpMyAdmin` for a web-based admin UI.

## Adding MySQL

The `AddMySql` extension registers a `MySqlServerResource` backed by `docker.io/library/mysql:9.6`:

```csharp
var mysql = builder.AddMySql("mysql");
```

**Signature:**
```csharp
IResourceBuilder<MySqlServerResource> AddMySql(
    this IDistributedApplicationBuilder builder,
    string name,
    IResourceBuilder<ParameterResource>? password = null,
    int? port = null)
```

Aspire auto-generates the root password. The default container port is **3306**.

## Adding Databases

```csharp
var inventoryDb = mysql.AddDatabase("inventory");
```

This creates a `MySqlDatabaseResource` as a child of the server.

## Connection String Format

MySQL uses a semicolon-delimited format:
```
Server={host};Port=3306;User ID=root;Password={password};Database=inventory
```

## phpMyAdmin Admin UI

Attach the phpMyAdmin web interface (`docker.io/library/phpmyadmin:5.2`) for visual database management:

```csharp
var mysql = builder.AddMySql("mysql")
    .WithPhpMyAdmin();
```

phpMyAdmin auto-connects to your MySQL server with the configured credentials.

## Init Scripts

MySQL supports `WithInitFiles` for running `.sql` scripts on first container startup:

```csharp
var mysql = builder.AddMySql("mysql")
    .WithInitFiles("./mysql-init");
```

Place scripts in the directory with numeric prefixes for execution order (e.g., `01-schema.sql`, `02-seed.sql`).

> **Note:** The older `WithInitBindMount` method is deprecated in favor of `WithInitFiles`, which copies files into the container rather than using bind mounts. This works better with remote Docker hosts.

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var mysql = builder.AddMySql("mysql")
    .WithDataVolume()
    .WithPhpMyAdmin();

var inventoryDb = mysql.AddDatabase("inventory");

builder.AddProject<Projects.InventoryApi>("inventory-api")
    .WithReference(inventoryDb);

builder.Build().Run();
```

## Key Takeaways

- **Resource types**: `MySqlServerResource` → `MySqlDatabaseResource`.
- **Container image**: `docker.io/library/mysql:9.6` on port 3306.
- **`WithPhpMyAdmin`**: Adds a `PhpMyAdminContainerResource` for web-based administration.
- **Init scripts**: `WithInitFiles("./scripts")` runs `.sql` files on first startup.
- **Connection string**: `Server={host};Port=3306;User ID=root;Password={pass};Database={db}`.
