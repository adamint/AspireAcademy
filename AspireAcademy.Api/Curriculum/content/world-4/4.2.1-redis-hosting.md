# Redis: Commander, Insight & Persistence

Redis is the most popular caching integration in Aspire, with a rich ecosystem of admin tools and persistence options.

## Adding Redis

The `AddRedis` extension registers a `RedisResource` backed by `docker.io/library/redis:8.6`:

```csharp
var cache = builder.AddRedis("cache");
```

**Signature:**
```csharp
IResourceBuilder<RedisResource> AddRedis(
    this IDistributedApplicationBuilder builder,
    string name,
    int? port = null,
    IResourceBuilder<ParameterResource>? password = null)
```

The default port is **6379**. No password is set by default — use the `password` parameter for authentication.

## Connection String Format

Redis uses a simple `host:port` format:
```
{host}:{port}
```

With a password:
```
{host}:{port},password={password}
```

## Admin UIs

### Redis Commander

A web-based CLI for Redis (`docker.io/rediscommander/redis-commander:latest`):

```csharp
var cache = builder.AddRedis("cache")
    .WithRedisCommander();
```

### Redis Insight

The official Redis graphical management tool (`docker.io/redis/redisinsight:3.0`):

```csharp
var cache = builder.AddRedis("cache")
    .WithRedisInsight();
```

Redis Insight provides memory analysis, slow-log inspection, and a visual key browser — ideal for debugging cache behavior.

## Persistence with WithPersistence

By default, Redis is purely in-memory. `WithPersistence` enables RDB snapshotting:

```csharp
var cache = builder.AddRedis("cache")
    .WithPersistence(interval: TimeSpan.FromSeconds(60), keysChangedThreshold: 5);
```

**Parameters:**
- `interval`: How often to snapshot (default: uses Redis defaults)
- `keysChangedThreshold`: Minimum key changes before snapshot (default: 1)

**Important:** `WithPersistence` alone doesn't survive container restarts — combine it with `WithDataVolume` for true durability:

```csharp
var cache = builder.AddRedis("cache")
    .WithPersistence()
    .WithDataVolume();
```

## Complete Example

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var cache = builder.AddRedis("cache")
    .WithRedisInsight()
    .WithPersistence(interval: TimeSpan.FromMinutes(1))
    .WithDataVolume();

builder.AddProject<Projects.WebApi>("web-api")
    .WithReference(cache);

builder.Build().Run();
```

## Key Takeaways

- **Resource type**: `RedisResource` (container, implements `IResourceWithConnectionString`).
- **Connection string**: `{host}:{port}` — simple and compact.
- **`WithRedisCommander`**: Web CLI for quick commands.
- **`WithRedisInsight`**: Rich graphical management and analysis.
- **`WithPersistence`**: Enables RDB snapshots; pair with `WithDataVolume` for durability.
