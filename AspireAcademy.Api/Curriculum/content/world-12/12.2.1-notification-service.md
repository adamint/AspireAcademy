# ResourceNotificationService Deep Dive

`ResourceNotificationService` is the central nervous system of a running Aspire application. It is a pub/sub system that broadcasts resource state changes. Every time a resource transitions from Starting to Running, a health check fails, or a container exits — this service is how the rest of the system finds out.

---

## Architecture

```
Producers                        Consumers
---------                        ---------
DCP state changes  --+     +---> Dashboard (gRPC -> Blazor)
Health check polls --+     +---> WaitFor mechanism
Custom publishers  --+---->+
Lifecycle hooks    --+     +---> Resource event publisher
Resource commands  --+     +---> Custom subscribers
```

## PublishUpdateAsync (from the source)

There are two overloads. The primary one takes a resource, a resource ID (for replicas), and a state factory:

```csharp
public Task PublishUpdateAsync(IResource resource, string resourceId,
    Func<CustomResourceSnapshot, CustomResourceSnapshot> stateFactory)
{
    var notificationState = GetResourceNotificationState(resourceId, resource);

    lock (notificationState)
    {
        var previousState = GetCurrentSnapshot(resource, notificationState);
        var newState = stateFactory(previousState);

        // Increment the per-resource snapshot version
        newState = newState with { Version = notificationState.GetNextVersion() };

        // Update commands and icons from annotations
        newState = UpdateCommands(resource, newState);
        newState = UpdateIcons(resource, newState);

        notificationState.LastSnapshot = newState;

        // Notify all subscribers
        OnResourceUpdated?.Invoke(new ResourceEvent(resource, resourceId, newState));
    }

    return Task.CompletedTask;
}
```

Key details:
- **`lock (notificationState)`** — updates are serialized per resource (no concurrent modifications to the same resource's state)
- **`stateFactory(previousState)`** — you receive the last snapshot and return a new one, using `with` expressions for immutability
- **`Version`** — auto-incremented per resource, used by consumers to detect changes
- **`UpdateCommands` / `UpdateIcons`** — reads `ResourceCommandAnnotation` and icon annotations to include in the snapshot

The simpler overload publishes to all resolved names of a resource (handles replicas):

```csharp
public async Task PublishUpdateAsync(IResource resource,
    Func<CustomResourceSnapshot, CustomResourceSnapshot> stateFactory)
{
    var resourceNames = resource.GetResolvedResourceNames();
    foreach (var resourceName in resourceNames)
    {
        await PublishUpdateAsync(resource, resourceName, stateFactory);
    }
}
```

## WatchAsync

Returns an `IAsyncEnumerable<ResourceEvent>` that yields every state change:

```csharp
public async IAsyncEnumerable<ResourceEvent> WatchAsync(
    [EnumeratorCancellation] CancellationToken cancellationToken = default)
```

The Dashboard subscribes to this to get real-time updates. So does the WaitFor mechanism.

## WaitForResourceAsync

Used internally by WaitFor and available for tests:

```csharp
public Task<string> WaitForResourceAsync(
    string resourceName,
    IEnumerable<string> targetStates,
    CancellationToken cancellationToken = default)
```

Watches the notification stream and returns when the resource reaches one of the target states.

## CustomResourceSnapshot

The immutable record at the heart of every notification:

```csharp
public sealed record CustomResourceSnapshot
{
    // Required
    public required ResourceStateSnapshot? State { get; init; }
    public required ImmutableArray<ResourcePropertySnapshot> Properties { get; init; }

    // Optional state
    public DateTime CreationTimeStamp { get; init; }
    public ImmutableArray<UrlSnapshot> Urls { get; init; }
    public ImmutableArray<VolumeSnapshot> Volumes { get; init; }
    public ImmutableArray<EnvironmentVariableSnapshot> EnvironmentVariables { get; init; }
    public ImmutableArray<HealthReportSnapshot> HealthReports { get; init; }
    public ImmutableArray<ResourceCommandSnapshot> Commands { get; init; }
    public HealthStatus? HealthStatus { get; init; }
    public int? ExitCode { get; init; }
    public long Version { get; init; }
    // ... more properties
}
```

Using `with` expressions:

```csharp
await notificationService.PublishUpdateAsync(resource,
    snapshot => snapshot with
    {
        State = new ResourceStateSnapshot("Running",
            KnownResourceStateStyles.Success),
        Properties = [
            .. snapshot.Properties,
            new ResourcePropertySnapshot("region",
                KnownProperties.Resource.Region, "East US")
        ]
    });
```

## Thread Safety

- **Multiple publishers** can call `PublishUpdateAsync` concurrently — the `lock` serializes updates per resource
- **Multiple subscribers** watch simultaneously — each `WatchAsync` enumerator gets its own independent stream
- **Snapshot immutability** prevents data races — `sealed record` with `with` expressions

> 🌍 **Real World:** The Dashboard you see in your browser is powered entirely by `WatchAsync`. Every resource row, every status pill, every URL link — all of it comes from `CustomResourceSnapshot` objects streamed through this service. When you click "Stop" on a resource, the Dashboard sends a command, DCP stops the container, and a new snapshot flows back through this exact pipeline to update the UI in milliseconds.
