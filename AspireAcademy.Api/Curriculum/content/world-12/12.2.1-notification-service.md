# ResourceNotificationService Deep Dive

`ResourceNotificationService` is the central nervous system of a running Aspire application. It is a **pub/sub system** that broadcasts resource state changes to every interested consumer.

Every time a resource transitions from `Starting` to `Running`, or a health check fails, or a container exits ‚Äî this service is how the rest of the system finds out.

## CustomResourceSnapshot

At the heart of notification is `CustomResourceSnapshot` ‚Äî a sealed record that captures the complete state of a resource at a point in time:

- **Resource state** (Starting, Running, Exited, FailedToStart)
- **Properties** ‚Äî key-value pairs visible in the Dashboard
- **Endpoint URLs** ‚Äî allocated and active endpoints
- **Health status** ‚Äî last known health check result
- **Exit code** ‚Äî for terminated resources

Snapshots are **immutable records**. Each state change produces a new snapshot.

## Publishing Updates

```csharp
await resourceNotificationService.PublishUpdateAsync(
    myResource,
    snapshot => snapshot with
    {
        State = new ResourceStateSnapshot("Running", KnownResourceStateStyles.Success)
    });
```

The callback receives the previous snapshot and returns a new one using `with` expressions.

## Subscribing

```csharp
await foreach (var resourceEvent in
    resourceNotificationService.WatchAsync(cancellationToken))
{
    Console.WriteLine($"{resourceEvent.Resource.Name}: {resourceEvent.Snapshot.State?.Text}");
}
```

## How WaitFor Uses It

When you write `api.WaitFor(db)`, at runtime:

1. DCP starts PostgreSQL
2. System subscribes to ResourceNotificationService for `db`
3. API is held in `Waiting` state
4. PostgreSQL health check passes ‚Üí `Running` state published
5. WaitFor detects `Running` + healthy ‚Üí API starts

## Custom Subscriber Example

```csharp
builder.Services.AddHostedService<ResourceStateLogger>();

class ResourceStateLogger(ResourceNotificationService notifications, ILogger<ResourceStateLogger> logger) : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        await foreach (var evt in notifications.WatchAsync(ct))
        {
            logger.LogInformation("{Name} ‚Üí {State}", evt.Resource.Name, evt.Snapshot.State?.Text);
        }
    }
}
```

## Thread Safety

- Multiple publishers can call `PublishUpdateAsync` concurrently ‚Äî serialized per resource
- Multiple subscribers watch simultaneously ‚Äî each gets an independent stream
- Snapshot immutability prevents data races
