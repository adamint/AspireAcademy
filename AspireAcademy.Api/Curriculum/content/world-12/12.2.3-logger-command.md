# ResourceLoggerService and ResourceCommandService

Two core services for per-resource log streaming and command execution.

---

## ResourceLoggerService

Every resource gets its own log stream. `ResourceLoggerService` manages these streams and makes them available to consumers (primarily the Dashboard).

```csharp
// Get a logger for a specific resource
ILogger logger = resourceLoggerService.GetLogger(resource);

// Write to the resource's log stream
logger.LogInformation("Processing batch of {Count} items", batchSize);
```

Output appears in the Dashboard's Console Logs view for that resource. This is different from the application's general log output — it is scoped to a single resource.

### Watching Logs

```csharp
await foreach (var logLine in resourceLoggerService.WatchAsync("api", ct))
{
    Console.WriteLine($"[{logLine.Timestamp}] {logLine.Content}");
}
```

### How DCP Captures Logs

For containers, DCP captures `stdout`/`stderr` from the Docker/Podman process and publishes log lines through `ResourceLoggerService`. For executables, DCP redirects the process's standard output.

## ResourceCommandService

Manages custom commands that can be executed on running resources.

### Registering Commands

```csharp
var commandService = app.Services
    .GetRequiredService<ResourceCommandService>();

commandService.RegisterCommand("api", new ResourceCommand(
    name: "clear-cache",
    displayName: "Clear Cache",
    executeCallback: async context =>
    {
        var client = context.ServiceProvider
            .GetRequiredService<IHttpClientFactory>()
            .CreateClient();
        await client.PostAsync("http://api/admin/clear-cache", null);
        return CommandResult.Success("Cache cleared successfully");
    },
    iconName: "Delete",
    isHighlighted: false));
```

### Where Commands Appear

1. **Dashboard** — action button in the resource row
2. **CLI** — `aspire do api clear-cache`
3. **MCP** — `execute_resource_command` tool (available to AI agents)

### Built-In Commands

Every resource automatically gets lifecycle commands:
- `start` — start a stopped resource
- `stop` — stop a running resource
- `restart` — stop then start

These are registered by DCP during resource creation and appear in the Dashboard for all resources.

> 🌍 **Real World:** Commands are how AI agents interact with running Aspire apps. When Copilot or Claude uses the Aspire MCP server, `execute_resource_command` calls into `ResourceCommandService` — the same pipeline that powers the Dashboard's action buttons. Your custom commands automatically become available to AI agents without any additional work.

### CommandResult

```csharp
public record CommandResult(bool Success, string? Message = null)
{
    public static CommandResult Success(string? message = null) => new(true, message);
    public static CommandResult Failure(string? message = null) => new(false, message);
}
```
