# Writing a Complete Hosting Integration

This lesson is a full walkthrough of building a production-quality hosting integration — the same pattern every official Aspire.Hosting.* package uses.

---

```
src/Aspire.Hosting.Meilisearch/
  Aspire.Hosting.Meilisearch.csproj
  MeilisearchResource.cs
  MeilisearchBuilderExtensions.cs
  PublicAPI.Shipped.txt
  PublicAPI.Unshipped.txt

tests/Aspire.Hosting.Meilisearch.Tests/
  MeilisearchResourceTests.cs      (app model tests - no Docker)
  MeilisearchManifestTests.cs      (manifest output tests)

```

## 2. Resource Class

(See previous lesson for the full `MeilisearchResource` implementation)

Key requirements:
- Inherit `ContainerResource` (or `Resource` for non-container resources)
- Implement `IResourceWithConnectionString`
- Define named endpoint constants
- Use `ReferenceExpression` for connection string format

## 3. Extension Methods

(See previous lesson for the full `MeilisearchBuilderExtensions` implementation)

Key patterns:
- Static extension methods on `IDistributedApplicationBuilder`
- Return `IResourceBuilder<T>` for fluent chaining
- Use `.WithImage()`, `.WithHttpEndpoint()`, `.WithHealthCheck()`
- Provide convenience methods (`.WithDataVolume()`, configuration options)

## 4. Tests

### App Model Test (No Docker)

```csharp
[Fact]
public void AddMeilisearch_CreatesResourceWithCorrectAnnotations()
{
    using var builder = TestDistributedApplicationBuilder.Create();
    builder.AddMeilisearch("search");

    var resource = Assert.Single(builder.Resources.OfType<MeilisearchResource>());
    Assert.Equal("search", resource.Name);

    var endpoint = Assert.Single(
        resource.Annotations.OfType<EndpointAnnotation>());
    Assert.Equal("http", endpoint.Name);
    Assert.Equal(7700, endpoint.TargetPort);

    Assert.Single(resource.Annotations.OfType<HealthCheckAnnotation>());
}
```

### Manifest Test

```csharp
[Fact]
public async Task Meilisearch_ManifestOutput()
{
    using var builder = TestDistributedApplicationBuilder.Create(
        DistributedApplicationOperation.Publish);
    var search = builder.AddMeilisearch("search");

    var manifest = await ManifestUtils.GetManifest(search.Resource);

    Assert.Equal("container.v0", manifest["type"]?.ToString());
    Assert.Contains("getmeili/meilisearch", manifest["image"]?.ToString());
}
```

### Integration Test (Docker)

```csharp
[Fact]
[RequiresDocker]
public async Task Meilisearch_AcceptsConnections()
{
    var appHost = await DistributedApplicationTestingBuilder
        .CreateAsync<TestAppHost>();
    await using var app = await appHost.BuildAsync();
    await app.StartAsync();

    var connStr = await app.GetConnectionStringAsync("search");
    var client = new HttpClient { BaseAddress = new Uri(connStr) };
    var response = await client.GetAsync("/health");

    Assert.Equal(HttpStatusCode.OK, response.StatusCode);
}
```

## 6. Package and Publish

```bash
dotnet pack src/Aspire.Hosting.Meilisearch/
dotnet nuget push artifacts/packages/Aspire.Hosting.Meilisearch.1.0.0.nupkg \
    --source https://api.nuget.org/v3/index.json
```

After publishing, anyone can:

```csharp
dotnet add package Aspire.Hosting.Meilisearch

var search = builder.AddMeilisearch("search");
```

> 🐛 **Common Gotcha:** If you change any public API (add a method, change a parameter) without updating `PublicAPI.Unshipped.txt`, the Roslyn analyzer will fail your CI build. Every public type, method, and property must be listed. When the PR ships, entries move from Unshipped to Shipped — this is how breaking changes are tracked across releases.
