# Writing Custom Resources

When no existing resource type fits your infrastructure, create your own. This lesson shows the exact pattern from the Aspire source.

---

## Step 1: Define the Resource Class

Every custom resource inherits from `Resource` (for abstract resources) or `ContainerResource` (for container-backed resources). Implement `IResourceWithConnectionString` if other resources need to connect to it.

```csharp
public class MeilisearchResource(string name) :
    ContainerResource(name), IResourceWithConnectionString
{
    // Define a named endpoint constant
    internal const string PrimaryEndpointName = "http";

    // Lazy endpoint reference
    private EndpointReference? _primaryEndpoint;
    public EndpointReference PrimaryEndpoint =>
        _primaryEndpoint ??= new(this, PrimaryEndpointName);

    // Connection string format
    // Uses ReferenceExpression to resolve at runtime
    public ReferenceExpression ConnectionStringExpression =>
        ReferenceExpression.Create(
            $"http://{PrimaryEndpoint.Property(EndpointProperty.Host)}:{PrimaryEndpoint.Property(EndpointProperty.Port)}");
}
```

Key points:
- **`EndpointReference`** is lazy — the actual port is not known until DCP allocates it
- **`ReferenceExpression`** contains placeholders resolved at runtime by DCP
- **`IResourceWithConnectionString`** enables `WithReference()` to inject the connection string

## Step 2: Write the Extension Method

```csharp
public static class MeilisearchBuilderExtensions
{
    public static IResourceBuilder<MeilisearchResource> AddMeilisearch(
        this IDistributedApplicationBuilder builder,
        string name,
        int? port = null)
    {
        var resource = new MeilisearchResource(name);

        return builder.AddResource(resource)
            .WithImage("getmeili/meilisearch")
            .WithImageTag("v1.7")
            .WithHttpEndpoint(
                port: port,
                targetPort: 7700,
                name: MeilisearchResource.PrimaryEndpointName)
            .WithHealthCheck(MeilisearchResource.PrimaryEndpointName);
    }

    public static IResourceBuilder<MeilisearchResource> WithDataVolume(
        this IResourceBuilder<MeilisearchResource> builder,
        string name = "meili-data")
    {
        return builder.WithVolume(name, "/meili_data");
    }

    public static IResourceBuilder<MeilisearchResource> WithMasterKey(
        this IResourceBuilder<MeilisearchResource> builder,
        IResourceBuilder<ParameterResource> masterKey)
    {
        return builder.WithEnvironment("MEILI_MASTER_KEY", masterKey);
    }
}
```

Note the pattern:
- `AddResource(resource)` registers it with the builder
- `.WithImage()` / `.WithImageTag()` set the container image
- `.WithHttpEndpoint()` declares the network endpoint
- `.WithHealthCheck()` adds an HTTP health check annotation
- Additional methods (`.WithDataVolume()`, `.WithMasterKey()`) extend the builder

## Step 3: Use It

```csharp
var search = builder.AddMeilisearch("search")
    .WithDataVolume()
    .WithMasterKey(builder.AddParameter("meili-key", secret: true));

builder.AddProject<Projects.Api>("api")
    .WithReference(search)
    .WaitFor(search);
```

The API receives `ConnectionStrings__search=http://localhost:XXXXX`.

## Step 4: Add Custom Annotations (Optional)

```csharp
public class MeilisearchIndexAnnotation(
    string indexName,
    string primaryKey) : IResourceAnnotation
{
    public string IndexName { get; } = indexName;
    public string PrimaryKey { get; } = primaryKey;
}

// Usage
builder.AddMeilisearch("search")
    .WithAnnotation(new MeilisearchIndexAnnotation("articles", "id"));
```

Custom annotations carry metadata that pipeline steps, event handlers, or custom publishers can read via `resource.Annotations.OfType<MeilisearchIndexAnnotation>()`.

> 🎲 **Fun Fact:** `ReferenceExpression` doesn't just build strings — it contains interpolation placeholders that DCP resolves at runtime. The expression `$"http://{PrimaryEndpoint.Property(EndpointProperty.Host)}:{PrimaryEndpoint.Property(EndpointProperty.Port)}"` becomes something like `http://localhost:52341` only after DCP allocates the port. This deferred resolution is what makes `EndpointReference` lazy — the actual values don't exist until the infrastructure is running.

## Step 5: Custom Health Check (Optional)

For health checks beyond simple HTTP:

```csharp
public class MeilisearchHealthCheck(string connectionString) : IHealthCheck
{
    public async Task<HealthCheckResult> CheckHealthAsync(
        HealthCheckContext context,
        CancellationToken cancellationToken = default)
    {
        try
        {
            var client = new HttpClient
                { BaseAddress = new Uri(connectionString) };
            var response = await client.GetAsync("/health", cancellationToken);
            var body = await response.Content.ReadFromJsonAsync<MeiliHealth>(
                cancellationToken: cancellationToken);

            return body?.Status == "available"
                ? HealthCheckResult.Healthy()
                : HealthCheckResult.Degraded($"Status: {body?.Status}");
        }
        catch (Exception ex)
        {
            return HealthCheckResult.Unhealthy("Meilisearch unreachable", ex);
        }
    }
}
```
