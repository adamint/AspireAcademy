# Resource-Level Events

Resource events fire for individual resources. Each carries the `IResource` that triggered it and the `IServiceProvider` for accessing services.

---

## The Complete Catalog

### BeforeResourceStartedEvent

Fires just before DCP starts a specific resource:

```csharp
public class BeforeResourceStartedEvent(
    IResource resource,
    IServiceProvider services) : IDistributedApplicationResourceEvent
{
    public IResource Resource { get; } = resource;
    public IServiceProvider Services { get; } = services;
}
```

### ResourceReadyEvent

Fires the **first time** a resource transitions to a healthy running state. This is the event that `WaitFor` uses internally:

```csharp
public class ResourceReadyEvent(
    IResource resource,
    IServiceProvider services) : IDistributedApplicationResourceEvent
{
    public IResource Resource => resource;
    public IServiceProvider Services => services;
}
```

### ConnectionStringAvailableEvent

Fires when a resource's connection string becomes available (resolved from placeholders to actual values):

```csharp
public class ConnectionStringAvailableEvent(
    IResource resource,
    IServiceProvider services) : IDistributedApplicationResourceEvent
{
    public IResource Resource => resource;
    public IServiceProvider Services => services;
}
```

### ResourceStoppedEvent

Fires when a resource exits or is stopped. Includes the final `ResourceEvent` snapshot:

```csharp
public class ResourceStoppedEvent(
    IResource resource,
    IServiceProvider services,
    ResourceEvent resourceEvent) : IDistributedApplicationResourceEvent
{
    public IResource Resource { get; } = resource;
    public IServiceProvider Services { get; } = services;
    public ResourceEvent ResourceEvent { get; } = resourceEvent;
}
```

### ResourceEndpointsAllocatedEvent

Fires when DCP allocates endpoints (port numbers) for a specific resource:

```csharp
public class ResourceEndpointsAllocatedEvent(
    IResource resource,
    IServiceProvider services) : IDistributedApplicationResourceEvent
{
    public IResource Resource { get; } = resource;
    public IServiceProvider Services { get; } = services;
}
```

### InitializeResourceEvent

The richest event — fires as a signal to initialize a resource. Provides direct access to eventing, logging, and notification services:

```csharp
public class InitializeResourceEvent(
    IResource resource,
    IDistributedApplicationEventing distributedApplicationEventing,
    ResourceLoggerService resourceLoggerService,
    ResourceNotificationService resourceNotificationService,
    IServiceProvider services) : IDistributedApplicationResourceEvent
{
    public IResource Resource { get; } = resource;
    public IDistributedApplicationEventing Eventing { get; }
        = distributedApplicationEventing;
    public ILogger Logger { get; } = resourceLoggerService.GetLogger(resource);
    public ResourceNotificationService Notifications { get; }
        = resourceNotificationService;
    public IServiceProvider Services { get; } = services;
}
```

`InitializeResourceEvent` is unique — it provides `Eventing`, `Logger`, and `Notifications` directly, so your handler can subscribe to further events, log to the resource's stream, and publish state updates without resolving them from DI.

## Subscribing to Resource Events

```csharp
// All resources
builder.Eventing.Subscribe<ResourceReadyEvent>(async (evt, ct) =>
{
    Console.WriteLine($"{evt.Resource.Name} is ready");
});

// Specific resource
var db = builder.AddPostgres("pg").AddDatabase("atlas-db");
builder.Eventing.Subscribe<ConnectionStringAvailableEvent>(
    db.Resource,
    async (evt, ct) =>
    {
        var connStr = await ((IResourceWithConnectionString)evt.Resource)
            .GetConnectionStringAsync(ct);
        Console.WriteLine($"Connection string: {connStr}");
    });
```
