# Application-Level Events

Application events fire at specific points in the AppHost lifecycle — before resources start, after endpoints are allocated, and after resources are created.

---

## BeforeStartEvent

The most important event. Fires **after `Build()` but before DCP starts any resources**. This is your last chance to modify the app model.

```csharp
// From the source:
public class BeforeStartEvent(
    IServiceProvider services,
    DistributedApplicationModel model) : IDistributedApplicationEvent
{
    public IServiceProvider Services { get; } = services;
    public DistributedApplicationModel Model { get; } = model;
}
```

Use cases:
- **Validate the app model** — check that required resources exist, annotations are correct
- **Register additional services** — add background services, health checks
- **Inspect the resource graph** — log all resources and their relationships
- **Conditional configuration** — add dev-only resources

```csharp
builder.Eventing.Subscribe<BeforeStartEvent>(async (evt, ct) =>
{
    // Validate every container has a health check
    foreach (var resource in evt.Model.Resources.OfType<ContainerResource>())
    {
        var hasHealthCheck = resource.Annotations
            .OfType<HealthCheckAnnotation>().Any();
        if (!hasHealthCheck)
        {
            evt.Services.GetRequiredService<ILogger<Program>>()
                .LogWarning("Container {Name} has no health check", resource.Name);
        }
    }
});
```

## AfterEndpointsAllocatedEvent

> **Note**: This event is deprecated in favor of the resource-specific `ResourceEndpointsAllocatedEvent`. It still works but emits a compiler warning.

Fires after DCP allocates ports for all endpoints, before resources start.

```csharp
[Obsolete("Use BeforeResourceStartedEvent or ResourceEndpointsAllocatedEvent instead.")]
public class AfterEndpointsAllocatedEvent(
    IServiceProvider services,
    DistributedApplicationModel model) : IDistributedApplicationEvent
{
    public IServiceProvider Services { get; } = services;
    public DistributedApplicationModel Model { get; } = model;
}
```

## AfterResourcesCreatedEvent

Fires after DCP has submitted all resources (but they may still be starting):

```csharp
[AspireExport(ExposeProperties = true)]
public class AfterResourcesCreatedEvent(
    IServiceProvider services,
    DistributedApplicationModel model) : IDistributedApplicationEvent
{
    public IServiceProvider Services { get; } = services;
    public DistributedApplicationModel Model { get; } = model;
}
```

## Lifecycle Sequence

```
Build()
  |
  v
BeforeStartEvent                    <-- your last chance to modify
  |
  v
DCP starts, allocates ports
  |
  v
AfterEndpointsAllocatedEvent        <-- deprecated, use resource events
  |
  v
Resources submitted to DCP
  |
  v
AfterResourcesCreatedEvent          <-- resources are starting
  |
  v
Individual resources: Starting -> Running
```
