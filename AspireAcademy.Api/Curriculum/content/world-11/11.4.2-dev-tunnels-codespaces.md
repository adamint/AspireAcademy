# Dev Tunnels and Codespaces

Aspire supports remote development through Dev Tunnels and GitHub Codespaces.

---

## Dev Tunnels

Dev Tunnels create secure HTTPS endpoints for your local services, shareable with teammates or external systems:

```csharp
builder.AddProject<Projects.Api>("api")
    .WithDevTunnel("api-tunnel");
```

This creates a public URL like `https://abc123.devtunnels.ms` that tunnels to your local API.

Use cases:
- Share your local app with a reviewer
- Test webhooks from external services (Stripe, GitHub)
- Demo work-in-progress features

## GitHub Codespaces

Aspire works in Codespaces — cloud-hosted development environments:

1. Add a `.devcontainer/devcontainer.json` to your repo
2. Configure Docker-in-Docker for container resources
3. Forward ports for the Dashboard and service endpoints

The Aspire VS Code extension works in Codespaces, providing the same code lenses, tree view, and debugging experience as local development.

## Dev Container Configuration

```json
{
    "image": "mcr.microsoft.com/devcontainers/dotnet:9.0",
    "features": {
        "ghcr.io/devcontainers/features/docker-in-docker:2": {},
        "ghcr.io/devcontainers/features/node:1": {},
        "ghcr.io/devcontainers/features/python:1": {}
    },
    "postCreateCommand": "dotnet tool install --global aspire",
    "forwardPorts": [18888]
}
```
