# Dev Tunnels and Codespaces

Aspire supports remote development through Dev Tunnels and GitHub Codespaces.

---

## Dev Tunnels

Dev Tunnels create secure public HTTPS endpoints for your local services:

```csharp
builder.AddProject<Projects.BazaarApi>("api")
    .WithDevTunnel("api-tunnel");
```

Creates a URL like `https://abc123.devtunnels.ms` that tunnels to your local API.

Use cases:
- Share your local app with teammates
- Test webhooks from Stripe, GitHub, Twilio
- Mobile testing against local API
- Demo sessions without deploying

## GitHub Codespaces

Aspire works in Codespaces — cloud-hosted dev environments with Docker support.

### Setup

Add `.devcontainer/devcontainer.json`:

```json
{
    "image": "mcr.microsoft.com/devcontainers/dotnet:9.0",
    "features": {
        "ghcr.io/devcontainers/features/docker-in-docker:2": {},
        "ghcr.io/devcontainers/features/node:1": {},
        "ghcr.io/devcontainers/features/python:1": {}
    },
    "postCreateCommand": "curl -sSL https://aspire.dev/install.sh | bash",
    "forwardPorts": [18888],
    "customizations": {
        "vscode": {
            "extensions": ["ms-dotnettools.dotnet-aspire"]
        }
    }
}
```

Key points:
- **docker-in-docker** required for container resources
- **node + python** for polyglot Bazaar services
- **forwardPorts** for Dashboard access
- Same config works with VS Code Dev Containers locally

### Port Forwarding

Codespaces automatically forwards ports for Aspire services. Dashboard, APIs, and frontends are accessible through GitHub port forwarding with HTTPS.
