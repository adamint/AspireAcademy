# Publish and Deploy Commands

Generate deployment artifacts and deploy to cloud targets.

---

## aspire publish

Generate deployment artifacts without deploying:

```bash
aspire publish                              # default target
aspire publish --target docker-compose      # Docker Compose
aspire publish --target kubernetes          # Kubernetes Helm
aspire publish --target azure               # Azure Bicep
aspire publish --output-path ./deploy       # specify output directory
```

The command:
1. Builds the AppHost in publish mode
2. Establishes a backchannel to the AppHost
3. Executes the publishing pipeline
4. Streams progress back to the CLI
5. Writes artifacts to the output path

## aspire deploy

Publish and deploy in one step:

```bash
aspire deploy --target azure                # publish + deploy to Azure
aspire deploy --target kubernetes           # publish + deploy to K8s cluster
aspire deploy --clear-cache                 # clear cached artifacts
```

`aspire deploy` runs the full pipeline: build → push → publish → deploy.

Each line corresponds to a pipeline step executing in the DAG.

> 🌍 **Real World:** In CI/CD pipelines, use `aspire publish` to generate artifacts in one stage, then deploy them using your existing deployment tooling (Helm, Terraform, Azure CLI). This separates artifact generation from deployment, giving you full control over release gates and approvals.
