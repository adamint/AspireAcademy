# CLI Architecture

The Aspire CLI is a .NET command-line tool with 40+ commands organized hierarchically.

---

## Command Structure

All commands inherit from `BaseCommand`:

```
aspire
├── new          — scaffold projects
├── init         — initialize Aspire in existing project
├── run          — run AppHost locally
├── start        — start AppHost (foreground)
├── stop         — stop running AppHost
├── ps           — list running resources
├── exec         — execute command on a resource
├── logs         — view resource logs
├── describe     — describe resources
├── do           — execute lifecycle commands
├── restore      — restore dependencies
├── wait         — wait for resources
├── publish      — generate deployment artifacts
├── deploy       — deploy to cloud
├── add          — add resources to AppHost
├── config       — manage configuration
├── secret       — manage secrets
├── certificate  — manage certificates
├── template     — manage templates
├── docs         — search documentation
├── doctor       — diagnose environment
├── telemetry    — manage telemetry
├── agent        — AI agent integration
│   └── mcp      — start MCP server
└── resource     — resource management
```

## The Backchannel

The CLI communicates with running AppHosts via a **JSON-RPC backchannel** over Unix sockets:

```
CLI  ──JSON-RPC──▶  AppHost
         ↑              │
         └──────────────┘
         responses
```

Commands like `ps`, `logs`, `exec`, and `describe` query the running AppHost through this channel. The AppHost must be running for these commands to work.

> 🌍 **Real World:** The command hierarchy mirrors the Aspire application lifecycle — scaffolding (`new`), development (`run`, `ps`, `logs`), production (`publish`, `deploy`), and operations (`agent mcp`, `doctor`). Commands like `ps`, `logs`, `exec`, and `describe` are backchannel commands that require a running AppHost — they don't work standalone.

## Global Options

Every command supports:
- `--debug` — enable debug logging
- `--log-level` — set log verbosity
- `--non-interactive` — disable interactive prompts
- `--no-logo` — suppress the Aspire logo
