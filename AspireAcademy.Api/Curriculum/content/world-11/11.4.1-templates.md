# Templates and the Template Catalog

`aspire new` uses a template system to scaffold projects in any language.

---

## Built-In Templates

| Template | Description | Languages |
|---|---|---|
| `aspire-starter` | Full app with AppHost, ServiceDefaults, API, frontend | C# |
| `aspire-empty` | Minimal AppHost | C# |
| `aspire-apphost-singlefile` | Single-file AppHost | C# |
| `aspire-ts-cs-starter` | TypeScript + C# combo | TypeScript/C# |
| `aspire-py-starter` | Python starter | Python |
| `aspire-js-frontend-starter` | JavaScript frontend | JavaScript |
| `aspire-xunit` / `aspire-nunit` / `aspire-mstest` | Test projects | C# |

## Using Templates

```bash
aspire new aspire-starter --name MyApp
aspire new aspire-py-starter --name MyPythonApp
aspire new aspire-empty --name MinimalApp
```

## Template Internals

Templates live in `Aspire.ProjectTemplates/templates/`. Each template contains:
- Template definition files (`.template.config/template.json`)
- Source files with placeholder substitution
- Post-creation actions (restore, code generation)

## Custom Templates

You can create custom templates following the `dotnet new` template format. Register them with:

```bash
dotnet new install ./path/to/template/
```

Then use them with `aspire new` or `dotnet new`.
