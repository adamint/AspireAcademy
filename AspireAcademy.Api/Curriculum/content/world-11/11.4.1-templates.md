# Templates and the Template Catalog

`aspire new` uses a template system to scaffold projects in any language.

---

## Built-In Templates

| Template | Description | Languages |
|---|---|---|
| `aspire-starter` | Full app: AppHost + ServiceDefaults + API + frontend | C# |
| `aspire-empty` | Minimal AppHost with just CreateBuilder | C# |
| `aspire-apphost-singlefile` | Single-file AppHost with #: directives | C# |
| `aspire-ts-cs-starter` | TypeScript AppHost + C# API | TypeScript/C# |
| `aspire-py-starter` | Python AppHost + Python API | Python |
| `aspire-js-frontend-starter` | JavaScript frontend with Aspire | JavaScript |
| `aspire-xunit` | xUnit test project with Aspire testing | C# |

## Using Templates

```bash
aspire new                                  # interactive
aspire new aspire-starter --name MyApp      # direct
aspire new aspire-py-starter --name Atlas   # Python
```

## What Gets Generated

For `aspire-starter --name Atlas`:

```
Atlas/
    Atlas.sln
    Atlas.AppHost/
        Program.cs         - AppHost with API + frontend
    Atlas.ServiceDefaults/
        Extensions.cs      - OpenTelemetry, health checks, resilience
    Atlas.Api/
        Program.cs         - Minimal API
    Atlas.Web/
        Program.cs         - Blazor frontend
```

For `aspire-py-starter --name Atlas`:

```
Atlas/
    apphost.py             - Python AppHost
    .modules/aspire_app/   - Generated Python SDK
    requirements.txt
    aspire.config.json
```

## Template Internals

Templates live in `src/Aspire.ProjectTemplates/templates/`. Each has a `.template.config/template.json` defining shortName, parameters, and postActions (dotnet restore, npm install, code generation).

## Custom Templates

Create your own following the `dotnet new` template format:

```bash
dotnet new install ./path/to/my-template/
aspire new my-custom-template --name MyProject
```
