# MCP Tools Reference

The Aspire MCP server exposes 14+ tools that AI agents can invoke.

---

## Resource Management

| Tool | Description |
|---|---|
| `list_resources` | List all resources with state, type, endpoints |
| `execute_resource_command` | Run lifecycle commands (start, stop, restart) |
| `select_apphost` | Select the active AppHost (multi-AppHost support) |
| `list_apphosts` | List all running AppHosts |

## Observability

| Tool | Description |
|---|---|
| `list_console_logs` | Get console output from a resource |
| `list_structured_logs` | Query structured (telemetry) log entries |
| `list_traces` | List distributed traces |
| `list_trace_structured_logs` | Get logs within a specific trace |

## Integrations and Diagnostics

| Tool | Description |
|---|---|
| `list_integrations` | List available Aspire integrations |
| `doctor` | Diagnose Aspire environment issues |
| `refresh_tools` | Refresh MCP tool definitions |

## Documentation

| Tool | Description |
|---|---|
| `list_docs` | List available documentation topics |
| `search_docs` | Search Aspire documentation |
| `get_doc` | Retrieve a specific documentation page |

## Example: AI Agent Debugging Session

An AI agent can:
1. `list_resources` → see all resources and their states
2. `list_console_logs("api")` → read API error logs
3. `list_traces` → find the failing trace
4. `list_trace_structured_logs(traceId)` → see all logs for that request
5. `search_docs("health checks")` → find relevant documentation
6. Suggest a fix based on the evidence
