# The MCP Server

Aspire exposes an MCP (Model Context Protocol) server that lets AI agents interact with running applications.

---

## Starting the MCP Server

```bash
aspire agent mcp
```

This starts a stdio-based MCP server that AI agents (Claude Code, VS Code Copilot, etc.) can connect to.

## Architecture

```
AI Agent (Claude Code / Copilot)
    ↓  MCP Protocol (stdio)
Aspire MCP Server
    ↓  JSON-RPC backchannel
Running AppHost
    ↓  gRPC
DCP (resources)
    ↓  OTLP
Dashboard (telemetry)
```

The MCP server acts as a bridge — it translates MCP tool calls into backchannel queries against the running AppHost.

## StdioMcpTransportFactory

The transport uses standard input/output for communication with the AI agent process. This is the standard MCP transport for command-line tools.

## IMcpNotifier

The MCP server can push notifications to connected agents:
- Resource state changes (started, stopped, crashed)
- Health check failures
- Log entries matching filters

## Dynamic Tool Refresh

When resources change (new resource added, resource restarted), the MCP server refreshes its tool list via `IMcpResourceToolRefreshService`. Connected agents see updated tools without reconnecting.
