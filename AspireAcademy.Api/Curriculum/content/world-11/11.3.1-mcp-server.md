# The MCP Server

Aspire exposes an MCP (Model Context Protocol) server that lets AI agents interact with running applications.

---

## Starting the MCP Server

```bash
aspire agent mcp
```

This starts a stdio-based MCP server. AI agents spawn this process and communicate via stdin/stdout.

## Architecture

```
AI Agent (Claude Code / Copilot / OpenCode)
    |
    |  MCP Protocol (JSON over stdio)
    v
Aspire CLI MCP Server
    |
    |  JSON-RPC backchannel (Unix socket)
    v
Running AppHost
    |
    |  gRPC
    v
DCP (resources, containers, processes)
    |
    |  OTLP
    v
Dashboard (telemetry store)
```

The MCP server bridges MCP tool calls into backchannel queries against the running AppHost.

> 🌍 **Real World:** The MCP server works with any MCP-compatible agent — Claude Code, VS Code Copilot, GitHub Copilot CLI, OpenCode, and custom agents. The protocol is agent-agnostic, so you're not locked into any specific AI tooling.

## StdioMcpTransportFactory

The transport reads JSON from stdin and writes JSON to stdout. This is the standard MCP transport for CLI tools. The AI agent spawns `aspire agent mcp` and communicates via I/O streams.

## IMcpNotifier

The MCP server pushes notifications to connected agents:

- Resource state changes (started, stopped, crashed)
- Health check failures
- Log entries matching filters

When a resource crashes, the connected AI agent is notified immediately for proactive debugging.

## Dynamic Tool Refresh

When the AppHost's resources change, `IMcpResourceToolRefreshService.RefreshToolsAsync()` updates the tool list. Connected agents discover new custom commands without reconnecting.

## VS Code Integration

The VS Code extension automatically registers the Aspire MCP server via `AspireMcpServerDefinitionProvider`. VS Code Copilot Chat gains access to all Aspire tools when you open an Aspire workspace.
