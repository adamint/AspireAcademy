# Agent Integration

Aspire auto-detects AI agent environments and configures itself to work with them.

---

## Supported Agents

| Agent | Scanner | How Detected |
|---|---|---|
| Claude Code | `ClaudeCodeAgentEnvironmentScanner` | Claude Code CLI process |
| VS Code Copilot | `VsCodeAgentEnvironmentScanner` | VS Code extension context |
| OpenCode | `OpenCodeAgentEnvironmentScanner` | OpenCode CLI process |
| Playwright | `PlaywrightSupport` | Playwright test context |
| GitHub Copilot CLI | `CopilotCliRunner` | Copilot CLI process |

## How Auto-Detection Works

When you run `aspire agent mcp`, the CLI:
1. Scans the environment for known agent indicators
2. Applies agent-specific configuration (prompt templates, tool filtering)
3. Starts the MCP server with appropriate settings

## Agent Configuration

Each agent scanner provides:
- Environment detection (is this agent running?)
- Default MCP tool set (which tools to expose)
- Prompt templates (how to format output for the agent)
- Configuration file paths (where agent configs live)

## Using Aspire with Claude Code

```bash
# In your Aspire project
aspire agent mcp
```

Claude Code discovers the MCP server and gains access to all Aspire tools. You can then ask Claude to:
- "What resources are running?"
- "Show me the logs from the API"
- "Find the slowest trace"
- "Why is the worker failing?"

## Using Aspire with VS Code Copilot

The VS Code extension automatically registers the Aspire MCP server. Copilot Chat can access Aspire tools when you are working in an Aspire workspace.
