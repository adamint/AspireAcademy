# Core Commands: run, start, stop, ps, exec, logs

The operational commands for managing running Aspire applications.

---

## aspire run

Start the AppHost and attach to the foreground:

```bash
aspire run                          # run in current directory
aspire run --project ./MyAppHost    # specify AppHost path
aspire run --no-build               # skip build step
```

## aspire start / stop

Start in the background:

```bash
aspire start                        # returns immediately
aspire stop                         # stops the running AppHost
```

## aspire ps

List running resources:

```bash
aspire ps
```

Output:
```
NAME       TYPE        STATE     ENDPOINTS
pg         Container   Running   tcp://localhost:5432
cache      Container   Running   tcp://localhost:6379
api        Project     Running   http://localhost:5001
frontend   Executable  Running   http://localhost:5173
ml         Executable  Running   http://localhost:8000
worker     Executable  Running   -
```

## aspire logs

View resource logs:

```bash
aspire logs api                     # logs for the API resource
aspire logs --follow                # stream all logs in real time
aspire logs pg --since 5m           # logs from the last 5 minutes
```

## aspire exec

Execute a command inside a resource:

```bash
aspire exec pg -- psql -U postgres  # open PostgreSQL shell
aspire exec cache -- redis-cli PING # ping Redis
```

## aspire describe

Show detailed information about a resource:

```bash
aspire describe api
```

Output includes endpoints, environment variables, health status, annotations, and connection strings.
