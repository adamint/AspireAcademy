# Polyglot AppHost Parsers

The VS Code extension can parse AppHost files in six languages to provide code lens support.

---

## Supported Languages

| Language | Parser | Detection |
|---|---|---|
| C# | Roslyn-based | `*.cs` with `CreateBuilder` |
| TypeScript | TypeScript AST | `apphost.ts` with `createBuilder` |
| Python | Python AST | `apphost.py` with `create_builder` |
| Go | Go AST | `apphost.go` with `CreateBuilder` |
| Java | Java AST | `AppHost.java` with `CreateBuilder` |
| Rust | Rust AST | `apphost.rs` with `create_builder` |

## What Parsers Extract

Each parser identifies:
- The AppHost entry point (where to place the Run/Debug code lens)
- Resource names (for tree view display)
- Resource relationships (for dependency visualization)

## Adding Support for New Languages

The parser architecture is extensible. Each parser implements a common interface that extracts:
1. Builder creation location (line number)
2. Resource declarations (name, type, line)
3. Reference declarations (source → target)

If Aspire adds AppHost support for a new language, the extension can add a parser for it.
