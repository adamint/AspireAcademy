# Polyglot AppHost Parsers

The VS Code extension parses AppHost files in C#, TypeScript, and Python to provide code lenses, tree views, and gutter decorations.

---

## Supported Languages

| Language | File Pattern | Entry Point Detection |
|---|---|---|
| **C#** | `*.cs` or `*.csproj` | `DistributedApplication.CreateBuilder` |
| **TypeScript** | `apphost.ts` | `createBuilder()` from `.modules/aspire.js` |
| **Python** | `apphost.py` | `create_builder()` from `aspire` module |

## What Parsers Extract

### 1. Builder Entry Point (line number)

The exact line where the builder is created. This is where the Run/Debug code lens appears.

### 2. Resource Declarations

```csharp
var cache = builder.AddRedis("cache");        // Name: cache, Type: Redis
var db = builder.AddPostgres("pg");           // Name: pg, Type: PostgreSQL
var api = builder.AddProject<Projects.Api>("api"); // Name: api, Type: Project
```

The parser extracts resource names and types for the tree view sidebar.

### 3. Reference Relationships

```csharp
api.WithReference(db);    // api depends on db
api.WithReference(cache); // api depends on cache
```

These build the dependency graph shown in the tree view.

## How Parsers Work

Each language parser uses regex and/or AST analysis to find builder creation, resource declarations, and reference calls. The parser interface is simple:

Input: file content (string)
Output: builder line number, resource list (name + type + line), reference list (source + target)

Example TypeScript parser logic:

```typescript
// Find builder creation
const builderMatch = text.match(
    /(?:const|let|var)\s+(\w+)\s*=\s*await\s+createBuilder/);

// Find resource declarations
const resourceRegex = /await\s+\w+\.(add\w+)\("(\w+)"/g;
```

## The Tree View

```
ASPIRE APPHOSTS
  Bazaar AppHost (apphost.cs)
    pg (PostgreSQL)
      bazaar-db (Database)
    cache (Redis)
    messaging (RabbitMQ)
    api (Project) -> pg, cache, messaging
    frontend (Vite) -> api
    ml (Python) -> messaging
    worker (Go) -> messaging
```

Resources show type icons and dependency arrows. Running resources show green/red indicators.

> 🌍 **Real World:** The parser's regex-based approach is intentionally simple — it doesn't need a full language server. This makes it fast enough to re-parse on every keystroke for real-time tree view updates, even in large AppHost files with dozens of resources.
