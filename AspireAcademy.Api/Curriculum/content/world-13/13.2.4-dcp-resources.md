# DCP Resource Types

DCP manages three primary resource types. Each is a C# class that mirrors a Kubernetes Custom Resource Definition (CRD), but optimized for local development.

---

## Container

Represents a Docker or Podman container. Created by `AddContainer()` and all container-based hosting integrations.

```csharp
// src/Aspire.Hosting/Dcp/Model/Container.cs (simplified)

internal sealed class Container : CustomResource, IKubernetesStaticMetadata
{
    public ContainerSpec? Spec { get; set; }
    public ContainerStatus? Status { get; set; }
}

internal sealed class ContainerSpec
{
    public string? Image { get; set; }          // e.g., "redis:7.4"
    public List<EnvVar>? Env { get; set; }       // environment variables
    public List<ContainerPort>? Ports { get; set; }  // port mappings
    public List<VolumeMount>? VolumeMounts { get; set; }
    public string? Command { get; set; }         // entrypoint override
    public List<string>? Args { get; set; }      // command arguments
    public ContainerLifetime? Lifetime { get; set; } // Session or Persistent
}

internal sealed class ContainerStatus
{
    public string? State { get; set; }           // Running, Exited, etc.
    public string? ContainerId { get; set; }     // Docker container ID
    public int? ExitCode { get; set; }
    public DateTime? StartTimestamp { get; set; }
    public DateTime? FinishTimestamp { get; set; }
    public List<ContainerPortStatus>? Ports { get; set; } // allocated ports
}
```

## Executable

Represents a local process. Created by `AddProject<T>()`, `AddExecutable()`, `AddNodeApp()`, `AddPythonApp()`, etc.

```csharp
internal sealed class Executable : CustomResource, IKubernetesStaticMetadata
{
    public ExecutableSpec? Spec { get; set; }
    public ExecutableStatus? Status { get; set; }
}

internal sealed class ExecutableSpec
{
    public string? ExecutablePath { get; set; }   // "dotnet", "go", "python3"
    public string? WorkingDirectory { get; set; }
    public List<string>? Args { get; set; }       // ["run", "--project", "..."]
    public List<EnvVar>? Env { get; set; }
    public string? LaunchProfile { get; set; }    // for .NET projects
}
```

## ExecutableReplicaSet

Multiple instances of an Executable. Created when you use `.WithReplicas(n)`:

```csharp
internal sealed class ExecutableReplicaSet : CustomResource, IKubernetesStaticMetadata
{
    public ExecutableReplicaSetSpec? Spec { get; set; }
    public ExecutableReplicaSetStatus? Status { get; set; }
}

internal sealed class ExecutableReplicaSetSpec
{
    public int Replicas { get; set; }             // number of instances
    public ExecutableSpec? Template { get; set; }  // spec for each replica
}
```

DCP creates `Replicas` individual processes, each with unique allocated ports.

## How the AppHost Translates Resources

```csharp
// In DcpExecutor (simplified)
if (resource is ContainerResource container)
{
    var dcpContainer = new Container
    {
        Metadata = new() { Name = resource.Name },
        Spec = new ContainerSpec
        {
            Image = container.GetContainerImageName(),
            Ports = GetPortMappings(resource),
            Env = GetEnvironmentVariables(resource),
            VolumeMounts = GetVolumeMounts(resource),
            Lifetime = GetContainerLifetime(resource)
        }
    };
    await _kubernetesService.CreateAsync(dcpContainer, ct);
}
```

## Container Lifetime in DCP

| Lifetime | DCP Behavior |
|---|---|
| `Session` | Container created on startup, `docker rm -f` on shutdown |
| `Persistent` | DCP checks if container exists. If yes, starts it. If no, creates it. Not removed on shutdown. |

## Watching State Changes

DCP reports state changes back to the AppHost via the watch API:

```csharp
await foreach (var (eventType, container) in
    _kubernetesService.WatchAsync<Container>(cancellationToken: ct))
{
    // eventType: Added, Modified, Deleted
    // container.Status.State: "Running", "Exited", etc.
    await UpdateResourceState(container.Metadata.Name, container.Status);
}
```

> 🌍 **Real World:** The `ContainerLifetime.Persistent` option is what enables database containers to survive AppHost restarts. When you use `.WithLifetime(ContainerLifetime.Persistent)`, DCP checks if a container with that name already exists before creating a new one. If it exists, DCP starts it instead of creating a duplicate. This means your database data persists across development sessions.
