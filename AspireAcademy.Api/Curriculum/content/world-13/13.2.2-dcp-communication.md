# DCP Communication

The Developer Control Plane (DCP) runs as a separate process. The AppHost communicates with it over gRPC using a Kubernetes-inspired API.

---

## DcpHost — Starting the DCP Process

`DcpHost` is a `BackgroundService` that manages the DCP process lifecycle:

```csharp
// src/Aspire.Hosting/Dcp/DcpHost.cs (simplified)

internal sealed class DcpHost : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        // 1. Find the DCP binary
        var dcpPath = _locations.DcpPath;

        // 2. Start DCP as a child process
        var process = Process.Start(new ProcessStartInfo
        {
            FileName = dcpPath,
            Arguments = BuildArguments(),
            WorkingDirectory = _locations.DcpDirectory,
            // ... environment variables for kubeconfig, socket paths
        });

        // 3. Wait for DCP to be ready
        await WaitForDcpReady(stoppingToken);

        // 4. Submit resources via DcpExecutor
        await _dcpExecutor.RunApplicationAsync(stoppingToken);

        // 5. On cancellation, tear down
        await _kubernetesService.StopServerAsync();
    }
}
```

## The Kubernetes-Inspired API (IKubernetesService)

DCP exposes a Kubernetes-style API. The AppHost uses `IKubernetesService` to create, watch, and delete resources:

```csharp
// src/Aspire.Hosting/Dcp/KubernetesService.cs

internal interface IKubernetesService
{
    Task<T> CreateAsync<T>(T obj, CancellationToken ct)
        where T : CustomResource, IKubernetesStaticMetadata;

    Task<T> GetAsync<T>(string name, string? ns, CancellationToken ct)
        where T : CustomResource, IKubernetesStaticMetadata;

    Task<List<T>> ListAsync<T>(string? ns, CancellationToken ct)
        where T : CustomResource, IKubernetesStaticMetadata;

    Task<T> DeleteAsync<T>(string name, string? ns, CancellationToken ct)
        where T : CustomResource, IKubernetesStaticMetadata;

    IAsyncEnumerable<(WatchEventType, T)> WatchAsync<T>(
        string? ns, CancellationToken ct)
        where T : CustomResource, IKubernetesStaticMetadata;

    Task<Stream> GetLogStreamAsync<T>(T obj, string logStreamType,
        CancellationToken ct, bool? follow, bool? timestamps,
        long? tail)
        where T : CustomResource, IKubernetesStaticMetadata;

    Task StopServerAsync(string resourceCleanup, CancellationToken ct);
}
```

This is not actual Kubernetes — DCP implements a compatible API locally. The `CustomResource` base class mimics Kubernetes CRDs.

## DcpExecutor — Submitting Resources

`DcpExecutor` translates the Aspire app model into DCP resource definitions:

```csharp
// src/Aspire.Hosting/Dcp/DcpExecutor.cs (simplified flow)

internal sealed class DcpExecutor
{
    public async Task RunApplicationAsync(CancellationToken ct)
    {
        var model = _serviceProvider
            .GetRequiredService<DistributedApplicationModel>();

        // Fire BeforeStartEvent
        await _eventing.PublishAsync(
            new BeforeStartEvent(_serviceProvider, model), ct);

        // Create DCP resources for each Aspire resource
        foreach (var resource in model.Resources)
        {
            if (resource is ContainerResource container)
            {
                await CreateContainerAsync(container, ct);
            }
            else if (resource is ProjectResource project)
            {
                await CreateExecutableAsync(project, ct);
            }
            else if (resource is ExecutableResource executable)
            {
                await CreateExecutableAsync(executable, ct);
            }
        }

        // Fire AfterResourcesCreatedEvent
        await _eventing.PublishAsync(
            new AfterResourcesCreatedEvent(_serviceProvider, model), ct);

        // Watch for state changes from DCP
        await WatchResourceChangesAsync(ct);
    }
}
```

## State Change Flow

```
DCP detects container is running
  |
  v
IKubernetesService.WatchAsync<Container>()
  yields (WatchEventType.Modified, container)
  |
  v
DcpExecutor.ProcessResourceChange()
  |
  v
ResourceNotificationService.PublishUpdateAsync(resource,
  snapshot => snapshot with { State = Running })
  |
  v
Subscribers notified:
  - Dashboard (gRPC push)
  - WaitFor mechanism (dependency check)
  - Your custom event handlers
```

## How Resources Map to DCP Types

| Aspire | DCP Type | Created By |
|---|---|---|
| `AddContainer("nginx", "nginx")` | `Container` | `CreateAsync<Container>(...)` |
| `AddRedis("cache")` | `Container` | Same (Redis is a container) |
| `AddProject<T>("api")` | `Executable` | `CreateAsync<Executable>(...)` |
| `.WithReplicas(3)` | `ExecutableReplicaSet` | `CreateAsync<ExecutableReplicaSet>(...)` |
| `AddPythonApp("ml", ...)` | `Executable` | Same (Python runs as a process) |

> 🎲 **Fun Fact:** DCP uses a Kubernetes-inspired API (Custom Resource Definitions, Watch semantics, Create/Get/Delete), but it's NOT actual Kubernetes. There's no kubelet, no etcd, no API server. DCP implements just enough of the K8s resource model to manage local containers and processes efficiently. This design choice means Aspire developers familiar with Kubernetes feel at home, while the system stays lightweight for local development.
