# The Backchannel Protocol

The CLI and AppHost communicate via JSON-RPC 2.0 over Unix domain sockets. This is the **backchannel** — a bidirectional channel that lets the CLI query and control a running AppHost.

---

## Socket Location

When an AppHost starts, it creates a socket file:

```
~/.aspire/cli/backchannels/auxi.sock.<process-id>
```

The CLI discovers running AppHosts by listing socket files in this directory. Each socket represents one running AppHost.

## Protocol Flow

```
CLI Process                              AppHost Process
    |                                        |
    +--- connect to Unix socket ------------>|
    |                                        |
    +--- {"jsonrpc":"2.0",                   |
    |     "method":"getDashboardInfo",  ---->|
    |     "id":1}                            |
    |                                        |
    |<--- {"jsonrpc":"2.0",                  |
    |      "id":1,                           |
    |      "result":{"urls":["..."]}}  ------|
    |                                        |
    +--- {"method":"watchResources"} ------->|
    |<--- streaming resource snapshots ------|
    |<--- streaming resource snapshots ------|
    |<--- ...                                |
```

## Backchannel Methods

| Method | Direction | Purpose |
|---|---|---|
| `getDashboardInfo` | CLI -> AppHost | Get Dashboard URL and configuration |
| `getResourceSnapshots` | CLI -> AppHost | Get current state of all resources |
| `watchResources` | CLI -> AppHost | Stream resource state changes (long-running) |
| `getConsoleLogs` | CLI -> AppHost | Stream console logs for a resource |
| `executeCommand` | CLI -> AppHost | Execute a lifecycle command on a resource |
| `stop` | CLI -> AppHost | Gracefully stop the AppHost |
| `resourceChanged` | AppHost -> CLI | Push notification when a resource changes |

## How RunCommand Uses the Backchannel

From `RunCommand.cs`:

```csharp
// Wait for the AppHost to start and expose its backchannel
var backchannel = await backchannelCompletionSource.Task
    .WaitAsync(cancellationToken);

// Get Dashboard URLs to display to the user
var dashboardUrls = await backchannel
    .GetDashboardUrlsAsync(cancellationToken);

// Watch for resource changes and display them
await foreach (var change in backchannel.WatchResourcesAsync(ct))
{
    DisplayResourceStatus(change);
}
```

## Publishing Activity Streaming

During `aspire publish`, the AppHost streams pipeline progress through the backchannel:

```json
{
    "method": "publishingActivity",
    "params": {
        "type": "stepStarted",
        "stepName": "build-api",
        "description": "Building API container image"
    }
}
```

The CLI renders these as progress bars:

```
[OK] build-prereq (0.2s)
[OK] build-api (12.3s)
[..] push-api ...
```

## BackchannelService (AppHost Side)

On the AppHost side, `BackchannelService` manages the socket:

1. Creates the socket file in `~/.aspire/cli/backchannels/`
2. Listens for incoming connections (one CLI at a time)
3. Routes JSON-RPC methods to appropriate handlers
4. Cleans up the socket file on shutdown
