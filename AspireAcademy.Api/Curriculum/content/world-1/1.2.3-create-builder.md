# DistributedApplication.CreateBuilder()

The entry point of every AppHost — what CreateBuilder sets up and returns.

## What CreateBuilder Initializes

```csharp
var builder = DistributedApplication.CreateBuilder(args);
```

This single line creates an `IDistributedApplicationBuilder` that sets up:

1. **Configuration** — loads `appsettings.json`, environment variables, command-line args, and user secrets
2. **Dependency injection** — the DI container for hosting infrastructure
3. **Resource collection** — an empty collection where `AddProject`, `AddRedis`, etc. register resources
4. **Pipeline** — ordered steps that execute on `Build().Run()`
5. **Eventing** — `IDistributedApplicationEventing` for lifecycle hooks

## The Builder Pattern

Every `Add*` method returns an `IResourceBuilder<T>` for fluent chaining:

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var redis = builder.AddRedis("cache");
var api = builder.AddProject<Projects.Api>("api")
                 .WithReference(redis)
                 .WithEnvironment("MY_VAR", "value");

builder.Build().Run();
```

## The args Parameter

Passing `args` is important. Aspire uses command-line arguments to switch modes:

- `--publisher` — generates manifests instead of running
- `--output-path` — where manifests are written
- Standard .NET hosting args (`--environment`, `--urls`) also work

## Build() and Run()

`builder.Build()` returns a `DistributedApplication`. Calling `.Run()` starts the orchestrator — launches all resources, opens the Dashboard, and blocks until shutdown. You can also use `.RunAsync()` for async control.

## Key Properties on the Builder

| Property | Purpose |
|----------|---------|
| `builder.Configuration` | Access app configuration |
| `builder.Services` | Register DI services |
| `builder.Resources` | Enumerate registered resources |
| `builder.Eventing` | Subscribe to lifecycle events |
| `builder.ExecutionContext` | Check if running or publishing |
