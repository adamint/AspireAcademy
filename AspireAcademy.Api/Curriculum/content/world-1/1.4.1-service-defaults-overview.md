# What AddServiceDefaults() Registers

Every service in an Aspire solution calls `builder.AddServiceDefaults()` in its `Program.cs`. This single line configures four cross-cutting concerns that every production-ready service needs.

## The Full Picture

Here's what the `AddServiceDefaults` extension method does, taken from the actual Aspire template:

```csharp
public static TBuilder AddServiceDefaults<TBuilder>(this TBuilder builder)
    where TBuilder : IHostApplicationBuilder
{
    builder.ConfigureOpenTelemetry();

    builder.AddDefaultHealthChecks();

    builder.Services.AddServiceDiscovery();

    builder.Services.ConfigureHttpClientDefaults(http =>
    {
        http.AddStandardResilienceHandler();
        http.AddServiceDiscovery();
    });

    return builder;
}
```

Let's walk through each registration.

## 1. OpenTelemetry (ConfigureOpenTelemetry)

This method sets up three telemetry pillars:

**Logging**: Configures `ILogger` to emit structured log entries via OTLP (OpenTelemetry Protocol):
```csharp
builder.Logging.AddOpenTelemetry(logging =>
{
    logging.IncludeFormattedMessage = true;
    logging.IncludeScopes = true;
});
```

**Metrics**: Registers automatic instrumentation for ASP.NET Core request metrics, outgoing HTTP client metrics, and .NET runtime metrics (GC, thread pool):
```csharp
builder.Services.AddOpenTelemetry()
    .WithMetrics(metrics =>
    {
        metrics.AddAspNetCoreInstrumentation()
            .AddHttpClientInstrumentation()
            .AddRuntimeInstrumentation();
    });
```

**Tracing**: Registers distributed trace collection for incoming HTTP requests and outgoing HTTP calls:
```csharp
.WithTracing(tracing =>
{
    tracing.AddAspNetCoreInstrumentation()
        .AddHttpClientInstrumentation();
});
```

All telemetry is exported via the **OTLP exporter**, which sends data to the Aspire Dashboard (or any OpenTelemetry-compatible backend like Jaeger or Azure Monitor).

## 2. Health Checks (AddDefaultHealthChecks)

Registers a basic health check and maps two endpoints:

```csharp
builder.Services.AddHealthChecks()
    .AddCheck("self", () => HealthCheckResult.Healthy(), ["live"]);
```

Later, `MapDefaultEndpoints()` exposes:
- **`/health`** — All registered health checks must pass (readiness probe)
- **`/alive`** — Only checks tagged `"live"` must pass (liveness probe)

The Dashboard monitors these endpoints and shows health status in real time.

## 3. Service Discovery

```csharp
builder.Services.AddServiceDiscovery();
```

This registers Aspire's service discovery system, which resolves service names (like `"api"`) to actual URLs (like `https://localhost:5123`). When you use `WithReference(api)` in the AppHost, the API's URL is injected as configuration, and service discovery resolves it transparently.

## 4. HTTP Resilience

```csharp
builder.Services.ConfigureHttpClientDefaults(http =>
{
    http.AddStandardResilienceHandler();
    http.AddServiceDiscovery();
});
```

`ConfigureHttpClientDefaults` applies configuration to **every** `HttpClient` created through the DI container. This means all outgoing HTTP calls automatically get retry policies, circuit breakers, timeouts, and service discovery — without any per-client configuration.
