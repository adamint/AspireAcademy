# Template Choices: C#, TypeScript, Python, Java

When you run `aspire new`, you pick a template that determines what service projects get scaffolded. The **AppHost** structure is always the same — only the service projects change.

## aspire-starter (Default)

The most common starting point. Generates:

- **AppHost** — C# project with `DistributedApplication.CreateBuilder()`
- **ServiceDefaults** — shared OpenTelemetry, health checks, resilience
- **ApiService** — a minimal API with a `/weatherforecast` endpoint
- **Web** — a Blazor frontend that calls the API

```csharp
var builder = DistributedApplication.CreateBuilder(args);
var api = builder.AddProject<Projects.MyApp_ApiService>("api");
builder.AddProject<Projects.MyApp_Web>("webfrontend")
       .WithExternalHttpEndpoints()
       .WithReference(api);
builder.Build().Run();
```

## aspire (Empty)

A minimal starting point — just the AppHost and ServiceDefaults with no service projects. Use this when you want to add your own projects from scratch.

## Polyglot Templates

Aspire supports non-.NET services through resource extensions:

### Node.js / JavaScript
```csharp
var frontend = builder.AddViteApp("frontend", "../frontend-app")
                      .WithHttpEndpoint(env: "PORT");
```

### Python
```csharp
var api = builder.AddPythonApp("flask-api", "../python-api", "app.py")
                 .WithHttpEndpoint(targetPort: 5000);
```

### Java
```csharp
var api = builder.AddJavaApp("spring-api", "../java-api");
```

## The AppHost Doesn't Change

Regardless of what language your services use, the AppHost is always C#. It uses the same `DistributedApplication.CreateBuilder()` entry point, the same `WithReference()` wiring, and the same Dashboard.

The template only determines the **starting service projects**. You can mix and match — add a Python service next to a .NET API next to a Node frontend — all orchestrated by the same AppHost.

## Choosing a Template

| Template | Best For |
|----------|----------|
| `aspire-starter` | Learning Aspire, .NET-only apps |
| `aspire` | Starting from scratch, polyglot apps |

Run `aspire new --list` or `dotnet new list aspire` to see all available templates.
