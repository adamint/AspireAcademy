# Installing the Aspire CLI

The Aspire CLI is a cross-platform tool that manages the full lifecycle of your Aspire application — from scaffolding to running to publishing.

---

## Installation

### macOS / Linux

```bash
# Install via the install script (recommended)
curl -sSL https://aspire.dev/install.sh | bash
```

### Windows

```powershell
irm https://aspire.dev/install.ps1 | iex
```

### Verify Installation

```bash
aspire --version
```

## aspire doctor

Before you create your first app, run `aspire doctor` to verify your development environment:

```bash
aspire doctor
```

`aspire doctor` checks:

| Check | What It Verifies |
|---|---|
| **.NET SDK** | Correct SDK version installed (skipped for non-.NET AppHosts) |
| **Dev Certificates** | HTTPS development certificate is trusted |
| **Container Runtime** | Docker (≥28.0) or Podman (≥5.0) is installed and running |
| **Deprecated Workloads** | No deprecated .NET Aspire workloads detected |

If any check fails, `aspire doctor` tells you exactly what to fix. Run it whenever something seems wrong.

## What About dotnet run?

You might wonder: do you use `aspire run` or `dotnet run`? Both work for C# AppHosts, but `aspire run` is the recommended entry point because:

- It works for **both** AppHost languages (C# and TypeScript)
- It provides richer console output with resource status
- It supports `--detach` for background execution
- It manages the backchannel communication between CLI and AppHost

For the rest of this course, we use `aspire run`.
