# Structured Logging and Metrics

ILogger structured logging, custom metrics with IMeterFactory, and how they flow to the Dashboard.

## Structured Logging

.NET's `ILogger` supports structured logging out of the box. Instead of string concatenation:

```csharp
// ❌ Unstructured — hard to search
logger.LogInformation($"Order {orderId} placed by {userId}");

// ✅ Structured — searchable, filterable
logger.LogInformation("Order {OrderId} placed by {UserId}", orderId, userId);
```

Structured logs preserve parameter names and values as separate fields. The Aspire Dashboard's **Structured Logs** view lets you filter by these fields.

## How Aspire Configures Logging

`AddServiceDefaults()` configures OpenTelemetry logging:

```csharp
builder.Logging.AddOpenTelemetry(logging =>
{
    logging.IncludeFormattedMessage = true;
    logging.IncludeScopes = true;
});
```

Logs are exported to the Dashboard via OTLP alongside traces and metrics.

## Log Levels

| Level | Use For |
|-------|---------|
| `Trace` | Detailed diagnostic info |
| `Debug` | Development-time debugging |
| `Information` | Normal operations |
| `Warning` | Unexpected but handled issues |
| `Error` | Failures that need attention |
| `Critical` | System-level failures |

## Metrics

Metrics track aggregate measurements over time. .NET 8+ uses `System.Diagnostics.Metrics`:

```csharp
private static readonly Meter s_meter = new("MyApp.Orders");
private static readonly Counter<int> s_ordersPlaced =
    s_meter.CreateCounter<int>("orders.placed");

public void PlaceOrder()
{
    s_ordersPlaced.Add(1);
}
```

Aspire configures built-in metric instruments automatically:

```csharp
builder.Services.AddOpenTelemetry()
    .WithMetrics(metrics =>
    {
        metrics.AddAspNetCoreInstrumentation()   // request duration, active requests
               .AddHttpClientInstrumentation()    // outbound request metrics
               .AddRuntimeInstrumentation();      // GC, thread pool, memory
    });
```

## The Dashboard Metrics View

The Dashboard displays metrics as time-series charts. Built-in metrics include:

- **HTTP request duration** — p50, p95, p99 latencies
- **Active requests** — concurrent request count
- **Runtime metrics** — GC collections, thread pool queue length, memory usage

Custom metrics appear automatically once registered with `AddMeter("MyApp.Orders")`.
