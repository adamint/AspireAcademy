# The Bazaar AppHost

Let us walk through the AppHost that wires Bazaar together. We will show it in C# — you will rewrite it in TypeScript later in the course.

---

## The Complete AppHost

```csharp
var builder = DistributedApplication.CreateBuilder(args);

// Infrastructure
var postgres = builder.AddPostgres("pg")
    .WithLifetime(ContainerLifetime.Persistent)
    .AddDatabase("bazaar-db");

var redis = builder.AddRedis("cache")
    .WithLifetime(ContainerLifetime.Persistent);

var rabbitmq = builder.AddRabbitMQ("messaging");

// C# API
var api = builder.AddProject<Projects.BazaarApi>("api")
    .WithReference(postgres)
    .WithReference(redis)
    .WithReference(rabbitmq)
    .WaitFor(postgres)
    .WaitFor(redis)
    .WaitFor(rabbitmq);

// TypeScript Frontend
builder.AddViteApp("frontend", "../bazaar-web")
    .WithReference(api)
    .WaitFor(api)
    .WithHttpEndpoint(port: 5173, env: "PORT");

// Python ML Service
builder.AddPythonApp("ml", "../bazaar-recs", "app:app")
    .WithReference(rabbitmq)
    .WithReference(api)
    .WaitFor(rabbitmq)
    .WithHttpEndpoint(port: 8000, env: "PORT");

// Go Worker
builder.AddExecutable("worker", "go", "../bazaar-worker", "run", ".")
    .WithReference(rabbitmq)
    .WithReference(api)
    .WaitFor(rabbitmq);

builder.Build().Run();
```

## Line by Line

### Infrastructure

```csharp
var postgres = builder.AddPostgres("pg")
    .WithLifetime(ContainerLifetime.Persistent)
    .AddDatabase("bazaar-db");
```

`AddPostgres("pg")` creates a PostgreSQL container resource. `.WithLifetime(ContainerLifetime.Persistent)` means the container survives AppHost restarts — your data is not lost when you stop and restart development. `.AddDatabase("bazaar-db")` creates a logical database within the PostgreSQL server.

### Service Wiring

```csharp
var api = builder.AddProject<Projects.BazaarApi>("api")
    .WithReference(postgres)
    .WithReference(redis)
    .WithReference(rabbitmq)
    .WaitFor(postgres)
    .WaitFor(redis)
    .WaitFor(rabbitmq);
```

`WithReference(postgres)` injects the PostgreSQL connection string as an environment variable (`ConnectionStrings__bazaar-db`). `WaitFor(postgres)` tells the orchestrator: do not start the API until PostgreSQL is running and healthy.

### Polyglot Services

```csharp
builder.AddViteApp("frontend", "../bazaar-web")
    .WithReference(api)
    .WaitFor(api);
```

`AddViteApp` orchestrates a Vite development server. It automatically detects the package manager (npm, yarn, pnpm), installs dependencies, and starts the dev server. `WithReference(api)` injects the API's URL so the frontend knows where to send requests.

```csharp
builder.AddPythonApp("ml", "../bazaar-recs", "app:app")
    .WithReference(rabbitmq)
    .WaitFor(rabbitmq);
```

`AddPythonApp` orchestrates a Python application. It sets up the virtual environment, installs pip dependencies, and runs the app. The third parameter `"app:app"` is the ASGI application entry point for uvicorn.

## What Happens at Runtime

When you run `aspire run`:

1. DCP starts PostgreSQL, Redis, and RabbitMQ containers
2. Health checks run on all three — `.WaitFor()` holds dependent services
3. Once infrastructure is healthy, the C# API starts
4. Once the API is healthy, the TypeScript frontend and Python ML service start
5. The Go worker starts once RabbitMQ is healthy
6. All six resources appear in the Dashboard with real-time status
