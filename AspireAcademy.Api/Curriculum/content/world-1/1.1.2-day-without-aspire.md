# A Day in the Life (Without Aspire)

It is 9:30 AM. You sit down to add a new feature to Bazaar. Before you write a single line of code, you need to get all four services running on your laptop.

---

## Step 1: Start the Infrastructure

You open a terminal and run Docker Compose to start PostgreSQL and Redis:

```bash
docker compose -f docker-compose.dev.yml up -d postgres redis rabbitmq
```

You wait 15 seconds for the health checks to pass. You check the logs to confirm PostgreSQL accepted connections.

## Step 2: Start the C# API

You open a second terminal. You need environment variables for the database connection string, Redis URL, and RabbitMQ connection:

```bash
export ConnectionStrings__postgres="Host=localhost;Port=5432;Database=bazaar;Username=bazaar;Password=secret"
export ConnectionStrings__redis="localhost:6379"
export ConnectionStrings__rabbitmq="amqp://guest:guest@localhost:5672"
dotnet run --project src/Bazaar.Api
```

The API starts on port 5001.

## Step 3: Start the Python ML Service

You open a third terminal. The Python service needs to know the API URL and the RabbitMQ connection:

```bash
cd src/bazaar-recs
source .venv/bin/activate
export API_URL=http://localhost:5001
export RABBITMQ_URL=amqp://guest:guest@localhost:5672
uvicorn app:app --port 8000
```

## Step 4: Start the Go Worker

Fourth terminal. The worker reads from RabbitMQ and writes to the API:

```bash
cd src/bazaar-worker
export API_URL=http://localhost:5001
export RABBITMQ_URL=amqp://guest:guest@localhost:5672
go run .
```

## Step 5: Start the TypeScript Frontend

Fifth terminal. The frontend needs the API URL:

```bash
cd src/bazaar-web
export VITE_API_URL=http://localhost:5001
npm run dev
```

## The Cost

You have five terminal windows open. You copy-pasted five connection strings. If you change the PostgreSQL password in Docker Compose, you have to update the C# API's environment variable manually — and you will forget, and it will fail silently, and you will spend 15 minutes debugging.

Now imagine a new team member joining. They clone the repo. They look at the README. The README is three pages long and already out of date because someone added Redis last week and forgot to update it.

**This is the inner-loop tax.** Every developer on your team pays it every morning. Every new dependency you add makes it worse.

## What Changes with Aspire

With Aspire, all five of those terminals become a single file — the AppHost — and a single command:

```bash
aspire run
```

One file declares all five services and their dependencies. Environment variables, connection strings, ports, and startup ordering are handled automatically. The Dashboard opens in your browser showing every service's status, logs, traces, and metrics in real time.
