# Atlas: The Story App

Throughout this course, you will build **Atlas** — a collaborative knowledge-sharing platform. Atlas is intentionally polyglot: four services, four languages, three pieces of infrastructure.

---

## The Four Services

### atlas-api (C# / ASP.NET)
The central API. Handles user requests, stores data in PostgreSQL, caches responses in Redis, and publishes events to RabbitMQ when content is created or updated.

### atlas-web (TypeScript / React / Vite)
The web frontend. A React application served by Vite's development server. Calls the atlas-api for all data. In production, this would be built and served as static files.

### atlas-ml (Python / FastAPI)
The recommendation engine. Listens on RabbitMQ for content events, uses scikit-learn to compute similarity scores, and exposes a `/recommend` endpoint that the API calls for personalized content suggestions.

### atlas-worker (Go)
A background processing service. Consumes messages from RabbitMQ — indexing content, generating thumbnails, sending notifications. Go's concurrency model makes it ideal for high-throughput background work.

## The Infrastructure

- **PostgreSQL** — the primary data store for users, content, and metadata
- **Redis** — caching layer for frequently accessed data and session storage
- **RabbitMQ** — message broker connecting the API to the worker and ML service

## Why Four Languages?

Atlas represents a realistic modern team. The backend engineers prefer C#. The frontend engineers prefer TypeScript. The data scientists prefer Python. The platform team chose Go for its lightweight concurrency. Aspire does not force them into one language — it lets each team use their best tool while providing unified orchestration, observability, and deployment.
