# Adding Your First Project Resource

Now it's your turn to write some Aspire code. In this challenge, you'll add a .NET project as a resource in the AppHost.

## What You Know So Far

- `DistributedApplication.CreateBuilder(args)` creates the builder
- The builder collects resources before `Build().Run()` starts everything
- Resources are the building blocks of your Aspire app

## The Key Method: `AddProject<T>()`

To register a .NET project, you call `AddProject<T>()` on the builder. The generic parameter `T` is an auto-generated class that points to the project's `.csproj` file. The string argument is the **resource name** — the logical name Aspire uses for this service in the Dashboard, logs, and service discovery.

```csharp
var api = builder.AddProject<Projects.ApiService>("api");
```

This returns an `IResourceBuilder<ProjectResource>`, which you can chain further configuration onto (like `WithReference()` or `WithEnvironment()`). For now, just adding the resource is enough.
