# DistributedApplication.CreateBuilder()

Every AppHost starts with a single line:

```csharp
var builder = DistributedApplication.CreateBuilder(args);
```

This static method is the entry point of the Aspire hosting API. Let's break down what it does and what it returns.

## What CreateBuilder Sets Up

`DistributedApplication.CreateBuilder(args)` creates and returns an `IDistributedApplicationBuilder`. Behind the scenes, this call initializes:

- **`IServiceCollection`** — The standard .NET dependency injection container. Aspire registers its own services here (resource orchestration, DCP communication, Dashboard integration), and you can add your own.
- **`ConfigurationManager`** — The standard .NET configuration system, pre-loaded with `appsettings.json`, environment variables, and command-line args. Aspire reads its own config (like `DOTNET_DASHBOARD_OTLP_ENDPOINT`) from here.
- **`IResourceCollection`** — An internal collection that stores every resource you add. When you call `AddProject`, `AddContainer`, or `AddRedis`, the resource gets added here.
- **Execution context** — Tracks whether you're running the app (`RunMode`) or generating a deployment manifest (`PublishMode`).

## The Builder Pattern

`IDistributedApplicationBuilder` follows the **builder pattern** — a design pattern where you configure an object step-by-step, then finalize it with a `Build()` call. You've seen this pattern with `WebApplication.CreateBuilder()` in ASP.NET Core.

```csharp
var builder = DistributedApplication.CreateBuilder(args);

// Step 1: Add resources (configuring the builder)
var redis = builder.AddRedis("cache");
var api = builder.AddProject<Projects.Api>("api")
                 .WithReference(redis);

// Step 2: Finalize
builder.Build().Run();
```

Between `CreateBuilder` and `Build()`, you're assembling a **resource graph** — a declaration of every resource and how they relate. Nothing is started yet. The builder is collecting your intent.

## Overloads

`CreateBuilder` has three public overloads:

```csharp
DistributedApplication.CreateBuilder()                // no args
DistributedApplication.CreateBuilder(args)             // string[] from Main
DistributedApplication.CreateBuilder(options)          // DistributedApplicationOptions
```

The `DistributedApplicationOptions` overload lets you configure advanced settings like disabling the Dashboard or setting the project directory explicitly. For most apps, passing `args` is all you need.
