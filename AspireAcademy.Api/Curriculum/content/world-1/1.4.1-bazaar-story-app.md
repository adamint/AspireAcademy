## The Four Services

### bazaar-api (C# / ASP.NET)
The central API. Handles product listings, orders, and user accounts. Stores data in PostgreSQL, caches popular products in Redis, and publishes events to RabbitMQ when orders are placed or inventory changes.

### bazaar-web (TypeScript / React / Vite)
The storefront. A React application served by Vite's development server. Displays product catalogs, shopping carts, and checkout flows. Calls the bazaar-api for all data. In production, this would be built and served as static files.

### bazaar-recs (Python / FastAPI)
The recommendation engine. Listens on RabbitMQ for purchase and browsing events, uses scikit-learn to compute product similarity scores, and exposes a `/recommend` endpoint that the API calls for personalized "you might also like" suggestions.

### bazaar-worker (Go)
A background processing service. Consumes messages from RabbitMQ — processing orders, generating invoices, updating inventory, sending confirmation emails. Go's concurrency model makes it ideal for high-throughput background work.

## The Infrastructure

- **PostgreSQL** — the primary data store for products, orders, users, and inventory
- **Redis** — caching layer for product catalogs, sessions, and shopping carts
- **RabbitMQ** — message broker connecting the API to the worker and recommendation service

## Why Four Languages?

Bazaar represents a realistic modern team. The backend engineers prefer C#. The frontend engineers prefer TypeScript. The data scientists prefer Python. The platform team chose Go for its lightweight concurrency. Aspire does not force them into one language — it lets each team use their best tool while providing unified orchestration, observability, and deployment.
