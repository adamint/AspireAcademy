# IResource — Everything Is a Resource

In Aspire, **everything is a resource**. A .NET project, a Redis container, a PostgreSQL database, a RabbitMQ broker, an executable — they're all resources. This is the foundational abstraction of Aspire's application model.

## The IResource Interface

At the root of the resource hierarchy is the `IResource` interface, defined in `Aspire.Hosting.ApplicationModel`:

```csharp
public interface IResource
{
    string Name { get; }
    ResourceAnnotationCollection Annotations { get; }
}
```

That's it — just two members:

- **`Name`**: A unique string identifier for the resource within the app model. This is the name you pass to methods like `AddProject("api")` or `AddRedis("cache")`. It appears in the Dashboard, in log entries, and in service discovery URLs.

- **`Annotations`**: A collection of `IResourceAnnotation` objects that describe the resource's behavior and configuration. Annotations are how Aspire attaches capabilities — endpoints, environment variables, container images, health checks — without requiring deep inheritance hierarchies.

## Why This Design Matters

The simplicity of `IResource` is intentional. Instead of encoding every possible capability into the interface (which would bloat it and create coupling), Aspire uses a **composition model**: a resource is a name plus a bag of annotations. Different resource types add different annotations.

This means:
- A `ContainerResource` has a `ContainerImageAnnotation` specifying its Docker image
- A `ProjectResource` has endpoint annotations for its Kestrel URLs
- A custom resource you create can have whatever annotations it needs

The `ResourceAnnotationCollection` extends `Collection<IResourceAnnotation>`, so you can add, remove, and query annotations. When Aspire's orchestrator (DCP) starts a resource, it reads these annotations to determine how to configure and launch it.

You'll see `IResource` everywhere in Aspire's API. Most extension methods are defined on `IResourceBuilder<T> where T : IResource`, which is why they work across all resource types.
