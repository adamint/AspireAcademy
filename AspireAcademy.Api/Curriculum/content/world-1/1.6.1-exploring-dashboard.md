# Exploring the Dashboard

Tour the Dashboard UI — resources table, console logs, structured logs, traces, and metrics.

## Opening the Dashboard

When you run an AppHost, the Dashboard URL is printed to the console:

```
Login to the dashboard at https://localhost:17178/login?t=abc123
```

The Dashboard is a Blazor web application that connects to the AppHost via gRPC and receives real-time updates.

## The Five Views

### 1. Resources

The home page. Shows every resource in a table:

| Column | Meaning |
|--------|---------|
| Name | Resource name from `AddProject("api")` |
| State | Running, Starting, Exited, etc. |
| Type | Project, Container, Executable |
| Source | Image name or project path |
| Endpoints | Clickable URLs to reach the resource |
| Health | Healthy ✅, Degraded ⚠️, Unhealthy ❌ |

Click a resource name to see its detail page with environment variables and console output.

### 2. Console Logs

Raw `stdout` and `stderr` from each resource. This is what you'd see in a terminal if you ran the process directly. Useful for startup errors, framework messages, and crash output.

### 3. Structured Logs

Parsed log entries from `ILogger` with:
- **Timestamp**, **Level** (Info, Warning, Error)
- **Message** with parameter values preserved
- **Resource** that emitted the log
- **TraceId/SpanId** linking logs to traces

You can filter by resource, log level, and free text.

### 4. Traces

Distributed traces showing request flow across services. Each trace is a waterfall:

```
Frontend (12ms total)
  └── GET /api/weather (8ms)
        └── Redis GET cache:weather (2ms)
```

Click a span to see tags, duration, and status code.

### 5. Metrics

Time-series charts for each metric instrument:
- HTTP request duration (histograms)
- Active requests (gauges)
- Runtime metrics (GC, thread pool)

## Real-Time Updates

The Dashboard streams updates via gRPC. When a resource's state changes, health checks complete, or new logs arrive, the UI updates instantly — no polling or refresh needed.
