# Aspire vs Docker Compose and Kubernetes

Aspire isn't the first tool for managing distributed apps. Here's how it compares to the alternatives and when each makes sense.

## Docker Compose

**Docker Compose** uses YAML files to define multi-container applications. It's widely adopted and works with any language.

| Dimension | Docker Compose | Aspire |
|---|---|---|
| **Language** | YAML (no type safety) | C# (compile-time checks) |
| **Service Discovery** | Manual (custom networks, DNS) | Automatic via `WithReference()` |
| **Telemetry** | None built-in (add Prometheus, Jaeger yourself) | OpenTelemetry + Dashboard out of the box |
| **IDE Support** | Limited (syntax highlighting only) | Full IntelliSense, refactoring, debugging |
| **.NET Integration** | Generic (no project-type awareness) | First-class (`AddProject<T>()`, launch profiles) |

Docker Compose is still valuable for production-like environments and non-.NET services. Aspire **complements** it — you can use Aspire for local development and Docker Compose (or Kubernetes) for deployment.

## Kubernetes

**Kubernetes** (K8s) is a production-grade container orchestration platform. It handles scaling, rolling deployments, self-healing, and networking.

| Dimension | Kubernetes | Aspire |
|---|---|---|
| **Primary Use** | Production deployment | Development-time orchestration |
| **Learning Curve** | Steep (YAML manifests, CRDs, operators) | Low (C# code you already know) |
| **Local Dev** | Possible (minikube, kind) but heavy | Lightweight — runs processes directly |
| **Scope** | Full cluster management | App topology and observability |

Aspire is **not** a Kubernetes replacement. It's a dev-time tool that can generate deployment manifests for K8s, Azure Container Apps, or other targets. Use Aspire to build and debug; use K8s to deploy and scale.

## Tye (Archived)

**Project Tye** was a .NET experiment for running microservices locally and deploying to Kubernetes. It was Aspire's spiritual predecessor but was archived in 2023. Tye lacked a typed resource model, had limited observability, and was never production-ready. Aspire took its best ideas — code-based orchestration and automatic service discovery — and built a complete, supported platform.

## Aspire's Unique Strength

Aspire's differentiator is the **compile-time resource graph with full IDE support**. Your app topology is a C# program: you can refactor it, debug it, test it, and use conditionals or loops to compose it. No other tool gives .NET developers this level of integration.
