# Creating a Project with `aspire new`

The fastest way to start an Aspire application is with the **Aspire CLI**. The `aspire new` command scaffolds a complete solution with all three project types wired together.

## Installing the Aspire CLI

```bash
curl -sSL https://aspire.dev/install.sh | bash
```

## Running the Command

```bash
aspire new
```

This launches an interactive wizard that lets you choose a template and name your project. You can also specify options directly:

```bash
aspire new aspire-starter --name MyApp
```

## What Gets Scaffolded

After running the command, you'll see this structure:

```
MyApp/
├── MyApp.sln                         ← Solution file
├── MyApp.AppHost/
│   ├── Program.cs                    ← Resource graph definition
│   ├── MyApp.AppHost.csproj          ← References Aspire.Hosting.AppHost
│   └── Properties/
│       └── launchSettings.json       ← Dashboard URL, ports
├── MyApp.ServiceDefaults/
│   ├── Extensions.cs                 ← AddServiceDefaults(), health checks, OTel
│   └── MyApp.ServiceDefaults.csproj  ← References OpenTelemetry, resilience, and service discovery packages
├── MyApp.ApiService/
│   ├── Program.cs                    ← API with weather endpoint
│   └── MyApp.ApiService.csproj       ← References ServiceDefaults
└── MyApp.Web/
    ├── Program.cs                    ← Blazor frontend
    └── MyApp.Web.csproj              ← References ServiceDefaults
```

## The `aspire.config.json` File

When you use the Aspire CLI, an `aspire.config.json` file may be created at the solution root. This file tells the CLI which project is the AppHost:

```json
{
  "appHost": {
    "path": "MyApp.AppHost/MyApp.AppHost.csproj"
  }
}
```

This allows you to run `aspire run` from the solution root without specifying the project path.

## Key Project References

The scaffolded projects are wired with specific NuGet packages:

- **AppHost** → `Aspire.Hosting.AppHost` (the hosting SDK that provides `DistributedApplication.CreateBuilder`)
- **ServiceDefaults** → references OpenTelemetry, health check, service discovery, and resilience packages
- **Service projects** → reference `ServiceDefaults` as a project reference

## The Generated AppHost Program.cs

The starter template generates an AppHost that looks like:

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var api = builder.AddProject<Projects.MyApp_ApiService>("api");

builder.AddProject<Projects.MyApp_Web>("webfrontend")
       .WithExternalHttpEndpoints()
       .WithReference(api);

builder.Build().Run();
```

This is the starting point you'll build on throughout this course. Every `AddProject`, `WithReference`, and `Build().Run()` will become second nature.
