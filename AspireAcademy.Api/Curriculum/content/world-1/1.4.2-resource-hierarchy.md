# Resource Type Hierarchy

Resource, ContainerResource, ProjectResource, ExecutableResource, ParameterResource — the class tree.

## The Hierarchy

```
IResource
  └── Resource (abstract base)
        ├── ContainerResource
        ├── ProjectResource
        ├── ExecutableResource
        └── ParameterResource
```

Each class adds capabilities through **marker interfaces** — not through deep inheritance.

## ContainerResource

Represents a Docker container:

```csharp
public class ContainerResource(string name, string? entrypoint = null)
    : Resource(name),
      IResourceWithEnvironment,
      IResourceWithArgs,
      IResourceWithEndpoints,
      IResourceWithWaitSupport
{
    public string? Entrypoint { get; set; }
}
```

Created by `builder.AddContainer("myapp", "nginx")` or integration methods like `AddRedis()`, `AddPostgres()`.

## ProjectResource

Represents a .NET project:

```csharp
public class ProjectResource : Resource,
    IResourceWithEnvironment,
    IResourceWithArgs,
    IResourceWithServiceDiscovery,
    IResourceWithWaitSupport
{
    public ProjectResource(string name) : base(name) { }
}
```

Created by `builder.AddProject<Projects.Api>("api")`. Key difference from containers: it implements `IResourceWithServiceDiscovery`, enabling automatic URL injection via `WithReference()`.

## ExecutableResource

Represents any executable process:

```csharp
public class ExecutableResource : Resource,
    IResourceWithEnvironment,
    IResourceWithArgs,
    IResourceWithEndpoints,
    IResourceWithWaitSupport
{
    public ExecutableResource(string name, string command, string workingDirectory)
        : base(name) { }

    public string Command { get; }
    public string WorkingDirectory { get; }
}
```

Used for non-.NET processes — Node.js apps, Python scripts, Go binaries.

## ParameterResource

Represents a configuration value (not a running process):

```csharp
public class ParameterResource : Resource,
    IManifestExpressionProvider,
    IValueProvider
{
    public bool Secret { get; }
    public ParameterDefault? Default { get; set; }
}
```

Created by `builder.AddParameter("api-key", secret: true)`. Parameters don't run — they provide values to other resources.

## Marker Interfaces

The real power is in the interfaces each resource implements:

| Interface | Meaning |
|-----------|---------|
| `IResourceWithEndpoints` | Has network endpoints (ports, URLs) |
| `IResourceWithEnvironment` | Can receive environment variables |
| `IResourceWithArgs` | Can receive command-line arguments |
| `IResourceWithConnectionString` | Produces a connection string |
| `IResourceWithServiceDiscovery` | Participates in service discovery |
| `IResourceWithWaitSupport` | Can wait for dependencies |

These interfaces determine which extension methods (`WithEndpoint`, `WithEnvironment`, `WithReference`, `WaitFor`) you can call on a resource builder.
