# First App Challenge

Time to put it all together. In this challenge, you'll write a complete AppHost that orchestrates two services with a shared dependency.

## Your Task

Create an AppHost that:

1. Adds a Redis cache named `"cache"`
2. Adds an API project named `"api"` that references the cache
3. Adds a web frontend named `"web"` that references the API
4. Makes the web frontend's HTTP endpoints externally accessible

## What You Need

Remember the core pattern:

```csharp
var builder = DistributedApplication.CreateBuilder(args);

// Add infrastructure
var cache = builder.AddRedis("cache");

// Add services with references
var api = builder.AddProject<Projects.Api>("api")
                 .WithReference(cache);

builder.AddProject<Projects.Web>("web")
       .WithExternalHttpEndpoints()
       .WithReference(api);

builder.Build().Run();
```

## Key Concepts

- **`AddRedis(name)`** registers a Redis container resource
- **`AddProject<T>(name)`** registers a .NET project; `T` is a generated type from `Projects.*`
- **`WithReference(resource)`** injects connection info (connection strings for data resources, URLs for service resources)
- **`WithExternalHttpEndpoints()`** marks the resource's HTTP endpoints as externally accessible (visible outside the orchestrator)
- **`Build().Run()`** starts everything and opens the Dashboard

## What Happens When You Run

When `Build().Run()` executes:

1. Redis starts as a container
2. The API project launches and receives a `ConnectionStrings__cache` environment variable pointing to Redis
3. The web frontend launches and receives service discovery URLs for the API
4. The Aspire Dashboard opens, showing all three resources, their logs, and traces
