# Resource States and Lifecycle

Starting, Running, Finished, Exiting, FailedToStart — CustomResourceSnapshot and ResourceNotificationService.

## The State Machine

Every resource transitions through a lifecycle tracked by `KnownResourceStates`:

```csharp
public static class KnownResourceStates
{
    public static readonly string NotStarted = nameof(NotStarted);
    public static readonly string Starting = nameof(Starting);
    public static readonly string Building = nameof(Building);
    public static readonly string Running = nameof(Running);
    public static readonly string Waiting = nameof(Waiting);
    public static readonly string Stopping = nameof(Stopping);
    public static readonly string Exited = nameof(Exited);
    public static readonly string Finished = nameof(Finished);
    public static readonly string FailedToStart = nameof(FailedToStart);
    public static readonly string RuntimeUnhealthy = nameof(RuntimeUnhealthy);
    public static readonly string Active = nameof(Active);

    public static readonly IReadOnlyList<string> TerminalStates =
        [Finished, FailedToStart, Exited];
}
```

## Typical State Flow

A healthy resource follows this path:

```
NotStarted → Starting → Running → Stopping → Exited
```

A container with a Dockerfile:

```
NotStarted → Building → Starting → Running → ...
```

A resource with `WaitFor` dependencies:

```
NotStarted → Waiting → Starting → Running → ...
```

A resource that crashes:

```
NotStarted → Starting → FailedToStart
```

## Terminal States

Three states are **terminal** — the resource won't transition further:

- **Finished** — process exited successfully (exit code 0)
- **Exited** — process exited (any exit code)
- **FailedToStart** — couldn't start at all

## CustomResourceSnapshot

State is tracked through `CustomResourceSnapshot`, a record that captures the full point-in-time state:

```csharp
public sealed record CustomResourceSnapshot
{
    public string? ResourceType { get; init; }
    public ResourceStateSnapshot? State { get; init; }
    public DateTimeOffset? CreationTimeStamp { get; init; }
    public int? ExitCode { get; init; }
    public HealthStatus? HealthStatus { get; init; }
    public ImmutableArray<UrlSnapshot> Urls { get; init; }
    public ImmutableArray<ResourcePropertySnapshot> Properties { get; init; }
}
```

## The Dashboard Connection

The Dashboard subscribes to `ResourceNotificationService` updates. Every state change pushes a new `CustomResourceSnapshot` to the Dashboard, which updates the resources table, status indicators, and health icons in real time.

## Health vs State

State and health are separate concepts:

- **State** = is the process running? (`Running`, `Exited`, etc.)
- **Health** = is it functioning correctly? (`Healthy`, `Degraded`, `Unhealthy`)

A resource can be `Running` but `Unhealthy` (e.g., a database container that's started but not yet accepting connections).
