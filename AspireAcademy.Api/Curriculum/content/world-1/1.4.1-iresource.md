# IResource — Everything Is a Resource

The base interface for all Aspire resources: Name and Annotations.

## The Interface

At the heart of Aspire's application model is a remarkably simple interface:

```csharp
public interface IResource
{
    string Name { get; }
    ResourceAnnotationCollection Annotations { get; }
}
```

That's it — just two members. Every resource in Aspire (projects, containers, databases, executables, parameters) implements `IResource`. The `Name` is the resource's logical identifier used in the Dashboard, logs, service discovery, and environment variable naming. The `Annotations` collection holds metadata that controls behavior.

## The Resource Base Class

Most resources extend the abstract `Resource` class:

```csharp
public abstract class Resource : IResource
{
    public virtual string Name { get; }
    public virtual ResourceAnnotationCollection Annotations { get; } = new();

    protected Resource(string name)
    {
        ModelName.ValidateName(nameof(Resource), name);
        Name = name;
    }
}
```

The constructor validates the name — it must be non-empty and contain only valid characters (letters, digits, hyphens, underscores).

## Why This Design Matters

The `IResource` interface is intentionally minimal. Aspire doesn't encode capabilities into a deep class hierarchy. Instead, it uses two extension points:

1. **Marker interfaces** — `IResourceWithEndpoints`, `IResourceWithEnvironment`, `IResourceWithConnectionString` — signal what a resource supports
2. **Annotations** — `EndpointAnnotation`, `HealthCheckAnnotation`, `ContainerMountAnnotation` — attach behavior without changing the class

This means you can create a custom resource by implementing `IResource` and adding the right annotations. The orchestrator doesn't care about the concrete type — it inspects interfaces and annotations to decide what to do.

## Resource Names in Practice

The name you pass to `AddProject("api")` or `AddRedis("cache")` becomes:

- The row label in the Dashboard
- The key in `ConnectionStrings__cache` environment variables
- The service discovery identifier in `services__api__https__0`
- The resource key in the Aspire manifest
