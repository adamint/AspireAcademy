# OpenTelemetry: Tracing with ActivitySource

How distributed tracing works with Activity, ActivitySource, and OTLP exporters.

## The Problem

When a user hits your web frontend, the request flows: Frontend → API → Database. If it's slow, which service is the bottleneck? Without tracing, you'd check three separate log files and guess.

## How .NET Tracing Works

.NET's tracing is built on `System.Diagnostics`:

```csharp
// Define a source for your service
private static readonly ActivitySource s_source = new("MyApp.Api");

// Create a span (called an "Activity" in .NET)
using var activity = s_source.StartActivity("ProcessOrder");
activity?.SetTag("order.id", orderId);
```

An `Activity` represents a unit of work (a **span** in OpenTelemetry terminology). Activities form a tree: a parent activity in the frontend creates child activities in the API and database layers.

## What Aspire Configures Automatically

`AddServiceDefaults()` registers OpenTelemetry tracing with built-in instrumentation:

```csharp
builder.Services.AddOpenTelemetry()
    .WithTracing(tracing =>
    {
        tracing.AddAspNetCoreInstrumentation()   // HTTP server spans
               .AddHttpClientInstrumentation()    // HTTP client spans
               .AddGrpcClientInstrumentation();   // gRPC client spans
    });
```

This means every HTTP request, outbound HTTP call, and gRPC call is automatically traced — no manual `ActivitySource` needed for infrastructure code.

## The Trace Propagation

When Service A calls Service B via `HttpClient`, the trace context (trace ID + span ID) propagates automatically via HTTP headers (`traceparent`). Service B's span becomes a child of Service A's span.

```
Frontend (trace: abc, span: 1)
  └── API call (trace: abc, span: 2)
        └── DB query (trace: abc, span: 3)
```

## OTLP Export

The data flows to the Dashboard via OTLP (OpenTelemetry Protocol). The AppHost injects the OTLP endpoint URL into each service as the `OTEL_EXPORTER_OTLP_ENDPOINT` environment variable. Services export telemetry there automatically.

## Adding Custom Spans

For business logic, create your own `ActivitySource`:

```csharp
private static readonly ActivitySource s_source = new("MyApp.OrderService");

public async Task<Order> ProcessOrder(int orderId)
{
    using var activity = s_source.StartActivity("ProcessOrder");
    activity?.SetTag("order.id", orderId);
    // ... business logic
}
```

Register it in `AddServiceDefaults()`:

```csharp
tracing.AddSource("MyApp.OrderService");
```
