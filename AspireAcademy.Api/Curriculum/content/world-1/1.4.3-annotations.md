# Annotations: Behavior Without Inheritance

How IResourceAnnotation attaches capabilities to resources without deep class hierarchies.

## The Pattern: Composition Over Inheritance

Instead of creating subclasses for every combination of features, Aspire uses **annotations** — small metadata objects attached to a resource's `Annotations` collection. The orchestrator inspects these annotations at runtime to decide what to do.

```csharp
public interface IResourceAnnotation { }
```

That's the entire interface — it's a marker. Any class implementing it can be attached to any resource.

## How Annotations Are Added

Most annotations are added by extension methods. When you call:

```csharp
builder.AddContainer("nginx", "nginx")
       .WithHttpEndpoint(port: 80);
```

The `WithHttpEndpoint` method creates an `EndpointAnnotation` and adds it to the container's `Annotations` collection:

```csharp
resource.Annotations.Add(new EndpointAnnotation
{
    UriScheme = "http",
    Port = 80,
    Name = "http"
});
```

## Key Built-In Annotations

| Annotation | Added By | Purpose |
|-----------|----------|---------|
| `EndpointAnnotation` | `WithEndpoint`, `WithHttpEndpoint` | Declares a network endpoint |
| `ContainerMountAnnotation` | `WithVolume`, `WithBindMount` | Mounts storage into a container |
| `ContainerLifetimeAnnotation` | `WithLifetime` | Controls container persistence |
| `HealthCheckAnnotation` | Integration packages | Registers health checks |
| `WaitAnnotation` | `WaitFor`, `WaitForCompletion` | Declares startup dependencies |
| `EnvironmentCallbackAnnotation` | `WithEnvironment` | Injects environment variables |
| `ResourceSnapshotAnnotation` | Orchestrator | Tracks resource state |

## Reading Annotations

You can query annotations by type:

```csharp
var endpoints = resource.Annotations.OfType<EndpointAnnotation>();
var hasHealthCheck = resource.Annotations.OfType<HealthCheckAnnotation>().Any();
```

## Why This Matters

Annotations make Aspire extensible. To add a new capability:

1. Define an annotation class implementing `IResourceAnnotation`
2. Add it to resources via an extension method
3. Read it in a pipeline step or lifecycle hook

No changes to `Resource`, `ContainerResource`, or any base class needed. This is how every hosting integration package works — `Aspire.Hosting.Redis` adds Redis-specific annotations to a `ContainerResource`.
