# Boss: Build a Single-File AppHost App

Build a 2-service app using a single-file AppHost with ServiceDefaults and the Dashboard.

## The Challenge

Combine everything from World 1 into a working application. You'll create a single-file AppHost that:

1. Uses the `#:sdk` directive to specify the Aspire SDK
2. Adds a Redis cache as infrastructure
3. Adds an API service that depends on Redis
4. Adds a web frontend that depends on the API
5. Runs everything with `Build().Run()`

## The Solution

```csharp
#:sdk Aspire.AppHost.Sdk@9.3.0
#:package Aspire.Hosting.Redis@9.3.0

var builder = DistributedApplication.CreateBuilder(args);

var cache = builder.AddRedis("cache");

var api = builder.AddProject("api", "./src/Api/Api.csproj")
                 .WithReference(cache)
                 .WaitFor(cache);

builder.AddProject("web", "./src/Web/Web.csproj")
       .WithExternalHttpEndpoints()
       .WithReference(api)
       .WaitFor(api);

builder.Build().Run();
```

## What Each Line Does

- **`#:sdk`** — specifies the Aspire SDK version (replaces `.csproj`)
- **`#:package`** — adds the Redis hosting integration
- **`AddRedis("cache")`** — starts a Redis container
- **`AddProject("api", path)`** — registers the API project (no `Projects.*` types in single-file)
- **`WithReference(cache)`** — injects `ConnectionStrings__cache` into the API
- **`WaitFor(cache)`** — API won't start until Redis is healthy
- **`WithExternalHttpEndpoints()`** — makes the web frontend accessible from the browser
- **`WithReference(api)`** — injects the API's URL into the frontend via service discovery

## What You Should See

When you run `aspire run`:

1. The Dashboard opens in your browser
2. Redis starts as a container (state: Starting → Running)
3. The API waits for Redis, then starts (state: Waiting → Starting → Running)
4. The web frontend waits for the API, then starts
5. All three resources show green health checks in the Dashboard

## Key Concepts Tested

- Single-file AppHost syntax (`#:sdk`, `#:package`)
- Resource model (`AddRedis`, `AddProject`)
- Dependency wiring (`WithReference`)
- Startup ordering (`WaitFor`)
- The Dashboard as your observability tool
