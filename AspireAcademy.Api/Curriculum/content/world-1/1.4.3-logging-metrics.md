# Structured Logging and Metrics

Tracing tracks individual requests. Logging and metrics give you the broader picture: what's happening across all requests, over time.

## Structured Logging with ILogger

You already know `ILogger` from ASP.NET Core. What makes it **structured** is the use of **message templates** instead of string interpolation:

```csharp
// ✅ Structured — properties are preserved as typed fields
logger.LogInformation("Order {OrderId} placed by {UserId}", orderId, userId);

// ❌ Interpolated — the message is just a flat string
logger.LogInformation($"Order {orderId} placed by {userId}");
```

With message templates, `OrderId` and `UserId` are captured as separate key-value pairs. The Dashboard's Structured Logs page lets you filter by these properties — for example, showing all logs where `UserId == "user-42"`.

### How Logs Flow to the Dashboard

ServiceDefaults configures OpenTelemetry logging:

```csharp
builder.Logging.AddOpenTelemetry(logging =>
{
    logging.IncludeFormattedMessage = true;
    logging.IncludeScopes = true;
});
```

- **`IncludeFormattedMessage`**: Sends the human-readable message alongside the structured properties
- **`IncludeScopes`**: Includes log scopes (additional context pushed onto the log pipeline, like request IDs)

These log entries are exported via OTLP to the Dashboard, where they appear in the Structured Logs page with full filtering and trace correlation.

## Custom Metrics with IMeterFactory

**Metrics** are numerical measurements collected over time: request counts, error rates, queue depths, latency percentiles.

### Creating a Custom Metric

Use `IMeterFactory` (from `System.Diagnostics.Metrics`) to create meters:

```csharp
public class OrderMetrics
{
    private readonly Counter<long> _ordersPlaced;
    private readonly Histogram<double> _orderProcessingTime;

    public OrderMetrics(IMeterFactory meterFactory)
    {
        var meter = meterFactory.Create("MyApp.Orders");
        _ordersPlaced = meter.CreateCounter<long>("orders.placed", "orders",
            "Number of orders placed");
        _orderProcessingTime = meter.CreateHistogram<double>("orders.processing_time", "ms",
            "Order processing duration");
    }

    public void RecordOrderPlaced() => _ordersPlaced.Add(1);

    public void RecordProcessingTime(double ms) => _orderProcessingTime.Record(ms);
}
```

- **`Counter<T>`**: A monotonically increasing value (total orders, total errors). Counters only go up.
- **`Histogram<T>`**: A distribution of values (latency, response sizes). Histograms track min, max, average, and percentiles.

### How Metrics Flow to the Dashboard

ServiceDefaults registers metrics instrumentation:

```csharp
.WithMetrics(metrics =>
{
    metrics.AddAspNetCoreInstrumentation()
        .AddHttpClientInstrumentation()
        .AddRuntimeInstrumentation();
});
```

The built-in instrumentation captures HTTP request metrics and .NET runtime metrics automatically. Your custom meters are also exported via OTLP. The Dashboard's Metrics page renders them as time-series charts where you can filter by dimension and toggle between chart and table views.
