# Resource Type Hierarchy

While `IResource` is the root interface, Aspire provides a hierarchy of concrete resource classes. Each represents a different kind of thing you can run.

## The Class Tree

```
IResource (interface)
  ‚îî‚îÄ‚îÄ Resource (abstract base class)
        ‚îú‚îÄ‚îÄ ContainerResource     ‚Äî Docker/OCI containers
        ‚îú‚îÄ‚îÄ ProjectResource       ‚Äî .NET projects

        ‚îî‚îÄ‚îÄ ParameterResource     ‚Äî Configuration values
```

## Resource (Base Class)

`Resource` is the abstract base class that implements `IResource`:

```csharp
public abstract class Resource : IResource
{
    public virtual string Name { get; }
    public virtual ResourceAnnotationCollection Annotations { get; } = new();

    protected Resource(string name) { /* validates name */ }
}
```

It provides the `Name` and `Annotations` properties, plus name validation. All concrete resource types inherit from this.

## ContainerResource

Represents a Docker or OCI container. Created via `builder.AddContainer("name", "image")`.

```csharp
public class ContainerResource(string name, string? entrypoint = null)
    : Resource(name), IResourceWithEnvironment, IResourceWithArgs, IResourceWithEndpoints
```

Key characteristics:
- Has a `ContainerImageAnnotation` specifying the image (e.g., `redis:7.4`)
- Supports environment variables, arguments, endpoints, volumes, and bind mounts
- Used for non-.NET dependencies like databases, caches, and message brokers

## ProjectResource

Represents a .NET project launched with `dotnet run`. Created via `builder.AddProject<T>("name")`.

```csharp
public class ProjectResource : Resource,
    IResourceWithEnvironment, IResourceWithArgs, IResourceWithServiceDiscovery
```

Key characteristics:
- Linked to a `.csproj` file via the generic `IProjectMetadata` type parameter
- Supports Kestrel endpoint configuration and launch profiles
- Participates in service discovery (other resources can reference it by name)

## ExecutableResource

Represents any executable that isn't a .NET project. Created via `builder.AddExecutable("name", "command", "workingDir")`.

```csharp
public class ExecutableResource : Resource,
    IResourceWithEnvironment, IResourceWithArgs, IResourceWithEndpoints
```

Use this for tools like Node.js servers, Python scripts, or custom binaries.

## ParameterResource

Represents a configuration value ‚Äî a string, secret, or connection string. Created via `builder.AddParameter("name")`.

```csharp
public class ParameterResource : Resource, IManifestExpressionProvider, IValueProvider
```

Key characteristics:
- Can be marked as `Secret` (values are masked in the Dashboard)
- Provides values at runtime via `GetValueAsync()`
- Used to pass API keys, passwords, and other configuration to resources

## When to Use Each

| Resource Type | Use When... |
|---|---|
| `ProjectResource` | You're running a .NET project (API, web app, worker) |
| `ContainerResource` | You need a Docker container (Redis, PostgreSQL, RabbitMQ) |
| `ExecutableResource` | You're running a non-.NET executable (Node.js, Python) |
| `ParameterResource` | You need to pass configuration values or secrets |
