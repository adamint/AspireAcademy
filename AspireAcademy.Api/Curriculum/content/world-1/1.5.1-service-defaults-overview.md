# What AddServiceDefaults() Registers

The full picture: OpenTelemetry, health checks, service discovery, and HTTP resilience.

## The Single Line

Every service project in an Aspire app calls this in `Program.cs`:

```csharp
builder.AddServiceDefaults();
```

This single line configures four production-ready cross-cutting concerns.

## 1. OpenTelemetry (Tracing + Metrics + Logging)

```csharp
builder.Services.AddOpenTelemetry()
    .WithTracing(tracing =>
    {
        tracing.AddAspNetCoreInstrumentation()
               .AddHttpClientInstrumentation()
               .AddGrpcClientInstrumentation();
    })
    .WithMetrics(metrics =>
    {
        metrics.AddAspNetCoreInstrumentation()
               .AddHttpClientInstrumentation()
               .AddRuntimeInstrumentation();
    });
```

Telemetry data is automatically exported to the Aspire Dashboard via OTLP (OpenTelemetry Protocol).

## 2. Health Checks

```csharp
builder.Services.AddHealthChecks()
    .AddCheck("self", () => HealthCheckResult.Healthy(), ["live"]);
```

This maps two endpoints:
- **`/health`** — all registered health checks (databases, dependencies, self)
- **`/alive`** — only checks tagged `"live"` (is the process running?)

The orchestrator polls `/health` to determine if a resource is healthy.

## 3. Service Discovery

```csharp
builder.Services.AddServiceDiscovery();
builder.Services.ConfigureHttpClientDefaults(http =>
{
    http.AddServiceDiscovery();
});
```

This enables automatic resolution of service URLs. When the API calls `http://catalog`, service discovery resolves it to the actual URL from environment variables injected by the AppHost.

## 4. HTTP Resilience

```csharp
builder.Services.ConfigureHttpClientDefaults(http =>
{
    http.AddStandardResilienceHandler();
});
```

Every `HttpClient` gets automatic retry (exponential backoff), circuit breaker, and timeout policies via Polly. No per-client configuration needed.

## Where It Lives

`AddServiceDefaults()` is defined in the **ServiceDefaults** project — a shared library referenced by every service. You can customize it by editing `Extensions.cs`.
