# Single-File AppHost: apphost.cs

The single-file AppHost format lets you define your entire application model in one `.cs` file — no `.csproj`, no solution file, no `Properties/launchSettings.json`. It's the fastest way to get started.

## The `#:sdk` Directive

A single-file AppHost starts with a special directive that tells the .NET SDK which Aspire version to use:

```csharp
#:sdk Aspire.AppHost.Sdk@9.3.0

var builder = DistributedApplication.CreateBuilder(args);
builder.Build().Run();
```

The `#:sdk` directive replaces what a `.csproj` file would do — it specifies the SDK and version. The rest is regular C# top-level statements.

## Adding Packages with `#:package`

To use hosting integrations (Redis, PostgreSQL, etc.), add `#:package` directives:

```csharp
#:sdk Aspire.AppHost.Sdk@9.3.0
#:package Aspire.Hosting.Redis@9.3.0

var builder = DistributedApplication.CreateBuilder(args);
var cache = builder.AddRedis("cache");
builder.Build().Run();
```

Each `#:package` directive is equivalent to a `<PackageReference>` in a `.csproj`.

## Referencing Projects with `#:project`

To add .NET projects as resources:

```csharp
#:sdk Aspire.AppHost.Sdk@9.3.0
#:project ./src/Api/Api.csproj

var builder = DistributedApplication.CreateBuilder(args);
builder.AddProject("api", "./src/Api/Api.csproj");
builder.Build().Run();
```

## The aspire.config.json File

The single-file AppHost uses `aspire.config.json` (instead of `launchSettings.json`) for configuration:

```json
{
  "appHost": {
    "path": "apphost.cs"
  }
}
```

This tells the Aspire CLI which file is the AppHost, so `aspire run` works from the directory.

## Running It

```bash
aspire run apphost.cs
# or from the directory with aspire.config.json:
aspire run
```

## When to Use Single-File

Single-file AppHosts are ideal for:

- **Quick prototypes** — spin up a Redis + API in seconds
- **Learning** — minimal ceremony, focus on the concepts
- **Simple apps** — when you don't need complex MSBuild configuration

The trade-off is less tooling support (no Visual Studio F5 debugging, no project references via `Projects.*` generated types).
