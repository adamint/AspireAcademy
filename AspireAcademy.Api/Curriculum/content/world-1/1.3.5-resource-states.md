# Resource States and Lifecycle

Every resource in Aspire goes through a lifecycle, transitioning between states as it starts, runs, and stops. Understanding these states helps you interpret what the Dashboard is showing and debug startup issues.

## Known Resource States

Aspire defines resource states in the `KnownResourceStates` static class:

| State | Meaning |
|---|---|
| **Starting** | The resource is being initialized (process launching, container pulling) |
| **Running** | The resource is up and healthy |
| **Waiting** | The resource is waiting for dependencies to become ready |
| **Stopping** | A graceful shutdown is in progress |
| **Exited** | The resource stopped (process exited, container stopped) |
| **Finished** | The resource completed successfully (common for one-shot tasks) |
| **FailedToStart** | The resource could not start (crash, port conflict, missing image) |
| **RuntimeUnhealthy** | The resource is running but failing health checks |
| **NotStarted** | The resource has not been started yet |

The states `Finished`, `FailedToStart`, and `Exited` are **terminal states** — the resource won't transition out of them without manual intervention (restart).

## CustomResourceSnapshot

Resource state is communicated via `CustomResourceSnapshot` — a record that captures a point-in-time view of a resource:

```csharp
public sealed record CustomResourceSnapshot
{
    public ResourceStateSnapshot? State { get; init; }
    public ImmutableArray<UrlSnapshot> Urls { get; init; }
    public ImmutableArray<EnvironmentVariableSnapshot> EnvironmentVariables { get; init; }
    public HealthStatus? HealthStatus { get; init; }
    // ... additional properties
}
```

## ResourceNotificationService

State changes are published through `ResourceNotificationService`. Aspire's Dashboard and internal monitors subscribe to it. You can also subscribe in tests or custom code:

```csharp
await resourceNotificationService.WaitForResourceAsync("api", KnownResourceStates.Running);
```

This is particularly useful in integration tests where you need to wait for a resource to be ready before running assertions.
