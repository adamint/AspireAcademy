# Kubernetes and Helm Publishing

Your platform team runs Kubernetes and expects Helm charts for every service. You don't want to hand-write Deployments, Services, StatefulSets, and PersistentVolumeClaims for each resource — especially when the Aspire AppHost already knows the topology. Aspire generates a complete Helm chart from your app model, with parameterized `values.yaml` your DevOps team can customize without touching the templates.

---

## Generating K8s Manifests

```bash
aspire publish --target kubernetes --output-path ./deploy/k8s
```

## What Gets Generated

```
deploy/k8s/
├── Chart.yaml
├── values.yaml
├── templates/
│   ├── api-deployment.yaml
│   ├── api-service.yaml
│   ├── frontend-deployment.yaml
│   ├── frontend-service.yaml
│   ├── pg-statefulset.yaml
│   ├── pg-service.yaml
│   ├── cache-deployment.yaml
│   └── cache-service.yaml
```

## KubernetesPublishingContext

The publishing engine:

1. Creates a `KubernetesResource` for each compute resource
2. Generates Deployment/StatefulSet YAML for each resource
3. Generates Service YAML for endpoint exposure
4. Collects parameters and secrets into Helm `values.yaml`
5. Applies `KubernetesServiceCustomizationAnnotation` for custom overrides

## Key Translations

| Aspire Concept | Kubernetes Equivalent |
|---|---|
| `AddProject("api")` | Deployment + Service |
| `WithReplicas(3)` | `spec.replicas: 3` |
| `AddPostgres("pg")` | StatefulSet + PersistentVolumeClaim |
| `WithReference(pg)` | Environment variables in pod spec |
| `WaitFor(pg)` | Init container or readiness probe |
| `AddParameter("key", secret: true)` | Kubernetes Secret |

> 🌍 **Real World:** Aspire's Kubernetes publisher generates a full Helm chart, not just raw manifests. This means you get `values.yaml` parameterization out of the box — your DevOps team can customize replica counts, image tags, and resource limits without modifying the generated templates.

## Customizing K8s Output

```csharp
builder.AddProject<Projects.Api>("api")
    .WithAnnotation(new KubernetesServiceCustomizationAnnotation(service =>
    {
        service.Spec.Type = "LoadBalancer";
    }));
```
