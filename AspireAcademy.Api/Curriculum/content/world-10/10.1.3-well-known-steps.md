# WellKnownPipelineSteps

Aspire defines a set of framework pipeline steps that hosting integrations and publishers use.

---

## The Steps

```csharp
public static class WellKnownPipelineSteps
{
    public const string Publish = "publish";
    public const string PublishPrereq = "publish-prereq";
    public const string Deploy = "deploy";
    public const string DeployPrereq = "deploy-prereq";
    public const string Build = "build";
    public const string BuildPrereq = "build-prereq";
    public const string Push = "push";
    public const string PushPrereq = "push-prereq";
    public const string ProcessParameters = "process-parameters";
    public const string Diagnostics = "diagnostics";
}
```

## How They Compose

```
build-prereq → build → push-prereq → push → publish-prereq → publish
                                                                  ↓
                                                              deploy-prereq → deploy
```

When you run `aspire publish`, only the publish chain executes. When you run `aspire deploy`, both publish and deploy chains execute.

> 🌍 **Real World:** The `process-parameters` step is where Aspire resolves user-supplied parameters (like Azure subscription IDs or Kubernetes namespaces). If you run `aspire publish` interactively, this is the step that prompts you for missing values.

## How Resources Add Steps

Hosting integrations add steps via `PipelineStepAnnotation`. When you call `AddPostgres("pg")`, the PostgreSQL integration adds steps for:
- Building the container image (if using a custom Dockerfile)
- Pushing the image to a registry
- Generating deployment manifests

These steps are automatically wired into the DAG with correct dependencies.
