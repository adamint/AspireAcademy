# Docker Compose Publishing

Aspire can generate a `docker-compose.yml` file from your app model — useful for teams that deploy with Compose.

---

## Generating Compose Files

```bash
aspire publish --output-path ./deploy/compose
```

## What Gets Generated

For Atlas, the output looks like:

```yaml
services:
  pg:
    image: postgres:17
    environment:
      POSTGRES_PASSWORD: ${PG_PASSWORD}
    volumes:
      - pg-data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD", "pg_isready"]

  cache:
    image: redis:7.4

  api:
    build: ../Atlas.Api
    environment:
      ConnectionStrings__atlas-db: "Host=pg;Port=5432;..."
      ConnectionStrings__cache: "cache:6379"
    depends_on:
      pg:
        condition: service_healthy
      cache:
        condition: service_started

  frontend:
    build: ../atlas-web
    environment:
      services__api__http__0: "http://api:5000"
    depends_on:
      api:
        condition: service_started

volumes:
  pg-data:
```

## Key Translations

| Aspire Concept | Docker Compose Equivalent |
|---|---|
| `AddPostgres("pg")` | `services.pg` with postgres image |
| `WithReference(pg)` | Environment variables pointing to service name |
| `WaitFor(pg)` | `depends_on` with `service_healthy` |
| `WithDataVolume("pg-data")` | Named volume in `volumes:` |
| `WithLifetime(Persistent)` | No direct equivalent (Compose containers always persist) |
