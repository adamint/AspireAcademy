# Manifest Publishing

The manifest publisher generates an `aspire-manifest.json` file that captures the complete app model as a portable JSON document.

---

## The Manifest Format

```json
{
    "resources": {
        "cache": {
            "type": "container.v0",
            "image": "docker.io/library/redis:7.4",
            "connectionString": "{cache.bindings.tcp.host}:{cache.bindings.tcp.port}"
        },
        "api": {
            "type": "project.v0",
            "path": "../Bazaar.Api/Bazaar.Api.csproj",
            "env": {
                "ConnectionStrings__cache": "{cache.connectionString}"
            },
            "bindings": {
                "http": { "scheme": "http", "protocol": "tcp" }
            }
        }
    }
}
```

## What It Captures

- Every resource and its type
- Container images and tags
- Project paths
- Environment variables with expression placeholders
- Endpoint bindings (ports, schemes)
- Connection string formats

> 🌍 **Real World:** The aspire-manifest.json format is Aspire's universal interchange format. Third-party tools like Radius, Pulumi, and custom enterprise deployment systems can consume it to deploy Aspire apps to any infrastructure — not just the targets Aspire ships with.

## Who Consumes It

The manifest is consumed by:
- `aspire deploy` — reads the manifest to deploy to a cloud target
- Docker Compose publisher — transforms manifest into docker-compose.yml
- Kubernetes publisher — transforms manifest into K8s YAML
- Azure Bicep publisher — transforms manifest into Bicep templates
- Custom publishers — any tool that understands the manifest format
