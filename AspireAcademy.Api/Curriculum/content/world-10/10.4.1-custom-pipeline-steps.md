# Writing Custom Pipeline Steps

You can inject your own steps into the publishing pipeline — before, after, or alongside the built-in steps.

---

## Adding a Custom Step

```csharp
builder.Eventing.Subscribe<BeforeStartEvent>((evt, ct) =>
{
    if (evt.Model.ExecutionContext.IsPublishMode)
    {
        var pipeline = evt.Services.GetRequiredService<IDistributedApplicationPipeline>();

        pipeline.AddStep(new PipelineStep
        {
            Name = "generate-api-docs",
            Description = "Generate OpenAPI documentation",
            Action = async ctx =>
            {
                // Generate docs
                await GenerateOpenApiDocs(ctx.PipelineContext.Model);
                ctx.PipelineContext.Summary.Add("API Docs", "Generated");
            },
            DependsOnSteps = [WellKnownPipelineSteps.BuildPrereq],
            RequiredBySteps = [WellKnownPipelineSteps.Publish]
        });
    }
    return Task.CompletedTask;
});
```

## Resource-Scoped Steps

Steps can be scoped to specific resources:

```csharp
pipeline.AddStep(new PipelineStep
{
    Name = "optimize-frontend",
    Resource = frontendResource,
    Action = async ctx => { /* optimize */ },
    DependsOnSteps = ["build"]
});
```

## Step Dependencies

Use `DependsOnSteps` and `RequiredBySteps` to position your step in the DAG:

- `DependsOnSteps: ["build"]` — runs after the build step
- `RequiredBySteps: ["publish"]` — must complete before publish

The pipeline validates the DAG for cycles and duplicate names at execution time.
