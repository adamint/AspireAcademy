# PipelineStep and DAG Execution

When you run `aspire publish`, Aspire executes a **pipeline** — a directed acyclic graph (DAG) of steps that build, push, and publish your application.

---

## PipelineStep

```csharp
public class PipelineStep
{
    public string Name { get; set; }
    public string? Description { get; set; }
    public Func<PipelineStepContext, Task> Action { get; set; }
    public List<string> DependsOnSteps { get; set; } = [];
    public List<string> RequiredBySteps { get; set; } = [];
    public List<string> Tags { get; set; } = [];
    public IResource? Resource { get; set; }
}
```

Each step has:
- **Name** — unique identifier
- **Action** — the async work to perform
- **DependsOnSteps** — steps that must complete before this one
- **RequiredBySteps** — steps that depend on this one

## DAG Execution

The pipeline executor:
1. Collects all steps from resource annotations
2. Runs configuration callbacks
3. Validates for duplicate names
4. Normalizes `RequiredBy` into `DependsOn`
5. Topologically sorts the graph
6. Executes steps with satisfied dependencies **concurrently**

Steps that have no dependencies on each other run in parallel, maximizing throughput.

## PipelineContext

The shared state across all steps:

```csharp
public class PipelineContext
{
    public DistributedApplicationModel Model { get; }
    public DistributedApplicationExecutionContext ExecutionContext { get; }
    public IServiceProvider Services { get; }
    public ILogger Logger { get; }
    public CancellationToken CancellationToken { get; }
    public PipelineSummary Summary { get; }
}
```

Every step can access the app model, DI services, and report summary information.
