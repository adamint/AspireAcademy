# Publish Events

The publishing pipeline fires events before and after execution, allowing you to hook into the lifecycle.

---

## BeforePublishEvent

Fires before the pipeline starts executing steps:

```csharp
builder.Eventing.Subscribe<BeforePublishEvent>((evt, ct) =>
{
    Console.WriteLine("Publishing starting...");
    return Task.CompletedTask;
});
```

## AfterPublishEvent

Fires after all pipeline steps have completed:

```csharp
builder.Eventing.Subscribe<AfterPublishEvent>((evt, ct) =>
{
    Console.WriteLine($"Published to: {evt.OutputPath}");
    return Task.CompletedTask;
});
```

## PublisherAdvertisementEvent

Fires during startup to discover available publishers:

```csharp
builder.Eventing.Subscribe<PublisherAdvertisementEvent>((evt, ct) =>
{
    evt.RegisterPublisher("custom-cloud", "My Custom Cloud Publisher");
    return Task.CompletedTask;
});
```

This is how the CLI knows which `--target` options are available.

## Backchannel Activity Streaming

When the CLI runs `aspire publish`, the AppHost streams progress back via the JSON-RPC backchannel:

```
[publish] Building container images... ██████████░░░░ 70%
[publish] Pushing to registry...       ████████████░░ 85%
[publish] Generating Bicep...          ██████████████ 100%
[publish] Complete. Output: ./deploy/azure/
```

Each pipeline step reports `PublishingActivity` objects that the CLI renders as progress.
