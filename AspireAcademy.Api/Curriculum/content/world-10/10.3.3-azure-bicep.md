# Azure Bicep Publishing

Aspire generates Azure Bicep templates that provision your entire application on Azure.

---

## Generating Bicep

```bash
aspire publish --target azure --output-path ./deploy/azure
```

## What Gets Generated

```
deploy/azure/
├── main.bicep
├── main.parameters.json
└── modules/
    ├── api.bicep
    ├── frontend.bicep
    ├── pg.bicep
    └── cache.bicep
```

## AzurePublishingContext

The publishing engine:

1. Creates an `AzureProvisioningOptions` with subscription, resource group, location
2. Builds an `Infrastructure` object representing all Azure resources
3. Maps each Aspire resource to Azure equivalents:
   - `AddPostgres` → Azure Database for PostgreSQL Flexible Server
   - `AddRedis` → Azure Cache for Redis
   - `AddProject` → Azure Container App
4. Generates parameterized Bicep with managed identity, networking, and role assignments

## Key Translations

| Aspire Concept | Azure Bicep Equivalent |
|---|---|
| `AddPostgres("pg")` | `Microsoft.DBforPostgresQL/flexibleServers` |
| `AddRedis("cache")` | `Microsoft.Cache/redis` |
| `AddProject("api")` | `Microsoft.App/containerApps` |
| `WithReference(pg)` | Managed identity + connection string |
| `AddParameter("key", secret: true)` | Bicep parameter with `@secure()` |
| `AddAzureStorage("storage")` | `Microsoft.Storage/storageAccounts` |

## Deploying

```bash
aspire deploy --target azure
```

This runs `aspire publish` to generate Bicep, then uses the Azure CLI (`az deployment`) to provision the resources.
