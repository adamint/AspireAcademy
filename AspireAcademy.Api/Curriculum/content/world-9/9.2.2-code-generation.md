# Code Generation Pipeline

The `ICodeGenerator` interface produces language-specific SDKs from the ATS metadata.

---

## The Interface

```csharp
public interface ICodeGenerator
{
    string Language { get; }
    Dictionary<string, string> GenerateDistributedApplication(AtsContext context);
}
```

Given an `AtsContext`, it produces a dictionary of `filename → content` — the generated SDK files.

## Implementations

The primary code generator produces the TypeScript AppHost SDK:

| Generator | Output | Key File |
|---|---|---|
| `AtsTypeScriptCodeGenerator` | `.modules/aspire.js` | TypeScript/JavaScript SDK |

> 💡 The Aspire codebase also contains code generators for Python, Go, Java, and Rust. These generate client-side bindings for orchestrated services, but only TypeScript is currently supported as a non-C# AppHost language.

## What Gets Generated

For each Handle type, the generator creates a **proxy class** with methods that call `invokeCapability`:

### TypeScript Example

```typescript
// Generated .modules/aspire.js
export class DistributedApplicationBuilderHandle {
    private _handle: string;

    async addRedis(name: string): Promise<RedisResourceBuilderHandle> {
        const result = await rpc.invoke("addRedis", {
            builderHandle: this._handle,
            name
        });
        return new RedisResourceBuilderHandle(result.$handle);
    }

    async addPostgres(name: string): Promise<PostgresServerResourceBuilderHandle> {
        // ...
    }
}
```

### Python Example

> Note: Python is not a supported AppHost language, but this shows how the same code generation pattern would produce client bindings for any language.

```python
# Example generated output
class DistributedApplicationBuilder:
    def __init__(self, handle: str):
        self._handle = handle

    def add_redis(self, name: str) -> "RedisResourceBuilder":
        result = rpc.invoke("addRedis", {
            "builderHandle": self._handle,
            "name": name
        })
        return RedisResourceBuilder(result["$handle"])
```

## Type Mapping

The TypeScript generator maps ATS types to language-native types:

| ATS Type | TypeScript |
|---|---|
| `string` | `string` |
| `int` | `number` |
| `bool` | `boolean` |
| Handle | `*Handle` class |
| DTO | Interface |
| Enum | Enum |

## When Code Is Generated

Code generation runs automatically when you:
- Run `aspire new` with a polyglot template
- Add a new hosting integration NuGet package
- Run `aspire restore`

The generated files are checked into your project so IDE autocomplete and type checking work immediately.
