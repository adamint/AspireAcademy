# ILanguageSupport and RuntimeSpec

`ILanguageSupport` defines how the Aspire CLI detects, scaffolds, and runs AppHosts in each language.

---

## The Interface

```csharp
public interface ILanguageSupport
{
    string Language { get; }
    bool CanHandle(string directory);
    Task<RuntimeSpec> GetRuntimeSpecAsync(string directory);
    Task ScaffoldAsync(string directory, ScaffoldOptions options);
}
```

## Detection

Each language support implementation has a detection pattern:

| Language | File Pattern | Implementation |
|---|---|---|
| C# | `apphost.cs` or `*.AppHost.csproj` | Built-in |
| TypeScript | `apphost.ts` | `TypeScriptLanguageSupport` |
| Python | `apphost.py` | `PythonLanguageSupport` |
| Go | `apphost.go` | `GoLanguageSupport` |
| Java | `AppHost.java` | `JavaLanguageSupport` |
| Rust | `apphost.rs` | `RustLanguageSupport` |

When you run `aspire run` in a directory, the CLI checks each `ILanguageSupport.CanHandle()` to determine which language to use.

## RuntimeSpec

`RuntimeSpec` describes how to run the AppHost:

```csharp
public record RuntimeSpec
{
    public string[] Initialize { get; init; }        // One-time setup
    public string[] InstallDependencies { get; init; } // Install packages
    public string[] Execute { get; init; }            // Run the app
    public string[] WatchExecute { get; init; }       // Run with file watching
}
```

Example for TypeScript:

```csharp
RuntimeSpec
{
    Initialize = ["npm", "init", "-y"],
    InstallDependencies = ["npm", "install"],
    Execute = ["npx", "tsx", "apphost.ts"],
    WatchExecute = ["npx", "tsx", "--watch", "apphost.ts"]
}
```

## Scaffolding

`ScaffoldAsync` creates the initial project structure for a new polyglot AppHost:

1. Creates the AppHost file (`apphost.ts`, `apphost.py`, etc.)
2. Creates package files (`package.json`, `requirements.txt`, etc.)
3. Runs code generation to create the `.modules/` SDK
