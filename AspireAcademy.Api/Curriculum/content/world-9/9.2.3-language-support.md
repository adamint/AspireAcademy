# ILanguageSupport and RuntimeSpec

`ILanguageSupport` defines how the Aspire CLI detects, scaffolds, and runs AppHosts in each language.

---

## The Interface

```csharp
public interface ILanguageSupport
{
    string Language { get; }
    bool CanHandle(string directory);
    Task<RuntimeSpec> GetRuntimeSpecAsync(string directory);
    Task ScaffoldAsync(string directory, ScaffoldOptions options);
}
```

## Detection

Each language support implementation has a detection pattern:

| Language | File Pattern | Implementation |
|---|---|---|
| C# | `apphost.cs` or `*.AppHost.csproj` | Built-in |
| TypeScript | `apphost.ts` | `TypeScriptLanguageSupport` |

When you run `aspire run` in a directory, the CLI checks each `ILanguageSupport.CanHandle()` to determine which language to use.

> 💡 The Aspire codebase also contains `ILanguageSupport` implementations for other languages (Python, Go, Java, Rust), but only C# and TypeScript are currently supported as AppHost languages. Services in those other languages are orchestrated via `AddPythonApp()`, `AddContainer()`, etc.

## RuntimeSpec

`RuntimeSpec` describes how to run the AppHost:

```csharp
public record RuntimeSpec
{
    public string[] Initialize { get; init; }        // One-time setup
    public string[] InstallDependencies { get; init; } // Install packages
    public string[] Execute { get; init; }            // Run the app
    public string[] WatchExecute { get; init; }       // Run with file watching
}
```

Example for TypeScript:

```csharp
RuntimeSpec
{
    Initialize = ["npm", "init", "-y"],
    InstallDependencies = ["npm", "install"],
    Execute = ["npx", "tsx", "apphost.ts"],
    WatchExecute = ["npx", "tsx", "--watch", "apphost.ts"]
}
```

## Scaffolding

`ScaffoldAsync` creates the initial project structure for a new TypeScript AppHost:

1. Creates the AppHost file (`apphost.ts`)
2. Creates package files (`package.json`)
3. Runs code generation to create the `.modules/aspire.js` SDK

## The CLI Lifecycle for Polyglot

```
aspire new --language typescript
    ↓
1. TypeScriptLanguageSupport.ScaffoldAsync()
    → creates apphost.ts, package.json
    ↓
2. ICodeGenerator.GenerateDistributedApplication()
    → creates .modules/aspire.js
    ↓
3. RuntimeSpec.InstallDependencies
    → runs npm install
```

```
aspire run
    ↓
1. TypeScriptLanguageSupport.CanHandle(".")
    → detects apphost.ts → true
    ↓
2. TypeScriptLanguageSupport.GetRuntimeSpecAsync()
    → returns RuntimeSpec with npx tsx commands
    ↓
3. Start RemoteHostServer process
    ↓
4. Execute "npx tsx apphost.ts"
    → apphost.ts connects to RemoteHost via Unix socket
    → builds the app model via RPC
    → DCP starts resources
```
