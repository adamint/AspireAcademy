# The JSON-RPC Protocol

Every polyglot AppHost call is a JSON-RPC 2.0 message over a Unix socket. Here is the wire format.

---

## Request Format

```json
{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "invokeCapability",
    "params": {
        "capabilityId": "addRedis",
        "args": {
            "builderHandle": "handle-1",
            "name": "cache"
        }
    }
}
```

## Response Format

```json
{
    "jsonrpc": "2.0",
    "id": 1,
    "result": {
        "$handle": "handle-3"
    }
}
```

The `$handle` key indicates a handle reference — the caller (TypeScript/Python) wraps this in a typed proxy object.

## Error Format

```json
{
    "jsonrpc": "2.0",
    "id": 1,
    "error": {
        "code": -32000,
        "message": "Resource name 'cache' is already in use",
        "data": {
            "type": "CapabilityException",
            "details": "..."
        }
    }
}
```

## AtsMarshaller

The `AtsMarshaller` handles type conversion between JSON and .NET:

| JSON Type | .NET Type | Category |
|---|---|---|
| `{ "$handle": "id" }` | Object reference | Handle type |
| `{ "name": "...", "port": 80 }` | DTO | Value type |
| `"Persistent"` | Enum value | Enum type |
| `"hello"` | `string` | Primitive |
| `42` | `int` | Primitive |
| `true` | `bool` | Primitive |

Handle types are passed by reference (only the ID crosses the wire). DTO types are passed by value (serialized as JSON). This distinction is defined by the Application Type System (ATS).

## Latency

Since the socket is local, round-trip latency is **microseconds**. A typical AppHost with 10 resources makes ~50 RPC calls during startup — total overhead is well under 10ms.
