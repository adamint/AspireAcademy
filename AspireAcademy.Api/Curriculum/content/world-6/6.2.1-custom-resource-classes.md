# Creating Custom Resource Classes

Every Aspire resource inherits from one of three base classes depending on what it represents:

- **`Resource`** — a logical resource (API gateway, feature flag service, external SaaS)
- **`ContainerResource`** — a Docker container managed by the orchestrator
- **`ExecutableResource`** — a local process launched on the host machine

## Implementing IResourceWithConnectionString

To allow other resources to call `.WithReference(yourResource)`, implement `IResourceWithConnectionString`. This interface requires two members: `ConnectionStringExpression` (a `ReferenceExpression` used in manifests) and `GetConnectionStringAsync()` (resolves the actual string at runtime).

```csharp
public class FeatureFlagResource(string name)
    : Resource(name), IResourceWithConnectionString
{
    // Define the endpoint this resource exposes
    public EndpointReference Endpoint => new(this, "http");

    // Expression used in manifest publishing and environment variables
    public ReferenceExpression ConnectionStringExpression =>
        ReferenceExpression.Create($"{Endpoint.Property(EndpointProperty.Url)}");

    // Runtime resolution delegates to the expression
    public ValueTask<string?> GetConnectionStringAsync(CancellationToken ct = default)
        => ConnectionStringExpression.GetValueAsync(ct);
}
```

## Choosing the Right Base Class

If your resource runs as a Docker container, inherit from `ContainerResource` to get built-in support for image configuration, volume mounts, port mappings, and container lifecycle management. If it's a local executable, use `ExecutableResource` for process management.

For everything else — external services, logical abstractions, or SaaS integrations — use `Resource` directly. You can still attach endpoints, environment variables, and health checks via annotations.

## Connection Properties

`IResourceWithConnectionString` also defines `GetConnectionProperties()`, which returns structured key-value metadata about the connection. The dashboard uses these properties to display connection details. Override this method when you want to expose specific properties like hostname, port, or database name separately from the raw connection string.
