# Boss: Publish to Docker Compose AND Kubernetes

Publish the same Aspire app to both Docker Compose and Kubernetes with custom configuration.

This boss challenge tests your mastery of the entire publish pipeline. You'll take a single Aspire app model and generate deployment artifacts for two different targets, each with custom configuration.

## The Challenge

Given an AppHost with a web frontend, an API, a PostgreSQL database, and a Redis cache, publish to both Docker Compose and Kubernetes with these requirements:

1. **Docker Compose**: Add restart policies and resource limits
2. **Kubernetes**: Set 3 replicas for the API with CPU/memory limits and a LoadBalancer service

## Solution

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var cache = builder.AddRedis("cache");
var postgres = builder.AddPostgres("pg")
    .WithDataVolume()
    .AddDatabase("catalogdb");

var api = builder.AddProject<Projects.CatalogApi>("api")
    .WithReference(cache)
    .WithReference(postgres);

var frontend = builder.AddProject<Projects.WebFrontend>("frontend")
    .WithReference(api)
    .WithExternalHttpEndpoints();

// Docker Compose target
builder.AddDockerComposeEnvironment("compose")
    .ConfigureComposeFile(compose =>
    {
        foreach (var (name, service) in compose.Services)
        {
            service.Restart = "unless-stopped";
        }
    });

// Kubernetes target
builder.AddKubernetesEnvironment("k8s");

api.PublishAsKubernetesService((resource, objects) =>
{
    var deployment = objects.OfType<Deployment>().First();
    deployment.Spec.Replicas = 3;
    var container = deployment.Spec.Template.Spec.Containers[0];
    container.Resources = new()
    {
        Limits = new() { ["cpu"] = "500m", ["memory"] = "256Mi" },
        Requests = new() { ["cpu"] = "100m", ["memory"] = "128Mi" }
    };
});

frontend.PublishAsKubernetesService((resource, objects) =>
{
    var service = objects.OfType<Service>().First();
    service.Spec.Type = "LoadBalancer";
});

builder.Build().Run();
```

## Publishing Both Targets

```bash
# Generate Docker Compose output
aspire publish --output-path ./compose-output --publisher docker-compose

# Generate Kubernetes output
aspire publish --output-path ./k8s-output --publisher kubernetes
```

## Key Takeaways

- The **same app model** produces artifacts for different platforms
- `ConfigureComposeFile` applies global Compose settings
- `PublishAsKubernetesService` customizes per-resource K8s output
- The manifest is generated once and consumed by each publish target
- Both outputs are standalone — no Aspire runtime required
