# Expression Resolution: From Config-Time Templates to Runtime Values

Aspire's expression system lets you write `ReferenceExpression.Create($"Host={pg.GetEndpoint("tcp")}")` at configuration time but resolve actual host/port values at runtime. Three source files power this:

## ReferenceExpression

At `src/Aspire.Hosting/ApplicationModel/ReferenceExpression.cs`, `ReferenceExpression` implements `IManifestExpressionProvider, IValueProvider, IValueWithReferences`. The static `Create(in ExpressionInterpolatedStringHandler)` method uses C#'s interpolated string handler pattern to capture the format string and parameter references **without evaluating them**.

Key properties:
- **Format** — The template string with `{0}`, `{1}` placeholders
- **ValueProviders** — The `IValueProvider` instances that fill those placeholders
- **IsConditional** — Supports `CreateConditional()` for optional expressions

At runtime, `GetValueAsync()` resolves all value providers and assembles the final string.

## EndpointReference

At `src/Aspire.Hosting/ApplicationModel/EndpointReference.cs`, `public sealed class EndpointReference` implements `IManifestExpressionProvider, IValueProvider`. Calling `.Property(EndpointProperty.Host)` returns an `EndpointReferenceExpression` that knows how to resolve to the actual allocated host, port, scheme, or full URL at runtime.

## ExpressionResolver

At `src/Aspire.Hosting/ApplicationModel/ExpressionResolver.cs`, `internal class ExpressionResolver` ties everything together. Key methods:

- **`ResolveAsync()`** — Entry point for resolving any expression
- **`EvalExpressionAsync()`** — Recursively resolves nested expressions, handling `ReferenceExpression` trees
- **`EvalValueProvider()`** — Resolves individual `IValueProvider` instances

## The Two-Phase Pattern

This separation is critical: at **config time** (in your AppHost `Program.cs`), expressions capture *what* to resolve. At **runtime** (when DcpExecutor starts resources), `ExpressionResolver` resolves *actual values* — allocated ports, generated passwords, real hostnames. This deferred evaluation is what makes Aspire's dependency injection of infrastructure work.
