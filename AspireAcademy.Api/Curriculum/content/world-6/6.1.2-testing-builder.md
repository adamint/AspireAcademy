# The DistributedApplicationTestingBuilder

Deep dive into the central class of Aspire's testing infrastructure and how to configure it for your test scenarios.

## Overview

`DistributedApplicationTestingBuilder` is a static class in the `Aspire.Hosting.Testing` namespace. It provides factory methods that load your AppHost project and return an `IDistributedApplicationTestingBuilder` — a test-aware application builder you can customize before starting the app.

## Creating a Testing Builder

### Generic Overload (Most Common)

The simplest way to create a testing builder references your AppHost via its project type:

```csharp
var builder = await DistributedApplicationTestingBuilder.CreateAsync<Projects.MyAppHost>();
```

The generic type parameter `TEntryPoint` points to the class generated by the AppHost project reference. Aspire locates the AppHost's `Program.cs` entry point and runs it to discover all resource definitions.

### Non-Generic Overload

If you need to specify the entry point type dynamically:

```csharp
var builder = await DistributedApplicationTestingBuilder.CreateAsync(typeof(Projects.MyAppHost));
```

Both overloads behave identically — the generic version is simply syntactic sugar.

### Passing Arguments

You can pass command-line arguments to the AppHost, just like running `dotnet run -- arg1 arg2`:

```csharp
var builder = await DistributedApplicationTestingBuilder.CreateAsync<Projects.MyAppHost>(
    ["--environment", "Testing"]
);
```

### Configuring the Builder

The most flexible overload lets you configure `DistributedApplicationOptions` and `HostApplicationBuilderSettings` directly:

```csharp
var builder = await DistributedApplicationTestingBuilder.CreateAsync<Projects.MyAppHost>(
    args: [],
    configureBuilder: (appOptions, hostSettings) =>
    {
        // Configure the host environment
        hostSettings.EnvironmentName = "Testing";
    }
);
```

## The IDistributedApplicationTestingBuilder Interface

The returned builder implements `IDistributedApplicationTestingBuilder`, which extends `IDistributedApplicationBuilder`. This means you have access to all the same APIs you use in your AppHost:

| Property | Description |
| --- | --- |
| `Services` | The `IServiceCollection` for registering or replacing services |
| `Configuration` | The `IConfigurationManager` for reading/writing config values |
| `Resources` | The resource collection containing all defined resources |
| `Environment` | The `IHostEnvironment` with environment name and paths |
| `ExecutionContext` | The `DistributedApplicationExecutionContext` (run vs publish mode) |
| `Eventing` | The eventing system for subscribing to lifecycle events |

### Key Methods

```csharp
// Add a new resource to the application
builder.AddResource(new ContainerResource("test-db"));

// Create a resource builder for further configuration
var resourceBuilder = builder.CreateResourceBuilder(myResource);

// Build the application (call this when you're done configuring)
await using var app = await builder.BuildAsync();
```

## Building and Running

Once you've configured the builder, call `BuildAsync()` to create the `DistributedApplication`:

```csharp
var builder = await DistributedApplicationTestingBuilder.CreateAsync<Projects.MyAppHost>();
await using var app = await builder.BuildAsync();
```

The `DistributedApplication` is both `IDisposable` and `IAsyncDisposable`. Using `await using` ensures all resources (containers, processes) are cleaned up when the test finishes.

### Starting the Application

`BuildAsync()` creates the app but doesn't start it. Call `StartAsync()` to launch all resources:

```csharp
await app.StartAsync();
```

This starts containers, launches projects, and begins health checks. You should wait for resources to be ready before making assertions.

## Interacting with the Running App

`DistributedApplication` provides extension methods (from `Aspire.Hosting.Testing`) for interacting with running resources:

### Creating HTTP Clients

```csharp
// Get an HttpClient pointed at a resource's default endpoint
var httpClient = app.CreateHttpClient("webfrontend");

// Specify a named endpoint
var apiClient = app.CreateHttpClient("apiservice", endpointName: "https");
```

The returned `HttpClient` has its `BaseAddress` set to the resource's actual URL (with the randomized port).

### Getting Connection Strings

```csharp
// Get the connection string for a database resource
var connectionString = await app.GetConnectionStringAsync("postgresdb");
```

### Getting Endpoints

```csharp
// Get the URI for a resource's endpoint
var endpoint = app.GetEndpoint("webfrontend");
// Returns something like: https://localhost:54321
```

### Waiting for Resources

Use `ResourceNotificationService` to wait for resources to be healthy before testing:

```csharp
await app.ResourceNotifications.WaitForResourceHealthyAsync("webfrontend");
await app.ResourceNotifications.WaitForResourceHealthyAsync("postgresdb");
```

A resource is considered healthy when it reaches the `Running` state and passes its health checks. If the resource has no health check annotations, reaching `Running` is sufficient.

## How It Differs from the Production Builder

The testing builder makes several adjustments compared to `DistributedApplication.CreateBuilder()`:

| Aspect | Production Builder | Testing Builder |
| --- | --- | --- |
| Port assignment | Uses configured ports | Randomizes ports to avoid conflicts |
| Execution | Runs immediately | Suspends until `BuildAsync()` is called |
| Lifecycle | Long-running process | Scoped to test lifetime |
| Dashboard | Launched by default | Not launched in tests |
| Disposal | Manual shutdown | Automatic via `IAsyncDisposable` |

## Complete Example

```csharp
[Fact]
public async Task WebFrontendReturnsOk()
{
    // Arrange
    var builder = await DistributedApplicationTestingBuilder.CreateAsync<Projects.MyAppHost>();
    await using var app = await builder.BuildAsync();
    await app.StartAsync();

    await app.ResourceNotifications.WaitForResourceHealthyAsync("webfrontend");

    // Act
    var httpClient = app.CreateHttpClient("webfrontend");
    var response = await httpClient.GetAsync("/");

    // Assert
    response.EnsureSuccessStatusCode();
    Assert.Equal("text/html", response.Content.Headers.ContentType?.MediaType);
}
```
