# Resilience Policies

When the Python ML service is slow or the Go worker drops a connection, your C# API should not crash. **Resilience policies** handle transient failures automatically.

---

## What ServiceDefaults Configures

```csharp
http.AddStandardResilienceHandler();
```

This adds a pipeline of four policies to every `HttpClient`:

| Policy | What It Does | Default |
|---|---|---|
| **Retry** | Retries failed requests with exponential backoff | 3 retries, 2s base delay |
| **Circuit Breaker** | Stops sending requests when failures exceed a threshold | 10% failure rate, 30s break |
| **Timeout** | Cancels requests that take too long | 30 seconds per attempt |
| **Total Timeout** | Limits total time across all retries | 2 minutes |

> 🏢 **Real-world story:** A team at a financial services company deployed a microservices app without retry policies. During a routine database failover (which took 4 seconds), every API request failed. With Aspire's `AddStandardResilienceHandler`, the retry policy would have transparently retried those requests after 2 seconds, and users would have experienced a brief pause instead of errors. The team added ServiceDefaults the next day. Total code change: one line.

## Custom Resilience Policies

```csharp
builder.Services.ConfigureHttpClientDefaults(http =>
{
    http.AddResilienceHandler("custom", pipeline =>
    {
        pipeline.AddRetry(new HttpRetryStrategyOptions
        {
            MaxRetryAttempts = 5,
            BackoffType = DelayBackoffType.Exponential,
            Delay = TimeSpan.FromSeconds(1)
        });
        pipeline.AddCircuitBreaker(new HttpCircuitBreakerStrategyOptions
        {
            SamplingDuration = TimeSpan.FromSeconds(60),
            FailureRatio = 0.2,
            BreakDuration = TimeSpan.FromSeconds(15)
        });
    });
});
```

## Resilience in Other Languages

Each language has its own resilience libraries:

- **Python**: `tenacity` for retries, `pybreaker` for circuit breakers
- **TypeScript**: `cockatiel`, `p-retry`
- **Go**: `github.com/avast/retry-go`, custom circuit breaker patterns

The patterns are the same — retry, circuit break, timeout — but the implementation is language-specific.
