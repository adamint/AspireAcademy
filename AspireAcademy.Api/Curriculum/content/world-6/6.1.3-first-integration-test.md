# Writing Your First Integration Test

A complete walkthrough of creating an integration test project, writing a test class, and running assertions against a live Aspire application.

## Setting Up the Test Project

### Create the Project

Start by creating an xUnit test project alongside your AppHost:

```dotnetcli
dotnet new xunit -n MyApp.Tests
```

### Add Required Packages

Install the Aspire testing package:

```dotnetcli
dotnet add package Aspire.Hosting.Testing
```

### Add a Project Reference to Your AppHost

Your test project needs a reference to the AppHost so the testing builder can discover it:

```xml
<ItemGroup>
  <ProjectReference Include="..\MyApp.AppHost\MyApp.AppHost.csproj" />
</ItemGroup>
```

This project reference generates a `Projects.MyApp_AppHost` type that you pass to `DistributedApplicationTestingBuilder.CreateAsync<T>()`.

## Writing the Test Class

### Using IAsyncLifetime for Setup and Teardown

Because starting an Aspire application involves async operations (launching containers, waiting for health checks), use xUnit's `IAsyncLifetime` to manage the lifecycle:

```csharp
using Aspire.Hosting.Testing;

public class WebFrontendTests : IAsyncLifetime
{
    private DistributedApplication _app = null!;

    public async Task InitializeAsync()
    {
        var builder = await DistributedApplicationTestingBuilder
            .CreateAsync<Projects.MyApp_AppHost>();

        _app = await builder.BuildAsync();
        await _app.StartAsync();

        // Wait for the resource under test to be ready
        await _app.ResourceNotifications
            .WaitForResourceHealthyAsync("webfrontend");
    }

    public async Task DisposeAsync()
    {
        await _app.DisposeAsync();
    }

    [Fact]
    public async Task HomepageReturnsSuccessStatusCode()
    {
        // Arrange
        var httpClient = _app.CreateHttpClient("webfrontend");

        // Act
        var response = await httpClient.GetAsync("/");

        // Assert
        Assert.Equal(HttpStatusCode.OK, response.StatusCode);
    }
}
```

### What Happens During the Test

1. **`InitializeAsync`** — the AppHost loads, containers start, ports are assigned, health checks pass
2. **Test methods run** — each `[Fact]` method interacts with the live application
3. **`DisposeAsync`** — containers stop, processes terminate, ports are released

## Adding More Test Methods

Once the application is running, you can write multiple tests against it. All test methods in the class share the same running application:

```csharp
[Fact]
public async Task HomepageReturnsHtml()
{
    var httpClient = _app.CreateHttpClient("webfrontend");

    var response = await httpClient.GetAsync("/");
    var content = await response.Content.ReadAsStringAsync();

    Assert.Contains("<html", content, StringComparison.OrdinalIgnoreCase);
}

[Fact]
public async Task ApiReturnsJsonData()
{
    var httpClient = _app.CreateHttpClient("apiservice");

    var response = await httpClient.GetAsync("/api/products");

    response.EnsureSuccessStatusCode();

    var contentType = response.Content.Headers.ContentType?.MediaType;
    Assert.Equal("application/json", contentType);
}

[Fact]
public async Task DatabaseConnectionStringIsAvailable()
{
    var connectionString = await _app.GetConnectionStringAsync("catalogdb");

    Assert.NotNull(connectionString);
    Assert.Contains("Host=", connectionString);
}
```

## Waiting for Multiple Resources

If your test depends on several resources, wait for all of them in `InitializeAsync`:

```csharp
public async Task InitializeAsync()
{
    var builder = await DistributedApplicationTestingBuilder
        .CreateAsync<Projects.MyApp_AppHost>();

    _app = await builder.BuildAsync();
    await _app.StartAsync();

    // Wait for all resources the tests will interact with
    await _app.ResourceNotifications.WaitForResourceHealthyAsync("webfrontend");
    await _app.ResourceNotifications.WaitForResourceHealthyAsync("apiservice");
    await _app.ResourceNotifications.WaitForResourceHealthyAsync("catalogdb");
}
```

## Using CancellationToken for Timeouts

Startup can be slow, especially when pulling container images for the first time. Use a `CancellationToken` to set a hard timeout:

```csharp
public async Task InitializeAsync()
{
    var builder = await DistributedApplicationTestingBuilder
        .CreateAsync<Projects.MyApp_AppHost>();

    _app = await builder.BuildAsync();

    using var cts = new CancellationTokenSource(TimeSpan.FromMinutes(5));
    await _app.StartAsync(cts.Token);

    await _app.ResourceNotifications
        .WaitForResourceHealthyAsync("webfrontend", cts.Token);
}
```

## Testing Response Content

Beyond status codes, you can parse and assert on response bodies:

```csharp
[Fact]
public async Task ApiReturnsExpectedProducts()
{
    var httpClient = _app.CreateHttpClient("apiservice");

    var products = await httpClient.GetFromJsonAsync<List<Product>>("/api/products");

    Assert.NotNull(products);
    Assert.NotEmpty(products);
    Assert.All(products, p => Assert.False(string.IsNullOrEmpty(p.Name)));
}
```

## Running the Tests

Run your tests with the standard `dotnet test` command:

```dotnetcli
dotnet test MyApp.Tests/MyApp.Tests.csproj
```

The first run may be slower as container images are pulled. Subsequent runs reuse cached images.

## Common Pitfalls

### Forgetting to Wait for Resources

If you call `CreateHttpClient` before the resource is healthy, you'll get connection refused errors:

```csharp
// Wrong — resource might not be ready yet
await app.StartAsync();
var client = app.CreateHttpClient("webfrontend");
var response = await client.GetAsync("/"); // may throw

// Correct — wait for the resource first
await app.StartAsync();
await app.ResourceNotifications.WaitForResourceHealthyAsync("webfrontend");
var client = app.CreateHttpClient("webfrontend");
var response = await client.GetAsync("/"); // works reliably
```

### Not Disposing the Application

If you don't dispose the `DistributedApplication`, containers and processes keep running after the test finishes. Always use `await using` or implement `DisposeAsync`:

```csharp
// Option 1: await using in a single test method
await using var app = await builder.BuildAsync();

// Option 2: IAsyncLifetime.DisposeAsync in a test class
public async Task DisposeAsync() => await _app.DisposeAsync();
```
