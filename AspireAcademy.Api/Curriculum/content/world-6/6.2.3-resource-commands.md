# Resource Commands

Resource commands add interactive buttons to the Aspire dashboard, letting developers trigger actions like resetting a database, clearing a cache, or restarting a service — all without leaving the browser.

## ResourceCommandAnnotation

The `ResourceCommandAnnotation` constructor accepts these parameters:

| Parameter | Type | Purpose |
|---|---|---|
| `name` | `string` | Unique identifier for the command |
| `displayName` | `string` | Label shown on the dashboard button |
| `updateState` | `Func<UpdateCommandStateContext, ResourceCommandState>` | Controls when the button is enabled/disabled |
| `executeCommand` | `Func<ExecuteCommandContext, Task<ExecuteCommandResult>>` | The action to run when clicked |
| `displayDescription` | `string?` | Tooltip text |
| `confirmationMessage` | `string?` | Prompt before execution |
| `iconName` | `string?` | Fluent UI icon name |
| `iconVariant` | `IconVariant?` | `Filled` or `Regular` icon style |
| `isHighlighted` | `bool` | Whether the command is visually emphasized |

## Adding a Command

```csharp
var db = builder.AddPostgres("pg").AddDatabase("mydb");

db.WithAnnotation(new ResourceCommandAnnotation(
    name: "reset-db",
    displayName: "Reset Database",
    updateState: ctx => ResourceCommandState.Enabled,
    executeCommand: async ctx =>
    {
        // Drop and recreate tables
        return ExecuteCommandResult.Success();
    },
    displayDescription: "Drops all tables and re-seeds data",
    confirmationMessage: "This will delete all data. Continue?",
    iconName: "DatabaseArrowDown",
    iconVariant: IconVariant.Regular));
```

## State Management

The `updateState` callback runs periodically to determine button state. Return `ResourceCommandState.Enabled`, `ResourceCommandState.Disabled`, or `ResourceCommandState.Hidden`. Use this to disable commands while a resource is starting or hide them in production.

## Execution Context

The `ExecuteCommandContext` passed to `executeCommand` includes `ServiceProvider` for resolving services and `ResourceName` for identifying the target. Return `ExecuteCommandResult.Success()` on success or `ExecuteCommandResult.Failure("reason")` to display an error in the dashboard.
