# AI Agents and the MCP Server

Aspire includes a Model Context Protocol (MCP) server that exposes your running application's state to AI agents and development tools. This enables AI-assisted debugging, monitoring, and application management through a standardized protocol.

## What is MCP?

The Model Context Protocol is an open standard that defines how AI agents interact with external tools and data sources. Aspire implements an MCP server that connects to a running AppHost and exposes its resources, logs, traces, and commands as structured tools that AI agents can invoke.

## Available MCP Tools

The Aspire MCP server provides several tools for querying application state:

| Tool | Purpose |
|------|---------|
| `list_resources` | View all resources, their types, state, and endpoints |
| `list_structured_logs` | Query structured log entries by resource |
| `list_traces` | Browse distributed traces across services |
| `list_console_logs` | Read console output from resources |
| `execute_resource_command` | Start, stop, or restart resources |

These tools give AI agents deep visibility into your running application — they can diagnose why a service is failing, analyze request traces across multiple services, or restart a misbehaving resource.

## How It Works

When the MCP server starts, it connects to your running AppHost and maintains a live connection to the application model. AI agents communicate with the MCP server using the standard protocol, sending tool invocation requests and receiving structured responses.

For example, an AI agent can call `list_resources` to discover all running services, then call `list_structured_logs` filtered to a specific resource to investigate an error, and finally use `list_traces` to trace a failing request across service boundaries.

This integration transforms Aspire applications into AI-observable systems where agents can autonomously investigate issues, reducing the manual debugging burden on developers.
