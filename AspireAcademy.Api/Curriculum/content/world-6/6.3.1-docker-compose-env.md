# AddDockerComposeEnvironment

Use AddDockerComposeEnvironment(name) to generate docker-compose.yaml from the Aspire app model.

The Docker Compose publish target transforms your Aspire app model into a standard `docker-compose.yaml` file that can run anywhere Docker is installed — no Aspire tooling required at runtime.

## Setting Up Docker Compose Publishing

In your AppHost, add a Docker Compose environment:

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var cache = builder.AddRedis("cache");
var postgres = builder.AddPostgres("pg").AddDatabase("catalogdb");

var api = builder.AddProject<Projects.CatalogApi>("api")
    .WithReference(cache)
    .WithReference(postgres);

builder.AddDockerComposeEnvironment("compose");

builder.Build().Run();
```

## Publishing

Run the publish command targeting Docker Compose:

```bash
aspire publish --output-path ./output --publisher docker-compose
```

This generates a `docker-compose.yaml` in the output directory.

## Generated Output

The generated `docker-compose.yaml` maps each resource to a service:

```yaml
services:
  cache:
    image: redis:7.4
    ports:
      - "6379"
  pg:
    image: postgres:17
    environment:
      POSTGRES_PASSWORD: "${PG_PASSWORD}"
    ports:
      - "5432"
  api:
    image: catalog-api:latest
    environment:
      ConnectionStrings__cache: "cache:6379"
      ConnectionStrings__catalogdb: "Host=pg;Database=catalogdb"
    depends_on:
      - cache
      - pg
```

## What Gets Mapped

- **Container resources** → Docker Compose services with their images
- **Project resources** → Services with built container images
- **Bindings** → Port mappings and environment variables
- **References** → `depends_on` relationships and connection strings
- **Parameters** → Environment variable substitution (`.env` file)

## Environment File

Parameters and secrets are written to a `.env` file alongside the compose file, keeping sensitive values out of the YAML:

```
PG_PASSWORD=generated-password-here
```

This makes the output fully self-contained and runnable with `docker compose up`.
