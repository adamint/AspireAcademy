# Running and Adding Integrations

Two of the most frequently used CLI commands are `aspire run` for launching your application and `aspire add` for incorporating new integrations — both designed to minimize friction in your development workflow.

## aspire run

The `aspire run` command builds your AppHost project, starts all registered resources, and launches the Aspire dashboard in your browser:

```bash
aspire run
```

Behind the scenes, this performs a `dotnet build`, starts containers for infrastructure resources, launches .NET projects, and opens the dashboard URL automatically. It is equivalent to `dotnet run` on your AppHost but with dashboard integration and better lifecycle management.

To target a specific AppHost when multiple exist:

```bash
aspire run --project ./src/MyApp.AppHost
```

Press `Ctrl+C` to gracefully shut down all resources, stopping containers and terminating service processes.

## aspire add

The `aspire add` command provides an interactive workflow for adding hosting integrations to your AppHost. Running it without arguments presents a searchable list of available integrations:

```bash
aspire add
```

The interactive selector lets you browse and filter integrations — databases, caches, message brokers, and cloud services. After selecting an integration, the CLI performs two actions automatically:

1. **Adds the NuGet package** — installs the corresponding `Aspire.Hosting.*` package to your AppHost project
2. **Adds boilerplate code** — inserts the `Add*()` call into your AppHost's `Program.cs`

For example, selecting Redis results in `Aspire.Hosting.Redis` being added to your project and `builder.AddRedis("cache")` being inserted into your code.

You can also skip the interactive flow by specifying the integration directly:

```bash
aspire add redis
```

This automation eliminates the manual steps of searching for package names and remembering method signatures.
