# Designing a Hosting Integration Package

A polished hosting integration follows Aspire's naming conventions and extension method patterns, making your custom resource feel native alongside built-in resources like Redis or PostgreSQL.

## The Three Method Patterns

### AddMyResource() — Registration

The primary entry point registers your resource with sensible defaults. It should configure endpoints, set the icon, and return an `IResourceBuilder<T>` for chaining.

```csharp
public static class FeatureFlagExtensions
{
    public static IResourceBuilder<FeatureFlagResource> AddFeatureFlags(
        this IDistributedApplicationBuilder builder,
        string name,
        int? port = null)
    {
        var resource = new FeatureFlagResource(name);

        return builder.AddResource(resource)
            .WithAnnotation(new ResourceIconAnnotation("Flag", IconVariant.Filled))
            .WithHttpEndpoint(port: port, name: "http")
            .WithHealthCheck("feature-flags-health");
    }
}
```

### WithFeature() — Optional Configuration

`With*` methods add optional capabilities. Each returns the builder for fluent chaining.

```csharp
public static IResourceBuilder<FeatureFlagResource> WithAdminUI(
    this IResourceBuilder<FeatureFlagResource> builder)
{
    return builder.WithHttpEndpoint(port: null, name: "admin")
        .WithAnnotation(new ResourceCommandAnnotation(
            "open-admin", "Open Admin", 
            ctx => ResourceCommandState.Enabled,
            async ctx => ExecuteCommandResult.Success()));
}
```

### RunAsEmulator() — Local Development

For resources backed by cloud services, provide an emulator option that runs a container locally.

```csharp
public static IResourceBuilder<FeatureFlagResource> RunAsEmulator(
    this IResourceBuilder<FeatureFlagResource> builder,
    Action<IResourceBuilder<ContainerResource>>? configure = null)
{
    // Replace with a container-based local implementation
}
```

## Packaging Guidelines

Package your integration as a NuGet package named `Aspire.Hosting.{Technology}`. Place extension methods in a descriptive static class. Include XML documentation on every public method — these generate IntelliSense tooltips. Add a README following the hosting integration template and include a `ConfigurationSchema.json` if your resource accepts structured configuration.
