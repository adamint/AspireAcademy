# PublishAsKubernetesService and Customization

Customize Kubernetes output per-resource with PublishAsKubernetesService and KubernetesResource.

The default Kubernetes output covers common scenarios, but production deployments often need custom annotations, resource limits, ingress rules, or sidecar containers. Aspire provides customization hooks at both the resource and global level.

## PublishAsKubernetesService

Customize how an individual resource maps to Kubernetes objects:

```csharp
var api = builder.AddProject<Projects.CatalogApi>("api")
    .WithReference(cache)
    .PublishAsKubernetesService((resource, objects) =>
    {
        // Customize the Deployment
        var deployment = objects.OfType<Deployment>().First();
        deployment.Spec.Replicas = 3;

        // Add resource limits
        var container = deployment.Spec.Template.Spec.Containers[0];
        container.Resources = new()
        {
            Limits = new Dictionary<string, string>
            {
                ["cpu"] = "500m",
                ["memory"] = "256Mi"
            },
            Requests = new Dictionary<string, string>
            {
                ["cpu"] = "100m",
                ["memory"] = "128Mi"
            }
        };
    });
```

## Adding Annotations and Labels

Common production requirements like monitoring, service mesh integration, or ingress:

```csharp
.PublishAsKubernetesService((resource, objects) =>
{
    var deployment = objects.OfType<Deployment>().First();

    // Add Prometheus scrape annotations
    deployment.Spec.Template.Metadata.Annotations = new()
    {
        ["prometheus.io/scrape"] = "true",
        ["prometheus.io/port"] = "8080"
    };

    // Add a service with specific type
    var service = objects.OfType<Service>().First();
    service.Spec.Type = "LoadBalancer";
});
```

## Adding Custom Objects

You can inject additional Kubernetes objects into the output:

```csharp
.PublishAsKubernetesService((resource, objects) =>
{
    // Add an HPA for auto-scaling
    objects.Add(new HorizontalPodAutoscaler
    {
        Metadata = new() { Name = resource.Name },
        Spec = new()
        {
            MinReplicas = 2,
            MaxReplicas = 10,
            TargetCPUUtilizationPercentage = 70
        }
    });
});
```

## Common Customizations

- **Replicas**: Set `deployment.Spec.Replicas` for scaling
- **Resource limits**: CPU and memory requests/limits
- **Service type**: `ClusterIP`, `LoadBalancer`, `NodePort`
- **Annotations**: Prometheus, Istio, cert-manager
- **Node selectors**: Target specific node pools
- **Tolerations**: Schedule on tainted nodes
