# ConfigureComposeFile and PublishAsDockerComposeService

Customize compose output with ConfigureComposeFile and per-resource customization with PublishAsDockerComposeService.

The default Docker Compose output works for most cases, but you often need to customize it — adding volumes, networks, resource limits, or extra services. Aspire provides two customization APIs.

## ConfigureComposeFile

`ConfigureComposeFile` lets you modify the entire generated `docker-compose.yaml` before it's written:

```csharp
builder.AddDockerComposeEnvironment("compose")
    .ConfigureComposeFile(compose =>
    {
        // Add a custom network
        compose.AddNetwork("backend", new()
        {
            Driver = "bridge"
        });

        // Add resource limits to a service
        if (compose.Services.TryGetValue("api", out var apiService))
        {
            apiService.Deploy = new()
            {
                Resources = new()
                {
                    Limits = new() { Cpus = "0.5", Memory = "512M" }
                }
            };
        }
    });
```

This callback receives the full compose model, so you can add networks, volumes, modify service definitions, or inject entirely new services.

## PublishAsDockerComposeService

For per-resource customization, use `PublishAsDockerComposeService` to control how an individual resource maps to a compose service:

```csharp
var api = builder.AddProject<Projects.CatalogApi>("api")
    .WithReference(cache)
    .PublishAsDockerComposeService((resource, service) =>
    {
        service.Restart = "unless-stopped";
        service.Labels = new Dictionary<string, string>
        {
            ["traefik.enable"] = "true",
            ["traefik.http.routers.api.rule"] = "Host(`api.local`)"
        };
    });
```

The callback receives both the Aspire resource and the compose service definition, letting you customize based on resource properties.

## Common Customizations

- **Restart policies**: Set `service.Restart = "unless-stopped"` for production resilience
- **Labels**: Add labels for reverse proxies (Traefik, Nginx)
- **Volumes**: Mount additional volumes for data persistence
- **Networks**: Isolate services into frontend/backend networks
- **Health checks**: Add compose-level health check overrides

## Combining Both Approaches

Use `ConfigureComposeFile` for global concerns (networks, volumes) and `PublishAsDockerComposeService` for per-resource settings. Both run during the compose generation pipeline step.
