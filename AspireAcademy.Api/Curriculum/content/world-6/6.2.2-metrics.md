# Metrics

Metrics are numerical measurements recorded over time — request counts, response latencies, error rates, queue depths.

---

## C# — IMeterFactory

```csharp
var meter = app.Services.GetRequiredService<IMeterFactory>()
    .Create("Bazaar.Api");

var requestCounter = meter.CreateCounter<long>("bazaar.api.requests");
var searchLatency = meter.CreateHistogram<double>("bazaar.api.search_latency_ms");

app.MapGet("/search", async (string query) =>
{
    requestCounter.Add(1, new KeyValuePair<string, object?>("endpoint", "search"));
    var sw = Stopwatch.StartNew();

    var results = await Search(query);

    searchLatency.Record(sw.ElapsedMilliseconds);
    return results;
});
```

## Python — OpenTelemetry Metrics

```python
from opentelemetry import metrics

meter = metrics.get_meter("bazaar.ml")
recommendation_counter = meter.create_counter("bazaar.ml.recommendations")

@app.post("/recommend")
async def recommend(request):
    results = compute_recommendations(request)
    recommendation_counter.add(len(results))
    return results
```

## TypeScript — OpenTelemetry Metrics

```typescript
import { metrics } from '@opentelemetry/api';

const meter = metrics.getMeter('bazaar.frontend');
const pageViews = meter.createCounter('bazaar.frontend.page_views');

function trackPageView(page: string) {
    pageViews.add(1, { page });
}
```

## Metric Types

| Type | Description | Example |
|---|---|---|
| **Counter** | Monotonically increasing count | Request count |
| **Histogram** | Distribution of values | Response latency |
| **UpDownCounter** | Value that goes up and down | Active connections |
| **ObservableGauge** | Sampled current value | CPU usage, queue depth |

> 🌍 **Real World:** Production systems typically start with three metrics per service: request count (Counter), request latency (Histogram), and error count (Counter). These three give you rate, latency, and error ratio — the foundation of SRE golden signals.

## Viewing in the Dashboard

The Dashboard's Metrics view shows:
- Time-series charts for each metric
- Percentile breakdowns for histograms
- Dimension filtering (e.g., by endpoint, status code)

All metrics flow via OTLP — the same protocol as traces and logs.
