# Final Boss: Build CloudMarket

This is the ultimate challenge — a capstone project that tests every concept from all six worlds of Aspire Academy. You will build **CloudMarket**, a multi-service e-commerce platform powered entirely by Aspire.

## The Mission

CloudMarket consists of three services, multiple infrastructure resources, custom resource types, eventing, observability, and a full integration test suite. Completing this challenge proves you have mastered the Aspire application model.

## The Ten Steps

1. **Scaffold** the solution using `aspire new` and the CLI tooling from Module 6.5
2. **Create three service projects**: `CatalogApi` for product management, `OrderApi` for order processing, and `WebFrontend` for the customer-facing UI
3. **Add PostgreSQL and Redis** — PostgreSQL with a `catalogdb` database for persistent storage, Redis as `cache` for response caching
4. **Add RabbitMQ** as `messaging` for asynchronous communication between CatalogApi and OrderApi when orders are placed
5. **Build a custom FeatureFlagResource** with a custom container icon and a toggle command, applying the patterns from Module 6.2
6. **Wire up eventing** — use `BeforeResourceStartedEvent` to seed the catalog database with sample products, and `AfterResourcesCreatedEvent` to log the complete application topology
7. **Configure observability** with OpenTelemetry metrics and distributed tracing across all three services via ServiceDefaults
8. **Write integration tests** using `DistributedApplicationTestingBuilder`, verifying CRUD operations through HTTP endpoints with proper fixture management
9. **Generate a deployment manifest** using `aspire publish` to produce the Aspire manifest JSON describing all resources and their relationships
10. **Document your public APIs** with XML documentation on all public types and methods

## Why This Matters

CloudMarket mirrors real-world distributed applications. Each step builds on previous modules — resources, custom types, eventing, testing, and CLI tooling all come together. Completing this challenge means you can architect, build, test, and deploy production-grade Aspire applications.
