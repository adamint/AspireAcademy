# Resource Notifications: The State Broadcasting System

Every time a resource changes state — starting, running, unhealthy, stopped — the Aspire dashboard updates in real time. This is powered by the notification system at `src/Aspire.Hosting/ApplicationModel/ResourceNotificationService.cs`.

## ResourceNotificationService

`ResourceNotificationService` is the central pub/sub hub built on `Channel<ResourceEvent>`. It acts as the single source of truth for all resource state in a running Aspire application.

### Publishing Updates

When DcpExecutor detects a container has started, it calls:

```csharp
PublishUpdateAsync(resource, snapshot => snapshot with { State = new("Running", KnownResourceStateStyles.Success) })
```

This functional update pattern ensures immutability — you receive the current `CustomResourceSnapshot` and return a new one with modifications. The service then broadcasts the updated snapshot to all subscribers.

### Waiting for Resources

`WaitForResourceHealthyAsync()` is what powers `WaitFor()` in your AppHost. It blocks the calling resource until health checks report a healthy status, enabling proper startup ordering.

## CustomResourceSnapshot

Defined at `src/Aspire.Hosting/ApplicationModel/CustomResourceSnapshot.cs`, this `sealed record` captures the complete state of a resource at a point in time:

- **State** — Current lifecycle state (Starting, Running, Stopped, etc.)
- **HealthStatus** — Healthy, Unhealthy, or Degraded from health check results
- **Properties** — Key-value metadata (container ID, image name, etc.)
- **Urls** — Exposed endpoint URLs for the dashboard
- **Commands** — Available actions (Start, Stop, Restart)
- **IconName** — Dashboard display icon

## Dashboard Integration

The dashboard subscribes to resource updates via `SubscribeResourcesAsync`, a gRPC server-streaming call. Each `PublishUpdateAsync()` triggers a push through this stream, causing the Blazor Server UI to re-render the affected resource row. This is why the dashboard updates without polling — it's a true push-based reactive pipeline.
