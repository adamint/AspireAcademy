# AddKubernetesEnvironment

Use AddKubernetesEnvironment(name) to generate Kubernetes manifests from the Aspire app model.

The Kubernetes publish target transforms your Aspire app model into standard Kubernetes manifests — Deployments, Services, ConfigMaps, and Secrets — that can be applied to any Kubernetes cluster.

## Setting Up Kubernetes Publishing

In your AppHost, add a Kubernetes environment:

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var cache = builder.AddRedis("cache");
var postgres = builder.AddPostgres("pg").AddDatabase("catalogdb");

var api = builder.AddProject<Projects.CatalogApi>("api")
    .WithReference(cache)
    .WithReference(postgres);

builder.AddKubernetesEnvironment("k8s");

builder.Build().Run();
```

## Publishing

Generate Kubernetes manifests:

```bash
aspire publish --output-path ./k8s-output --publisher kubernetes
```

## Generated Manifests

For each resource, Aspire generates appropriate Kubernetes objects:

**Project resources** produce:
- `Deployment` with container spec and environment variables
- `Service` with port mappings from bindings
- `ConfigMap` for non-sensitive configuration
- `Secret` for connection strings and passwords

**Container resources** produce:
- `Deployment` with the specified container image
- `Service` exposing declared ports

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: api
spec:
  replicas: 1
  selector:
    matchLabels:
      app: api
  template:
    spec:
      containers:
        - name: api
          image: catalog-api:latest
          env:
            - name: ConnectionStrings__cache
              value: "cache:6379"
          ports:
            - containerPort: 8080
```

## Resource Mapping

| Aspire Resource | Kubernetes Objects |
|---|---|
| `project.v0` | Deployment + Service + ConfigMap |
| `container.v0` | Deployment + Service |
| `parameter.v0` (secret) | Secret |
| `parameter.v0` (plain) | ConfigMap |

## Applying to a Cluster

The generated manifests are standard Kubernetes YAML:

```bash
kubectl apply -f ./k8s-output/
```

No Aspire runtime or tooling is needed on the cluster — the output is pure Kubernetes.
