# The Publishing Pipeline

Behind `aspire publish` lies a pipeline that serializes your distributed application model into JSON. Understanding this pipeline lets you customize how resources appear in the manifest and handle publish-vs-run differences.

## ManifestPublishingContext

When Aspire publishes a manifest, it creates a `ManifestPublishingContext` containing a `Utf8JsonWriter`. Each resource type has a corresponding write method that serializes its properties, bindings, and environment variables into the JSON output. Custom resources can participate by implementing `IManifestExpressionProvider` to control their manifest representation.

## Publish Mode vs. Run Mode

`ExecutionContext.IsPublishMode` is the key boolean that distinguishes between local development and manifest generation. Use it to branch behavior in your AppHost:

```csharp
var cache = builder.AddRedis("cache");

if (builder.ExecutionContext.IsPublishMode)
{
    // In publish mode — configure for production Azure Cache
    cache.PublishAsAzureRedis();
}
else
{
    // In run mode — use local container
    cache.WithImageTag("7.4");
}
```

This pattern is essential for resources that behave differently in development versus production — local containers in run mode, managed cloud services in publish mode.

## IManifestExpressionProvider

Implement `IManifestExpressionProvider` on your custom resource to control how it appears in the manifest. The interface requires a `ValueExpression` property that returns the string expression used when other resources reference yours.

```csharp
public class MyResource : Resource, IManifestExpressionProvider
{
    public string ValueExpression =>
        $"{{{Name}.bindings.http.url}}";
}
```

## Custom Publishing

For advanced scenarios, you can create entirely custom publishers that consume the `DistributedApplicationModel` and generate output in any format — Terraform files, Pulumi programs, or proprietary deployment descriptors. The model's strongly-typed resource graph gives you full access to every resource, endpoint, and relationship defined in the AppHost.
