# Health Checks and Resource Icons

Two annotations dramatically improve the dashboard experience: health checks provide real-time status monitoring, and icons give your custom resources a professional visual identity.

## Wiring Health Checks

The `WithHealthCheck()` extension method connects a standard `IHealthCheck` implementation to a resource. The dashboard polls these checks and displays live status (Healthy, Degraded, Unhealthy) with color-coded indicators.

```csharp
// Register the health check in DI
builder.Services.AddHealthChecks()
    .AddCheck("feature-flags-health", () =>
    {
        // Check if the feature flag service is responsive
        return HealthCheckResult.Healthy("Service is responding");
    });

// Wire it to your resource
var flags = builder.AddResource(new FeatureFlagResource("flags"))
    .WithHealthCheck("feature-flags-health");
```

The health check name passed to `WithHealthCheck()` must match a check registered in the DI container via `AddHealthChecks()`. Multiple health checks can be attached to a single resource — the dashboard aggregates them into an overall status.

## Resource Icons

`ResourceIconAnnotation` sets the icon displayed next to your resource in the dashboard. Icons come from the Fluent UI icon set and accept an `IconVariant` of either `Filled` or `Regular`.

```csharp
builder.AddResource(new FeatureFlagResource("flags"))
    .WithAnnotation(new ResourceIconAnnotation("Flag", IconVariant.Filled))
    .WithHealthCheck("feature-flags-health");
```

Common icon names include `Database`, `Server`, `Cloud`, `Flag`, `Key`, `Shield`, and `Globe`. Browse the [Fluent UI Icons](https://developer.microsoft.com/fluentui#/styles/web/icons) catalog for the full set.

## Combining Both

Health checks and icons work together to create a polished dashboard experience. A healthy resource shows a green checkmark beside its custom icon; an unhealthy one shows a red indicator — making issues immediately visible without clicking into resource details.
