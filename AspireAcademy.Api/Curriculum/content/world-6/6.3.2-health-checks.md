# Health Checks: Built-In and Custom

Health checks tell Aspire whether a service is ready to receive traffic. They drive `WaitFor` behavior and Dashboard status indicators.

---

## Built-In Health Checks

Hosting integrations register health checks automatically:

| Integration | Health Check |
|---|---|
| PostgreSQL | SQL connection attempt |
| Redis | PING command |
| RabbitMQ | Management API check |
| SQL Server | SQL connection attempt |
| MongoDB | Admin command |

Client integrations also register health checks:

```csharp
builder.AddNpgsqlDataSource("bazaar-db"); // registers PostgreSQL health check
builder.AddRedisClient("cache");          // registers Redis health check
```

> 🐛 **Common Gotcha:** `WaitFor(db)` waits for health checks to pass, not just for the container to start. If your database container is running but hasn't finished initialization (e.g., running init scripts), `WaitFor` correctly holds dependent services until the health check (SQL connection attempt) succeeds.

## Custom Health Checks

```csharp
builder.Services.AddHealthChecks()
    .AddCheck("ml-service", async (ct) =>
    {
        var client = new HttpClient();
        var response = await client.GetAsync("http://ml/health", ct);
        return response.IsSuccessStatusCode
            ? HealthCheckResult.Healthy()
            : HealthCheckResult.Unhealthy("ML service unreachable");
    });
```

## Health Endpoints

`AddServiceDefaults()` maps two endpoints:

| Endpoint | Purpose | Checks |
|---|---|---|
| `/health` | Readiness probe | All registered health checks |
| `/alive` | Liveness probe | Always returns 200 (process alive) |

## In the Dashboard

The Dashboard shows health status per resource:
- **Green** — all health checks passing
- **Yellow** — health checks not yet evaluated
- **Red/Orange** — one or more health checks failing

## How WaitFor Uses Health Checks

`WaitFor(db)` subscribes to `ResourceNotificationService` and waits until the resource state is `Running` **and** all registered health checks pass. This is why hosting integrations register health checks — so `WaitFor` knows when the database is truly ready, not just when the container started.
