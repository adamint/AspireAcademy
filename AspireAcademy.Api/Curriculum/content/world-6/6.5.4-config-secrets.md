# Configuration and Secrets Management

The Aspire CLI provides two commands for managing application settings: `aspire config` for general configuration and `aspire secret` for sensitive values. Both integrate with .NET's standard configuration system so values flow seamlessly to your running resources.

## aspire config

The `aspire config` command reads and writes configuration values for your AppHost project:

```bash
# Set a configuration value
aspire config set Parameters:CacheSize "256"

# Read a configuration value
aspire config get Parameters:CacheSize

# List all configuration
aspire config list
```

Values set through `aspire config` are stored in the AppHost's configuration sources and are available at runtime through the standard `IConfiguration` interface. This is useful for non-sensitive parameters like feature flags, resource sizing, or environment-specific settings.

## aspire secret

The `aspire secret` command wraps .NET's Secret Manager with Aspire-aware functionality, making it easy to store sensitive values like connection strings, API keys, and passwords:

```bash
aspire secret set ConnectionStrings:mydb "Host=localhost;Database=myapp;Username=admin;Password=secret"
```

Secrets are stored in the user's local secret store — never in source control. At runtime, the configuration system automatically merges secrets with other configuration sources, so your code accesses them through `IConfiguration` like any other value.

To view stored secrets:

```bash
aspire secret list
```

## Configuration Flow

The relationship between these commands and your application follows .NET's configuration precedence. Values set via `aspire secret` take priority over `appsettings.json`, and environment variables override both. This layered approach means you can define defaults in configuration files, override them with secrets for local development, and use environment variables in production — all without changing code.

Understanding this flow is essential for managing credentials across development, staging, and production environments.
