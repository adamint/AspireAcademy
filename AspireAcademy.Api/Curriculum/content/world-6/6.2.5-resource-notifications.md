# Resource Notifications and State Updates

`ResourceNotificationService` is the mechanism for pushing real-time state updates to the Aspire dashboard. Custom resources use it to report their current status, properties, URLs, and health — keeping the dashboard synchronized with external systems.

## Publishing Updates

The core method is `PublishUpdateAsync`, which takes a resource and a transformation function over `CustomResourceSnapshot`:

```csharp
await notificationService.PublishUpdateAsync(resource,
    snapshot => snapshot with
    {
        State = new("Running", KnownResourceStateStyles.Success),
        Properties = [new("version", "1.2.3"), new("region", "us-east-1")],
        Urls = [new("Dashboard", "https://flags.example.com", isInternalUrl: false)]
    });
```

## CustomResourceSnapshot

`CustomResourceSnapshot` is a sealed record with these key properties:

| Property | Type | Purpose |
|---|---|---|
| `ResourceType` | `string` | The type of resource displayed in the dashboard |
| `State` | `ResourceStateSnapshot` | Display name and style (Success, Warning, Error) |
| `Properties` | `ImmutableArray<ResourcePropertySnapshot>` | Key-value metadata shown in details |
| `HealthStatus` | `HealthStatus?` | Overall health indicator |
| `Urls` | `ImmutableArray<UrlSnapshot>` | Clickable links in the dashboard |
| `EnvironmentVariables` | `ImmutableArray<EnvironmentVariableSnapshot>` | Env vars for display |
| `Volumes` | `ImmutableArray<VolumeSnapshot>` | Volume mounts |
| `Commands` | `ImmutableArray<CommandSnapshot>` | Available commands |
| `Relationships` | `ImmutableArray<RelationshipSnapshot>` | Resource relationships |
| `CreationTimeStamp` | `DateTime?` | When the resource was created |
| `StartTimeStamp` | `DateTime?` | When the resource started |
| `StopTimeStamp` | `DateTime?` | When the resource stopped |
| `ExitCode` | `int?` | Exit code if the resource has stopped |
| `IconName` | `string?` | Override the displayed icon |
| `IconVariant` | `IconVariant?` | Filled or Regular |

## Building a Background Monitor

For resources that represent external services, create a `BackgroundService` that periodically polls the external system and publishes updates:

```csharp
public class FlagServiceMonitor(
    ResourceNotificationService notifications,
    FeatureFlagResource resource) : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        while (!ct.IsCancellationRequested)
        {
            var healthy = await CheckExternalService(ct);
            await notifications.PublishUpdateAsync(resource,
                s => s with { State = healthy
                    ? new("Running", KnownResourceStateStyles.Success)
                    : new("Degraded", KnownResourceStateStyles.Warning) });
            await Task.Delay(TimeSpan.FromSeconds(10), ct);
        }
    }
}
```

## Waiting for Resources

`ResourceNotificationService` also exposes `WaitForResourceHealthyAsync()`, which blocks until a resource reports a healthy state. Use this in integration tests or orchestration logic that depends on a resource being fully operational.
