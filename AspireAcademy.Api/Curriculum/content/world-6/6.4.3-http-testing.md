# HTTP Testing with CreateHttpClient

The most common integration test pattern involves making HTTP requests to your services and asserting on responses. Aspire provides `CreateHttpClient` to streamline this workflow by returning an `HttpClient` with the base address pre-configured to the resource's allocated endpoint.

## Creating an HTTP Client

The `CreateHttpClient` extension method on `DistributedApplication` takes a resource name and returns a ready-to-use client:

```csharp
var client = app.CreateHttpClient("apiservice");
```

The returned `HttpClient` has its `BaseAddress` set to the dynamically allocated URL, so you can make requests using relative paths immediately — no manual endpoint resolution needed.

## End-to-End REST API Testing

Here is a complete pattern for testing a REST API endpoint:

```csharp
var client = app.CreateHttpClient("apiservice");

var response = await client.GetAsync("/api/products");
response.EnsureSuccessStatusCode();

var products = await response.Content
    .ReadFromJsonAsync<Product[]>();
Assert.NotEmpty(products);
```

This validates the entire request pipeline: routing, controller logic, database access, serialization, and response formatting.

## Testing Write Operations

For CRUD operations, chain requests to verify persistence:

```csharp
var client = app.CreateHttpClient("apiservice");

var newProduct = new { Name = "Widget", Price = 9.99 };
var postResponse = await client.PostAsJsonAsync(
    "/api/products", newProduct);
postResponse.EnsureSuccessStatusCode();

var created = await postResponse.Content
    .ReadFromJsonAsync<Product>();
var getResponse = await client.GetAsync(
    $"/api/products/{created!.Id}");
var fetched = await getResponse.Content
    .ReadFromJsonAsync<Product>();

Assert.Equal("Widget", fetched!.Name);
```

## Targeting Specific Endpoints

If a resource exposes multiple endpoints, specify which one to target:

```csharp
var client = app.CreateHttpClient(
    "apiservice", endpointName: "https");
```

This is useful for services with separate HTTP and HTTPS endpoints, or resources exposing both REST and gRPC interfaces on different ports.
