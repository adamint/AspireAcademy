# The Aspire CLI Architecture

The Aspire CLI is a standalone command-line tool that streamlines every phase of Aspire development — from scaffolding new projects to running, configuring, and publishing applications. It is compiled using Native AOT for fast startup and minimal memory footprint.

## Installation

The CLI is installed as a .NET global tool:

```bash
dotnet tool install -g aspire
```

Once installed, all commands are accessible under the `aspire` prefix. The CLI automatically detects your project structure by following project references from the current directory to discover AppHost projects.

## Command Hierarchy

The CLI organizes functionality into a clear command hierarchy:

| Command | Purpose |
|---------|---------|
| `aspire new` | Scaffold new Aspire solutions from templates |
| `aspire run` | Build and launch the AppHost with dashboard |
| `aspire add` | Add hosting integrations interactively |
| `aspire publish` | Generate deployment manifests |
| `aspire config` | Read and write AppHost configuration |
| `aspire secret` | Manage secrets via .NET Secret Manager |
| `aspire doctor` | Diagnose environment issues |
| `aspire deploy` | Deploy an Aspire application |
| `aspire logs` | View resource logs |
| `aspire mcp` | Start the Model Context Protocol server |

Each command is self-documenting — append `--help` to any command to see available options and usage examples.

## AppHost Discovery

When you run an Aspire CLI command, it scans the current directory and its parent directories for `.csproj` files that reference `Aspire.Hosting`. This automatic discovery means you can run `aspire run` from anywhere within your solution and the CLI will find and launch the correct AppHost.

If multiple AppHost projects exist, the CLI prompts you to select one. You can also specify the project explicitly with the `--project` flag.

## Native AOT Compilation

The CLI is compiled ahead-of-time to native code, eliminating JIT compilation overhead. This means commands like `aspire --version` execute in milliseconds rather than the seconds typical of .NET tools, making it feel responsive even for simple queries.
