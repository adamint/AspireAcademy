# TelemetryRepository: In-Memory Telemetry Storage

At `src/Aspire.Dashboard/Otlp/Storage/TelemetryRepository.cs`, `public sealed partial class TelemetryRepository` is the dashboard's telemetry database — an in-memory store optimized for fast writes and bounded memory usage.

## Storage Architecture

TelemetryRepository uses `CircularBuffer` ring buffers with configurable size limits. When the buffer is full, the oldest entries are evicted automatically. This keeps memory consumption predictable regardless of how long the dashboard runs.

### Write Methods

Each telemetry signal has a dedicated ingestion method:

- **`AddTraces()`** — Stores distributed trace spans, linking them into trace trees
- **`AddLogs()`** — Stores structured log entries with resource association
- **`AddMetrics()`** — Stores metric data points (counters, histograms, gauges)

These methods are called by the OTLP gRPC services after protobuf deserialization.

### Read Methods

Query methods retrieve telemetry for UI rendering:

- **`GetTraces()`** — Returns traces with filtering and pagination
- **`GetLogs()`** — Returns logs filtered by resource, severity, or text

## The Watcher Pattern

The real-time magic lives in `TelemetryRepository.Watchers.cs`. Two async streaming methods enable push-based UI updates:

- **`WatchSpansAsync()`** — Yields new trace spans as they arrive
- **`WatchLogsAsync()`** — Yields new log entries as they arrive

Both use `BoundedChannel` with `BoundedChannelFullMode.DropOldest` for O(1) streaming — if a slow subscriber falls behind, old notifications are dropped rather than causing backpressure.

## Subscription Debouncing

To prevent UI thrashing during high-throughput scenarios, watchers implement a **100ms debounce interval**. Multiple telemetry events arriving within 100ms are batched into a single UI notification. This keeps the Blazor rendering pipeline efficient while maintaining near-real-time responsiveness. Without debouncing, a burst of 1000 trace spans would trigger 1000 re-renders; with it, just a handful.
