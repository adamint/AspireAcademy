# Boss Challenge: Build a Feature Flag Service Integration

Put everything from Module 6.2 together by building a complete Aspire hosting integration for a feature flag service. This challenge tests your ability to create custom resources, annotations, commands, health checks, and notifications.

## Your Mission

Build an `Aspire.Hosting.FeatureFlags` integration that includes:

### 1. Resource Class

Create `FeatureFlagResource` inheriting from `Resource` and implementing `IResourceWithConnectionString`. It should expose an HTTP endpoint and generate a connection string from that endpoint's URL.

### 2. Extension Method

Create `AddFeatureFlags(this IDistributedApplicationBuilder builder, string name)` that:

- Registers the resource with an HTTP endpoint
- Attaches a `ResourceIconAnnotation` using the "Flag" icon with `IconVariant.Filled`
- Wires a health check named `"feature-flags-health"`

### 3. Dashboard Command

Add a "Reset Flags" command using `ResourceCommandAnnotation` that:

- Shows a confirmation message before executing
- Uses the "ArrowReset" icon
- Returns `ExecuteCommandResult.Success()` after clearing flag state

### 4. Background Monitor

Create a `BackgroundService` that uses `ResourceNotificationService.PublishUpdateAsync()` to report the resource state every 10 seconds, including a `version` property in the snapshot.

## Success Criteria

- Other resources can call `.WithReference(flags)` and receive a valid connection string
- The dashboard displays the Flag icon and real-time health status
- The "Reset Flags" button appears and prompts for confirmation
- The resource state updates periodically in the dashboard

## Hints

- Look at built-in integrations like Redis or PostgreSQL for patterns
- Use `ReferenceExpression.Create()` for `ConnectionStringExpression`
- Register health checks via `builder.Services.AddHealthChecks()`
- Inject `ResourceNotificationService` into your `BackgroundService`
