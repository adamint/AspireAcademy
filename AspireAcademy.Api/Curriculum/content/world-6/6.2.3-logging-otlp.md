# Structured Logging and OTLP

Structured logging captures log entries as queryable data — not just text strings. Combined with OTLP, logs flow to the Dashboard alongside traces and metrics.

---

## C# — ILogger

```csharp
app.MapGet("/search", (string query, ILogger<Program> logger) =>
{
    logger.LogInformation("Search requested for {Query} at {Time}",
        query, DateTime.UtcNow);
    // ...
});
```

The `{Query}` placeholder creates a **structured property** — you can filter logs by `Query` in the Dashboard.

## Python — structlog or logging

```python
import structlog

logger = structlog.get_logger()

@app.post("/recommend")
async def recommend(request):
    logger.info("recommendation_requested", user_id=request.user_id, count=10)
```

## TypeScript — pino or winston

```typescript
import pino from 'pino';
const logger = pino();

app.get('/search', (req, res) => {
    logger.info({ query: req.query.q }, 'search requested');
});
```

## The Three Signals via OTLP

All three observability signals flow through the same pipeline:

```
Service                    OTLP                     Dashboard
  ├─ Traces ─────────────────────────────────────▶ Traces view
  ├─ Metrics ────────────────────────────────────▶ Metrics view
  └─ Logs ───────────────────────────────────────▶ Structured Logs view
```

**OTLP** (OpenTelemetry Protocol) is the standard wire protocol. The Dashboard runs an OTLP receiver that ingests all three signals and stores them in memory for visualization.

## Console Logs vs Structured Logs

The Dashboard has two log views:

| View | Source | Format |
|---|---|---|
| **Console Logs** | stdout/stderr from the process | Raw text |
| **Structured Logs** | OpenTelemetry log exporter | Queryable properties |

Console logs are always available (DCP captures stdout). Structured logs require OpenTelemetry configuration — which ServiceDefaults provides automatically for .NET services.

For non-.NET services, configure the OpenTelemetry SDK to export to the OTLP endpoint that the Dashboard provides.

> 🐛 **Common Gotcha:** Using string interpolation (`$"Search for {query}"`) instead of message templates (`"Search for {Query}", query`) in C# defeats structured logging. The Dashboard will show the message as plain text with no queryable properties.
