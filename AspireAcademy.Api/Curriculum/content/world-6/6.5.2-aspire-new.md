# Scaffolding Projects with aspire new

The `aspire new` command scaffolds complete Aspire solutions from built-in templates, giving you a fully structured starting point with AppHost, ServiceDefaults, and service projects already wired together.

## Available Templates

Aspire ships with two primary templates:

**aspire-starter** creates a full application with a Web frontend, API backend, AppHost, and ServiceDefaults project. This is ideal for learning or prototyping because it includes working service-to-service communication out of the box:

```bash
aspire new aspire-starter --name MyApp
```

This generates four projects: `MyApp.AppHost`, `MyApp.ServiceDefaults`, `MyApp.ApiService`, and `MyApp.Web` — all properly connected with project references and resource registrations.

**aspire-empty** creates a minimal solution with just the AppHost and ServiceDefaults. Use this when you want full control over your service architecture:

```bash
aspire new aspire-empty --name MyApp
```

## What Gets Generated

The scaffolded AppHost includes `builder.AddProject` calls for each service project, `WithReference` chains for connecting services, and `ServiceDefaults` configuration for OpenTelemetry, health checks, and service discovery.

The ServiceDefaults project contains shared configuration that every service project references — resilience policies, telemetry exporters, and health check endpoints.

## Customization Options

Specify an output directory with `--output`:

```bash
aspire new aspire-starter --name MyApp --output ./src
```

After scaffolding, you can immediately run the application:

```bash
cd MyApp
aspire run
```

The generated solution builds and runs without modification, launching the dashboard where you can observe all services, their endpoints, logs, and traces. From this baseline, you add your own services, resources, and integrations incrementally.
