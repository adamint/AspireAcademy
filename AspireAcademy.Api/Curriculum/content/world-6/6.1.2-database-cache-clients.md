# Database and Cache Clients

Client integrations for databases and caches — the service-side registration for hosting integrations.

---

## Npgsql (PostgreSQL)

```csharp
builder.AddNpgsqlDataSource("atlas-db");
```

Registers:
- `NpgsqlDataSource` in DI
- Connection string from `ConnectionStrings::atlas-db`
- OpenTelemetry tracing for SQL queries
- Health check (connection test)

Usage:
```csharp
app.MapGet("/users", async (NpgsqlDataSource db) =>
{
    await using var cmd = db.CreateCommand("SELECT * FROM users");
    // ...
});
```

## Entity Framework Core

```csharp
builder.AddNpgsqlDbContext<AtlasDbContext>("atlas-db");
```

Registers the `DbContext` with Npgsql provider, OpenTelemetry, and connection pooling.

## SqlClient (SQL Server)

```csharp
builder.AddSqlServerClient("catalog");
```

## MongoDB

```csharp
builder.AddMongoDBClient("analytics");
```

## StackExchange.Redis

```csharp
builder.AddRedisClient("cache");
// or for IDistributedCache:
builder.AddRedisDistributedCache("cache");
// or for output caching:
builder.AddRedisOutputCache("cache");
```

## What Client Integrations Add

Every client integration provides:

| Feature | Description |
|---|---|
| **DI registration** | The typed client is ready to inject |
| **Connection string** | Read from `ConnectionStrings::<name>` |
| **OpenTelemetry** | Traces for queries, cache hits/misses |
| **Health check** | Validates connectivity |
| **Retry policies** | Default resilience via HttpClient pipeline |
