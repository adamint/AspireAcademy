# Manifest Expressions and Resolution

How {resource.connectionString} and {resource.bindings.http.url} expressions resolve at publish time.

Manifest expressions are the glue that connects resources together at deployment time. Instead of hardcoding values, the manifest uses **expression placeholders** that resolve to concrete values when the deployment target processes the manifest.

## Expression Syntax

Expressions use curly braces with dot-notation paths:

```
{resource.property}
```

Common expression patterns:

| Expression | Resolves To |
|---|---|
| `{cache.connectionString}` | The connection string for the `cache` resource |
| `{db.bindings.tcp.host}` | The hostname of the `db` resource's TCP binding |
| `{db.bindings.tcp.port}` | The port number of the TCP binding |
| `{param.value}` | The value of a `parameter.v0` resource |

## How Expressions Appear in the Manifest

Resources reference each other through expressions in `env` and `connectionString` fields:

```json
{
  "resources": {
    "postgres": {
      "type": "container.v0",
      "connectionString": "Host={postgres.bindings.tcp.host};Port={postgres.bindings.tcp.port};Username=postgres;Password={pg-password.value}"
    },
    "pg-password": {
      "type": "parameter.v0",
      "value": "{pg-password.inputs.value}",
      "inputs": {
        "value": { "type": "string", "secret": true }
      }
    },
    "api": {
      "type": "project.v0",
      "env": {
        "ConnectionStrings__postgres": "{postgres.connectionString}"
      }
    }
  }
}
```

## Resolution at Deploy Time

The deployment target (Docker Compose, Kubernetes, Azure) resolves expressions by:

1. **Building a dependency graph** from expression references
2. **Resolving leaf values** first (parameters, known ports)
3. **Substituting expressions** with concrete values
4. **Generating target-specific configuration** (environment variables, secrets, config maps)

## Parameter Inputs

Parameters with `inputs` require values at publish time. The `ProcessParameters` pipeline step prompts for missing values or reads them from configuration:

```json
{
  "type": "parameter.v0",
  "inputs": {
    "value": { "type": "string", "secret": true }
  }
}
```

Expressions make the manifest portable — the same manifest can target Docker Compose, Kubernetes, or Azure, with each target resolving expressions appropriately.
