# Global Lifecycle Events

Global events fire once during the application lifecycle and affect the entire distributed application — not individual resources. They are your primary tool for cross-cutting startup logic.

## BeforeStartEvent

`BeforeStartEvent` fires before the orchestrator begins starting any resources. This is the ideal place for validation, dynamic resource injection, or environment setup. The event carries both `Services` and `Model`, so you can inspect and modify the full resource graph.

```csharp
builder.Eventing.Subscribe<BeforeStartEvent>(async (evt, ct) =>
{
    // Validate all databases have connection strings
    foreach (var resource in evt.Model.Resources.OfType<IResourceWithConnectionString>())
    {
        var cs = await resource.GetConnectionStringAsync(ct);
        if (string.IsNullOrEmpty(cs))
        {
            throw new InvalidOperationException(
                $"Resource '{resource.Name}' has no connection string configured.");
        }
    }
});
```

## AfterResourcesCreatedEvent

`AfterResourcesCreatedEvent` fires after all resources have been instantiated and added to the model, but before they fully start running. Use it for late-binding configuration that depends on the complete resource graph being available.

```csharp
builder.Eventing.Subscribe<AfterResourcesCreatedEvent>(async (evt, ct) =>
{
    var logger = evt.Services.GetRequiredService<ILogger<Program>>();
    logger.LogInformation("All {Count} resources created.",
        evt.Model.Resources.Count());
});
```

## IDistributedApplicationLifecycleHook

For structured, reusable lifecycle participation, implement `IDistributedApplicationLifecycleHook`. This interface defines methods like `BeforeStartAsync` and `AfterResourcesCreatedAsync` that the orchestrator calls automatically. Register your hook with `builder.Services.AddLifecycleHook<MyHook>()`.

```csharp
public class ValidationHook : IDistributedApplicationLifecycleHook
{
    public Task BeforeStartAsync(DistributedApplicationModel model, CancellationToken ct)
    {
        // Validate model before anything starts
        return Task.CompletedTask;
    }
}
```

Lifecycle hooks are ideal when your startup logic is complex enough to warrant its own class rather than an inline callback.
