# ServiceDefaults — The Full Picture

`AddServiceDefaults()` is a single line in your .NET service that configures all cross-cutting concerns. It lives in the `ServiceDefaults` project shared across your solution.

---

## What It Registers

The `Extensions.cs` file in ServiceDefaults calls:

```csharp
public static IHostApplicationBuilder AddServiceDefaults(this IHostApplicationBuilder builder)
{
    // OpenTelemetry
    builder.ConfigureOpenTelemetry();

    // Health checks
    builder.AddDefaultHealthChecks();

    // Service discovery
    builder.Services.AddServiceDiscovery();
    builder.Services.ConfigureHttpClientDefaults(http =>
    {
        http.AddStandardResilienceHandler();  // Resilience
        http.AddServiceDiscovery();            // Service discovery for HttpClient
    });

    return builder;
}
```

> 🎲 **Fun Fact:** `AddServiceDefaults()` is the only line that separates an ordinary ASP.NET Core app from a fully observable Aspire service. One line gives you distributed tracing, metrics, structured logging, health checks, service discovery, and HTTP resilience.

## Breaking It Down

### OpenTelemetry
- Registers `ActivitySource` for distributed tracing
- Configures OTLP exporter to send traces, metrics, and logs to the Dashboard
- Registers default instrumentation (HTTP, runtime metrics)

### Health Checks
- Maps `/health` (readiness) and `/alive` (liveness) endpoints
- `/health` includes all registered health checks
- `/alive` is a simple 200 OK (process is running)

### Service Discovery
- Registers the configuration-based service discovery provider
- Reads `services__*` environment variables injected by `WithReference`
- Resolves hostnames like `http://api` to actual URLs

### HTTP Resilience
- Adds retry, circuit breaker, and timeout policies to all `HttpClient` instances
- Uses `Microsoft.Extensions.Resilience` (Polly-based)

## Non-.NET Services

`AddServiceDefaults()` is .NET-specific. For TypeScript, Python, and Go, you configure each concern independently using OpenTelemetry SDKs and language-native libraries. The environment variables injected by Aspire are the same regardless of language.
