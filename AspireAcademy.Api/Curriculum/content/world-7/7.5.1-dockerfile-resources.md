# Dockerfile Resources

Aspire provides multiple ways to build container images from Dockerfiles. Whether you have an existing Dockerfile or want to generate one programmatically, the hosting APIs handle image building, tagging, and pushing as part of the publish pipeline.

## WithDockerfile — Existing Dockerfiles

The most common approach references an existing Dockerfile:

```csharp
builder.AddDockerfile("frontend", "../Frontend")
    .WithDockerfile("../Frontend", "Dockerfile.prod", stage: "final");
```

`WithDockerfile` creates a `DockerfileBuildAnnotation` recording the build context path, Dockerfile path, and optional build stage. It also calls `EnsureBuildAndPushPipelineAnnotations()` to register `build-<resource>` and `push-<resource>` pipeline steps.

## AddDockerfile — Container from Dockerfile

`AddDockerfile` is a convenience on the builder that creates a container resource and configures it for Dockerfile builds in one call:

```csharp
var frontend = builder.AddDockerfile("frontend", "../Frontend");
```

## WithDockerfileFactory — Dynamic Dockerfiles

For generated Dockerfiles, `WithDockerfileFactory` accepts a callback that produces Dockerfile content at build time:

```csharp
builder.AddContainer("sidecar", "scratch")
    .WithDockerfileFactory(ctx => $"""
        FROM mcr.microsoft.com/dotnet/runtime:10.0
        COPY --from=build /app /app
        ENTRYPOINT ["dotnet", "/app/Sidecar.dll"]
        """);
```

The factory receives a `DockerfileFactoryContext` and returns the Dockerfile as a string. The content is written to a temporary file in a dedicated subdirectory (to avoid scanning large temp trees) before being passed to the container runtime.

## WithDockerfileBuilder — Fluent DSL

`WithDockerfileBuilder` provides a fluent API for programmatic Dockerfile construction:

```csharp
builder.AddContainer("worker", "scratch")
    .WithDockerfileBuilder(ctx =>
    {
        ctx.Builder
            .From("mcr.microsoft.com/dotnet/sdk:10.0", "build")
            .Copy(".", "/src")
            .Run("dotnet publish -o /app")
            .From("mcr.microsoft.com/dotnet/runtime:10.0")
            .Copy("--from=build", "/app", "/app")
            .EntryPoint("dotnet", "/app/Worker.dll");
    });
```

Multiple callbacks can be chained via `DockerfileBuilderCallbackAnnotation`. On materialization, all callbacks execute in order, and the result is serialized to UTF-8 with LF line endings (no BOM).

## Build Arguments and Secrets

```csharp
var npmToken = builder.AddParameter("npm-token", secret: true);

builder.AddDockerfile("frontend", "../Frontend")
    .WithBuildArg("NODE_ENV", "production")
    .WithBuildSecret("npm_token", npmToken);
```

`WithBuildArg` stores values in `DockerfileBuildAnnotation.BuildArguments`. `WithBuildSecret` stores them in `.BuildSecrets`. Note that secret parameters are **rejected** by `WithBuildArg` — you must use `WithBuildSecret` to prevent accidental exposure in build logs.

## Base Image Override

`WithDockerfileBaseImage` attaches a `DockerfileBaseImageAnnotation` to override the base images:

```csharp
builder.AddProject<Projects.Api>("api")
    .WithDockerfileBaseImage(
        buildImage: "mcr.microsoft.com/dotnet/sdk:10.0",
        runtimeImage: "mcr.microsoft.com/dotnet/aspnet:10.0");
```
