# Kubernetes Environment

Publishing to Kubernetes generates a complete **Helm chart** from your Aspire app model. The entry point is `AddKubernetesEnvironment()`, which configures how every resource maps to Kubernetes objects.

## Setting Up Kubernetes Publishing

```csharp
var builder = DistributedApplication.CreateBuilder(args);

var k8s = builder.AddKubernetesEnvironment("k8s");

var cache = builder.AddRedis("cache");
var api = builder.AddProject<Projects.Api>("api")
    .WithReference(cache);

builder.Build().Run();
```

## KubernetesEnvironmentResource

This resource carries defaults for the entire Kubernetes deployment:

| Property | Default | Purpose |
|----------|---------|---------|
| `HelmChartName` | `"aspire"` | Name in `Chart.yaml` |
| `HelmChartVersion` | `"0.1.0"` | Chart version |
| `HelmChartDescription` | `"Aspire Helm Chart"` | Chart description |
| `DefaultStorageType` | `"emptyDir"` | Volume type for resources needing storage |
| `DefaultStorageClassName` | `null` | Kubernetes StorageClass for PVCs |
| `DefaultStorageSize` | `"1Gi"` | Default PVC size |
| `DefaultStorageReadWritePolicy` | `"ReadWriteOnce"` | PVC access mode |
| `DefaultImagePullPolicy` | `"IfNotPresent"` | Container image pull policy |
| `DefaultServiceType` | `"ClusterIP"` | Kubernetes Service type |

## Extension Methods

| Method | Purpose |
|--------|---------|
| `AddKubernetesEnvironment(builder, name)` | Registers the Kubernetes environment |
| `WithProperties(...)` | Configures environment-level properties |
| `PublishAsKubernetesService<T>(Action<KubernetesResource>)` | Customize a resource's Kubernetes output |

## KubernetesResource — The Per-Resource Wrapper

Each compute resource gets wrapped in a `KubernetesResource` object that holds:

- **`Workload`** — either a `Deployment` or `StatefulSet`
- **`ConfigMap`** — non-secret configuration values
- **`Secret`** — sensitive values (passwords, connection strings)
- **`Service`** — the Kubernetes `Service` exposing endpoints
- **`AdditionalResources`** — extra Kubernetes objects (PVCs, etc.)

## Pipeline Integration

`KubernetesInfrastructure` subscribes to `BeforeStartEvent` and creates `KubernetesResource` wrappers for all compute resources. It attaches `DeploymentTargetAnnotation` to each resource so the pipeline knows how to handle it. The `publish-<env>` step then serializes everything into Helm chart files.

## Per-Resource Customization

```csharp
builder.AddRedis("cache")
    .PublishAsKubernetesService<RedisResource>(k8sResource =>
    {
        k8sResource.Service.Spec.Type = "LoadBalancer";
    });
```

This attaches a `KubernetesServiceCustomizationAnnotation` that runs during chart generation, letting you modify any Kubernetes object before it's serialized to YAML.
