# Blazor + gRPC + OTLP

The Aspire Dashboard is built from three technology layers.

---

## Blazor Server (UI)

The Dashboard is an ASP.NET Blazor Server application. Blazor renders HTML on the server and pushes updates to the browser via SignalR WebSocket. This enables **real-time updates** without polling.

Components:
- `ResourcesPage` — the resources table
- `ConsoleLogs` — real-time log streaming
- `StructuredLogs` — queryable log viewer
- `Traces` — waterfall trace viewer
- `Metrics` — time-series charts

## gRPC Resource Service

The Dashboard receives resource state from the AppHost via a gRPC service. The AppHost pushes updates whenever a resource changes state:

```
AppHost → ResourceNotificationService → gRPC → Dashboard → Blazor Component
```

This is how the Dashboard knows a container started, a health check passed, or a process exited — in real time.

## OTLP Receiver

The Dashboard runs an OTLP (OpenTelemetry Protocol) receiver that ingests:
- **Traces** (spans from distributed requests)
- **Metrics** (counters, histograms, gauges)
- **Logs** (structured log entries)

Services export to this receiver via the `OTEL_EXPORTER_OTLP_ENDPOINT` environment variable, which DCP injects automatically.

## Data Storage

All data is stored **in memory**. The Dashboard is for development — not production monitoring. When you stop the AppHost, all traces, metrics, and logs are lost. For production, export OTLP to Seq, Application Insights, Jaeger, or Prometheus.
