# How Resources Flow to the Dashboard

The data flow from DCP state change to Dashboard UI update.

---

## The Flow

```
1. DCP detects container is Running
   ↓
2. DcpExecutor receives gRPC notification
   ↓
3. ResourceNotificationService.PublishUpdateAsync()
   ↓
4. CustomResourceSnapshot created (immutable record)
   ↓
5. All subscribers notified:
   ├── Dashboard gRPC service → pushes to Blazor
   ├── WaitFor mechanism → checks if dependency is ready
   └── Custom subscribers → your code
   ↓
6. Blazor component re-renders
   ↓
7. Browser updates via SignalR
```

## Latency

The entire flow — from container state change to browser update — takes **milliseconds**. There is no polling. Every component uses push-based notification:
- DCP → AppHost: gRPC streaming
- AppHost → Dashboard: gRPC streaming
- Dashboard → Browser: SignalR WebSocket

## What the Dashboard Receives

Each `CustomResourceSnapshot` contains:
- Resource name and type
- Current state (Starting, Running, Exited, etc.)
- Endpoint URLs (allocated by DCP)
- Health check status
- Custom properties (container ID, environment variables)
- Exit code (for terminated resources)
