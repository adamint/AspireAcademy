# Customizing the Pipeline

Aspire's pipeline is fully extensible. You can inject custom steps, reorder existing ones, or add configuration callbacks — all through the `IDistributedApplicationPipeline` interface accessible via `builder.Eventing` and resource annotations.

## Adding Custom Steps

Use `IDistributedApplicationPipeline.AddStep()` to insert your own work into the DAG:

```csharp
pipeline.AddStep("notify-slack", async ctx =>
{
    var summary = ctx.Summary;
    await SendSlackNotification($"Deploy complete: {summary.Items.Count} items");
},
dependsOn: WellKnownPipelineSteps.Deploy);
```

The `dependsOn` parameter accepts a string (step name), a `PipelineStep` object, or even a collection. Similarly, `requiredBy` lets you declare that another step depends on yours.

## Pipeline Configuration Callbacks

`AddPipelineConfiguration()` registers a callback that runs *after* all steps are collected but *before* execution. This is your chance to inspect, reorder, or modify the step graph:

```csharp
pipeline.AddPipelineConfiguration(async ctx =>
{
    var buildSteps = ctx.GetSteps("build-compute");
    foreach (var step in buildSteps)
    {
        // Add a pre-build lint step for each resource
        ctx.Steps.Add(new PipelineStep
        {
            Name = $"lint-{step.Resource?.Name}",
            Action = async stepCtx => await LintDockerfile(step.Resource!),
            DependsOnSteps = ["build-prereq"],
            RequiredBySteps = [step.Name]
        });
    }
});
```

`PipelineConfigurationContext.GetSteps(tag)` returns all steps matching a tag. `GetSteps(resource)` returns steps associated with a specific resource. `GetSteps(resource, tag)` combines both filters.

## Resource-Level Step Factories

Resources inject steps through `PipelineStepAnnotation` and `PipelineStepFactoryContext`. When a resource needs pipeline participation, it adds a `WithPipelineStepFactory()` annotation that receives the `PipelineStepFactoryContext` (which includes both the `PipelineContext` and the `IResource`) and returns one or more steps.

## Reporting Progress

Steps report progress through `IReportingStep` and `IReportingTask`:

```csharp
Action = async ctx =>
{
    await ctx.ReportingStep.UpdateStatusAsync("Building image...");
    // ... do work ...
    await ctx.ReportingStep.SucceedAsync("Image built successfully");
}
```

The `SucceedAsync`, `WarnAsync`, and `FailAsync` extension methods (from `PublishingExtensions`) set the final status. The `PipelineSummary` collects key-value pairs that are displayed to the user after the pipeline completes — use `ctx.Summary.Add("Image", "myapp:latest")` to surface important outputs.
