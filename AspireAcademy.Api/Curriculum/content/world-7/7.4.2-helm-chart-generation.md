# Helm Chart Generation

When the Kubernetes publish step runs, `KubernetesPublishingContext` generates a complete Helm chart directory structure from your app model.

## Output Structure

```
output/
├── Chart.yaml
├── values.yaml
└── templates/
    ├── api/
    │   ├── deployment.yaml
    │   ├── service.yaml
    │   ├── configmap.yaml
    │   └── secret.yaml
    └── cache/
        ├── deployment.yaml
        ├── service.yaml
        ├── configmap.yaml
        └── secret.yaml
```

## Chart.yaml

Generated from `KubernetesEnvironmentResource` properties:

```yaml
apiVersion: v2
name: aspire
version: 0.1.0
appVersion: 0.1.0
description: Aspire Helm Chart
type: application
keywords:
  - aspire
  - kubernetes
kubeVersion: ">= 1.18.0-0"
```

The `HelmChart` model supports full Helm metadata: `Dependencies`, `Maintainers`, `Home`, `Sources`, `Icon`, and `Annotations`.

## values.yaml

All parameterized values are extracted into `values.yaml`, organized by resource:

```yaml
parameters:
  admin-password: ""
secrets:
  api:
    connectionstrings-cache: ""
config:
  api:
    services-cache-tcp-0: ""
```

This separation lets operators customize values at deploy time with `helm install --set` or a values override file.

## Helm Template Expressions

Aspire expressions are transformed into Helm template syntax by `HelmExtensions`:

| Aspire Expression | Helm Expression |
|-------------------|----------------|
| `{admin-password.value}` | `{{ .Values.parameters.admin-password }}` |
| `{cache.connectionString}` | `{{ .Values.secrets.cache.connectionstring }}` |
| `{api.bindings.http.port}` | `{{ .Values.config.api.bindings-http-port }}` |

Helper methods like `ToHelmExpression()`, `ToHelmParameterExpression()`, `ToHelmSecretExpression()`, and `ToHelmConfigExpression()` handle the translation. Numeric values get Helm pipe conversions (`| int`, `| int64`, `| float64`) where appropriate.

## Naming Conventions

`HelmExtensions` provides consistent naming functions:

| Method | Example Input | Output |
|--------|--------------|--------|
| `ToKubernetesResourceName` | `"MyApi"` | `"myapi"` |
| `ToConfigMapName` | `"api"` | `"api-config"` |
| `ToSecretName` | `"api"` | `"api-secrets"` |
| `ToDeploymentName` | `"api"` | `"api"` |
| `ToServiceName` | `"api"` | `"api"` |
| `ToPvcName` | `"cache"` | `"cache-pvc"` |

## YAML Serialization

The YAML serializer uses camelCase naming convention, custom converters for `ByteArray` and `IntOrString` types, and special quoting logic via `HelmExtensions.ShouldDoubleQuoteString()` — strings containing Helm expressions (`{{ }}`) are always double-quoted to prevent YAML parsing issues.
