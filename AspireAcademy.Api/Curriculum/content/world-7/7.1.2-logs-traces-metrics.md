# Structured Logs, Traces, and Metrics

The Dashboard's three remaining views show queryable logs, distributed traces, and time-series metrics.

---

## Structured Logs

Unlike console logs (raw text), structured logs have **properties** you can query:

| Property | Example |
|---|---|
| Severity | Information, Warning, Error |
| Category | Bazaar.Api.SearchController |
| Message | Search requested for "aspire" |
| TraceId | abc123def456 |
| SpanId | 789ghi012 |
| Custom | Query=aspire, ResultCount=42 |

Filter by severity, category, or any custom property. Click a trace ID to jump to the trace view.

## Traces

The trace view shows distributed request flows as **waterfall timelines**:

```
Frontend  ████████████████████████████████████████  200ms
  └─ API  ██████████████████████████████           150ms
      ├─ PostgreSQL  █████████                      50ms
      └─ ML Service  ████████████████               80ms
```

Each bar is a **span**. Click to see:
- Duration
- HTTP method and status code
- Database query text
- Custom attributes (tags)
- Errors and exceptions

## Metrics

Time-series charts for numerical measurements:

- **Request rate** — requests per second by endpoint
- **Error rate** — 4xx and 5xx responses over time
- **Response latency** — p50, p95, p99 percentiles
- **Custom metrics** — anything you record with `Counter`, `Histogram`, etc.

All three views consume data from the OTLP receiver. Any service that exports traces, metrics, or logs via OTLP appears automatically — regardless of language.
