# Resource Commands and the Interaction Service

Resources can expose actions beyond Start/Stop — and the Dashboard and CLI can prompt users for input.

---

## Resource Commands in the Dashboard

Click a resource's action menu to see available commands:
- **Start / Stop / Restart** — built-in lifecycle commands
- **Browse** — opens the resource's HTTP endpoint
- **View Logs** — switches to the Console Logs view for this resource
- **Custom commands** — registered by your AppHost code

## The Interaction Service

Some commands need user input. The `IInteractionService` provides a way to prompt users:

```csharp
var answer = await interactionService.PromptAsync(
    resource,
    "Enter the migration name:",
    "AddNewTable");
```

- **In the Dashboard**: Shows a dialog with an input field
- **In the CLI**: Prompts in the terminal

This lets you build interactive workflows — database migrations, seed scripts, configuration changes — that work from both the Dashboard and CLI.

## Custom Commands in the Dashboard

When you register a custom command (like "Seed Data" from lesson 4.3.3), it appears as a button in the Dashboard's resource action menu. Click it, and the command executes against the running resource.
