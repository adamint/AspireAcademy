# Resource Commands and the Interaction Service

Resources expose **commands** that users trigger from the Dashboard UI or CLI. The **Interaction Service** handles cases where commands need user input.

---

## Built-In Commands

Every resource automatically gets lifecycle commands from DCP:

| Command | Icon | Description |
|---|---|---|
| **Start** | Play | Start a stopped resource |
| **Stop** | Stop | Stop a running resource |
| **Restart** | Refresh | Stop and start |
| **Browse** | Globe | Open HTTP endpoint in browser |

These appear in the resource's action dropdown in the Dashboard.

## Custom Commands

Register custom commands in your AppHost:

```csharp
builder.Eventing.Subscribe<AfterResourcesCreatedEvent>((evt, ct) =>
{
    var commandService = evt.Services
        .GetRequiredService<ResourceCommandService>();

    commandService.RegisterCommand("api", new ResourceCommand(
        name: "seed-database",
        displayName: "Seed Database",
        executeCallback: async context =>
        {
            var client = context.ServiceProvider
                .GetRequiredService<IHttpClientFactory>()
                .CreateClient();
            var response = await client.PostAsync(
                "http://api/admin/seed", null);

            return response.IsSuccessStatusCode
                ? CommandResult.Success("Database seeded")
                : CommandResult.Failure($"Seed failed: {response.StatusCode}");
        },
        iconName: "Database",
        isHighlighted: true
    ));

    return Task.CompletedTask;
});
```

Custom commands appear alongside built-in commands in:
- **Dashboard** — action dropdown button
- **CLI** — `aspire do api seed-database`
- **MCP** — `execute_resource_command` tool (AI agents)

## The Interaction Service

Some commands need user input. The `IInteractionService` provides prompts:

```csharp
commandService.RegisterCommand("db", new ResourceCommand(
    "run-migration", "Run Migration",
    async context =>
    {
        var interaction = context.ServiceProvider
            .GetRequiredService<IInteractionService>();

        var migrationName = await interaction.PromptAsync(
            context.Resource,
            "Enter migration name:",
            defaultValue: "AddNewTable");

        if (migrationName is null)
            return CommandResult.Failure("Cancelled by user");

        // Run the migration...
        return CommandResult.Success($"Migration '{migrationName}' applied");
    }
));
```

The prompt renders as:
- **Dashboard**: A modal dialog with an input field and OK/Cancel
- **CLI**: An interactive terminal prompt
- **MCP**: The AI agent receives a prompt request

## CommandResult

```csharp
public record CommandResult(bool Success, string? Message = null)
{
    public static CommandResult Success(string? msg = null) => new(true, msg);
    public static CommandResult Failure(string? msg = null) => new(false, msg);
}
```

The Dashboard shows results as toast notifications. The CLI prints to terminal.
