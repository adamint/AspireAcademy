# Compose Customization

Aspire provides multiple hooks to customize the generated Docker Compose output — from low-level service manipulation to high-level resource-specific overrides.

## PublishAsDockerComposeService

For fine-grained control over how a specific resource appears in the compose file, use `PublishAsDockerComposeService`:

```csharp
builder.AddRedis("cache")
    .PublishAsDockerComposeService<RedisResource>((composeService, service) =>
    {
        service.Restart = "always";
        service.Deploy = new DeployConfig
        {
            Resources = new Resources
            {
                Limits = new ResourceLimits { Cpus = "0.5", Memory = "256M" }
            }
        };
    });
```

The callback receives both the `DockerComposeServiceResource` (Aspire's wrapper) and the `Service` (the compose YAML model), giving you direct access to every compose property.

## ConfigureComposeFile

For global compose file modifications, use `ConfigureComposeFile()` on the environment resource:

```csharp
compose.ConfigureComposeFile(file =>
{
    file.AddNetwork(new Network { Name = "external-net", External = true });
    file.AddVolume(new Volume { Name = "shared-data" });
});
```

This runs after all resources are translated but before YAML serialization, so you can add networks, volumes, configs, or modify any service.

## Environment Variable Placeholders

When compose services reference parameters, the values become environment variable placeholders. Three helper methods create these placeholders:

- `AsEnvironmentPlaceholder(IManifestExpressionProvider, DockerComposeServiceResource)` — converts any expression provider to a `${VAR}` placeholder
- `AsEnvironmentPlaceholder(ParameterResource, DockerComposeServiceResource)` — converts a parameter directly
- `AsEnvironmentPlaceholder(IResourceBuilder<ParameterResource>, DockerComposeServiceResource)` — accepts a builder

## ConfigureEnvFile

The `.env` file content can be customized:

```csharp
compose.ConfigureEnvFile((envFile) =>
{
    // Add custom environment variables
});
```

## Docker Compose Dashboard

The Dashboard is a first-class compose service. Configure it via `WithDashboard()`:

```csharp
compose.WithDashboard(dashboard =>
{
    dashboard.WithEnvironment("DASHBOARD__FRONTEND__AUTHMODE", "Unsecured");
});
```

`DockerComposeAspireDashboardResource` exposes `PrimaryEndpoint` (web UI) and `OtlpGrpcEndpoint` (telemetry ingestion). All other services automatically send OTLP data to this endpoint when the dashboard is enabled.

## Service Customization Annotations

Under the hood, `PublishAsDockerComposeService` attaches a `DockerComposeServiceCustomizationAnnotation` to the resource. The publishing context applies all customization annotations after building the base service configuration but before YAML serialization. You can also add these annotations directly for more programmatic control.
