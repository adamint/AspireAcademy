# Manifest JSON Structure

The Aspire manifest is a JSON document with a top-level `resources` object. Each key is a resource name, and each value describes the resource's type, configuration, and connections.

```json
{
  "resources": {
    "cache": { ... },
    "api": { ... },
    "pg": { ... },
    "pgdb": { ... }
  }
}
```

## Resource Type Schemas

Every resource has a `type` field using the format `kind.version`:

### `parameter.v0` — Parameters

```json
"admin-password": {
  "type": "parameter.v0",
  "value": "{admin-password.inputs.value}",
  "inputs": {
    "value": {
      "type": "string",
      "secret": true
    }
  }
}
```

Parameters define inputs that are resolved at deploy time. The `inputs` object describes the schema — `type` can be `"string"`, and `secret: true` marks sensitive values. A `default` field provides fallback values.

### `container.v0` / `container.v1` — Containers

```json
"cache": {
  "type": "container.v0",
  "image": "redis:8.6",
  "connectionString": "{cache.bindings.tcp.host}:{cache.bindings.tcp.port}",
  "bindings": {
    "tcp": {
      "scheme": "tcp",
      "protocol": "tcp",
      "transport": "tcp",
      "targetPort": 6379
    }
  }
}
```

Container resources include `image` (or `build` for Dockerfile-based), optional `connectionString`, `bindings`, `env`, `volumes`, `bindMounts`, `containerFiles`, and `args`.

### `project.v0` / `project.v1` — .NET Projects

```json
"api": {
  "type": "project.v1",
  "path": "../Api/Api.csproj",
  "deployment": { "replicas": 1 },
  "env": {
    "ConnectionStrings__pgdb": "{pgdb.connectionString}",
    "services__cache__tcp__0": "{cache.bindings.tcp.url}"
  },
  "bindings": {
    "http": { "scheme": "http", "protocol": "tcp", "transport": "http" },
    "https": { "scheme": "https", "protocol": "tcp", "transport": "http" }
  }
}
```

### `value.v0` — Value Resources

Used for connection-string-only resources and conditional expressions:

```json
"pgdb": {
  "type": "value.v0",
  "connectionString": "{pg.connectionString};Database=pgdb"
}
```

### `executable.v0` — Executables

```json
"worker": {
  "type": "executable.v0",
  "workingDirectory": "../Worker",
  "command": "node",
  "args": ["index.js"],
  "env": { ... }
}
```

## Build Blocks

Dockerfile-based resources replace `image` with a `build` object:

```json
"frontend": {
  "type": "container.v0",
  "build": {
    "context": "../Frontend",
    "dockerfile": "Dockerfile",
    "stage": "final",
    "args": { "NODE_ENV": "production" },
    "secrets": { "npm_token": "{npm-token.value}" }
  }
}
```

The `context` and `dockerfile` paths are relative to the manifest location.
