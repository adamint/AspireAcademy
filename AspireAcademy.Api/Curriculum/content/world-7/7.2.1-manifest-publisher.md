# The Manifest Publisher

Before Aspire can publish to Docker Compose or Kubernetes, it needs a portable description of your entire application — every resource, endpoint, connection string, and environment variable. That's the **manifest**, a JSON file produced by `ManifestPublishingContext`.

## ManifestPublishingContext

`ManifestPublishingContext` is the engine that serializes your `DistributedApplicationModel` into JSON. It accepts a `Utf8JsonWriter`, a manifest output path, and the `ExecutionContext`. Key methods include:

- `WriteResourceAsync(IResource)` — dispatches to type-specific writers
- `WriteContainerAsync(ContainerResource)` — handles container images and Dockerfile builds
- `WriteParameterAsync(ParameterResource)` — serializes parameters with input schemas
- `WriteBindings(IResource)` — emits endpoint definitions (scheme, protocol, transport, target port)
- `WriteEnvironmentVariablesAsync(IResource)` — resolves and writes all env vars
- `WriteConnectionString(IResource)` — emits the connection string expression
- `WriteCommandLineArgumentsAsync(IResource)` — emits args arrays

## Triggering Manifest Publishing

Manifest publishing is wired into the pipeline via `ManifestPublishingExtensions.AddManifestPublishing()`, which registers a `publish-manifest` step. You can also trigger it manually:

```bash
aspire publish --publisher manifest --output-path ./manifest
```

The output path can be a file (`.json`) or a directory — if it's a directory, the publisher writes `aspire-manifest.json` inside it.

## How Resources Are Serialized

`WriteResourceAsync` checks the resource type and dispatches accordingly:

1. **ParameterResource** → `WriteParameterAsync`
2. **ContainerResource** → `WriteContainerAsync`
3. **ProjectResource** → writes project path and deployment config
4. **ExecutableResource** → writes working directory, command, and args
5. Resources implementing `IManifestExpressionProvider` → custom expression output
6. Unsupported resources → `{ "error": "This resource does not support generation in the manifest." }`

The `TryAddDependentResources()` method recursively tracks referenced resources (via connection strings, endpoints, and parameters) to ensure every dependency is included in the manifest, even if not explicitly registered.

## Relative Paths

All file paths in the manifest are normalized relative to the manifest directory via `GetManifestRelativePath()`. This makes the manifest portable across machines — a project path becomes `../src/Api/Api.csproj` rather than an absolute path.
