# Resources and Console Logs

The Dashboard's first two views show what is running and what it is saying.

---

## Resources View

A table of every resource in your application:

| Column | Description |
|---|---|
| **Name** | Resource name from the AppHost |
| **State** | Starting, Running, Waiting, Exited, FailedToStart |
| **Type** | Project, Container, Executable |
| **Source** | Docker image or project path |
| **Endpoints** | Clickable URLs for HTTP endpoints |
| **Actions** | Start, Stop, Restart, Browse, custom commands |

Color-coded state indicators:
- 🟢 **Green** — Running and healthy
- 🟡 **Yellow** — Starting or Waiting (for WaitFor)
- 🔴 **Red** — FailedToStart or unhealthy
- ⚪ **Gray** — Exited (terminated, possibly successfully)

## Console Logs

Real-time stdout/stderr from every resource. Select a resource from the dropdown to filter.

Console logs capture:
- Application output (`Console.WriteLine`, `print()`, `fmt.Println`)
- Framework output (ASP.NET startup messages, uvicorn startup)
- Error output (stack traces, crash information)

Logs are streamed in real time — no polling, no delays. DCP captures stdout/stderr from every process and container and forwards them to the Dashboard via `ResourceLoggerService`.
