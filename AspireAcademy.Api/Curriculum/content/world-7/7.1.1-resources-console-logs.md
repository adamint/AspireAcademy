# Resources and Console Logs

The Dashboard's Resources view is the first thing you see when you start an Aspire application. It is your real-time command center for every resource in the app model.

---

## The Resources Table

Every resource registered in your AppHost appears as a row:

| Column | Description | Example |
|---|---|---|
| **Name** | Resource name from AddRedis("cache"), AddProject("api"), etc. | `cache`, `api`, `frontend` |
| **State** | Current lifecycle state with color indicator | Running (green), Starting (blue), Waiting (yellow) |
| **Type** | Resource category | Project, Container, Executable |
| **Source** | Docker image or project path | `redis:7.4`, `Bazaar.Api.csproj` |
| **Endpoints** | Clickable HTTP/TCP URLs | `http://localhost:5001` |
| **Actions** | Start, Stop, Restart, Browse, custom commands | Buttons in dropdown |

## State Indicators

| Color | State | Meaning |
|---|---|---|
| Green | Running | Active, healthy, all health checks passing |
| Blue spinner | Starting | Container pulling/starting or process launching |
| Yellow | Waiting | Blocked by WaitFor — dependency not yet ready |
| Red | FailedToStart | Crashed, bad config, or dependency failure |
| Orange | RuntimeUnhealthy | Running but health checks failing |
| Gray | Exited | Terminated (check exit code for success/failure) |

## Resource Details Panel

Click a resource name to expand its details:

- **Properties** — key-value pairs: container ID, image, allocated ports
- **Environment Variables** — every injected env var (connection strings, service URLs)
- **Endpoints** — full URLs with copy buttons
- **Health Checks** — status of each registered check with last check time
- **Volumes** — mounted volumes and bind mounts

## Console Logs

Select a resource from the Console Logs dropdown to see its real-time stdout/stderr:

```
[13:42:01] info: Microsoft.Hosting.Lifetime[14]
              Now listening on: http://[::]:5001
[13:42:01] info: Microsoft.Hosting.Lifetime[0]
              Application started.
[13:42:02] info: Bazaar.Api.SearchController[0]
              Search index ready with 1,247 documents
```

Console logs are **streamed in real time** — no polling. DCP captures stdout/stderr from every container and process via `ResourceLoggerService` and pushes them to the Dashboard through the gRPC streaming connection.

### How It Works Under the Hood

```
Container stdout  --> DCP captures --> ResourceLoggerService
                                            |
Process  stdout   --> DCP captures -->     |
                                            v
                                    Dashboard gRPC subscription
                                            |
                                            v
                                    ConsoleLogs.razor component
                                            |
                                            v
                                    Browser (SignalR push)
```
